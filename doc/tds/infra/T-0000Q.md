# TDS: docker compose preflight 端口冲突检测扩展 (Task ID: T-0000Q)

## 一、需求背景

- **关联需求**：QA 战报 BUG-INFRA-006（模拟）暴露 `npm run e2e:up` 启动失败，报错 `Error: address already in use`（端口冲突）。
- **影响测试用例**：所有 E2E 用例无法在本地执行，开发者需手动定位冲突进程（`lsof -i :5432` / `kill -9`），体验差。
- **业务影响**：
  - 新人 onboarding 时间 +15 分钟（troubleshooting 端口冲突）。
  - CI 环境若有残留进程（如上次失败未清理），导致 E2E 任务连续失败。
- **现状**：
  - `scripts/dev/preflight.sh`（T-0000G）已实现 5 端健康检查（PG/Redis/AppServer/AdminServer/Web），但**仅检测服务是否可达**，不检测端口是否已被占用。
  - `scripts/dev/e2e-up.sh` 直接执行 `docker compose up -d`，若端口冲突则 compose 启动失败，错误信息被 docker 日志淹没。
- **本 Task 目标**：
  1. 在 `e2e-up.sh` 中，在 `docker compose up` **之前**检测关键端口（5432/6379/3000/3001/5173）是否已被占用。
  2. 若冲突，输出清晰的错误提示（包含占用进程信息，支持 macOS/Linux）。
  3. 可选：提供自动清理选项（`--force` flag 自动 kill 冲突进程，但需二次确认避免误杀）。

## 二、技术方案

### 核心策略

**Phase 1（本 Task 交付）**：
- 在 `e2e-up.sh` 开头增加 `check_ports()` 函数，检测 5 个关键端口。
- 若冲突，打印彩色错误信息（红色高亮端口 + 占用进程 PID/名称）+ 退出码 1。
- 提示开发者手动清理命令（如 `kill -9 <PID>`）。

**Phase 2（未来可选）**：
- 增加 `--force` flag，自动 kill 冲突进程（需二次确认提示）。
- 支持 Windows（通过 `netstat -ano` 或 PowerShell）。

### 改造点位：e2e-up.sh 增加端口检测

**文件**：`scripts/dev/e2e-up.sh`

```bash
#!/bin/bash
set -euo pipefail

# 颜色定义
RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
NC='\033[0m' # No Color

# 关键端口列表（服务名 : 端口）
declare -A PORTS=(
  ["PostgreSQL"]=5432
  ["Redis"]=6379
  ["AppServer"]=3000
  ["AdminServer"]=3001
  ["Web"]=5173
)

# 函数：检测单个端口是否被占用（跨平台）
check_port() {
  local port=$1
  local service_name=$2

  if [[ "$OSTYPE" == "darwin"* ]]; then
    # macOS: 使用 lsof
    local pid=$(lsof -ti :$port 2>/dev/null || true)
  else
    # Linux: 使用 ss 或 netstat
    local pid=$(ss -tulnp 2>/dev/null | grep ":$port " | awk '{print $7}' | grep -oP 'pid=\K[0-9]+' || \
                netstat -tulnp 2>/dev/null | grep ":$port " | awk '{print $7}' | cut -d'/' -f1 || true)
  fi

  if [[ -n "$pid" ]]; then
    # 获取进程名称
    local process_name=$(ps -p $pid -o comm= 2>/dev/null || echo "unknown")
    echo -e "${RED}✗ Port $port ($service_name) is already in use by PID $pid ($process_name)${NC}"
    return 1
  else
    echo -e "${GREEN}✓ Port $port ($service_name) is available${NC}"
    return 0
  fi
}

# 函数：批量检测所有端口
check_all_ports() {
  local conflicts=0
  for service in "${!PORTS[@]}"; do
    if ! check_port "${PORTS[$service]}" "$service"; then
      conflicts=$((conflicts + 1))
    fi
  done

  if [[ $conflicts -gt 0 ]]; then
    echo ""
    echo -e "${RED}ERROR: $conflicts port(s) are already in use.${NC}"
    echo -e "${YELLOW}Please stop the conflicting processes manually:${NC}"
    echo ""
    for service in "${!PORTS[@]}"; do
      local port="${PORTS[$service]}"
      if [[ "$OSTYPE" == "darwin"* ]]; then
        local pid=$(lsof -ti :$port 2>/dev/null || true)
      else
        local pid=$(ss -tulnp 2>/dev/null | grep ":$port " | awk '{print $7}' | grep -oP 'pid=\K[0-9]+' || true)
      fi
      if [[ -n "$pid" ]]; then
        echo "  kill -9 $pid  # $service (port $port)"
      fi
    done
    echo ""
    exit 1
  fi
}

# 主流程
echo "🔍 Checking port availability..."
check_all_ports

echo ""
echo "🚀 Starting docker compose..."
docker compose -f docker-compose.yml up -d

echo ""
echo "⏳ Waiting for services to be ready..."
./scripts/dev/preflight.sh

echo ""
echo -e "${GREEN}✅ E2E environment is ready!${NC}"
```

**关键点**：
1. **跨平台兼容**：
   - macOS：`lsof -ti :<port>`（最可靠）。
   - Linux：优先 `ss -tulnp`（现代），fallback 到 `netstat -tulnp`（legacy）。
2. **进程信息**：通过 `ps -p <PID> -o comm=` 获取进程名称（如 `postgres` / `node`）。
3. **彩色输出**：使用 ANSI 转义码（`\033[0;31m` 红色，`\033[0;32m` 绿色），在终端中高亮显示。
4. **退出码**：若有冲突，返回 1（阻止后续 docker compose 执行）。

### 可选扩展：--force 自动清理

**在 `e2e-up.sh` 开头增加参数解析**：

```bash
# 参数解析
FORCE_KILL=0
while [[ $# -gt 0 ]]; do
  case $1 in
    --force)
      FORCE_KILL=1
      shift
      ;;
    *)
      echo "Unknown option: $1"
      exit 1
      ;;
  esac
done

# 修改 check_all_ports 函数
check_all_ports() {
  local conflicts=0
  local pids_to_kill=()

  for service in "${!PORTS[@]}"; do
    local port="${PORTS[$service]}"
    # ... （检测逻辑同上）
    if [[ -n "$pid" ]]; then
      conflicts=$((conflicts + 1))
      pids_to_kill+=("$pid")
    fi
  done

  if [[ $conflicts -gt 0 ]]; then
    if [[ $FORCE_KILL -eq 1 ]]; then
      echo -e "${YELLOW}⚠️  Found $conflicts conflicting process(es). Killing...${NC}"
      for pid in "${pids_to_kill[@]}"; do
        kill -9 "$pid" 2>/dev/null || true
        echo "  Killed PID $pid"
      done
      sleep 2  # 等待端口释放
    else
      # （现有错误提示逻辑）
      exit 1
    fi
  fi
}
```

**使用方式**：`npm run e2e:up -- --force`（package.json 需支持透传参数）。

## 三、TDD 验收标准

### 功能用例

1. **U-1**（端口空闲时正常启动）：所有端口未被占用，运行 `./scripts/dev/e2e-up.sh`，输出 5 行绿色 `✓ Port <N> available`，docker compose 成功启动。
2. **U-2**（单个端口冲突）：手动启动 PostgreSQL（占用 5432），运行 `e2e-up.sh`，输出红色错误 + 提示 `kill -9 <PID>`，脚本退出码 1，docker compose 未启动。
3. **U-3**（多端口冲突）：同时占用 5432/6379/3000，输出 3 行红色错误 + 3 条 kill 命令提示。
4. **U-4**（进程名称显示）：PostgreSQL 进程占用 5432，错误信息包含 `PID 12345 (postgres)`（进程名称准确）。
5. **U-5**（--force 自动清理，可选）：运行 `e2e-up.sh --force`，自动 kill 冲突进程 + 继续启动 compose。

### 跨平台用例

6. **P-1**（macOS）：在 macOS（Intel/M1）环境下运行，端口检测正确（使用 `lsof`）。
7. **P-2**（Linux）：在 Ubuntu 20.04/22.04 环境下运行，端口检测正确（使用 `ss` 或 `netstat`）。

### 集成用例

8. **I-1**（npm scripts 集成）：`npm run e2e:up` 调用 `e2e-up.sh`，端口冲突时终端显示彩色错误（不被 npm 日志淹没）。
9. **I-2**（CI 环境）：在 GitHub Actions 中（假设有残留进程），端口检测失败后 job 标记为 failed，日志清晰显示冲突端口。

### 回归用例

10. **R-1**：无端口冲突时，`e2e-up.sh` 执行时间增加 <2s（端口检测开销可忽略）。
11. **R-2**：修改 `e2e-up.sh` 后，现有 E2E 套件全部通过（`npm run e2e:local`）。

### 用户体验

12. **UX-1**（错误信息清晰）：首次遇到端口冲突的新人，看到错误提示后能在 1 分钟内自行解决（通过复制 `kill` 命令执行）。
13. **UX-2**（Runbook 更新）：`doc/tests/E2E_RUNBOOK.md` §故障排查追加 Q11（端口冲突），链向本 TDS。
