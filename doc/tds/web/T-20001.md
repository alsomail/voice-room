#### [HIGH] LoginForm.tsx:72 — Alert 组件使用了错误的 prop `title`，错误信息无法显示

**文件**：`app/web/src/pages/login/LoginForm.tsx:72`

**问题**：Ant Design 的 `Alert` 组件用于显示文本内容的 prop 是 `message`，而非 `title`。
`title` 会被 React 作为原生 HTML 全局属性渲染在根 DOM 元素上（即鼠标悬停 tooltip），
不会出现在可见文本内容中。当前代码：

```tsx
// ❌ 错误：title 是 HTML 属性，不渲染为可见文本
<Alert
  data-testid="alert-error"
  type="error"
  title={error}
  showIcon
/>
```

**直接影响**：登录失败时，Alert 仅显示一个错误图标，错误文本完全不可见，
直接违反验收标准「3. 登录失败提示」。

**修复**：将 `title={error}` 改为 `message={error}`：

```tsx
// ✅ 正确：message 是 Alert 的内容 prop
<Alert
  data-testid="alert-error"
  type="error"
  message={error}
  showIcon
/>
```

**备注**：测试套件中 `toHaveTextContent('Invalid admin credentials')` 断言应当无法通过，
声称的"100% 测试通过"需核实。

---

#### [MEDIUM] apiClient.ts:64-65 — 未检查 HTTP 响应状态即直接解析 JSON

**文件**：`app/web/src/core/network/apiClient.ts:64`

**问题**：`response.json()` 在 HTTP 502/504 等返回 HTML 错误页面时会抛出
`SyntaxError: Unexpected token '<'`，用户会看到技术性的 JSON 解析错误信息。
应先检查 `response.ok` 或在 `response.json()` 外层加 try/catch，
返回友好提示。

#### [MEDIUM] apiClient.ts — fetch 请求无超时控制

**文件**：`app/web/src/core/network/apiClient.ts:64`

**问题**：原生 `fetch` 默认无超时，Admin Server 无响应时请求将无限挂起，
登录按钮 loading 状态永不消失。应使用 `AbortController` 添加超时（建议 15s）。

#### [MEDIUM] LoginForm.tsx:38 — `localStorage.getItem` 在每次渲染时执行

**文件**：`app/web/src/pages/login/LoginForm.tsx:38`

**问题**：`savedUsername` 和 `initialValues` 在函数体内直接计算，
每次 `loading`/`error` 状态变更触发重渲染都会重新调用 `localStorage.getItem`。
`initialValues` 仅在 Form 挂载时有效，应使用 `useState` 懒初始化：

```tsx
// ✅ 只在挂载时读取一次 localStorage
const [initialValues] = useState<LoginFormValues>(() => {
  const saved = localStorage.getItem(REMEMBER_KEY);
  return { username: saved ?? '', password: '', remember: !!saved };
});
```

#### [HIGH] LoginForm.tsx:70 — Alert 组件使用了已废弃的 `message` prop，应使用 `title`

**文件**：`app/web/src/pages/login/LoginForm.tsx:70`

**问题**：上轮 Review 判断错误，将 `title={error}` 标记为 bug 并要求改为 `message={error}`。
经核实 **antd v6.3.6** 的类型声明及实现：

- `title` — **当前正确 prop**（`/** Content of Alert */`）
- `message` — **已废弃**（`@deprecated please use 'title' instead`）

antd v6 实现中：`const mergedTitle = title ?? message;`，并在非 production 环境输出：

```
Warning: [antd: Alert] `message` is deprecated. Please use `title` instead.
```

该警告已在测试输出中实际出现（3 条），证明"修复"引入了 deprecated API 使用。

**当前错误代码**：
```tsx
// ❌ message 在 antd v6 中已 deprecated
<Alert
  data-testid="alert-error"
  type="error"
  message={error}
  showIcon
  className={styles.errorAlert}
/>
```

**正确代码**（回退至原始写法）：
```tsx
// ✅ title 是 antd v6 Alert 的正确 content prop
<Alert
  data-testid="alert-error"
  type="error"
  title={error}
  showIcon
  className={styles.errorAlert}
/>
```

**证据**：
- `app/web/node_modules/antd/es/alert/Alert.d.ts` 第 44 行：`/** Content of Alert */ title?: React.ReactNode;`
- `app/web/node_modules/antd/es/alert/Alert.d.ts` 第 47-49 行：`/** @deprecated please use 'title' instead. */ message?: React.ReactNode;`
- `app/web/node_modules/antd/es/alert/Alert.js` 第 84 行：`const mergedTitle = title ?? message;`（message 为回退兼容，非主路径）
- 测试输出：`Warning: [antd: Alert] 'message' is deprecated. Please use 'title' instead.`（3 条）

**功能影响**：因 antd v6 保留了 `message` 的向后兼容回退，40 个测试均通过，AC 表面满足，但代码质量不达标，且在测试/开发环境输出告警，未来 antd major 版本升级时存在 breaking change 风险。

**返修要求**：将 `message={error}` 改回 `title={error}`。
