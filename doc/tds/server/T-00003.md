# TDS: 手机号一键登录接口 (Task ID: T-00003)

## 一、背景与目标
- **关联需求**：product.md - 手机号一步登录
- **前置依赖**：T-00002 (短信验证码发送接口)
- **本 Task 的目标**：实现 POST `/api/v1/auth/login` 一步登录接口，验证码正确后登录，手机号不存在自动注册

## 二、方案设计
### 核心数据流
Client → POST /api/v1/auth/login {phone, code} → Controller → AuthService → Redis HGET sms:code:{phone} 校验 → 校验通过 → 查 users 表 → 不存在则 INSERT (自动注册) → shared::jwt::encode_jwt() → 返回 {token, user, is_new} → DEL sms:code:{phone}

### 涉及文件清单
| 文件路径 | 变更类型 | 说明 |
|----------|----------|------|
| `app/server/src/modules/auth/controller.rs` | 修改 | 增加 login handler |
| `app/server/src/modules/auth/service.rs` | 修改 | 登录 + 自动注册逻辑 |
| `app/server/src/modules/auth/repository.rs` | 新增 | users 表 CRUD |
| `app/server/src/modules/auth/dto.rs` | 新增 | LoginRequest, LoginResponse |
| `app/shared/src/jwt/encode.rs` | 引用 | JWT 签发 |

### 接口 / 信令定义
参见 `doc/protocol.md` §二 2.2 POST /api/v1/auth/login

## 三、TDD 验收用例
- [ ] 正向：验证码正确，已有用户登录成功，返回 JWT + 用户信息 + is_new=false
- [ ] 正向：验证码正确，新手机号自动注册，返回 JWT + 用户信息 + is_new=true
- [ ] 正向：新用户默认昵称格式为 "用户XXX"（随机后缀）
- [ ] 正向：JWT 有效期 30 天，iss="voiceroom"
- [ ] 异常：验证码错误返回 401 (40103)
- [ ] 异常：验证码过期返回 401 (40104)
- [ ] 异常：验证码尝试超过 5 次返回 401 (40105)
- [ ] 异常：校验成功后 Redis key 被删除（一次性作废）

**审查范围**：controller.rs、service.rs、dto.rs、repository.rs、routes.rs、redis_store/mod.rs、common/error.rs、shared/jwt/token.rs

**验收标准核查**：
1. 🟢 验证码错误/过期返回 401：`InvalidVerificationCode`(40103)、`VerificationCodeExpired`(40104)、`VerificationCodeMaxAttempts`(40105) 均正确映射 401，与 shared::error::ErrorCode 枚举数值一致。

3. 🟢 JWT 30天有效期：`TOKEN_EXPIRES_SECS = 30*24*3600`；claims(sub/iss/exp/iat) 齐全；复用 `shared::jwt::encode_token()`；`jwt_secret` 来自环境变量注入，无硬编码。
4. 🟡 "基于 msg_id 幂等"（Tasks.md §验收标准#4）：`LoginRequest` 无 `msg_id` 字段，但实际幂等通过 OTP 一次性原子消费实现（`VERIFY_CODE_LUA` DEL），与 protocol.md §1.6 明确定义的"可重试+自动注册"策略一致。`msg_id` 在 protocol.md 中仅属于 WebSocket 信令（§5.3），不适用 HTTP 端点。Tasks.md 该条款为历史描述偏差，非代码缺陷。

**安全性**：
- OTP 防重放：Lua 原子 DEL，同一 OTP 二次使用返回 40104 ✅
- 并发注册：OTP 原子消费序列化入口，DB phone UNIQUE 兜底 ✅  
- request_id：所有错误路径调用 `err_response(e, rc.request_id())` ✅
- 5xx 信息屏蔽：`safe_message()` 正确隐藏内部细节 ✅

**低优先级观察（不阻塞）**：
- L-01：被封用户 OTP 在封禁检查前已被消费（行为符合安全原则，未在 TDS 文档化）
