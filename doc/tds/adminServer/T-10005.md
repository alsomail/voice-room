# T-10005 — Admin Server 房间详情接口

## 接口规格

```
GET /api/v1/admin/rooms/:id
Authorization: Bearer <admin-jwt>
```

### 路径参数

| 参数 | 类型 | 说明 |
|------|------|------|
| id | string (UUID) | 房间 ID |

### 成功响应（200）

```json
{
  "code": 0,
  "message": "ok",
  "request_id": "xxx",
  "data": {
    "room_id": "uuid",
    "title": "Room Title",
    "status": "active",
    "room_type": "normal",
    "member_count": 5,
    "max_members": 50,
    "owner": {
      "user_id": "uuid",
      "nickname": "UserNick",
      "avatar": "https://..."
    },
    "mic_slots": [],
    "created_at": "2024-01-01T00:00:00Z",
    "updated_at": "2024-01-01T00:00:00Z"
  }
}
```

> **注意**：`mic_slots` 在 MVP 阶段固定返回空数组，后续迭代填充麦位信息。

### 错误码

| HTTP | code | 说明 |
|------|------|------|
| 401 | 40101 | 无 Token / Token 无效 / C端 JWT |
| 401 | 40102 | Token 已过期 |
| 403 | 40301 | 角色无 RoomRead 权限（如 finance） |
| 400 | 40003 | 路径参数 id 不是合法 UUID |
| 404 | 40400 | 房间不存在或已软删除 |

### 与 C 端的差异

| 场景 | C 端 | 后台（本接口） |
|------|------|---------------|
| closed 房间 | 404 | **200**（后台需要查看历史房间） |
| 软删除房间 | — | 404/40400 |

## SQL 设计

```sql
-- 按 id 查询房间详情（不过滤 status，仅过滤 deleted_at IS NULL）
SELECT r.id, r.title, r.status, r.room_type, r.member_count, r.max_members,
       r.owner_id, u.nickname AS owner_nickname, u.avatar AS owner_avatar,
       r.created_at, r.updated_at
FROM rooms r
JOIN users u ON r.owner_id = u.id
WHERE r.id = $1
  AND r.deleted_at IS NULL;
```

### 发现问题列表

#### 🟡 MEDIUM（建议修复）

**MEDIUM-01：D-08 集成测试对 `updated_at` / `created_at` 仅做存在性断言，未验证 RFC3339 格式**

```rust
// 当前实现（bootstrap/mod.rs D-08）
assert!(json["data"]["updated_at"].as_str().is_some(), "D-08: updated_at 必须存在");

// 建议补充格式验证
let updated_at_str = json["data"]["updated_at"].as_str().unwrap();
assert!(
    DateTime::parse_from_rfc3339(updated_at_str).is_ok(),
    "D-08: updated_at 应为合法 RFC3339 格式，实际: {updated_at_str}"
);
```

> 风险：若未来 `to_rfc3339()` 被意外替换为其他格式，现有测试无法捕获。

#### 🟢 LOW（可选优化）

```rust
// 当前
```
