# TDS: 订单查询 API (Task ID: T-10025)

## 一、背景与目标

- **关联需求**：[phase1_payment_billing.md §2.1 In Scope — Admin 订单查询](../../product/phase1_payment_billing.md)；[payment_api.md §9.7 Admin REST](../../protocol/payment_api.md#97-admin-restadmin-server)
- **本 Task 的目标**：为 Admin Server 实现订单分页列表与详情两个只读接口，支持多维度过滤、完整状态时间戳、Google 原始响应 JSON 回传，并在每次查询时写审计日志。

---

## 二、方案设计

### 核心数据流

```
Web Admin (T-20030)
  │
  │  GET /api/v1/admin/payments/orders
  │      ?user_id=&state=&provider=&created_from=&created_to=
  │       &amount_min=&amount_max=&page=1&page_size=20
  ▼
[JWT 中间件] decode_token(iss="voiceroom-admin")
  │  失败 → 401
  ▼
AdminAuthContext.require_permission(PaymentRead)
  │  失败 → 403 (40301)
  ▼
list_orders_handler(Query<ListOrdersQuery>)
  ├─ page_size 上限 100，超出截断
  ├─ created_from / created_to 合法性校验（from ≤ to）
  ▼
PaymentOrderRepo::list_orders(filter) — shared DB (admin_server_user)
  SQL: SELECT o.*, s.diamonds, s.display_price_usd
       FROM payment_orders o
       JOIN payment_skus s ON o.sku_id = s.sku_id
       WHERE (user_id = $1 OR $1 IS NULL)
         AND (state = $2 OR $2 IS NULL)
         AND (provider = $3 OR $3 IS NULL)
         AND (created_at >= $4 OR $4 IS NULL)
         AND (created_at <= $5 OR $5 IS NULL)
         AND (amount_micros >= $6 OR $6 IS NULL)
         AND (amount_micros <= $7 OR $7 IS NULL)
       ORDER BY created_at DESC
       LIMIT $8 OFFSET $9
  ▼
AuditLogger::log_action(action="list_payment_orders",
                        detail={filters, page, page_size})  — fire-and-forget
  ▼
Response 200 { code:0, data:{ orders:[…], total, page, page_size } }

─────────────────────────────────────────────────────────

Web Admin (T-20030 detail Drawer)
  │
  │  GET /api/v1/admin/payments/orders/:id
  ▼
[JWT + PaymentRead]
  ▼
detail_order_handler(Path<Uuid>)
  ▼
PaymentOrderRepo::find_by_id(order_id)
  SQL: SELECT o.*, s.diamonds, s.display_price_usd,
              o.state_history,       -- JSONB
              o.provider_response_raw  -- JSONB（Google Purchases.products:get 原始响应）
       FROM payment_orders o
       JOIN payment_skus s ON o.sku_id = s.sku_id
       WHERE o.order_id = $1
  │  不存在 → 404 (40904 ORDER_NOT_FOUND)
  ▼
AuditLogger::log_action(action="view_payment_order_detail",
                        target_type="payment_order", target_id=order_id)
  ▼
Response 200 {
  order_id, user_id, sku_id, provider,
  amount_micros, currency, country_code, state,
  state_history: [{state, ts, source}, …],
  provider_response_raw: { … },     // Google 原始 JSON
  purchase_token_masked: "oojkl…XXXX",
  provider_order_id,
  risk_flags,
  created_at, verified_at, credited_at, acked_at, failed_at, failed_reason
}
```

### 接口 / 信令定义

**ListOrdersQuery 参数（补充 §9.7 未列细节）**：

| 参数 | 类型 | 默认值 | 约束 |
|------|------|--------|------|
| `user_id` | `UUID?` | — | 可选 |
| `state` | `String?` | — | 枚举值见 §9.2.3 |
| `provider` | `String?` | — | `google_play` \| `apple_iap` \| `mock` |
| `created_from` | `DateTime?` | — | ISO-8601 UTC |
| `created_to` | `DateTime?` | — | ISO-8601 UTC，须 ≥ created_from |
| `amount_min` | `i64?` | — | micros 单位 |
| `amount_max` | `i64?` | — | micros 单位，须 ≥ amount_min |
| `page` | `u32` | 1 | ≥ 1 |
| `page_size` | `u32` | 20 | 1–100，超 100 自动截断 |

**性能策略**：
- 利用 `idx_orders_user_created(user_id, created_at DESC)` 加速按用户过滤
- `idx_orders_state_pending(state, created_at)` 加速状态过滤
- 无复杂索引时，跨全表聚合场景由 `EXPLAIN ANALYZE` 验证 P95 < 200ms

### Round 1
- **状态**：🔴 未通过（BLOCKED）
- **审查意见**：
  - **[P0-1] 模块未挂载**：`app/adminServer/src/modules/mod.rs` 缺少 `pub mod payment;`，payment 目录下的所有文件均未被编译器编译进 crate，等同于死代码。需在该文件追加此声明。
  - **[P0-2] controller.rs 不存在**：`list_orders_handler` / `detail_order_handler` 两个 handler 文件均未创建。`payment/mod.rs` 虽声明 `pub mod controller;`，但源文件缺失（因 P0-1 掩盖了编译错误）。
  - **[P0-3] 路由未注册**：`app/adminServer/src/bootstrap/mod.rs` 中未发现 `/api/v1/admin/payments/orders` 及 `/api/v1/admin/payments/orders/:id` 任何路由注册，HTTP 端点完全不可达。
  - **[P0-4] 集成测试缺失**：TDD 验收用例 OQ-01～OQ-15 / PROTO-1/2 均未实现，`tests/payment_orders_test.rs` 文件不存在。TDD 阶段必须先有测试再有实现，此条为红线。
- **改进建议**：
  1. 先在 `modules/mod.rs` 追加 `pub mod payment;`，验证 `cargo build` 通过（会暴露缺失文件的编译错误）。
  2. 创建 `payment/controller.rs`：实现 `list_orders_handler` + `detail_order_handler`，使用 `AuditLogger::log_action` 写审计日志（fire-and-forget），`OrderFilter` 从 `ListOrdersQuery` 转换，`find_by_id` 不存在返回 404/40904。
  3. 在 `bootstrap/mod.rs` 注册两条路由（`GET /api/v1/admin/payments/orders` + `GET /api/v1/admin/payments/orders/:id`），绑定 `PaymentRead` 权限中间件。
  4. 创建 `tests/payment_orders_test.rs` 集成测试，至少覆盖 OQ-01（分页）、OQ-10（audit log）、OQ-11（401）、OQ-12（403）、OQ-13（404）、OQ-15（page_size 截断）、PROTO-1（URL 字面量）。
