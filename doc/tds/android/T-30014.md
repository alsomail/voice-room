## 二、方案设计

### §2.1 核心数据流

```
RoomUiState.messages: List<ChatMessageUi>
        │
        ▼
ChatMessageList(messages, modifier)
   ├── LaunchedEffect(messages.size) → animateScrollToItem(lastIndex)
   ├── remember { messages.distinctBy { it.messageId } }   ← 去重
   └── LazyColumn
         ├── USER_TEXT    → UserMessageItem  (Row → Column → nickname + content)
         └── SYSTEM_NOTICE → SystemNoticeItem (Box centered → gray Text)
```

### §2.2 数据模型扩展（存于 `RoomUiState.kt`）

```kotlin
/** 消息类型枚举（T-30014 新增） */
enum class MessageType {
    USER_TEXT,      // 普通用户文字消息（左对齐，显示昵称）
    SYSTEM_NOTICE,  // 系统通知（居中，灰色，无昵称头像）
}

/** 聊天消息 UI 模型（T-30014 扩展） */
data class ChatMessageUi(
    val messageId: String,
    val senderNickname: String? = null,   // SYSTEM_NOTICE 时为 null
    val content: String,
    val timestamp: Long,
    val messageType: MessageType = MessageType.USER_TEXT,
)
```

> **向后兼容**：`senderNickname` 由 `String` 改为 `String? = null`；`messageType` 带默认值
> `MessageType.USER_TEXT`，现有调用方无需修改。

### §2.3 UI 规格

| 类型 | 容器 | 对齐 | 内容 | testTag |
|------|------|------|------|---------|
| `USER_TEXT` | `Row → Column` | 左对齐（fillMaxWidth） | 昵称（primary 色，labelSmall）+ 内容（bodyMedium） | `user_message_{index}` |
| `SYSTEM_NOTICE` | `Box(contentAlignment=Center)` | 居中 | 内容（labelSmall，onSurface 60% alpha） | `system_message_{index}` |

两类 Item 均使用 `semantics(mergeDescendants = true)` 以支持 `assertTextContains`。

### §2.4 自动滚动机制

```kotlin
LaunchedEffect(deduplicated.size) {
    if (deduplicated.isNotEmpty()) {
        listState.animateScrollToItem(deduplicated.lastIndex)
    }
}
```

- 监听列表长度变化（新消息到来），触发平滑滚动至末尾
- 使用 `deduplicated` 而非原始 `messages`，避免重复 key 引发 LazyColumn 崩溃

### §2.5 消息去重

在 Composable 内部用 `remember(messages)` 计算去重列表：
```kotlin
val deduplicated = remember(messages) { messages.distinctBy { it.messageId } }
```

后续 T-30017 ViewModel 层亦会做去重，双层保险。

  **[MEDIUM] CL-06 — 时钟控制非标准用法（不阻止合并，建议后续修复）**
  - 文件：`ChatMessageListTest.kt:187`
  - 问题：`mainClock.advanceTimeBy(1_000)` 在默认 `autoAdvance = true` 状态下调用，按 Compose 测试框架规范，手动推进时钟须先暂停自动模式，否则该调用为无效 no-op。当前测试能通过是因为 `waitForIdle()` 本身会 flush 协程调度器，`animateScrollToItem` 在 idle 时已完成，`advanceTimeBy` 并无实质作用，但会误导后续维护者认为此处做了时钟控制。
  - 建议修复：
    ```kotlin
    // 方案 A（明确手动控制）
    composeTestRule.mainClock.autoAdvance = false
    composeTestRule.setContent { ChatMessageList(messages = messages, modifier = Modifier.height(300.dp)) }
    composeTestRule.mainClock.advanceTimeBy(1_000)
    composeTestRule.waitForIdle()
    // 方案 B（简洁，依赖 waitForIdle 足够）
    composeTestRule.setContent { ... }
    composeTestRule.waitForIdle()  // 去掉多余的 advanceTimeBy
    ```

  **[LOW] UserMessageItem — Row 包裹单个 Column 存在冗余**
  - 文件：`ChatMessageList.kt:92`
  - 问题：`Row { Column { … } }` 中 Row 只含一个子元素，对布局无贡献，增加一层 measure/layout 开销。
  - 建议：直接用 `Column(modifier = modifier)` 替换，Row 可在将来需要头像时再引入。

  **[LOW] 昵称与消息内容之间缺少间距**
  - 文件：`ChatMessageList.kt:93-106`
  - 问题：`Column` 未设 `verticalArrangement` 或 `Spacer`，昵称与内容紧贴，视觉层次弱。
  - 建议：`Column(verticalArrangement = Arrangement.spacedBy(2.dp))`。
