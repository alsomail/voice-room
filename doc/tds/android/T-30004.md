# TDS: 用户信息 Repository (Task ID: T-30004)

## 一、背景与目标
- **关联需求**：product.md - 用户资料
- **本 Task 的目标**：封装用户信息的获取、缓存和订阅。通过 Retrofit 拉取 `/users/me`，使用 Room Database 本地缓存，并通过 Flow 提供实时数据订阅能力。
- **前置依赖**：T-00005 (users/me 接口), T-30003

## 二、方案设计

```
登录成功 → UserRepository.fetchMe() → Retrofit GET /users/me → UserEntity → Room Database 缓存
其他页面 → UserRepository.observeMe() → Room DAO Flow → 实时订阅用户信息变更
```

#### [MEDIUM] M01 — `data class` 自动 `toString()` 暴露 PII 字段

**文件：** `domain/user/UserProfile.kt` / `data/remote/model/UserMeResponseData.kt`

**问题：** Kotlin `data class` 自动生成的 `toString()` 会将所有字段序列化为字符串，包括 `phone`（手机号，PII）和 `coinBalance`（财务数据）。若上层调用方（ViewModel 日志、Timber、Firebase Crashlytics 等崩溃上报 SDK）无意间打印 `result.toString()` 或 `profile.toString()`，敏感数据将进入日志流，违反数据隐私规范。

```kotlin
// 现状：data class 自动生成 toString() 包含所有字段
data class UserProfile(
    val phone: String,      // 泄漏："+966512345678"
    val coinBalance: Long,  // 泄漏：1000
    ...
)

// 建议：重写 toString() 遮蔽敏感字段
override fun toString(): String =
    "UserProfile(id=$id, nickname=$nickname, vipLevel=$vipLevel, phone=***, coinBalance=***)"
```

`UserMeResponseData` 同样建议处理（或转换完成后立即丢弃，不长期持有）。

#### [MEDIUM] M02 — `parseBody` 与 `RetrofitAuthRepository` 完全重复，违反 DRY

**文件：** `data/user/RetrofitUserRepository.kt:45-66` vs `data/auth/RetrofitAuthRepository.kt:53-74`

**问题：** `parseBody` 函数两处实现完全相同（仅注释语言不同）。随着 Repository 数量增加（未来用户信息更新、礼物、房间等），每个 Repository 都会各自复制这段逻辑，导致：错误处理策略变更时需同步修改多处，极易遗漏。

```kotlin
// 建议：提取到 data/remote/NetworkExtensions.kt 共享扩展函数
fun <T> retrofit2.Response<ApiResponse<T>>.parseBody(gson: Gson): T {
    if (isSuccessful) {
        val apiBody = body() ?: throw ApiException(-1, "Empty response body")
        if (apiBody.code == 0 && apiBody.data != null) return apiBody.data
        throw ApiException(apiBody.code, apiBody.message)
    }
    val errorJson = errorBody()?.string()
    if (!errorJson.isNullOrBlank()) {
        runCatching {
            val type = object : TypeToken<ApiResponse<Nothing>>() {}.type
            val errorBody: ApiResponse<Nothing> = gson.fromJson(errorJson, type)
            throw ApiException(errorBody.code, errorBody.message)
        }.onFailure { if (it is ApiException) throw it }
    }
    throw ApiException(code(), "HTTP ${code()}: ${message()}")
}
```

#### [LOW] L01 — `Gson` 实例未通过 DI 注入，各 Repository 独立构造

**文件：** `data/user/RetrofitUserRepository.kt:26`

**问题：** `private val gson = Gson()` 每个 Repository 独立创建实例。`Gson` 线程安全且重量级，应作为单例通过 DI（Hilt/手动注入）共享，与 `RetrofitAuthRepository` 保持一致的注入模式。

```kotlin
// 建议：通过构造器注入
class RetrofitUserRepository(
    private val apiService: UserApiService,
    private val gson: Gson          // 由 DI 提供单例
) : IUserRepository { ... }
```
