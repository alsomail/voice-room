# TDS: WS MuteUser/UnmuteUser 信令 + 双重拦截 (Task ID: T-00029)

## 一、背景与目标
- **关联需求**：E-10 禁麦 / 禁言 ([phase1_room_governance §3.6](../../product/phase1_room_governance.md))
- **本 Task 的目标**：新增 MuteUser/UnmuteUser WS 信令；禁麦用户自动下麦；SendMessage/TakeMic 前置拦截。

## 二、方案设计

### 信令
**`MuteUser` C→S**
```json
{ "type":"MuteUser", "msg_id":"uuid",
  "payload":{ "room_id":"uuid", "target_user_id":"uuid",
              "type":"mic"|"chat", "duration_sec":300, "reason":"..." } }
```
- duration_sec ∈ [60, 86400]；0 视为"解除"走 UnmuteUser
- type 必填

**`UnmuteUser` C→S**
```json
{ "type":"UnmuteUser", "msg_id":"uuid",
  "payload":{ "room_id":"uuid", "target_user_id":"uuid", "type":"mic"|"chat" } }
```

**广播 `UserMuted` S→房间所有人**
```json
{ "type":"UserMuted", "payload":{
    "room_id":"uuid", "target_user_id":"uuid", "type":"mic",
    "duration_sec":300, "expires_at":"RFC3339", "operator_id":"uuid" } }
```
- duration_sec=0 即解除广播

错误码：`40301` 权限不足、`40002` payload 非法、`40400` target 不在房间、`40302` target 为房主。

### Redis Key
| Key | TTL | 含义 |
|-----|-----|------|
| `mic_muted:{room_id}:{user_id}` | duration_sec | 禁麦中 |
| `chat_muted:{room_id}:{user_id}` | duration_sec | 禁言中 |

### 处理流程
```rust
handle_mute(type=mic):
  1. 权限：ctx.user_id ∈ {owner, admin} 且 target ≠ owner
  2. SETEX mic_muted:... duration reason
  3. INSERT room_mute_records(type='mic', duration_sec, reason, operator_id)
  4. 若 target 在麦 → room_manager.leave_mic + 广播 MicLeft forced=true
  5. 广播 UserMuted 给房间所有人
```

### SendMessage / TakeMic 前置拦截
- **SendMessage**（T-00016）处理前：`GET chat_muted:{room_id}:{user_id}`，存在则返回 `CHAT_MUTED(40305)`
- **TakeMic**（T-00014）处理前：`GET mic_muted:{room_id}:{user_id}`，存在则返回 `MIC_MUTED(40306)`

### 送礼不受影响
T-00020 不检查 mute key（符合权限矩阵）
