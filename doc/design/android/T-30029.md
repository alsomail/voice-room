# T-30029: 礼物接收者选择器

> **归属端**: Android
> **模块**: UI/Gift
> **前置依赖**: T-30028 (礼物面板)
> **参考**: [competitors.md §A.1](../../product/competitors.md)

---

## 需求描述

礼物面板中间的横向滚动组件，用于选择礼物接收者。只显示当前在麦用户，默认选中主麦（1号麦）。

## 组件布局

```
┌──────────────────────────────────┐
│ 接收者                            │  ← labelSmall, 左对齐
│ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐         │
│ │👑│ │👤│ │👤│ │👤│ │👤│  >       │  ← LazyRow, 48dp 头像
│ │主│ │阿│ │哈│ │法│ │萨│         │  ← 昵称缩略（最多2字符）
│ └──┘ └──┘ └──┘ └──┘ └──┘         │
│  ✓                                │  ← 选中项金色光圈 + 底部对勾
└──────────────────────────────────┘
```

## 交互逻辑

| 操作 | 行为 |
|------|------|
| 面板打开 | 默认选中 1 号主麦（若无人则选第一个在麦用户） |
| 点击头像 | 该项选中（金色光圈 + 对勾）+ 清除其他选中 |
| 麦位变化（WS） | 列表实时刷新，若选中用户下麦则回退到主麦 |
| 无人在麦 | 显示占位"当前无人在麦" + 礼物面板"送出"按钮禁用 |

## 异常流程

| 场景 | 处理 |
|------|------|
| 所有人下麦 | 清除选中 + 占位提示 |
| 选中用户突然下麦 | 自动切换到下一个在麦用户（若无则清空） |

## 通用组件

- `AvatarWithFrame`（T-30018）：base 头像
- `MicUserChip`（新）：头像 + 缩略昵称 + 选中光圈组合

## TDD 验收标准

1. 面板打开时默认选中 1 号主麦用户
2. 点击其他麦位用户后选中切换
3. WS 推送麦位变化后列表实时更新
4. 选中用户下麦时自动切换到主麦或下一位
5. 无人在麦时显示"当前无人在麦"占位 + 送出按钮禁用
6. 选中状态暴露给 ViewModel（供 T-30030 SendGift 使用）
