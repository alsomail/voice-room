## 三、TDD 验收用例
- [ ] **隐身**：king（invisibility=all）出现在观众席查询时被剔除；admin/房主仍可见；MemberSnapshot 同步过滤
- [ ] **优先抢麦**：king（weight=10.0）vs 普通用户（weight=1.0）100ms 内并发抢同一坑位 100 次 → king 成功率 ≥ 85%（softmax 抽签）
- [ ] **礼物折扣**：duke（10%）送 1000 钻礼物 → 实际扣款 900 钻；gift_transactions.discount_pct=10, original_price=1000；房间 GiftReceived.payload 仍显示原价（产品规则）
- [ ] **折扣 floor 至千位**：duke 送 1234 钻 → discount=123 → 实付 1111；测断言 floor((1234*90)/100/1000)*1000=1000... → 实际产品规则按 phase1_nobility §3.5.8 校核；测试与 phase1_nobility 完全一致
- [ ] **免密**：duke 进入密码房 + 房主开关 respect_owner_switch=false → 200 直接入；房主开 respect_owner_switch=true → 400 PASSWORD_REQUIRED
- [ ] **配置驱动**：admin 修改 `privileges.gift_discount.percent` → 缓存 invalidation 后下一次送礼立即生效（< 60s）

### 🔴 协议路径绑定
- [ ] **PROTO-1**：观众席响应中隐身用户 user_id 缺席（grep 测试断言）
- [ ] **PROTO-2**：TakeMicResult 在并发场景下高等级用户先返回 success
- [ ] **PROTO-3**：SendGiftResult.charged_diamonds < gift.price（折扣验证）
- [ ] **PROTO-4**：JoinRoom 不带 password 时 duke 用户返 200（免密钩子触发）
