# TDS: Android productFlavors {local/staging/prod} (Task ID: T-30050)

> **主设计**：[T-0000E](./T-0000E.md)（§2.4.4 Android BuildConfig 字段冻结）

## 一、背景与目标
- 修复体检缺口 C7：Android 仅 debug/release，无环境维度，切换需重改 local.properties 全量重编。
- 引入 `productFlavors`，三档 APK 同设备共存。

## 二、方案设计

### 涉及文件清单
| 文件路径 | 变更类型 | 说明 |
|---|---|---|
| `app/android/app/build.gradle.kts` | 修改 | 增加 `flavorDimensions += "env"` 与 3 个 flavor |
| `app/android/app/src/local/AndroidManifest.xml` | 新增（如需） | 仅 local 启用 `usesCleartextTraffic=true` |
| `app/android/app/src/main/AndroidManifest.xml` | 修改 | 移除全局 cleartext 配置（改由 flavor 控制） |
| `app/android/local.properties.example` | 新增 | 文档化 `voiceRoomApiBaseUrl` 等覆盖键 |

### flavor 定义（核心）
```kotlin
android {
    flavorDimensions += "env"
    productFlavors {
        create("local") {
            dimension = "env"
            applicationIdSuffix = ".local"
            versionNameSuffix = "-local"
            buildConfigField("String", "API_BASE_URL", "\"http://10.0.2.2:3000/api\"")
            buildConfigField("String", "WS_URL", "\"ws://10.0.2.2:3000/ws\"")
            buildConfigField("String", "ANALYTICS_ENDPOINT", "\"http://10.0.2.2:3000/api/v1/events/batch\"")
            buildConfigField("String", "APP_ENVIRONMENT", "\"local\"")
            manifestPlaceholders["usesCleartextTraffic"] = "true"
        }
        create("staging") {
            dimension = "env"
            applicationIdSuffix = ".stg"
            versionNameSuffix = "-stg"
            buildConfigField("String", "API_BASE_URL", "\"https://stg-api.example.com/api\"")
            buildConfigField("String", "WS_URL", "\"wss://stg-api.example.com/ws\"")
            buildConfigField("String", "ANALYTICS_ENDPOINT", "\"https://stg-api.example.com/api/v1/events/batch\"")
            buildConfigField("String", "APP_ENVIRONMENT", "\"staging\"")
            manifestPlaceholders["usesCleartextTraffic"] = "false"
        }
        create("prod") {
            dimension = "env"
            // 无 applicationIdSuffix
            buildConfigField("String", "API_BASE_URL", "\"https://api.example.com/api\"")
            buildConfigField("String", "WS_URL", "\"wss://api.example.com/ws\"")
            buildConfigField("String", "ANALYTICS_ENDPOINT", "\"https://api.example.com/api/v1/events/batch\"")
            buildConfigField("String", "APP_ENVIRONMENT", "\"prod\"")
            manifestPlaceholders["usesCleartextTraffic"] = "false"
        }
    }
}
```

### 与现有 `local.properties` override 的关系
保留现有 `resolveConfigValue()` 的 3 级 fallback；flavor 提供的是**默认值层**，`local.properties` 与环境变量仍可覆盖。优先级：
1. `local.properties` 的 `voiceRoomApiBaseUrl`（最高）
2. 环境变量 `VOICE_ROOM_API_BASE_URL`
3. flavor 默认值
4. 旧硬编码 `dev-api.example.com`（删除）

### Build Variant 矩阵
- `localDebug`、`localRelease`
- `stagingDebug`、`stagingRelease`
- `prodDebug`、`prodRelease`

CI 默认构建 `localDebug` + `prodRelease`。

### applicationId 同设备共存
- local: `com.voice.room.android.local.debug`（local + debug suffix `.debug`）
- staging: `com.voice.room.android.stg.debug`
- prod release: `com.voice.room.android`
三者 `applicationId` 不同 → 同设备并存。
