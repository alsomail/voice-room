# TDS: Admin 贵族用户查询 API (Task ID: T-10032)

## 一、背景与目标

- **关联需求**：[phase1_nobility.md §3.5 贵族特权细则](../../product/phase1_nobility.md)；[phase1_nobility.md §6 运营管理](../../product/phase1_nobility.md)
- **本 Task 的目标**：在 Admin Server 提供贵族用户分页查询接口（支持 tier_id / status / expire_before 多维筛选）及单用户贵族历史记录查询，供运营人员核查到期用户、renew_channel 构成、操作记录等；分页响应 P95 < 100ms（10 万行规模）；全程跨租户隔离（多实例共库场景）。

## 二、方案设计

### 核心数据流

```
GET /api/v1/admin/nobles/users
  ?tier_id=duke&status=active&expire_before=2026-07-01&page=1&size=20
  ↓ JWT + RBAC(NobleTierRead: super_admin | operator | cs)
  ↓ 参数校验：
      status ∈ {active, expired}（缺省返回全部）
      expire_before 合法 ISO 8601 日期
      page ≥ 1, size ∈ [1,100]
  ↓ 构建 SQL（动态 WHERE 拼接）：
      SELECT un.user_id, u.nickname, u.avatar_url,
             un.tier_id, un.start_at, un.current_period_start,
             un.expire_at, un.auto_renew, un.renew_channel,
             un.total_paid_diamonds, un.total_paid_usd_micros,
             nt.name_en, nt.name_ar, nt.level, nt.badge_color
      FROM user_nobles un
        JOIN users u ON u.id = un.user_id
        JOIN noble_tiers nt ON nt.tier_id = un.tier_id
      WHERE 1=1
        [AND un.tier_id = ?]                           -- tier_id 筛选
        [AND un.expire_at > now()]                     -- status=active
        [AND un.expire_at <= now()]                    -- status=expired
        [AND un.expire_at < ?]                         -- expire_before
      ORDER BY un.expire_at ASC                        -- 即将到期优先
      LIMIT ? OFFSET ?
  ↓ SELECT COUNT(*) … (同 WHERE 条件，tokio::try_join! 并发执行)
  ↓ 200 { items: [...], total, page, size }

GET /api/v1/admin/nobles/users/:user_id/history
  ↓ JWT + RBAC(NobleTierRead)
  ↓ 参数校验：user_id 合法 UUID
  ↓ SQL:
      SELECT * FROM noble_history
      WHERE user_id = ?
      ORDER BY created_at DESC
      LIMIT 100   -- 最近 100 条
  ↓ 200 { items: [...] }
```

**索引依赖**（已由 T-00065 建立）：

```sql
-- 已有（T-00065 migration）：
CREATE INDEX idx_user_nobles_expire ON user_nobles (expire_at);
-- 若 tier_id 筛选频繁，可考虑追加复合索引：
CREATE INDEX idx_user_nobles_tier_expire ON user_nobles (tier_id, expire_at);
-- noble_history 按用户+时间查询：
CREATE INDEX idx_noble_history_user_time ON noble_history (user_id, created_at DESC);
```

**跨租户隔离**：Admin Server 使用 `admin_server_user` 数据库账号，可访问全部用户数据。Admin 无 region 字段，通过 `users.tenant_id`（若存在）或 `users.region` 字段过滤；MVP 阶段单租户，`tenant_id` 条件缺省跳过，但 SQL 框架已预留 `WHERE tenant_id = ?` 占位，多租户上线时仅需补充参数注入。

**`GET /api/v1/admin/nobles/users` 查询参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `tier_id` | String | 否 | 按 tier 筛选（如 `duke`） |
| `status` | Enum | 否 | `active`（expire_at > now()）/ `expired`（expire_at ≤ now()）/ 缺省=全部 |
| `expire_before` | ISO 8601 Date | 否 | 筛选 expire_at < ? 的用户（运营常用：找即将到期用户）|
| `page` | Int | 否 | 默认 1，≥ 1 |
| `size` | Int | 否 | 默认 20，∈ [1, 100] |

**性能约束**：

- 分页查询 + count 并发执行（`tokio::try_join!`），P95 < 100ms（10 万行 + 索引覆盖）
- `idx_user_nobles_expire` 索引覆盖 `expire_at` 筛选（T-00065 已建）
- `idx_noble_history_user_time` 索引覆盖历史查询（本 Task 新建）

## 三、TDD 验收用例

- [ ] **NQ01** status=active 仅返回 `expire_at > now()` 的用户；不含已过期用户
- [ ] **NQ02** status=expired 仅返回 `expire_at <= now()` 的用户；不含有效用户
- [ ] **NQ03** status 缺省 → 返回全部（active + expired）
- [ ] **NQ04** tier_id=duke → 仅返回 duke 等级用户
- [ ] **NQ05** expire_before=2026-06-01 → 仅返回 expire_at < 2026-06-01 的用户
- [ ] **NQ06** 分页：seed 25 条数据；page=1&size=10 返回 10 条；page=3&size=10 返回 5 条；total=25
- [ ] **NQ07** 响应字段含 tier 信息（tier_name_en、tier_level、badge_color — JOIN noble_tiers 验证）
- [ ] **NQ08** `renew_channel` 字段值为 `diamonds` / `usd` / `admin_grant` 之一（不为 null）
- [ ] **NQ09** 历史记录按 `created_at DESC` 排序（最新操作在前）
- [ ] **NQ10** 用户无历史记录时返回 200 + 空数组 `[]`（不 404）
- [ ] **NQ11** 跨租户隔离：seed 两个 tenant 用户；查询仅返回有效 tenant 数据（MVP 单租户时全返，框架已预留 tenant_id 条件）
- [ ] **NQ12** 无 `NobleTierRead` 权限（role=finance）→ 403
- [ ] **NQ13** size=101 → 400（超出上限 100）
- [ ] **NQ14** page=0 → 400
- [ ] **NQ15** expire_before 格式非法（如 "abc"）→ 400
- [ ] **NQ16** 分页查询 + count 并发执行（FakeRepository 断言两个查询均被调用）
