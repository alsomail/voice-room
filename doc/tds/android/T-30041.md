### 预设
```kotlin
enum class KickReason(@StringRes val label: Int, val key: String) {
    Harassment(R.string.kick_reason_harassment, "harassment"),
    Spam(R.string.kick_reason_spam, "spam"),
    Abuse(R.string.kick_reason_abuse, "abuse"),
    Other(R.string.kick_reason_other, "other")
}
```

### State
```kotlin
data class KickDialogState(
    val selected: KickReason = KickReason.Harassment,
    val customText: String = "",
    val submitting: Boolean = false
) {
    val canSubmit: Boolean
        get() = !submitting && (selected != KickReason.Other || customText.isNotBlank())
}
```

### 布局
- AlertDialog，不可外部 dismiss（`properties = DialogProperties(dismissOnClickOutside = false)`）

- 底部 [取消] [确认]（`GoldButton` 在 canSubmit 为 false 时置灰）

## 五、Reviewer意见
### Round 1
- **状态**：❌ 未通过（WARNING — 存在 1 个 HIGH 级别问题）
- **审查意见**：
  1. **[HIGH] WS JSON 字符串拼接未转义**（`RoomViewModel.kt:487`）：`kickUser` 用模板字符串将 `reason` 直接拼入 JSON，当用户在"其他"输入框输入含 `"` 或 `\` 的文本时，WS 信令 JSON 格式损坏，服务端将无法解析。UI 层只限制 100 字符，未过滤特殊字符，属系统性问题（同文件多处 WS 消息拼接均有此隐患）。
  2. **[MEDIUM] KR41-04 测试标签错位**：TDS KR41-04 定义为"点击外部不关闭"，`KickDialogStateTest` 中 KR41-04 标注的却是 `submitting=true → canSubmit=false` 行为，测试可追溯性混乱。`dismissOnClickOutside=false` 实现正确，但没有以 KR41-04 命名的自动化追踪。
  3. **[LOW] `dismissOnBackPress = false` 超出 TDS 声明**：实现额外屏蔽了系统返回键，TDS 未说明，建议补充产品意图。
  4. **[LOW] `KickReason` 枚举缺少 `@StringRes label`**：与 TDS 设计不符，使用硬编码中文字符串，后续多语言扩展需回补。
- **改进建议**：修复 `kickUser` 中 JSON 拼接，改用 Gson/kotlinx.serialization 序列化或手动转义 `"` 和 `\`；校正 KR41-04 测试标签；在 TDS 中补充 `dismissOnBackPress=false` 说明。
