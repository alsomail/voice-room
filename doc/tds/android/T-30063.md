# TDS: 客户端 verify + ack + 容错 (Task ID: T-30063)

## 一、背景与目标
- **关联需求**：[phase1_payment_billing.md §6.3 / §0 acknowledgePurchase 3 天兜底](../../product/phase1_payment_billing.md)。
- **目标**：onPurchasesUpdated → 写 DataStore `pending_purchases` → 调 verify → 收 ACKED 后 BillingPort.acknowledge → 删本地。App 启动时 `PendingPurchaseResumer` 全量重试。余额由 WS BalanceUpdated 推送刷新（红线 #1，禁止本地推断）。

## 二、方案设计
### 核心数据流
```
DomainPurchase 进入 → DataStore.pendingPurchases += token
   ↓ scope=Repository
PendingPurchaseQueue::process(token):
   1. POST /payments/google/verify { order_id, purchase_token }
      ├─ 成功 → 收到响应 state=ACKED 或 CREDITED
      ├─ 网络失败 → 5min 后指数退避重试，最多 3 次
      └─ 40901 INVALID_PURCHASE → 永久失败，标记本地 token 为 FAILED 不再重试
   2. 成功后 BillingPort.acknowledge(token)
   3. DataStore.pendingPurchases -= token
   4. 余额由 WS BalanceUpdated（reason=payment_credit）刷新（不本地 +=）
```

## 三、TDD 验收用例
- [ ] 杀进程后冷启动可恢复 pending（DataStore 持久化）
- [ ] 验证失败网络错 → 5min 重试 3 次，最终标 FAILED
- [ ] 同 token 永不重复消费（DataStore + Server 双幂等）
- [ ] 充值成功 Toast "已到账 +N 钻石" + 跳钱包页
- [ ] 余额由 WS BalanceUpdated 推送刷新；本地不直接 += （断言 ViewModel 不存在 `balance.value += diamonds` 调用）
- [ ] BillingPort.acknowledge 在 verify 成功后必调一次
