# TDS: Web Admin Zod schema 接入（HTTP DTO 强校验，WS 暂缓）(Task ID: T-00102)

## 一、背景与目标

- **关联需求**：用户决策 #3 — 跨语言 E2E 不强制 Web，但 Web 仍存在 HTTP DTO 字段失配风险（手写 TypeScript interface vs server `dto.rs`）。本 Task 范围仅覆盖 **HTTP REST 层**，WS 层留 follow-up（Web 当前 WS 使用面较窄）。
- **本 Task 的目标**：基于 T-00100 的 `doc/protocol/schemas/http/*.schema.json` 在 Web 侧建立 **Zod 校验层**：所有 `apiClient.*` 出/入参出栈/入栈强校验；schema mismatch 立刻在开发模式抛错（生产模式打 sentry 替代品的 console.error）。

## 三、TDD 验收用例

- [ ] **ZOD-1**：构造 mock server 响应缺少必填字段（如 `users[0].id`），apiClient 抛 `ZodError`，业务层 catch 并打 console.error。
- [ ] **ZOD-2**：构造响应字段类型错误（如 `total: "100"` 字符串），抛 `ZodError`。
- [ ] **ZOD-3**：所有现有 8+ apiClient endpoint 覆盖率 100%（grep `z.object` 命中数 ≥ endpoint 数）。
- [ ] **ZOD-4**：`additionalProperties: false` 等价 — Zod `.strict()` 拒绝未知字段，但允许 codegen 配置时降级为 warn（避免 server 滚动添加字段就 break web）。
