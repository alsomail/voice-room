# TDS: 贵族 tier CRUD API (Task ID: T-10030)

## 一、背景与目标

- **关联需求**：[phase1_nobility.md §3.5 贵族特权细则](../../product/phase1_nobility.md)；[phase1_nobility.md §6 运营管理](../../product/phase1_nobility.md)
- **本 Task 的目标**：在 Admin Server 提供贵族等级配置的 CRUD 接口（GET/POST/PUT/DELETE `/api/v1/admin/nobles/tiers`），含 privileges JSONB Schema 校验、上架/下架开关（软删快照保护）、所有写操作写 `admin_logs` 审计；同时向 App Server 发布缓存失效事件，保证运营改 tier 配置后 C 端实时感知。

## 二、方案设计

### 核心数据流

```
GET /api/v1/admin/nobles/tiers?page=&size=
  ↓ JWT 校验 + RBAC(NobleTierRead: super_admin | operator | cs)
  ↓ SELECT noble_tiers ORDER BY level ASC LIMIT ? OFFSET ?
  ↓ 200 { items:[...], total, page, size }

POST/PUT /api/v1/admin/nobles/tiers[/:id]
  ↓ JWT 校验 + RBAC(NobleTierWrite: super_admin | operator)
  ↓ privileges JSON Schema 校验（jsonschema crate + NobilityPrivileges.schema.json）
  ↓ 正整数校验：monthly_diamonds > 0, monthly_usd > 0,
               privileges.monthly_stipend.percent ∈ [0,100]
  ↓ level UNIQUE 检查（冲突 → 409）
  BEGIN TX
    INSERT / UPDATE noble_tiers
    INSERT admin_logs { action=tier_create/tier_update, target_id=tier_id, detail={before,after} }
  COMMIT
  ↓ Redis PUBLISH admin:events { type:"noble_tiers_invalidate", tier_id }  ← fire-and-forget
  ↓ 200/201

DELETE /api/v1/admin/nobles/tiers/:id   (软删，is_active=false)
  ↓ JWT + RBAC(NobleTierWrite)
  BEGIN TX
    SELECT noble_tiers WHERE tier_id=? (不存在 → 404)
    UPDATE noble_tiers SET is_active=false   ← 已持有用户保留至 expire_at（快照机制）
    INSERT admin_logs { action=tier_delete }
  COMMIT
  ↓ Redis PUBLISH noble_tiers_invalidate
  ↓ 200
```

**快照机制说明**：`noble_tiers` 只做软删（`is_active=false`），绝不物理删除；`user_nobles.tier_id` 仍指向原记录，持有者贵族特权不受影响，直至 `expire_at`。

### 接口 / 信令定义

字段清单、privileges JSONB Schema、错误码详见：

- 数据模型：[nobility_api.md §10.2.1 `noble_tiers` 表](../../protocol/nobility_api.md#1021-noble_tiers-表)
- privileges JSONB Schema：[nobility_api.md §10.2.3](../../protocol/nobility_api.md#1023-noble_tiersprivileges-jsonb-schema)
- Admin REST 接口总表：[nobility_api.md §10.5](../../protocol/nobility_api.md#105-admin-rest)
- 错误码（40914 TIER_INACTIVE / 40917 PRIVILEGES_SCHEMA_INVALID）：[nobility_api.md §10.6](../../protocol/nobility_api.md#106-错误码表e-09)

**POST/PUT body 关键校验规则**（不在协议中重复定义，仅列约束摘要）：

| 字段 | 类型 | 约束 |
|------|------|------|
| `monthly_diamonds` | BIGINT | > 0，正整数，禁止浮点 |
| `monthly_usd` | NUMERIC(10,2) | > 0 |
| `level` | SMALLINT | 1..6，全局 UNIQUE |
| `privileges.monthly_stipend.percent` | INT | 0..100（含边界）|
| `privileges.gift_discount.percent` | INT | 0..100 |
| `privileges` | JSONB | 必须通过 `NobilityPrivileges.schema.json` Draft 2020-12 校验 |
| `entrance_animation_url` | TEXT | LV3+ 必填，LV1/2 可 NULL；URL 必须指向 CDN 白名单前缀 |
| `bgm_url` | TEXT | LV2+ 必填；URL 白名单同上 |

**admin_logs 格式**：

```jsonc
// tier_create
{ "action": "tier_create", "target_type": "noble_tier", "target_id": "<tier_id>",
  "detail": { "after": { /* 完整 tier 对象 */ } } }

// tier_update
{ "action": "tier_update", "target_type": "noble_tier", "target_id": "<tier_id>",
  "detail": { "before": { /* 修改前快照 */ }, "after": { /* 修改后快照 */ } } }

// tier_delete（软删）
{ "action": "tier_delete", "target_type": "noble_tier", "target_id": "<tier_id>",
  "detail": { "name_en": "Duke", "level": 5, "is_active_before": true } }
```

## 三、TDD 验收用例

从任务清单复制并扩充边界用例：

- [ ] **TC01** POST 创建成功 → 201，level UNIQUE，`admin_logs` +1
- [ ] **TC02** POST privileges JSON Schema 不通过 → 400 + `code=40917` + 字段路径（如 `$.monthly_stipend.percent`）
- [ ] **TC03** POST `monthly_diamonds=0` → 400（正整数校验）
- [ ] **TC04** POST `monthly_diamonds=-1` → 400（正整数校验）
- [ ] **TC05** POST `privileges.monthly_stipend.percent=101` → 400（0..100 范围校验）
- [ ] **TC06** POST level 重复 → 409
- [ ] **TC07** PUT 修改 monthly_diamonds：已持有 `user_nobles` 不受影响（快照机制验证：user 仍使用旧价格续费至 expire_at）
- [ ] **TC08** DELETE（软删）：`is_active` → false，已持有用户 `user_nobles` 记录仍存在且 expire_at 不变
- [ ] **TC09** GET list 分页：page=2 & size=2，返回正确偏移；total 字段准确
- [ ] **TC10** GET list 仅返回 `is_active=true` 的 tier（软删后消失）
- [ ] **TC11** 无 `NobleTierWrite` 权限（role=cs）→ 403
- [ ] **TC12** 写操作后 Redis publish `noble_tiers_invalidate` 事件（FakeEventPublisher 断言已调用）
- [ ] **TC13** `entrance_animation_url` 非白名单 CDN 前缀 → 400
- [ ] **TC14** LV3+ 不填 `entrance_animation_url` → 400（level=3 必填）

### Round 1

- **状态**：🔴 未通过（BLOCKED）
- **审查意见**：
  - **[P0-1] nobility 模块完全不存在**：`app/adminServer/src/modules/nobility/` 目录及其下所有文件（`mod.rs`、`dto.rs`、`tier_controller.rs`、`tier_service.rs`、`repository.rs`、`privileges_validator.rs` 等）均未创建，`modules/mod.rs` 也无 `pub mod nobility;` 声明。
  - **[P0-2] 权限枚举缺失**：`auth/context.rs` 的 `Permission` 枚举未添加 `NobleTierRead` / `NobleTierWrite`，RBAC 矩阵尚未配置对应角色权限，无法进行任何权限校验。
  - **[P0-3] 路由未注册**：`GET/POST/PUT/DELETE /api/v1/admin/nobles/tiers` 四条路由均未在 bootstrap 注册。
  - **[P0-4] privileges 校验器缺失**：`privileges_validator.rs`（`validate_privileges()` 函数 + jsonschema crate 集成）完全缺失，无法对 POST/PUT body 做 JSON Schema 校验（要求返回 `code=40917 + 字段路径`）。
  - **[P0-5] Redis 缓存失效事件缺失**：写操作后应 PUBLISH `admin:events { type: "noble_tiers_invalidate" }`，实现完全缺失。
  - **[P0-6] 集成测试缺失**：TC01～TC14 / PROTO-1/2/3 均未实现，`tests/tier_crud_test.rs` 文件不存在。
- **改进建议**：
  1. 创建 `modules/nobility/` 目录，从 `mod.rs` 起依次建立各子模块文件，并在 `modules/mod.rs` 追加 `pub mod nobility;`。
  2. 在 `auth/context.rs` 追加 `NobleTierRead` / `NobleTierWrite` 枚举值，更新 RBAC 矩阵（super_admin+operator+cs 有 Read，super_admin+operator 有 Write）。
  3. 创建 `privileges_validator.rs`，使用 `jsonschema` crate 加载 `NobilityPrivileges.schema.json` 做 Draft 2020-12 校验；失败时返回含字段路径的错误响应（40917）。
  4. tier CRUD 必须在同一事务内完成 INSERT/UPDATE + INSERT admin_logs，事务后 fire-and-forget `noble_tiers_invalidate`。
  5. 集成测试重点覆盖 TC02（schema 校验失败 40917 含字段路径）、TC08（软删后 user_nobles 不变）、TC10（GET list 仅返回 is_active=true）、TC12（Redis publish 断言）。
