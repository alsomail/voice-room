# TDS: 财务汇总 API (Task ID: T-10028)

## 一、背景与目标

- **关联需求**：[phase1_payment_billing.md §2.1 In Scope — Web 财务报表](../../product/phase1_payment_billing.md)；[payment_api.md §9.7 Admin REST](../../protocol/payment_api.md#97-admin-restadmin-server)
- **本 Task 的目标**：为财务/运营提供日粒度或月粒度的支付汇总报表，支持按货币分组、USD 折算、退款负数分离、时区对齐（Asia/Riyadh +03:00），100 万订单聚合 P95 < 2s。

## 二、方案设计

### 核心数据流

```
Web Admin (T-20033 财务报表页)
  │
  │  GET /api/v1/admin/payments/reports
  │      ?granularity=day&from=2026-05-01&to=2026-05-31
  ▼
[JWT 中间件] decode_token(iss="voiceroom-admin")
  │  失败 → 401
  ▼
AdminAuthContext.require_permission(PaymentReport)
  │  失败 → 403 (40301)
  ▼
summary_handler(Query<ReportQuery>)
  ├─ granularity ∉ {day, month} → 400 (40003)
  ├─ from 或 to 解析失败 → 400 (40003)
  ├─ from > to → 400 (40003 DATE_RANGE_INVALID)
  ├─ 区间长度校验：day 粒度最长 366 天，month 粒度最长 24 个月
  ▼
ReportQuery::aggregate(granularity, from, to)
  │
  │  SQL（PostgreSQL，利用 AT TIME ZONE 时区转换 + FILTER 聚合）：
  │
  │  WITH period_data AS (
  │    SELECT
  │      date_trunc($1,  -- 'day' or 'month'
  │                credited_at AT TIME ZONE 'Asia/Riyadh'
  │      )::DATE                                          AS period,
  │      currency,
  │      amount_micros,
  │      state,
  │      provider
  │    FROM payment_orders
  │    WHERE
  │      credited_at >= $2                               -- from (UTC)
  │      AND credited_at < $3                            -- to+1day (UTC, 半开区间)
  │      AND state IN ('ACKED','CREDITED','REFUNDED')
  │      AND provider != 'mock'                          -- 隔离 dev_mock 数据
  │  )
  │  SELECT
  │    period,
  │    currency,
  │    SUM(amount_micros) FILTER (WHERE state != 'REFUNDED')  AS gmv_micros,
  │    SUM(amount_micros) FILTER (WHERE state = 'REFUNDED')   AS refund_micros,
  │      -- 退款为正值 micros；计算时取负
  │    COUNT(*) FILTER (WHERE state != 'REFUNDED')            AS order_count,
  │    COUNT(*) FILTER (WHERE state = 'REFUNDED')             AS refund_count
  │  FROM period_data
  │  GROUP BY period, currency
  │  ORDER BY period ASC, currency ASC
  ▼
ReportService::apply_fx_rates(rows, config.payment.exchange_rates)
  │
  │  config 结构（config/{profile}.toml）：
  │    [payment.exchange_rates]
  │    SAR = 0.2666        # 1 SAR = 0.2666 USD
  │    AED = 0.2723
  │    USD = 1.0000
  │    DEFAULT = 1.0       # 未知货币兜底，Sentry P2 告警
  │
  ├─ 对每个 (period, currency) 行：
  │     gmv_usd  = gmv_micros  / 1_000_000.0 * exchange_rates[currency]
  │     refund_usd = -(refund_micros / 1_000_000.0 * exchange_rates[currency])
  │                              ↑ 退款携带负号
  │     avg_ticket_usd = if order_count > 0 { gmv_usd / order_count } else { 0.0 }
  │
  ├─ 按 period 合并多货币：
  │     merged[period].gmv_usd       += gmv_usd (每种货币)
  │     merged[period].gmv_by_currency[currency] = gmv_local_amount
  │     merged[period].order_count   += order_count
  │     merged[period].refund_count  += refund_count
  │     merged[period].refund_amount_usd += refund_usd
  ▼
Response 200：
{
  "code": 0,
  "data": {
    "granularity": "day",
    "from": "2026-05-01",
    "to": "2026-05-31",
    "series": [
      {
        "date": "2026-05-01",          // 按 Asia/Riyadh 时区对齐的日期
        "gmv_usd": "1234.56",          // 成交额 USD，字符串保留 2 位小数
        "gmv_by_currency": {
          "SAR": "4632.57",
          "USD": "0.00"
        },
        "order_count": 142,
        "refund_count": 3,
        "refund_amount_usd": "-45.67", // 退款携带负号
        "avg_ticket_usd": "8.69"
      }
    ],
    "totals": {
      "gmv_usd": "38765.43",
      "order_count": 4412,
      "refund_count": 89,
      "refund_amount_usd": "-1423.10",
      "avg_ticket_usd": "8.78"
    }
  }
}
```

### 性能策略

| 手段 | 说明 |
|------|------|
| **索引命中** | 使用 `idx_orders_state_pending(state, created_at)` 作为基础；财务查询建议新增 `idx_orders_credited_at(credited_at, state, provider)` 覆盖索引 |
| **AT TIME ZONE 下推** | `credited_at AT TIME ZONE 'Asia/Riyadh'` 在 PostgreSQL 层计算，避免应用层 OOM |
| **Mock 数据隔离** | `provider != 'mock'` 谓词排除 dev 数据，不混入财务报表 |
| **分区表预留** | `payment_orders` 若按月分区（T-00050 延后项），此查询可自动剪枝至目标月份分区 |
| **预聚合物化视图（可选）** | 若 100 万订单场景下 P95 仍不达标，可建 `payment_daily_summary` 物化视图，由 cron 每小时刷新；当天数据实时查询补充 |

> **P95 目标**：100 万订单、day 粒度、31 天范围，聚合 SQL P95 < 2s（PostgreSQL 同库，`admin_server_user` 全权账号，EXPLAIN ANALYZE 验证）。

### 接口 / 信令定义

路径字面量与权限详见 [payment_api.md §9.7](../../protocol/payment_api.md#97-admin-restadmin-server)。

**Query 参数**：

| 参数 | 类型 | 默认值 | 约束 |
|------|------|--------|------|
| `granularity` | `String` | 必填 | `day` \| `month` |
| `from` | `Date (YYYY-MM-DD)` | 必填 | Asia/Riyadh 时区日期 |
| `to` | `Date (YYYY-MM-DD)` | 必填 | `from ≤ to` |

**时区细节**：`from` / `to` 以 Asia/Riyadh 时区 00:00 为边界，转换为 UTC 时偏移 +03:00（即 `from` 对应 UTC `from - 3h`，`to` 对应 UTC `to+1day - 3h`）。

**权限**：`payment.report`（finance / super_admin 拥有；operator / cs 无此权限）。

## 三、TDD 验收用例

### 正向用例
- [ ] **RP-01**：`granularity=day&from=2026-05-01&to=2026-05-03`，有 3 条 ACKED 订单（各一天）→ series 含 3 个日期条目
- [ ] **RP-02**：退款订单（state=REFUNDED）在对应日期 `refund_count +1`，`refund_amount_usd` 为负数
- [ ] **RP-03**：`gmv_by_currency` 字段按货币分组（SAR / USD 分别有值）
- [ ] **RP-04**：USD 折算：SAR 订单 `amount_micros=9990000`（9.99 SAR）× `exchange_rates.SAR=0.2666` → `gmv_usd` ≈ `2.66`（误差 < 0.01）
- [ ] **RP-05**：`totals.gmv_usd` = 所有日期 `gmv_usd` 之和（允许浮点精度 ±0.01 USD）
- [ ] **RP-06**：`granularity=month` 时，同月订单合并为一条 series 条目，`date` 格式为 `2026-05`
- [ ] **RP-07**：`provider=mock` 的订单不出现在报表中（dev_mock 隔离）
- [ ] **RP-08**：`from=2026-05-01&to=2026-05-01` 仅含当天 Asia/Riyadh 时区内的订单

### 时区验收
- [ ] **RP-09**：Asia/Riyadh 23:59 的订单归入当天，UTC 对应 20:59；次日 00:00 +03 对应 UTC 21:00 的订单归入次日

### 性能验收
- [ ] **RP-P1**：100 万订单，`granularity=day&from=T-365&to=T` 聚合，EXPLAIN ANALYZE 显示索引命中，P95 < 2s

### 异常用例
- [ ] **RP-10**：`granularity=week` → 400 (40003)
- [ ] **RP-11**：`from=2026-05-31&to=2026-05-01`（from > to）→ 400 (40003 DATE_RANGE_INVALID)
- [ ] **RP-12**：`from` 或 `to` 缺省 → 400 (40003)
- [ ] **RP-13**：role=`cs` 调用 → 403 (40301)
- [ ] **RP-14**：`exchange_rates` 中缺少 `EUR` 货币配置，存在 EUR 订单时 → 使用 DEFAULT=1.0 兜底 + Sentry warn 日志

### 配置化验收
- [ ] **RP-15**：修改 config `exchange_rates.SAR=0.3` 后重启，`gmv_usd` 按新汇率计算

### 🔴 协议路径绑定的强制验收

- [ ] **PROTO-1**（覆盖绑定表 #1）：集成测试断言 URL 字面量为 `/api/v1/admin/payments/reports`（完全一致）；`summary_handler` 被触发（tracing span）
- [ ] **PROTO-2**：响应体 series 元素字段 `date`、`gmv_usd`、`gmv_by_currency`、`order_count`、`refund_count`、`refund_amount_usd`、`avg_ticket_usd` 均存在且类型正确（`gmv_usd` / `refund_amount_usd` 为字符串型小数，`order_count` 为整数）；`refund_amount_usd` 值为负数
