# TDS: App Server 出栈 envelope schema 自检 + HTTP DTO `deny_unknown_fields` (Task ID: T-00103)

## 一、背景与目标

- **关联需求**：T-00100 已冻结字段，但 server 是发布方；若 server 代码改字段名而忘记同步 protocol/，下游所有客户端会被静默打断。本 Task 在 server 出栈处加 **dev/test profile 自检**：每条 WS broadcast / HTTP response 在发出前，按 schema 校验；mismatch 时 panic（dev/test）或 ERROR + 埋点（prod 兼容）。
- **本 Task 的目标**：
  1. WS 出栈：`broadcast_to_room_inner` / `send_to_user` 加 schema 校验中间层（dev/test profile 启用，prod 默认关）。
  2. HTTP DTO：所有 `dto.rs` 的 Response struct 加 `#[serde(deny_unknown_fields)]`（输入侧）+ Response 编译期 schema 比对单测。
  3. ping/pong 兼容期实现（T-00108 协调）：dev/test 阶段 server 双发 `Ping` + `ping`，下个版本切单。

## 二、方案设计

### 涉及文件清单

| 文件 | 变更 | 说明 |
|------|------|------|
| `app/server/src/ws/schema_guard.rs` | 新增 | dev/test 启用的 schema 校验中间层；加载 `doc/protocol/schemas/ws/*.schema.json` |
| `app/server/src/ws/connection.rs` | 修改 | broadcast_to_room_inner / send_to_user 调用 schema_guard |
| `app/server/src/modules/*/dto.rs` | 修改 | 所有 Response 加 `#[serde(deny_unknown_fields)]`（仅 Deserialize 场景，Response 通常只 Serialize 不需要；但 Request struct 需要） |
| `app/server/tests/protocol_schema_test.rs` | 新增 | 抽样 8 条出栈 envelope，断言 schema validate 通过 |
| `app/server/Cargo.toml` | 修改 | 加 `jsonschema = "0.18"` dev-dep |

## 三、TDD 验收用例

- [ ] **GUARD-1**：dev profile 下，故意构造一条不符 schema 的 broadcast envelope（缺 `payload.mic_index`），单测断言 schema_guard panic。
- [ ] **GUARD-2**：prod profile 下不 panic，仅 ERROR 日志 + 埋点；不影响业务流。
- [ ] **DENY-1**：所有 Request DTO 加 `deny_unknown_fields` 后，发送含未知字段的 JSON，返回 400。
- [ ] **PING-COMPAT**：dev profile server 出栈同时含 `Ping` 和 `ping`（兼容期，下版本删 `ping`）。
- [ ] **REGRESSION**：现有 server 集成测试 0 回归（含 13/13 chat、9/9 gift HTTP、12/12 gift WS 等）。
