### 公告弹出时机
```kotlin
onEnterRoom { room ->
    if (room.announcement.isNotBlank()) {
        val last = seenStore.get(room.id)
        if (last == null || now - last > 24.hours) {
            showPopup(room.announcement)
            seenStore.save(room.id, now)
        }
    }
}
```

## 五、Reviewer意见
### Round 1
- **状态**：❌ 未通过（2 个 HIGH 级别问题须修复后重新 Review）
- **审查意见**：
  - **HIGH-01 AnnouncementSeenStore 未注册为 Application 级单例**：`RoomViewModel` 构造参数 `announcementSeenStore` 使用默认值 `InMemoryAnnouncementSeenStore()`，而 `AppContainer` 未将其注册为 Application 级单例（对比 `kickCooldownStore` 已正确注册）。每次用户退出房间后 ViewModel 被销毁并重建，历史已读记录全部丢失，AN43-02（24h 内再进房不弹窗）在生产路径中完全失效。
  - **HIGH-02 AN43-07 测试内容与验收要求完全错位**：标注为 AN43-07 的测试方法实际测试的是 "RoomInfoUpdated 更新 title"（AN43-06 的变体），而非 TDS 要求的 "Owner 👑 / Admin 🛡️ / member 无徽章"。`RoleBadge` 三种角色渲染逻辑无任何测试覆盖。
- **改进建议**：
  1. 在 `AppContainer` 新增 `val announcementSeenStore: AnnouncementSeenStore = InMemoryAnnouncementSeenStore()` 字段，并在 `RoomViewModel.Factory` 中注入（参考 `kickCooldownStore` 模式）。
  2. 将现有标注为 AN43-07 的测试重命名为 AN43-06c，并新增真正的 AN43-07 测试：验证 `Role.fromString()` 三种映射（owner/admin/unknown→Member）及 `AdminChanged` 触发 `audienceState` 角色字段正确更新。
  3. LOW（可延后）：`AnnouncementIcon` 改用 `IconButton` 包装以满足 48dp 触摸目标；`RoleBadge` 后续迭代改用矢量图标替换 emoji。
