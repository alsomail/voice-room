# T-10004 — Admin Server 房间列表接口

## 接口规格

```
GET /api/v1/admin/rooms
Authorization: Bearer <admin-jwt>
```

### 查询参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|-------|------|
| page | u32? | 1 | 页码（>= 1） |
| page_size | u32? | 20 | 每页数量（1~100） |
| status | string? | — | 过滤状态：`active` / `closed` |
| keyword | string? | — | 按 title 关键词过滤（大小写不敏感） |

### 成功响应（200）

```json
{
  "code": 0,
  "message": "ok",
  "request_id": "xxx",
  "data": {
    "total": 42,
    "page": 1,
    "page_size": 20,
    "items": [
      {
        "id": "uuid",
        "title": "Room Title",
        "status": "active",
        "room_type": "normal",
        "member_count": 5,
        "max_members": 50,
        "owner_id": "uuid",
        "owner_nickname": "UserNick",
        "owner_avatar": "https://...",
        "created_at": "2024-01-01T00:00:00Z"
      }
    ]
  }
}
```

### 错误码

| HTTP | code | 说明 |
|------|------|------|
| 401 | 40101 | 无 Token / Token 无效 / C端 JWT |
| 401 | 40102 | Token 已过期 |
| 403 | 40301 | 角色无 RoomRead 权限（如 finance） |
| 400 | 40003 | 参数校验失败（page/page_size/status 非法） |

## SQL 设计

```sql
-- Count
SELECT COUNT(*)
FROM rooms r
JOIN users u ON r.owner_id = u.id
WHERE r.deleted_at IS NULL
  AND ($1::text IS NULL OR r.status = $1)
  AND ($2::text IS NULL OR LOWER(r.title) LIKE '%' || LOWER($2) || '%');

-- List (sorted by created_at DESC)
SELECT r.id, r.title, r.status, r.room_type, r.member_count, r.max_members,
       r.owner_id, u.nickname AS owner_nickname, u.avatar AS owner_avatar,
       r.created_at
FROM rooms r
JOIN users u ON r.owner_id = u.id
WHERE r.deleted_at IS NULL
  AND ($1::text IS NULL OR r.status = $1)
  AND ($2::text IS NULL OR LOWER(r.title) LIKE '%' || LOWER($2) || '%')
ORDER BY r.created_at DESC
LIMIT $3 OFFSET $4;
```

### 一、安全性

#### 1.1 Admin JWT 鉴权（✅ 正确）

| 场景 | 预期 | 实际 |
|------|------|------|
| 无 token | 401/40101 | ✅ `AdminAuthContext::from_request_parts` 缺失 header → `Unauthorized` |
| C 端 JWT（iss="voiceroom"）| 401/40101 | ✅ `decode_token(..., "voiceroom-admin")` 通过 `set_issuer` 校验 iss，C 端 token 被 jsonwebtoken 库在校验阶段直接拒绝 → `Unauthorized` |
| 过期 JWT | 401/40102 | ✅ `ErrorKind::ExpiredSignature` → `AppError::TokenExpired` |
| 签名无效 | 401/40101 | ✅ decode 失败 → `AppError::Unauthorized` |
| sub 非 UUID 格式 | 401/40101 | ✅ `Uuid::parse_str` 失败 → `AppError::Unauthorized` |

关键路径：`shared/jwt/token.rs::decode_token` 使用 `Validation::set_issuer` 将 iss 校验委托给 jsonwebtoken 库，**在签名校验阶段同步完成**，不存在 iss 绕过风险。

#### 1.2 RBAC 权限检查（✅ 正确）

- RBAC 检查位于 **controller 层**（`controller.rs:30`），在 service 调用之前执行，符合分层职责要求。
- `finance` 角色：`has_permission(RoomRead) = false` → `AppError::Forbidden` → HTTP 403/40301。
- `super_admin / operator / cs` 均已验证有 `RoomRead` 权限（集成测试 E-04, `rooms_super_admin_can_access`, `rooms_cs_can_access`）。

### 三、SQL 安全

- `count_rooms` 和 `find_rooms` 均使用 **SQLx 参数化绑定**（`$1~$4`）。
- status、keyword、limit、offset 全部通过 `.bind()` 传入，**无 SQL 注入风险**。

#### 🟡 MEDIUM 注意：keyword 中的 LIKE 通配符行为不一致

**问题**：PostgreSQL SQL 中 keyword 参数嵌入 `LIKE '%' || LOWER($2) || '%'`，PostgreSQL 会将 `$2` 中的 `%` 和 `_` 解释为 LIKE 通配符。而 `FakeAdminRoomRepository` 使用 Rust `String::contains()` 进行字面量匹配，两者行为不一致。

**示例**：keyword=`"test_room"` → Postgres 将 `_` 视为"任意单字符"，而 Fake 视为字面量下划线。

**影响**：不影响常规 alphanumeric 关键词搜索；若用户传入含 `%`/`_` 的 keyword，Postgres 行为与预期（字面量包含搜索）不符。  

**建议修复**（下次迭代）：在 service 层对 keyword 做 LIKE 特殊字符转义，或在 SQL 层使用 `replace($2, '%', '\%')` 等方式。
