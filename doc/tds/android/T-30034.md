### AnalyticsPort 接口
```kotlin
interface AnalyticsPort {
    fun track(event: String, properties: Map<String, Any?> = emptyMap())
    fun setUser(userId: String?, traits: Map<String, Any?> = emptyMap())
    fun captureException(throwable: Throwable, extras: Map<String, Any?> = emptyMap())
    fun setConsent(mode: ConsentMode)  // All / CrashOnly / None
}

enum class ConsentMode { All, CrashOnly, None }
```

### SentryAnalytics 实现关键点
```kotlin
class SentryAnalytics @Inject constructor(
    @ApplicationContext ctx: Context,
    private val filter: SensitiveFilter
) : AnalyticsPort {
    init {
        SentryAndroid.init(ctx) { opts ->
            opts.dsn = BuildConfig.SENTRY_DSN
            opts.environment = BuildConfig.BUILD_TYPE // dev / staging / prod
            opts.beforeSend = BeforeSendCallback { evt, _ ->
                filter.scrub(evt)
                evt
            }
            opts.tracesSampleRate = 0.1
            opts.isEnableAnrDetection = true
            opts.isAttachThreads = true
        }
    }
    override fun captureException(t: Throwable, extras: Map<String, Any?>) {
        Sentry.captureException(filter.scrubThrowable(t)) { scope ->
            extras.forEach { (k, v) -> scope.setExtra(k, v?.toString() ?: "null") }
        }
    }
    // track/setUser：此类不负责业务事件上报，仅转发给 EventReportClient（T-30035）
    // MVP 暂让 SentryAnalytics 作为单一 Port，内部组合 EventReportClient
    // ...
}
```

## 五、Reviewer意见
### Round 1
- **状态**：✅ 通过
- **审查意见**：
  - CRITICAL 0 项。防腐层架构完整，`AnalyticsPort` 接口设计规范，`SensitiveFilter` 脱敏逻辑正确（phone/JWT），`BuildConfig.SENTRY_DSN` 注入合规，静态检查脚本 A34-01 通过。
  - **[HIGH-01]** `io.sentry:sentry-android` 未作为 Gradle 依赖添加，`DefaultSentryHub` 所有 Sentry SDK 调用均为 stub（实际只写 `Log.e()`）；TDS §4 已明确标注 MVP 阶段可接受，激活步骤清晰。
  - **[HIGH-02]** `AppContainer.fromBuildConfig()` 注入 `NoopAnalytics()` 而非 `SentryAnalytics`；SDK 激活时需同步切换，属 MVP 有意选择。
  - **[MEDIUM-01]** `SensitiveFilter.scrubThrowable()` 将所有异常包装为 `RuntimeException`，导致 Sentry 异常类型分组失效；建议接入 SDK 前改为在 `beforeSend` callback 内脱敏 message，保留原始异常类型。
  - **[MEDIUM-02]** `SentryAnalytics.setUser()` 未检查 `ConsentMode.None`，None 模式下仍写入用户身份；建议加守卫：`if (currentConsent == ConsentMode.None) return`。
  - **[LOW-01]** `check_no_sentry_imports.sh` 的 `--exclude-dir="impl"` 过宽，建议改为 path-level 过滤。
- **改进建议**：
  - T-30035 或独立 Task 中完成 Sentry SDK 依赖添加 + `DefaultSentryHub` 激活 + `AppContainer` 切换为 `SentryAnalytics`
  - `scrubThrowable` 类型丢失问题在正式接入 Sentry 前修复
  - `setUser` ConsentMode.None 守卫可在本轮或下一 Task 补充
