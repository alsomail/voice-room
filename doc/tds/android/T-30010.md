### §2.3 WS 消息格式（来自 protocol.md）

发送：
```json
{"type":"JoinRoom","roomId":"xxx","msgId":"uuid"}
{"type":"LeaveRoom","roomId":"xxx"}
{"type":"SendMessage","roomId":"xxx","content":"hello","msgId":"uuid"}
```

接收（Server → Client）：
| type | 处理 |
|------|------|
| `RoomSnapshot` | 初始化 RoomUiState（micSlots/onlineCount/roomName） |
| `UserJoined` | onlineCount++ |
| `UserLeft` | onlineCount-- |
| `MicTaken` | 更新 micSlots 对应 slot |
| `MicLeft` | 清空 micSlots 对应 slot |
| `MessageReceived` | 追加 chatMessages，去重 msgId |
| `RoomClosed` | 发出 NavigateBack UiEffect |

```kotlin
    val events: Flow<RoomEvent> = _events.receiveAsFlow()

    fun joinRoom(roomId: String) { ... }
    fun leaveRoom() { ... }
    fun sendMessage(content: String) { ... }
    override fun onCleared() { leaveRoom() }
}
```

### Round 1

#### HIGH-1｜CancellationException 被 `catch (e: Exception)` 吞噬（`joinRoom()`）

**文件**：`feature/room/RoomViewModel.kt` 第 88 行

`joinRoom()` 内 `viewModelScope.launch { try { ... } catch (e: Exception) { ... } }` 中，`CancellationException` 是 `Exception` 的子类，会被此 catch 块拦截并将 UI 状态错误地设为 `RoomViewState.Error`，同时吞噬协程取消信号，导致父 scope 无法正确感知子协程终止。

**修复方案**：

```kotlin
} catch (e: CancellationException) {
    throw e   // 保持协程取消语义，必须 rethrow
} catch (e: Exception) {
    _uiState.value = RoomViewState.Error(e.message ?: "Unknown error")
}
```

---

#### HIGH-2｜`seenMsgIds` 无界增长，缺少 TODO 注释

**文件**：`feature/room/RoomViewModel.kt` 第 64 行

`seenMsgIds: MutableSet<String>` 随消息数量无限增长，长时间在线（高流量房间/数小时）时内存持续上升且不释放。MVP 阶段可接受，但**必须添加 TODO 注释**明确技术债。

**修复方案**：

```kotlin
// TODO(T-30010): seenMsgIds 无界增长，长时间在线时内存持续上升。
//                MVP 可接受；后续应改为 LRU 固定上限（如 maxSize=1000）或定期清理。
private val seenMsgIds = mutableSetOf<String>()
```

---

#### HIGH-3｜`onCleared()` 可见性提升为 `public`，破坏封装

**文件**：`feature/room/RoomViewModel.kt` 第 122 行

`ViewModel.onCleared()` 基类为 `protected`，是框架通过 `ViewModelStore.clear()` 调用的内部钩子，不应对外暴露。`public override` 使外部（Activity/Fragment/其他 ViewModel）可以错误地主动调用，且测试因此直接调用 `viewModel.onCleared()`（VM-11），耦合了框架内部行为。

**修复方案**：

```kotlin
// 去掉 public 修饰符，恢复 protected
override fun onCleared() {
    super.onCleared()
    leaveRoom()
}
```

VM-11 测试改为通过 `ViewModelStore` 间接触发：

```kotlin
// VM-11 修正写法
val store = ViewModelStore()
val vm = ViewModelProvider(store, factory)[RoomViewModel::class.java]
store.clear()  // 触发 onCleared()，间接验证 leaveRoom() 被调用
assertTrue(fakeWsClient.state.value is WebSocketState.Disconnected)
```

#### MEDIUM-1｜Gson 为隐式传递依赖

**文件**：`app/build.gradle.kts`

Gson（`JsonParser`）在代码中直接使用，但 `build.gradle.kts` 未显式声明。当前通过 `converter-gson:2.11.0` 的传递依赖获取，若 Retrofit 升级调整依赖树，Gson 可能消失导致编译错误且难以追踪。

**建议**：在 `build.gradle.kts` 中补充直接依赖：

```kotlin
implementation("com.google.code.gson:gson:2.11.0")
```

#### MEDIUM-2｜`RoomSnapshot.toRoomUiState()` 扩展函数位置偏离单一职责

**文件**：`feature/room/RoomViewModel.kt` 第 226 行

Data 层到 UI 层的映射逻辑放在 ViewModel 文件中，违反单一职责原则。

**建议**：迁移至 `data/room/RoomSnapshotMapper.kt` 或 `feature/room/RoomUiStateMapper.kt`，便于独立测试和复用。

---

#### MEDIUM-3｜`StateFlow<WebSocketState>` 传递消息可能丢失重复内容

**文件**：`feature/room/RoomViewModel.kt` 第 131 行 / `core/ws/WebSocketState.kt`

`StateFlow` 具有 `distinctUntilChanged` 语义，若服务端短时间内推送两条内容完全相同的消息（`WebSocketState.Message(text)` data class equals 相等），第二条会被静默丢弃，`handleWsMessage` 永远不会处理。此为 T-30008 架构设计的已知权衡，建议在 T-30008 TDS 中记录，或后续将 WS 消息通道改为 `SharedFlow(replay=0)`。

---

#### LOW-1｜缺少 `@HiltViewModel` 注解

TDS §2.4 设计中包含 `@HiltViewModel`，实现中缺失。若当前不使用 Hilt，建议从 TDS 设计删除相关注解，避免误导。

#### LOW-2｜`Channel.UNLIMITED` 无 backpressure 边界

对于导航事件量级通常可接受；若 UI 长时间后台不消费，可改为 `Channel(capacity = 64)` 并在 `trySend` 失败时 log 警告。
