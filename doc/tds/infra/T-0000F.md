# TDS: 根 .env.example 修订 + 三档 profile 模板 (Task ID: T-0000F)

> **主设计**：[T-0000E](./T-0000E.md)（请先阅读其 §2.4 字段契约）

## 一、背景与目标
- 修复体检缺口 C1/C3/C13：`.env.example` 数据库密码 typo + 缺少三档 profile 示例。
- 输出 4 个 example 文件，使新人 `cp` 后改少量字段即可启动 E2E。

## 二、方案设计

### 涉及文件清单
| 文件路径 | 变更类型 | 说明 |
|---|---|---|
| `.env.example` | 修改 | 修复 `app_server_pwd → app_server_pass`；新增 `E2E_PROFILE`、`E2E_ALLOW_WRITES`、`VITE_ADMIN_API_BASE_URL`（虽然 Web 用，但同时在 root example 列出便于发现） |
| `tests/scripts/env/.env.local.example` | 新增 | local 全字段（指 127.0.0.1）+ Midscene 占位 |
| `tests/scripts/env/.env.staging.example` | 新增 | staging 占位 URL（`https://stg-*.example.com`），DB/Redis 字段留空（远端通常不直连） |
| `tests/scripts/env/.env.prod.example` | 新增 | prod 占位 URL；`E2E_ALLOW_WRITES=0` 默认；DB/Redis 留空；明确注释「仅 read-only smoke」 |
| `.gitignore` | 修改 | 确保 `.env`、`tests/scripts/env/.env.local`、`.env.staging`、`.env.prod` 全部忽略 |

### 关键约束
- 三档 example 字段**完全对齐** T-0000E §2.4.1 的字段表，禁止个性化扩字段。
- `prod` example 顶部必须有醒目注释：「⚠️ 此环境仅允许 @prod-safe 标签的只读用例」。

## 三、TDD 验收用例
- [ ] `diff <(grep -oE '^[A-Z_]+=' .env.example | sort) <(grep -oE '^[A-Z_]+=' tests/scripts/env/.env.local.example | sort)` 字段集合一致
- [ ] `git status` 在 `cp .env.example .env` 后不显示新追踪的 `.env`
- [ ] 三档 example 中所有 URL 均为合法 URL（可用 `node -e "new URL(...)"` 校验）
- [ ] `prod.example` 中 `E2E_ALLOW_WRITES=0`
