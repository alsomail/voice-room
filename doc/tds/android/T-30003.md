# TDS: JWT 拦截器 (Task ID: T-30003)

## 一、背景与目标
- **关联需求**：所有需要认证的 C 端 API
- **本 Task 的目标**：实现 OkHttp AuthInterceptor，自动为每个请求附加 JWT token，并处理 401 未授权响应（清除 token + 跳转登录页）。
- **前置依赖**：T-00004 (JWT 中间件), T-30002

## 二、方案设计
### 核心数据流
```
OkHttp Request → AuthInterceptor → TokenManager.getToken()
  → 有 token？→ 添加 Authorization: Bearer xxx Header → 发送请求
  → 收到 401？→ TokenManager.clearToken() → 跳转登录页
  → 无 token → 直接发送请求（公开接口如 send-code/login）
```

### 涉及文件清单
| 文件路径 | 变更类型 | 说明 |
|----------|----------|------|
| `app/android/app/src/main/java/com/voiceroom/core/network/AuthInterceptor.kt` | 新增 | OkHttp Interceptor |
| `app/android/app/src/main/java/com/voiceroom/core/network/NetworkModule.kt` | 修改 | 注册 AuthInterceptor |
| `app/android/app/src/main/java/com/voiceroom/core/network/UnauthorizedHandler.kt` | 新增 | 401 处理逻辑（清除 token + 跳转登录） |

### 接口 / 信令定义
无新增接口。本 Task 为基础设施层，拦截所有 HTTP 请求自动注入 `Authorization` Header。

## 三、TDD 验收用例
- [ ] 正向：每个请求自动携带 `Authorization: Bearer xxx` Header
- [ ] 正向：无 token 时不附加 Header（公开接口如 send-code/login）
- [ ] 异常：收到 401 响应 → 清除 DataStore token → 导航到 LoginScreen
- [ ] 异常：收到 401 时不应该无限重试（避免循环）
