# TDS: 余额不足引导弹窗 (Task ID: T-30032)

## 一、背景与目标
- **关联需求**：E-07 余额不足转化 → 未来充值入口 ([设计文档](../../design/android/T-30032.md))
- **本 Task 的目标**：点送出时服务端返回 40290 或本地判 balance<total 时弹 `InsufficientBalanceDialog`，引导进入 WalletScreen。

### Dialog 设计
- 标题："钻石不足" / "رصيدك غير كافٍ"
- 正文：
  - "当前余额：💎 {current}"
  - "所需：💎 {required}"
  - "差：💎 {required - current}"
- 按钮：[取消] [去充值 GoldButton]
- `dismissOnClickOutside=false`, `dismissOnBackPress=true`
- 点"去充值" → `navController.navigate("wallet")` + 关闭礼物面板
- 点"取消" → 关闭 Dialog，礼物面板保留选中态

### 触发场景
1. Server 返回 `code=40290` (INSUFFICIENT_BALANCE)
2. 本地按钮点送出前已判断 balance<total（按钮本应 disabled，保守双重拦截）

### testTag

## 五、Reviewer意见
### Round 1
- **状态**：未通过
- **审查意见**：
  - **[HIGH] onGoToWallet 回调二义性**：`GiftPanelBottomSheet` 的 `onGoToWallet` 参数同时被传入 `InsufficientBalanceDialog`（按钮点击回调）和 `LaunchedEffect` 的 `NavigateToWallet` 处理器（导航回调），两种语义混用。若 RoomScreen 传 `{ vm.onGoToWallet() }`：Dialog 按钮触发 → VM emit 事件 → LaunchedEffect 再次调用 `vm.onGoToWallet()` → 第二次 emit → 潜在无限循环；若传导航 callback：VM 状态管理完全被绕过，`showInsufficientDialog` 和 `selectedGiftId` 均不会被清除。根本原因是"触发 VM 动作"与"执行导航副作用"被混入同一参数。
  - **[MEDIUM] I32-05 测试覆盖缺口**：TDS 验收用例 I32-05 要求验证"外部点击不关闭"，但测试文件中 I32-05 实际测试的是"dismiss 后选中态保留"（与 I32-03 高度重叠），`dismissOnClickOutside=false` 行为无任何 Instrumented UI 测试覆盖。
  - **[LOW-1] testTag 命名不一致**：TDS §testTag 写 `btn_goto_wallet`，Review 规格写 `btn_go_to_wallet`，建议统一避免 E2E 误读。
  - **[LOW-2] KDoc @param onSelectRecipient 重复**：`GiftPanelBottomSheet.kt` 第 72/76 行重复定义，属复制遗留。
- **改进建议**：
  1. **[必须]** 将 `onGoToWallet` 拆分为 `onGoToWalletClick`（Dialog 按钮 → `vm.onGoToWallet()`）和 `onNavigateToWallet`（LaunchedEffect NavigateToWallet 事件 → 实际导航），明确分离 ViewModel 调用与导航副作用两条职责。
  2. **[建议]** 补充 Instrumented 测试覆盖 TDS I32-05 "外部点击不关闭"场景（模拟点击 Dialog 外部，断言 Dialog 仍可见）。
  3. **[可选]** 统一 TDS §testTag 与 Review 规格中的 `btn_goto_wallet` 拼写；删除 KDoc 中重复的 `@param onSelectRecipient`。
