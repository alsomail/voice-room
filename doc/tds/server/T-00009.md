## 接口规范

### `GET /api/v1/rooms/:id`

**描述**：获取指定房间的详情（含房主信息、麦位列表）。无需鉴权。

**路径参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| id | UUID string | 房间 ID |

**成功响应** `200 OK`

```json
{
  "code": 0,
  "request_id": "xxx",
  "data": {
    "room_id": "uuid-string",
    "title": "房间标题",
    "room_type": "normal",
    "member_count": 5,
    "max_members": 50,
    "owner": {
      "user_id": "uuid-string",
      "nickname": "用户昵称",
      "avatar": "https://example.com/avatar.jpg"
    },
    "mic_slots": [],
    "created_at": "2024-01-01T00:00:00+00:00"
  }
}
```

**错误响应**

| 场景 | HTTP 状态码 | code |
|------|------------|------|
| id 不是合法 UUID | 400 | 40003 |
| 房间不存在 / closed / soft-deleted | 404 | 40400 |

---

## 设计决策

### UUID 解析策略

使用 `Path<String>` + 手动 `Uuid::parse_str`，而非 `Path<Uuid>`，原因：
- Axum 提取器对无效 UUID 会返回 422，而非项目统一的 400 (code=40003)
- 手动解析可精确控制错误类型为 `AppError::ValidationError`

### 房间可见性过滤

仅 `status = 'active' AND deleted_at IS NULL` 的房间对外可见：
- `closed` 状态 → 404
- `deleted_at IS NOT NULL` (soft-delete) → 404

### mic_slots MVP 实现

固定返回空数组 `[]`，后续麦位功能（T-00012+）实现后再填充。

### 认证

公开接口，无需 `Authorization` header。
