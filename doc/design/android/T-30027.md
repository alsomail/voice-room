## 需求描述

新增 `WalletScreen` 钱包页，展示用户钻石余额与交易流水。入口：个人中心的"钻石余额"卡片点击。

## 交互逻辑

| 操作 | 行为 |
|------|------|
| 进入页面 | 调用 `GET /wallet/balance` + `GET /wallet/transactions?page=1` |
| 点击"充值" | Toast："充值功能即将上线" |
| 下拉刷新 | 重新拉取余额 + 流水第一页 |
| 滚动到底 | 加载下一页（分页 20 条/页） |
| WS `BalanceUpdated` | 余额卡片数字动画变化 + 顶部插入新流水 |

## 异常流程

| 场景 | 处理 |
|------|------|
| 网络断开 | 显示 Room 本地缓存；Snackbar"离线模式，可能非最新" |
| API 失败 | Skeleton → 错误占位图 + 重试按钮 |
| 流水为空 | 插画 + "还没有交易记录" |

## 通用组件

- `GoldButton`（T-30018）
- `AvatarWithFrame`（T-30018）
- 新组件 `BalanceCard`：金色渐变卡片（余额 + 操作按钮），未来贵族/充值页复用
- 新组件 `TransactionItem`：流水项（图标/标题/金额/时间）

## TDD 验收标准

1. 进入页面后余额显示为 API 返回值（金色大字）
2. 流水列表分页加载，滚动到底自动加载下一页
3. 下拉刷新重新拉取
4. 收到 WS `BalanceUpdated` 事件后：余额数字更新 + 顶部插入新流水
5. 网络断开时显示本地缓存 + Snackbar 提示
6. 点击充值按钮显示 Toast "充值功能即将上线"
7. 流水项点击暂无行为（Phase 1 MVP）
