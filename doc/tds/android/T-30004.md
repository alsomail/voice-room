# TDS: 用户信息 Repository (Task ID: T-30004)

## 一、背景与目标
- **关联需求**：product.md - 用户资料
- **本 Task 的目标**：封装用户信息的获取、缓存和订阅。通过 Retrofit 拉取 `/users/me`，使用 Room Database 本地缓存，并通过 Flow 提供实时数据订阅能力。
- **前置依赖**：T-00005 (users/me 接口), T-30003

## 二、方案设计
### 核心数据流
```
登录成功 → UserRepository.fetchMe() → Retrofit GET /users/me → UserEntity → Room Database 缓存
其他页面 → UserRepository.observeMe() → Room DAO Flow → 实时订阅用户信息变更
```

### 接口 / 信令定义
- `GET /users/me` — 获取当前登录用户信息，需要 JWT 认证

### Room Entity 设计
```kotlin
@Entity(tableName = "users")
data class UserEntity(
    @PrimaryKey val id: String,
    val phone: String,
    val nickname: String,
    val avatar: String?,
    val coinBalance: Long,
    val vipLevel: Int,
    val createdAt: String,
    val updatedAt: Long = System.currentTimeMillis()
)
```
