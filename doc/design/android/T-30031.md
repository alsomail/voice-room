# T-30031: 送礼特效播放器 + 弹幕样式

> **归属端**: Android
> **模块**: UI/Gift/Animation
> **前置依赖**: T-30030 (SendGift 客户端)
> **参考**: [competitors.md §A.1 特效分层](../../product/competitors.md)

---

## 需求描述

实现接收到 `GiftReceived` WS 广播后的三层特效：
- **L1 聊天弹幕**（T1/T2 礼物）：仅聊天区气泡
- **L2 麦位光圈**（T3 礼物）：接收者麦位金色闪烁 2s
- **L3 全屏 Lottie**（T4/T5 礼物）：全屏动画覆盖层 5-8s

## 架构

```
GiftReceivedEvent (WS)
   │
   ├─ effect_level == L1 → GiftBubbleMessage 追加到聊天区
   ├─ effect_level == L2 → GiftBubbleMessage + MicSlotGlowEffect(receiver_id) 2s
   └─ effect_level == L3 → GiftBubbleMessage + FullScreenLottieOverlay 5-8s
```

## L1 弹幕消息样式

```
┌────────────────────────────────┐
│  [金色昵称] 送给 [金色昵称] 🐪 x66  │  ← 聊天区一行
│  (左对齐 + 礼物小图标 16dp)       │
└────────────────────────────────┘
```

- 发送者昵称：金色 `#D4AF37`, bold
- "送给" 文字：白色
- 接收者昵称：金色
- 礼物图标：16dp，从 `gift.icon_url` Coil 加载
- 数量：白色 `x66` 格式
- 连击聚合消息数量仍显示聚合后的 count

## L2 麦位光圈特效

- 触发：接收者对应麦位（含主麦）
- 动画：金色圆形光圈从 1.0 缩放到 1.3 + 渐隐，循环 3 次，总时长 2s
- 实现：`Modifier.drawBehind` 绘制径向渐变 + `animateFloatAsState`
- 若接收者已下麦：降级为 L1（仅弹幕）

## L3 全屏 Lottie 覆盖层

- 覆盖整个 RoomScreen（`Box` 顶层 z=100）
- 背景半透明黑 `rgba(0,0,0,0.3)`
- 中央播放 Lottie 动画（使用 `airbnb/lottie-compose`）
- 动画 URL 从 `gift.animation_url` 下载（缓存到 Coil DiskCache）
- 持续时长：T4 = 5s / T5 = 8s
- 用户交互：
  - 点击背景 → 跳过（立即结束）
  - 动画期间房间其他交互不中断（可继续聊天、上下麦）
  - 动画覆盖层不拦截 Lottie 区域外的点击

## 补偿渲染规则

- WS 重连后调用 `GET /gifts/recent?room_id=` 拉最近 20 条
- 历史消息**只渲染 L1 弹幕**，不回放 L2/L3 动画（避免信息轰炸）

## 异常流程

| 场景 | 处理 |
|------|------|
| Lottie 加载失败 | 降级为 L2 光圈特效 |
| 并发多个 L3 动画 | 排队播放（队列深度 3，超出丢弃） |
| 动画播放中用户退出房间 | 取消动画 + 释放资源 |

## 通用组件

- `GiftBubbleMessage`（新）：聊天区礼物弹幕样式
- `MicSlotGlowModifier`（新）：麦位光圈 Modifier，可复用到其他场景
- `FullScreenLottieOverlay`（新）：全屏 Lottie 覆盖层容器，未来贵族进场/节日特效复用

## TDD 验收标准

1. T1/T2 礼物收到广播后仅弹出 GiftBubbleMessage
2. T3 礼物额外在接收者麦位播放 2s 金色光圈
3. T4/T5 礼物播放全屏 Lottie 覆盖层 5/8s
4. 点击 L3 覆盖层背景立即结束动画
5. 连击礼物仅播一次动画（监听 msg_id 去重）
6. WS 重连补偿消息只渲染 L1，不回放 L2/L3
7. Lottie 加载失败降级为 L2
8. 动画期间聊天输入、上下麦按钮仍可交互
