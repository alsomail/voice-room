### 表单 State
```kotlin
data class CreateRoomUiState(
    val title: String = "",
    val coverUrl: String = "",           // 从 T-30037 返回
    val category: RoomCategory = CHAT,   // 枚举
    val announcement: String = "",
    val passwordEnabled: Boolean = false,
    val password: String = "",           // 6 位数字
    val submitting: Boolean = false,
    val error: String? = null
) {
    val canSubmit: Boolean
        get() = title.isNotBlank() && title.length <= 30
             && announcement.length <= 200
             && (!passwordEnabled || password.matches(Regex("\\d{6}")))
             && coverUrl.isNotEmpty()
             && !submitting
}
```

### 提交
```kotlin
fun submit() {
    viewModelScope.launch {
        _state.update { it.copy(submitting = true) }
        val req = CreateRoomRequest(
            title = state.title,
            room_type = if (state.passwordEnabled) "password" else "normal",
            cover_url = state.coverUrl,
            category = state.category.key,
            announcement = state.announcement.ifBlank { null },
            password = state.password.ifBlank { null }
        )
        runCatching { roomApi.createRoom(req) }
            .onSuccess { navigateToRoom(it.room_id) }
            .onFailure { handleError(it) }
    }
}
```

### Round 1
- **状态**：❌ 未通过（BLOCKED）
- **审查意见**：
  - **[HIGH-01] `IRoomRepository` 接口未更新，新字段静默丢失**：`IRoomRepository.createRoom()` 仍只有 `(title, type, password)` 三参数；`submit()` 中 `coverUrl`/`category`/`announcement` 完全未传入仓库层，后端永远收不到这三个字段，T-30036 核心功能形同虚设。`CreateRoomRequest` 已正确添加字段，但接口不匹配导致字段被丢弃。
  - **[HIGH-02] `CreateRoomScreen` 缺少封面选择 UI，提交按钮永久禁用**：`onSelectCover` 回调参数声明了但在 Composable 内部从未调用，屏幕上无任何 CoverPreview 组件，`viewModel.updateCoverUrl()` 无法被触发，`coverUrl` 永为空，`canSubmit` 中 `coverUrl.isNotEmpty()` 永为 `false`，提交按钮永久禁用，整个新版表单无法提交。
  - **[MEDIUM-01] `canSubmit` 与 `validate()` 标题长度计算不一致**：`canSubmit` 用 `title.length`（UTF-16 code units），旧版用 `codePointCount()`（Unicode 字符）；含 emoji 时行为不同，应统一为 `codePointCount`。
  - **[LOW-01] `RoomCategory` 硬编码中文标签**：TDS 设计为 `@StringRes val labelRes: Int`，实现改为 `val label: String`，不支持 i18n。
- **改进建议**：
  1. 更新 `IRoomRepository.createRoom()` 接口加入 `coverUrl/category/announcement` 参数，同步更新 `RetrofitRoomRepository` 及所有测试 Fake；
  2. 在 `CreateRoomScreen` 增加封面选择区域（CoverPreview 点击 → `onSelectCover()` → `updateCoverUrl()`）；
  3. `canSubmit` 中改用 `title.codePointCount(0, title.length) <= 30`；
  4. （可选）`RoomCategory` 改回 `@StringRes` 支持 i18n。

### Round 6 Follow-up（BUG-GOVERNANCE-FORM-VALIDATE，2026-04-30）

**结论：🟢 修复**

- **根因**：`CreateRoomBottomSheet.kt` 内 [创建] 按钮 `enabled = !isLoading`，
  没有检查 `title.isNotBlank()`，用户在空标题下仍可点击提交，触发 400。
  历史上 `CreateRoomFormState.canSubmit` 已实现该校验，但 BottomSheet 路径
  使用本地 Compose state（`var title by remember`），与 formState 解耦后未同步该不变量。
- **修复**：
  - `CreateRoomBottomSheet.kt` 提交按钮 `enabled` 表达式扩展为
    `!isLoading && isFormValid`，其中 `isFormValid` 包含 `title.isNotBlank()`、
    `trim().codePointCount` 在 `1..MAX_TITLE_LENGTH`，并在 `selectedType=="password"`
    时强制 `password.isNotBlank()`。
  - `CreateRoomFormState.canSubmit` 既有不变量保持不变。
- **测试**：新增 `CreateRoomFormValidationTest`（7 个用例），覆盖空标题、纯空白、
  密码房空密码、加载中等路径，本地 `:app:testLocalDebugUnitTest --tests CreateRoomFormValidationTest` 全绿。
