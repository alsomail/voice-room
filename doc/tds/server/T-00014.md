# TDS: App Server 麦位上麦接口 (Task ID: T-00014)

## 一、背景与目标

- **关联需求**：语音房间核心功能——麦位管理（T-00012 进入房间的延伸操作）
- **前置依赖**：T-00012（进入房间逻辑 ✅ Done）、T-00013（离开房间逻辑 ✅ Done）
- **本 Task 的目标**：处理 `TakeMic` WS 信令，通过内存写锁保证并发安全，校验禁麦状态、麦位占用情况，成功后原子占用麦位并广播 `MicTaken` 事件

## 二、方案设计

### 信令格式（对齐 protocol.md）

**请求（客户端 → 服务端）**：
```json
{
  "type": "TakeMic",
  "msg_id": "uuid",
  "payload": { "mic_index": 0 }
}
```
> `mic_index` 取值范围：0-8（共 9 个麦位）

**成功响应（服务端 → 请求方）**：
```json
{
  "type": "TakeMicResult",
  "msg_id": "uuid",
  "code": 0,
  "payload": { "mic_index": 0 },
  "timestamp": 1713312000
}
```

**错误码**：
| code | 含义 | 触发条件 |
|------|------|---------|
| `40002` | 参数非法 | payload 缺失，或 mic_index 不在 0-8 范围 |
| `40301` | 用户已在麦上 | 该 user_id 已占用其他麦位 |
| `40302` | 用户被禁麦 | user_id 在 `room_state.banned_mics` 中 |
| `40303` | 麦位已被占用 | `mic_slots[mic_index]` 为 `Some(_)` |
| `40400` | 用户不在房间 | `registry.get_room_id(connection_id)` 返回 None |

**广播（服务端 → 房间所有成员，含请求方）**：
```json
{
  "type": "MicTaken",
  "payload": { "mic_index": 0, "user_id": "uuid" },
  "timestamp": 1713312000
}
```

### 核心逻辑：take_mic_slot（原子操作）

```rust
/// 原子检查并占用麦位。持写锁期间完成"检查 + 设置"，保证并发安全。
pub fn take_mic_slot(&self, mic_index: usize, user_id: Uuid) -> Result<(), TakeMicError> {
    let mut slots = self.mic_slots.write().unwrap_or_else(|e| e.into_inner());
    // 检查用户是否已在任意麦位
    if slots.iter().any(|s| *s == Some(user_id)) {
        return Err(TakeMicError::AlreadyOnMic);
    }
    // 检查目标麦位是否被占用
    if slots[mic_index].is_some() {
        return Err(TakeMicError::SlotOccupied);
    }
    slots[mic_index] = Some(user_id);
    Ok(())
}

#[derive(Debug, PartialEq)]
pub enum TakeMicError {
    AlreadyOnMic,
    SlotOccupied,
}
```

**关键**：`take_mic_slot` 是同步函数，写锁在函数返回时立即释放，不跨越任何 `.await` 点。

### handle_take_mic 处理流程（7 步）

| 步骤 | 操作 | 失败行为 |
|------|------|---------|
| 1 | 解析 `payload.mic_index`，校验 0-8 范围 | → code:40002 |
| 2 | `registry.get_room_id(connection_id)` | → code:40400 |
| 3 | `room_manager.get_room(room_id)` | → code:40400（防御性） |
| 4 | `room_state.banned_mics.contains(&user_id)` | → code:40302 |
| 5 | `room_state.take_mic_slot(mic_index, user_id)` | AlreadyOnMic→40301；SlotOccupied→40303 |
| 6 | 广播 `MicTaken` 给 `registry.get_connections_in_room(room_id)` | 失败不阻断 |
| 7 | 返回 `TakeMicResult { code:0, payload:{ mic_index } }` | — |

### 并发安全分析

`std::sync::RwLock` 写锁在单节点进程内保证原子性，满足 MVP 并发抢麦需求。
若未来扩展多节点，仅需替换 `take_mic_slot` 实现为 Redis SET NX，接口不变。
