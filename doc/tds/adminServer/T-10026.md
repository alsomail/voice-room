# TDS: 手动补单 / 退款 API (Task ID: T-10026)

## 一、背景与目标

- **关联需求**：[phase1_payment_billing.md §2.1 In Scope — Admin 手动补单](../../product/phase1_payment_billing.md)；[payment_api.md §9.7 Admin REST](../../protocol/payment_api.md#97-admin-restadmin-server)
- **本 Task 的目标**：为 super_admin 提供手动补单（FAILED→CREDITED）与退款（ACKED/CREDITED→REFUNDED）两个写操作接口；所有操作写审计日志、触发余额变更推送；双写幂等，禁止对终态 ACKED 订单重复补单。

## 二、方案设计

### 核心数据流

```
── 补单 (recredit) ──────────────────────────────────────────────

Web Admin (T-20031 弹窗)
  │
  │  POST /api/v1/admin/payments/orders/:id/recredit
  │  Body: { "reason": "客诉核实后补单 #4567" }
  ▼
[JWT 中间件] decode_token(iss="voiceroom-admin")
  │  失败 → 401
  ▼
AdminAuthContext — require_role(super_admin)
  │  非 super_admin → 403 (40301)
  ▼
recredit_handler(Path<Uuid>, Json<RecreditRequest>)
  ├─ reason 空字符串 → 400 (40003 VALIDATION_ERROR)
  ▼
PaymentAdminService::recredit_order(order_id, admin_id, reason)
  │
  BEGIN TRANSACTION
  ├─ SELECT * FROM payment_orders WHERE order_id=$1 FOR UPDATE
  │    不存在 → 404 (40904 ORDER_NOT_FOUND)，ROLLBACK
  ├─ if state IN (ACKED, CREDITED, REFUNDED)
  │    → 409 (40905 ORDER_ALREADY_FINALIZED)，ROLLBACK   ← 红线
  ├─ if state NOT IN (FAILED, CANCELLED)
  │    → 400 (只能补单 FAILED / CANCELLED 订单)，ROLLBACK
  ├─ 查 payment_skus WHERE sku_id = order.sku_id  取 diamonds
  ├─ UPDATE payment_orders
  │     SET state = 'CREDITED',
  │         credited_at = now(),
  │         state_history = state_history || [{state:'CREDITED',ts,source:'admin_recredit'}]
  │     WHERE order_id = $1
  ├─ UPDATE users
  │     SET diamond_balance = diamond_balance + sku.diamonds
  │     WHERE id = order.user_id
  ├─ INSERT wallet_transactions (
  │     user_id, type='recharge', amount=sku.diamonds,
  │     balance_after=<new_balance>,
  │     source='admin_recredit',
  │     operator_id=admin_id, reason=reason
  │   )
  ├─ INSERT admin_logs (
  │     admin_id, action='order_recredit',
  │     target_type='payment_order', target_id=order_id,
  │     detail={reason, diamonds:sku.diamonds, sku_id:order.sku_id},
  │     ip=<from request>
  │   )
  COMMIT
  │
  ▼
EventPublisher::publish(admin:events {
  type: "balance_updated",
  payload: { user_id, new_balance, delta: sku.diamonds, reason: "admin_recredit" },
  admin_id, ts
})  — fire-and-forget（失败仅 warn）
  │
  App Server 订阅 → 推送 WS BalanceUpdated(reason='admin_recredit')
  │
  ▼
Response 200 { code:0, data:{ order_id, new_state:"CREDITED", diamonds_credited } }

── 退款 (refund) ────────────────────────────────────────────────

POST /api/v1/admin/payments/orders/:id/refund
  Body: { "reason": "..." }
  ▼
[JWT + super_admin]
  ▼
refund_handler → PaymentAdminService::refund_order(order_id, admin_id, reason)
  BEGIN TRANSACTION
  ├─ SELECT FOR UPDATE
  ├─ if state NOT IN (ACKED, CREDITED)
  │    → 400 (只允许对已入账订单退款)
  ├─ UPDATE payment_orders SET state='REFUNDED', failed_at=now(),
  │         state_history = state_history || [{state:'REFUNDED',ts,source:'admin_refund'}]
  ├─ UPDATE users SET diamond_balance = diamond_balance - sku.diamonds
  │     （允许余额为负；wallet_transactions.amount 为负值）
  ├─ INSERT wallet_transactions (source='admin_refund', amount=-sku.diamonds, ...)
  ├─ INSERT admin_logs (action='order_refund', ...)
  COMMIT
  │
  ▼
EventPublisher::publish(balance_updated, reason='admin_refund')
  ▼
Response 200 { code:0, data:{ order_id, new_state:"REFUNDED", diamonds_deducted } }
```

### 接口 / 信令定义

路径字面量、错误码权限定义详见 [payment_api.md §9.7](../../protocol/payment_api.md#97-admin-restadmin-server)。

**RecreditRequest / RefundRequest**：

```jsonc
{ "reason": "客诉处理，已人工核实 Google 收款记录 GPA.xxxx" }
```
- `reason`：必填，2–500 字符；空字符串 → 400 (40003)

**RecreditResponse**：

```jsonc
{
  "code": 0,
  "data": {
    "order_id": "uuid",
    "new_state": "CREDITED",
    "diamonds_credited": 600
  }
}
```

**错误码**（除通用鉴权码外，特殊于本 Task）：

| code | HTTP | 触发场景 |
|------|------|---------|
| `40905` | 409 | ORDER_ALREADY_FINALIZED — 已 ACKED/CREDITED/REFUNDED 禁止再补单 |
| `40904` | 404 | ORDER_NOT_FOUND |
| `40003` | 400 | VALIDATION_ERROR — reason 为空 |
| `40301` | 403 | Forbidden — 非 super_admin |

## 三、TDD 验收用例

### 正向用例
- [ ] **RC-01**：super_admin 对 FAILED 订单 recredit → 200，订单 state=CREDITED，余额+钻石数
- [ ] **RC-02**：recredit 成功后 `wallet_transactions` 新增一条 `source='admin_recredit'`
- [ ] **RC-03**：recredit 成功后 `admin_logs` 含 `operator_id` + `reason` + `detail.diamonds`
- [ ] **RC-04**：`state_history` 新增 `{state:"CREDITED", source:"admin_recredit"}` 节点
- [ ] **RC-05**：super_admin 对 ACKED 订单 refund → 200，state=REFUNDED，余额-钻石数（允许负值）
- [ ] **RC-06**：refund 成功后 `wallet_transactions` `amount` 字段为负数
- [ ] **RC-07**：Redis Pub/Sub `balance_updated` 事件被发布（`reason` 字段为 `admin_recredit` / `admin_refund`）

### 异常用例
- [ ] **RC-08**：role=`operator` 调用 recredit → 403 (40301)
- [ ] **RC-09**：role=`finance` 调用 refund → 403 (40301)
- [ ] **RC-10**：`reason=""` → 400 (40003)
- [ ] **RC-11**：对 ACKED 订单 recredit → 409 (40905 ORDER_ALREADY_FINALIZED)
- [ ] **RC-12**：order_id 不存在 → 404 (40904)
- [ ] **RC-13**：对 PENDING 状态订单 refund → 400（非终态不可退款）
- [ ] **RC-14**：DB 事务中途（admin_logs INSERT）异常 → 全量回滚，余额不变，订单 state 不变

### 幂等性验收
- [ ] **RC-15**：同 `order_id` 重复调用 recredit（并发 2 次）→ 只有一次成功（SELECT FOR UPDATE 保证），另一次返回 409

- [ ] **PROTO-1**（绑定表 #1）：集成测试断言 `POST /api/v1/admin/payments/orders/:id/recredit` 路径字面量完全一致；`recredit_handler` 被调用（tracing span）；`admin_logs.action` = `order_recredit`
- [ ] **PROTO-2**（绑定表 #2）：`refund_handler` 被调用；`wallet_transactions.source` = `admin_refund`；`amount` 字段为负数

## 五、Reviewer意见

> 状态枚举：🟢 通过 | 🔴 未通过

### Round 1
- **状态**：🔴 未通过（BLOCKED）
- **审查意见**：
  - **[P0-1] admin_service.rs 不存在**：`PaymentAdminService::recredit_order` / `refund_order` 核心原子事务逻辑完全缺失。`payment/mod.rs` 声明了 `pub mod admin_service;` 但源文件不存在（因 `modules/mod.rs` 未挂载 payment 模块，编译错误被掩盖）。
  - **[P0-2] recredit/refund handler 不存在**：`controller.rs` 文件缺失，`recredit_handler` / `refund_handler` 均未实现。
  - **[P0-3] 路由未注册**：`POST /api/v1/admin/payments/orders/:id/recredit` 与 `POST /api/v1/admin/payments/orders/:id/refund` 均未在 bootstrap 中注册，端点不可达。
  - **[P0-4] 原子事务缺失（红线）**：TDS 要求 `SELECT FOR UPDATE + UPDATE payment_orders + UPDATE users + INSERT wallet_transactions + INSERT admin_logs` 在同一 `BEGIN … COMMIT` 块内完成，但实现完全缺失。补单不可重入验证（ACKED/CREDITED 状态返回 409）无法被测试。
  - **[P0-5] Redis Pub/Sub 事件未实现**：补单/退款后 `EventPublisher::publish(balance_updated)` 调用完全缺失。
  - **[P0-6] 集成测试缺失**：RC-01～RC-15 / PROTO-1/2/3 均未实现，`tests/payment_recredit_test.rs` 文件不存在。
- **改进建议**：
  1. 先解决 T-10025 的 P0-1（模块挂载），再创建 `admin_service.rs`，在事务内按 TDS 流程实现 recredit/refund（SELECT FOR UPDATE → 状态机校验 → UPDATE payment_orders → UPDATE users → INSERT wallet_transactions → INSERT admin_logs）。
  2. 重点确保状态机单向性：ACKED/CREDITED/REFUNDED 状态必须返回 409/40905，FAILED/CANCELLED 才可 recredit，ACKED/CREDITED 才可 refund。
  3. 在 `controller.rs` 追加 handler，并通过 `require_role("super_admin")` 校验（非 permission，是强角色校验）。
  4. 创建 `tests/payment_recredit_test.rs`，至少覆盖 RC-01（recredit 成功）、RC-11（重复 recredit 409）、RC-08（operator 403）、RC-14（DB 异常全量回滚）、PROTO-3（Redis payload reason 字段精确匹配）。
- **🔴 Changelog 行**：已在 `doc/tasks/index.md` changelog 表追加版本行
