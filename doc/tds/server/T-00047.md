## 二、方案设计

### 核心数据流

```
                                 ┌────────────────────────────────────────────┐
[Android] RoomViewModel          │ ⭐ 主路径（C→S WebSocket）                 │
   .sendMessage(content) ──────▶ │ wsClient.sendEnvelope(type="SendMessage")  │
   （无 Retrofit POST /chat-msgs）│                                            │
                                 └─────────────────┬──────────────────────────┘
                                                   │
                                                   ▼
                              app/server/src/room/handler/chat.rs
                                  ::handle_send_message
                              ① content 校验 1..=500 chars
                              ② NOT_IN_ROOM / CHAT_MUTED 检查
                              ③ msg_id 幂等去重（processed_msg_ids）
                              ④ filter_content(content)
                              ⑤ chat_repo.insert_message → DB id
                              ⑥ broadcast_to_room(RoomMessage envelope)
                              ⑦ ACK SendMessageResult{code:0}
                                                   │
                                                   ▼
                                          ┌──────────────────┐
                                          │ RoomMessage      │
                                          │ envelope (§6.8.2)│
                                          └────────┬─────────┘
                                                   │ broadcast_to_room
                                                   │（注入 envelope.msg_id UUID v4）
                                                   ▼
                              ↘ 房间内所有 WS 连接（含发送者）

[运营/后端]                    ┌────────────────────────────────────────────┐
POST /api/v1/chat-messages ──▶ │ 备路径（HTTP REST，不上客户端）            │
   { room_id, content }        │                                            │
                                 └─────────────────┬──────────────────────────┘
                                                   ▼
                              app/server/src/modules/chat/controller.rs
                                  ::send_chat_message_handler
                              ① JWT auth
                              ② room_id UUID + content 1..=500 校验
                              ③ filter_content(content) ⚠️ 本 Task 新增
                              ④ chat_repo.insert_message → DB id
                              ⑤ broadcast_to_room(RoomMessage envelope) /
                                   降级 broadcast_to_room_no_state
                              ⑥ HTTP 200 { msg_id }
                                                   │
                                                   ▼
                              （envelope 与 WS 路径**逐字段等价**，仅 envelope.msg_id 各自独立）
```

### 实现约束（红线对齐）

1. **REST 路径补齐 `filter_content` 是 PROTO-2 等价的硬前提**：当前实现 `payload.content` 直接用 `req.content`（未净化），WS 路径用 `filter_content(content)`，违反 PROTO-2。本 Task 必须在 `send_chat_message_handler` 第 3 步（`insert_message` 之前）注入 `let filtered = crate::room::filter::filter_content(&req.content);`，并将 `filtered` 同时用于 DB 写入与广播 envelope。
2. **不引入新依赖**：复用 `room::filter::filter_content`（既有，T-00016 落地）、既有 `broadcast_to_room` / `broadcast_to_room_no_state`、既有 `chat_repo.insert_message`。
3. **不动 WS 路径**：`handle_send_message` 不在本 Task 范围内做任何修改（守住 BUG-CHAT-WS 修复链 Round 22 ✅ 状态）。
4. **不引入幂等 / muted 检查到 REST 路径**：REST 备路径定位为运营 / 后端工具，禁言由调用方自约束；HTTP `Idempotency-Key` 由后续独立 Task 接入。在 TDS 的"实施差异"章节明确记录差异范围。
5. **DoD 反向回填铁律**：DoD 阶段必须把绑定表 #1/#2/#3 三行写入 `doc/arch/server/index.md` 的「🔌 协议入口索引」表格；并在 `doc/arch/server/room.md` 末尾新增同名小节。两端互加跨端反向链接。

  - `payload.content` 相等（均为 `filter_content` 后）

  - `envelope.msg_id` 均为合法 UUID v4 且**互不相等**（设计如此）

### 边界与异常

- [ ] **B-1**：REST `content` 为空串 / 全 ASCII 空白 / 超过 500 chars → 400，**不**触发 `filter_content`、**不**写 DB、**不**广播。
- [ ] **B-2**：REST `content` 为纯敏感词（filter 后变空串）→ 当前 `filter_content` 行为：保留语义（替换为遮罩字符）；DB 落库的 content 与 envelope `payload.content` 仍相等（PROTO-2 不破）。在测试中显式断言"过滤后非空"以避免后续 filter 实现变更引入空 content 入库。
- [ ] **B-3**：REST 路径房间不存在（`room_manager.get_room` 返回 None）→ 仍走 `broadcast_to_room_no_state` 分支，HTTP 200，DB 落库；envelope `payload.content` 为净化后内容。
