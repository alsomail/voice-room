### 组件功能
- 时间筛选：`Radio.Group`：[最近 1h, 24h, 7d, 30d, 自定义]
- 自定义时：`DatePicker.RangePicker`（限 30 天）
- 事件名过滤：`Select mode=multiple`，选项来自 `ANALYTICS_EVENTS` 常量（前端枚举，避免频繁查后端）
- 时间线列表：倒序，每项展示
  - event_name（Tag 高亮色按类别）
  - server_ts 格式化
  - 设备/app_version/network_type（小字）
  - properties 折叠 JSON（`JsonView` 组件，关键字 highlight）
- 分页：Ant Design Pagination，默认 20/页
- 导出 CSV 按钮：当前筛选下最多 1000 条；文件名 `user_{id}_events_{ts}.csv`

### CSV 导出实现
前端遍历分页 API 最多 50 页（1000 条），合并后 `Papa.unparse` 生成 Blob 下载。若筛选结果超 1000 条，提示用户"结果过多，已导出前 1000 条"。

## Review R1 修复记录（2026-04-22）

### HIGH-1 修复：XSS 防护（`EventTimelineItem.tsx`）
- **问题**：`highlightText` 函数直接将 `JSON.stringify` 原始输出注入 `dangerouslySetInnerHTML`，不转义 `<`、`>`、`&` 等 HTML 特殊字符，存在 XSS 漏洞。
- **修复**：新增 `escapeHtml()` 纯函数，在 `highlightText` 中先对文本 HTML 转义，再对关键字 HTML 转义后执行正则替换，确保 `<mark>` 包裹的仅是转义后的安全文本。
- **新增测试**（TDD RED→GREEN）：
  - `properties 含 <script> 时，highlight 激活后渲染内容不含原始 <script> 标签`
  - `highlight 为空时，properties 含 HTML 特殊字符仍被转义`
  - `highlight 关键字本身含 HTML 特殊字符时不产生 XSS`

### HIGH-2 修复：CSV 导出 limit 改为 100（`EventStreamTab.tsx:195`）
- **问题**：`limit: 20` 导出 1000 条需最多 50 次 API 调用。
- **修复**：将 `baseParams.limit` 改为 `100`，导出调用次数从 50 次降至 10 次。

### MEDIUM-1 修复：CSV 导出添加 AbortController（`EventStreamTab.tsx`）
- **问题**：组件卸载后 CSV 导出请求仍在进行，缺乏取消机制。
- **修复**：新增 `exportAbortRef`，`handleExportCsv` 中创建独立 `AbortController`，每次导出前取消前一次请求；组件 `unmount` 的 useEffect cleanup 中也调用 `exportAbortRef.current?.abort()`；`AbortError` 不向用户报错（主动取消）。

- **测试结果（修复后）**：
  - 新增 3 个 XSS 安全测试，全量 **450 个测试全部通过**（36 个测试文件）
  - `EventTimelineItem.tsx`：Stmts **98.82%** / Branch **85.71%** / Funcs **100%**（较修复前大幅提升）
  - `EventStreamTab.tsx`：Stmts 84.27%

### Round 1（2026-04-22）

- **状态**：❌ 未通过（改回 TDD）
- **审查意见**：
  - **[HIGH-1] XSS 漏洞**：`EventTimelineItem.tsx` 使用 `dangerouslySetInnerHTML` 渲染 `JSON.stringify(event.properties)` 的结果，而 `JSON.stringify` 不转义 HTML 特殊字符（`<`、`>`、`&`）。当 `highlight` prop 激活时，服务端 properties 中若含 `<img onerror=...>` 等 payload 将被执行。当前 `highlight` 未传入（死代码），但 TDS 要求关键字高亮功能迟早上线，危险 API 不应入库。
  - **[HIGH-2] CSV 导出 limit=20 过低**：导出 1000 条需最多 50 次 API 调用，后端允许 `limit=100`，应改为 `limit: 100` 使调用次数降至 10 次。
  - **[MEDIUM-1] CSV 导出无 AbortSignal**：组件卸载后导出请求仍在进行，应复用 AbortController 模式。
  - **[LOW-1] eslint-disable 注释**：`useEffect` 处 `eslint-disable-next-line` 缺乏说明，建议补充注释或确认可移除。
- **改进建议**：
  1. `highlightText` 中先对输入字符串做 `escapeHtml`（`<`→`&lt;` 等），再做正则替换 `<mark>`；或引入 `DOMPurify.sanitize()`。
  2. `handleExportCsv` 中将 `limit: 20` 改为 `limit: 100`。
  3. 为 `handleExportCsv` 中的循环 `listUserEvents` 调用传入 `AbortSignal`，并在组件 unmount 时取消。
