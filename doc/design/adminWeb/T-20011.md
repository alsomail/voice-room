# T-20011: 活水房间监控增强

> **归属端**: Web (Admin)  
> **模块**: Room  
> **前置依赖**: T-20004 (RoomsPage), T-20005 (RoomDetailModal), T-10004 (房间列表后台接口)

---

## 需求描述

在现有房间管理页面上增加"活水监控"能力：标记疑似异常房间（长期无活跃/人数异常），增加房间活跃度筛选条件。

## 页面设计（增强 RoomsPage）

### 新增筛选条件

在现有工具栏的状态下拉框旁，新增"活跃度"筛选：

```
搜索框 | 状态: [全部 ▼] | 活跃度: [全部 ▼] | [刷新]
                              ├─ 全部
                              ├─ 活跃 (≥5人在线)
                              ├─ 冷清 (<5人在线且存在>1h)
                              └─ 疑似异常 (0人在线且未关闭)
```

### 房间列表增强

| 列名 | 说明 | 新增 |
|------|------|------|
| 房间ID | 已有 | — |
| 房间名 | 已有 | — |
| 房主 | 已有 | — |
| 在线人数 | 已有 | — |
| **活跃状态** | **新增** | Tag: 活跃(绿) / 冷清(黄) / 异常(红) |
| **持续时长** | **新增** | 房间创建至今的时长 (如 "2h 35m") |
| 创建时间 | 已有 | — |
| 操作 | 已有 | — |

### 活跃状态 Tag 规则

| Tag | 颜色 | 条件 |
|-----|------|------|
| 活跃 | 绿色 `success` | 在线人数 ≥ 5 |
| 冷清 | 黄色 `warning` | 在线人数 1-4 且存在 > 1 小时 |
| 异常 | 红色 `error` | 在线人数 = 0 且状态为 active |
| 正常 | 蓝色 `processing` | 其他情况 |

## 交互逻辑

1. 页面加载时自动计算各房间活跃状态（前端根据 `member_count` 和 `created_at` 计算）
2. "活跃度"筛选为前端过滤（不需要新后端 API，复用现有 T-10004 接口数据）
3. 异常房间行高亮背景 `rgba(231, 76, 60, 0.1)`

## 自动化测试锚点

| data-testid | 元素 | 断言 |
|-------------|------|------|
| `activity-filter` | 活跃度筛选下拉 | 可选择全部/活跃/冷清/异常 |
| `room-activity-tag-{roomId}` | 活跃状态 Tag | 颜色与条件匹配 |
| `room-duration-{roomId}` | 持续时长列 | 格式为 "Xh Ym" |

## TDD 验收标准

1. 房间列表新增"活跃状态"和"持续时长"两列
2. 活跃状态 Tag 颜色根据规则正确渲染
3. "活跃度"筛选可过滤房间列表
4. 异常房间（0人+active）行背景高亮
5. **所有现有房间管理功能（搜索/分页/关闭/详情）不可回归**
