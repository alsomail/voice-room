# TDS: AppServer config 补全 + 新增 staging.toml (Task ID: T-00040)

> **主设计**：[T-0000E](../infra/T-0000E.md) §2.2（加载链）、§2.4.2（toml 字段冻结）、§2.10（文件清单第 13~14 行）、§2.11 Step 4（服务端 config 对称化）、§2.12 R6（加载链优先级）。
> **下游联动**：T-10020（AdminServer 对称）、T-0000H（globalSetup 远端 staging/prod 对接）、T-0000J（用例 baseURL）、T-0000L（RUNBOOK 启动 SOP）。

## 一、背景与目标

### 关联需求
- 模块 9：E2E 测试基建。修复 T-0000E 体检结论 **C5：AppServer config 字段过少**（`dev.toml` 1 行、`test.toml` 3 行、`prod.toml` 3 行，多环境隔离形同虚设）。

### 一句话目标
**让 `APP_PROFILE=local|staging|prod` 切换 AppServer 时，加载链 `default.toml → {profile}.toml → ENV` 全程可见、关键字段缺失启动期 fail-fast（退出码 78），且现有单测 0 回归。**

### 现状 vs 目标
| 维度 | 现状（2026-05-31） | 目标 |
|---|---|---|
| `default.toml` 章节 | `[app] [server] [log] [database]`（缺 `[redis] [jwt]`） | 补全 6 章节，与 §2.4.2 字段表 1:1 对齐 |
| `dev.toml` 行数 | 1 个章节、4 行 | 与 default diff 化（仅覆写差异字段） |
| `test.toml` | port=3001（与 AdminServer port 冲突） | 改 port=4000，避免端口冲突 |
| `staging.toml` | **不存在** | 新增，`environment="staging"`，全部敏感字段走 ENV |
| `prod.toml` | 3 行 | 字段对齐 + `log.format=json` 显式锁定 |
| Profile 入口 | `APP_ENV` / `APP__ENVIRONMENT`（无 `APP_PROFILE`） | 增加 `APP_PROFILE` 主入口（与 T-0000E §2.2 一致），保留 `APP_ENV` 别名向后兼容 |
| `DATABASE_URL` 缺失 | `main.rs:49 expect` 裸 panic | `ServerSettings::load()` 内 fail-fast，退出码 78、明确错误信息 |
| `REDIS_URL` 缺失 | `main.rs:61 unwrap_or("redis://127.0.0.1:6379")` 静默回退 | 非 `dev` profile 必填；`dev` 允许 fallback 但日志 WARN |
| Invalid profile | `default_for(environment)` 静默接受任意字符串 | 启动期白名单校验 → 退出码 78 |
| 加载链可见性 | 无启动日志摘要 | 启动期打印生效配置摘要（不含密钥明文） |

### 不做（Out of Scope）
1. **不引入** `config` crate（保持现有自研 loader，避免与 AdminServer 异构带来的迁移风险——AdminServer 在 T-10020 才决定是否引入）。
2. **不修改** `redis_store.rs` / `database.rs` 等下游消费方接口（仅改 `infrastructure/config.rs` 与 `main.rs` 的取值路径）。
3. **不引入** TOML 中以 `${ENV_VAR}` 字面值占位符（自研 loader 不做插值，env 走 `apply_env_overrides` / `env::var` 直接覆盖；TOML 内仅放非敏感默认值）。
4. **不调整** SMS provider / Twilio 配置（仍在 main.rs 单独 `env::var`，由独立任务处理）。

## 二、方案设计

### 2.1 核心数据流（启动期 config 加载链）

```
                    cargo run / docker run
                            │
                            ▼
         ServerSettings::load()  ──┐
                            │     │ 1. dotenvy::from_path(.env)            （已存在）
                            │     │ 2. 解析 APP_PROFILE（新）
                            │     │      ├─ env: APP_PROFILE > APP_ENV > APP__ENVIRONMENT
                            │     │      └─ 默认 "dev"，白名单 {dev,test,staging,prod}
                            │     │ 3. 校验 profile 合法性 → 不合法 exit 78
                            │     │ 4. settings = default_for(profile)
                            │     │ 5. apply(load_config_file("default.toml"))
                            │     │ 6. apply(load_config_file("{profile}.toml"))
                            │     │ 7. apply_env_overrides()（APP__SERVER__*, APP__LOG__*, 新增 APP__JWT__*, APP__DATABASE__*）
                            │     │ 8. 注入敏感字段：
                            │     │      ├─ DATABASE_URL → settings.database.url（必填，缺则 exit 78）
                            │     │      ├─ REDIS_URL    → settings.redis_url（profile≠dev 必填，缺则 exit 78；dev WARN 回退）
                            │     │      └─ JWT_SECRET   → require_jwt_secret()（已存在，沿用 anyhow::Error → exit 78）
                            │     │ 9. log_summary()：tracing::info!(profile, host, port, jwt_secret_len, redis_url_redacted, db_url_redacted, ...)
                            ▼     │
                  ServerSettings  ▲
                            │     │ 任一步骤失败：
                            ▼     │   eprintln!("CONFIG ERROR: …") + std::process::exit(78)
                       main.rs    │   （EX_CONFIG, sysexits.h）
                            │     │
                            ▼
                 后续 pool / redis / sms / state / serve …
```

**关键不变量**：
- 加载链顺序 **与 T-0000E §2.2 严格一致**：`default → {profile}.toml → ENV`，env 优先级最高。
- 任何字段在 `default.toml` 都有合理默认（即使会被 profile 或 env 覆盖），保证 TOML 可读性。
- 敏感字段（`DATABASE_URL` `REDIS_URL` `JWT_SECRET`）**永不写入 TOML**，仅由 ENV 注入；TOML 中相应位置以注释标注「由 ENV 注入」。
- 启动期 fail-fast 路径**统一退出码 78**（EX_CONFIG），错误前缀 `CONFIG ERROR:`，便于 grep + preflight 抓取。

### 2.4 加载链（详细步骤）

```rust
// app/server/src/infrastructure/config.rs（新签名草图，仅供说明）
impl ServerSettings {
    pub fn load() -> anyhow::Result<Self> {
        // Step 1: dotenv（已有）
        load_dotenv();

        // Step 2: 解析 profile 入口
        let profile = Profile::from_env()?; // APP_PROFILE > APP_ENV > APP__ENVIRONMENT > "dev"

        // Step 3: 白名单校验（dev/test/staging/prod）
        // Profile::from_env 内部即 fail，错误信息：
        //   CONFIG ERROR: invalid APP_PROFILE='xxx'; expected one of [dev,test,staging,prod]

        // Step 4: 默认值
        let mut settings = Self::default_for(profile.as_str());

        // Step 5/6: TOML 分层
        let dir = config_dir();
        settings.apply(load_config_file(dir.join("default.toml"))?);
        settings.apply(load_config_file(dir.join(format!("{profile}.toml")))?);

        // Step 7: ENV override（非敏感字段）
        settings.apply_env_overrides();

        // Step 8: 注入敏感字段（fail-fast）
        settings.database.url = Some(require_env(
            "DATABASE_URL",
            "DATABASE_URL must be set; refused to start"
        )?);
        settings.jwt_secret = require_jwt_secret(env::var("JWT_SECRET").ok())?;
        settings.redis_url = match env::var("REDIS_URL") {
            Ok(v) if !v.trim().is_empty() => Some(v),
            _ if profile.allow_redis_fallback() => {
                tracing::warn!("REDIS_URL not set, fallback to redis://127.0.0.1:6379 (dev only)");
                Some("redis://127.0.0.1:6379".to_owned())
            }
            _ => bail_config_error("REDIS_URL", "REDIS_URL must be set for non-dev profile")?,
        };

        // Step 9: 启动摘要日志（脱敏）
        settings.log_summary();
        Ok(settings)
    }
}
```

`main.rs` 在调用 `ServerSettings::load()` 之后，对 `anyhow::Error` 做统一拦截，**输出 `CONFIG ERROR:` 前缀 + `std::process::exit(78)`**，避免依赖 `?` 触发的 anyhow `Debug` 默认输出。

```rust
// main.rs 顶部新增 helper
fn fatal_config(err: anyhow::Error) -> ! {
    eprintln!("CONFIG ERROR: {err:#}");
    std::process::exit(78);
}

// main 内：
let settings = ServerSettings::load().unwrap_or_else(|e| fatal_config(e));
```

### 2.5 错误契约（fail-fast 错误信息表）

| 触发条件 | stderr 前缀 | 错误正文（≤ 100 字） | 退出码 |
|---|---|---|---|
| `APP_PROFILE` 不在白名单 | `CONFIG ERROR: ` | `invalid APP_PROFILE='xxx'; expected one of [dev,test,staging,prod]` | 78 |
| `DATABASE_URL` 缺失 / 空 | `CONFIG ERROR: ` | `DATABASE_URL must be set; refused to start (profile=staging)` | 78 |
| `JWT_SECRET` 缺失 | `CONFIG ERROR: ` | `JWT_SECRET must be set; refusing to fall back to a hardcoded default` | 78 |
| `JWT_SECRET` 空白 | `CONFIG ERROR: ` | `JWT_SECRET must not be empty` | 78 |
| `JWT_SECRET=change-me-in-production` | `CONFIG ERROR: ` | `JWT_SECRET still equals the placeholder 'change-me-in-production'; refuse to start` | 78 |
| `REDIS_URL` 缺失（staging/prod/test） | `CONFIG ERROR: ` | `REDIS_URL must be set for non-dev profile (profile=staging)` | 78 |
| TOML 解析失败 | `CONFIG ERROR: ` | `failed to parse {path}: {toml_error}` | 78 |
| `APP__SERVER__PORT` 解析失败 | （WARN，不 fail） | `tracing::warn!` 保留原值 | — |

> **可定位性原则**：错误正文必须包含「字段名 + profile 值 + 修复 hint（关联 ENV 名）」，便于在 preflight.sh 与 RUNBOOK 内 grep。错误前缀 `CONFIG ERROR:` 是与 T-0000G preflight 的契约（preflight 见此前缀即可定位 server 启动失败而非端口被占）。
