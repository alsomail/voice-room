## 一、背景与目标

### 关联需求
- 模块 9（E2E 测试基建）P0 第 4 个 Task，落地 T-0000E §2.11 Migration Path **Step 3：E2E 调度器接管**。

### 一句话目标
**让 Playwright 启动期成为「单一编排入口」**：依据 `E2E_PROFILE` 加载 env → 调 preflight → 按需触发 seed → 注入 `process.env` 给 worker；结束后按需 reset；并通过 fixture 在 prod profile 下兜底拦截写操作。

### 与现状偏差（驱动本设计）
当前 `playwright.config.ts`（第 9~11 行）仅 `dotenv.config({ path: .env })`，`baseURL` 注释（第 44 行），无 `globalSetup`/`globalTeardown`。所有用例靠裸 `process.env.X` 读值，缺字段时进入 `test()` 才崩溃；profile 切换无入口；prod 误用零防护。

### 不在本 Task 范围内
1. `.env.example` / `.env.{profile}.example` 模板内容 → T-0000F 已冻结；
2. seed/reset/preflight 脚本本体 → T-0000G 已交付；
3. `npm run e2e:local` / `e2e:staging` / `e2e:prod-smoke` 等一键命令 → T-0000I；

5. Midscene Key 注入策略文档 → T-0000K。

## 二、方案设计

### 2.1 核心数据流

```
┌────────────────────────── playwright 进程 (主线程) ──────────────────────────┐
│                                                                              │
│  playwright.config.ts                                                        │
│    │ ① require.resolve('./tests/scripts/support/globalSetup')               │
│    │ ② require.resolve('./tests/scripts/support/globalTeardown')            │
│    │ ③ use.baseURL = (lazy)                                                 │
│    ▼                                                                         │
│  ┌──── globalSetup.ts ──────────────────────────────────────────────┐       │
│  │  Step1: env = loadE2EEnv()                          ◄── envLoader │       │
│  │           │  (读 .env → tests/scripts/env/.env.<profile>)         │       │
│  │           │  缺字段 → throw MissingEnvError → process.exit(78)    │       │
│  │  Step2: await runPreflight(env)                  ◄── T-0000G ③   │       │
│  │           │  (spawn 'bash scripts/dev/preflight.sh --profile P')  │       │
│  │           │  非 0 → throw PreflightFailedError → process.exit(<rc>)│      │
│  │  Step3: if local && allowWrites && E2E_SEED!=0:                  │       │
│  │           await runSeed(env)                     ◄── T-0000G ①   │       │
│  │           │  (spawn 'bash scripts/dev/seed-e2e.sh')              │       │
│  │           │  parse scripts/dev/.seed-output.env → process.env    │       │
│  │  Step4: writeProcessEnv(env)  // 规范化字段写入 process.env       │       │
│  │  Step5: persist resolved env → globalThis.__E2E_ENV__（fixture 用）│      │
│  └──────────────────────────────────────────────────────────────────┘       │
│                                                                              │
│  ┌──── playwright workers (并行) ───────────────────────────────────┐       │
│  │  test fixture(prodSafeGuard, e2eEnv) → 读 process.env / globalThis│      │
│  │    └─ profile=prod && !@prod-safe → test.skip()                  │       │
│  │    └─ profile=prod && writes 类 fixture → test.skip()            │       │
│  └──────────────────────────────────────────────────────────────────┘       │
│                                                                              │
│  ┌──── globalTeardown.ts ───────────────────────────────────────────┐       │
│  │  if profile==='local' && E2E_RESET!=0:                          │       │
│  │     await runReset(env)                          ◄── T-0000G ②   │       │
│  │     (spawn 'bash scripts/dev/reset-e2e.sh --yes')                │       │
│  │  其余 profile：skip（远端从不自动 reset）                         │       │
│  └──────────────────────────────────────────────────────────────────┘       │
└──────────────────────────────────────────────────────────────────────────────┘
```

> **不变量**：
> 1. envLoader 是**唯一** env 读取源；任何 `process.env.X`（worker 侧）的值必须经过 globalSetup 注入。**禁止** worker 内再 `dotenv.config()`。
> 2. globalSetup 失败必须**进程级**退出（fail-fast），禁止允许任何 `test()` 进入再崩溃。
> 3. 调用三脚本一律通过 `child_process.spawn`（非 `execSync`），便于流式打印与 2s 超时控制。

#### 2.3.1 类型定义（与 T-0000E §2.7 字段顺序一一对应）

```typescript
// tests/scripts/support/types.ts
export type E2EProfile = 'local' | 'staging' | 'prod';

export interface E2EEnv {
  profile: E2EProfile;
  allowWrites: boolean;             // E2E_ALLOW_WRITES，prod 默认 false

  // HTTP / WS 端点（4 项必填，所有 profile）
  appServerBaseUrl: string;         // APP_SERVER_BASE_URL
  adminServerBaseUrl: string;       // ADMIN_SERVER_BASE_URL
  adminWebUrl: string;              // ADMIN_WEB_URL
  appWsUrl: string;                 // APP_WS_URL

  // 仅 local 必填的直连项
  databaseUrl?: string;             // DATABASE_URL（local 必填，staging/prod 留空）
  redisUrl?: string;                // REDIS_URL（同上）

  // Android 用例条件必填（非 Android 用例可缺）
  androidAppId?: string;            // ANDROID_APP_ID

  // Token 套件（5 个常驻 + 2 个过期），所有 profile 必填
  tokens: {
    valid: string;                  // E2E_VALID_TOKEN
    expired: string;                // E2E_EXPIRED_TOKEN
    admin: string;                  // E2E_ADMIN_TOKEN
    op: string;                     // E2E_OP_TOKEN
    cs: string;                     // E2E_CS_TOKEN
    fin: string;                    // E2E_FIN_TOKEN
    expiredAdmin: string;           // E2E_EXPIRED_ADMIN_TOKEN
  };

  // Seed 回填 ID 三件套（local 由 seed 写入；staging/prod 手填）
  ids: {
    roomId: string;                 // E2E_ROOM_ID
    userAId: string;                // E2E_USER_A_ID
    userBId: string;                // E2E_USER_B_ID
  };

  // Midscene LLM（WEB 用例条件必填，缺则 WEB 用例 skip）
  midscene: {
    apiKey: string;                 // MIDSCENE_MODEL_API_KEY
    modelName: string;              // MIDSCENE_MODEL_NAME 默认 gpt-4o
    baseUrl?: string;               // MIDSCENE_OPENAI_BASE_URL
    cache: boolean;                 // MIDSCENE_CACHE 默认 true
  };

  // CI 软门禁（默认 false，CI 中置 true）
  ciReady: boolean;                 // CI_E2E_READY
}

export class MissingEnvError extends Error {
  constructor(
    public readonly profile: E2EProfile,
    public readonly missingFields: string[],
    public readonly envFilePath: string,
  ) {
    super(MissingEnvError.format(profile, missingFields, envFilePath));
    this.name = 'MissingEnvError';
  }
  static format(profile: string, fields: string[], path: string): string {
    return [
      `[E2E envLoader] MissingEnvError: profile=${profile} missing ${fields.length} required field(s):`,
      ...fields.map(f => `  - ${f}`),
      `Hint: copy ${path}.example to ${path} and fill in values.`,
      `Reference: doc/tds/infra/T-0000F.md §2.3 field table.`,
    ].join('\n');
  }
}
```

#### 2.3.2 函数签名

```typescript
// tests/scripts/support/envLoader.ts
export function loadE2EEnv(opts?: { cwd?: string }): E2EEnv;
// 行为：纯函数 + fail-fast；不写 process.env（Step4 的 writeProcessEnv 单独负责）。
//      多次调用结果一致（除非在调用之间手动改了 process.env / 文件系统）。
```

#### 2.3.3 加载链（顺序严格冻结）

```
1. process.env.E2E_PROFILE 解析 → profile（默认 'local'，非 local|staging|prod 立即抛 InvalidProfileError）
2. dotenv.config({ path: <cwd>/.env, override: false })            ← 主 .env（开发本地常用；CI 通常不存在）
3. dotenv.config({ path: <cwd>/tests/scripts/env/.env.<profile>, override: true })
                                                                    ← profile 专属配置（覆盖 .env）
4. 真实 process.env（dotenv override:true 已写入）成为字段读取源；
   注：CI / 手动 export 的环境变量在 dotenv 之前已存在，dotenv 不会覆盖它们（override:false 默认）；
   profile 文件用 override:true 是为了让 .env.staging 能改写 .env 的开发值，
   但**真实 shell 已 export 的值优先级最高**（在 step2/3 之前已存在，dotenv 仍不会覆盖 process.env 既有值）——
   这与 dotenv v17 的标准语义一致：dotenv `override:true` 仅覆盖**之前 dotenv.config 写入的值**，不覆盖原生 shell env。
5. 校验 profile 对应的「必填字段集」（见 §2.3.4），缺一即收集进 missingFields；
6. missingFields.length > 0 → throw new MissingEnvError(profile, missingFields, '<cwd>/tests/scripts/env/.env.<profile>')
7. 类型转换：bool（`1`/`true`/`yes` → true；`0`/`false`/`no` → false；其他抛 InvalidEnvError）；url（new URL() 抛即 InvalidEnvError）；
8. 返回不可变 E2EEnv 对象（Object.freeze）。
```

> **优先级总结（高→低）**：原生 shell export > `tests/scripts/env/.env.<profile>` > 根 `.env` > 默认值。

### 2.4 globalSetup 流程详解

#### 2.4.1 步骤拆解

```typescript
// tests/scripts/support/globalSetup.ts（伪代码骨架，非完整实现）
import { spawn } from 'node:child_process';
import { loadE2EEnv } from './envLoader';
import dotenv from 'dotenv';
import fs from 'node:fs';
import path from 'node:path';

export default async function globalSetup() {
  const t0 = Date.now();
  let env;
  try {
    // Step 1: load env（fail-fast）
    env = loadE2EEnv();
    console.log(`[E2E setup] profile=${env.profile} allowWrites=${env.allowWrites}`);

    // Step 2: preflight（spawn bash）
    await runPreflight(env);

    // Step 3: seed（仅 local && allowWrites && E2E_SEED!=0；默认开）
    const seedFlag = process.env.E2E_SEED ?? '1';
    if (env.profile === 'local' && env.allowWrites && seedFlag !== '0') {
      await runSeed(env);
      // 解析 seed 回填文件并 export
      const seedOut = path.resolve('scripts/dev/.seed-output.env');
      if (fs.existsSync(seedOut)) {
        const parsed = dotenv.parse(fs.readFileSync(seedOut));
        for (const [k, v] of Object.entries(parsed)) process.env[k] = v;
        // seed 后必须重新 loadE2EEnv，让 ids/tokens 重算
        env = loadE2EEnv();
      }
    }

    // Step 4: 规范化字段写入 process.env，让 worker 端可读
    writeProcessEnv(env);

    // Step 5: 跨进程传递（Playwright 不共享内存，写文件让 fixture 读）
    fs.writeFileSync(
      path.resolve('tests/scripts/.e2e-runtime.json'),
      JSON.stringify(env, null, 2),
      { mode: 0o600 },
    );

    console.log(`[E2E setup] OK in ${Date.now() - t0}ms`);
  } catch (err: any) {
    console.error(`[E2E setup] FAILED:`, err.message ?? err);
    process.exit(err.exitCode ?? 1);
  }
}
```

#### 2.4.2 各步骤异常处理

| Step | 错误类型 | 退出码 | 日志格式 | 可恢复 |
|------|----------|--------|----------|--------|
| 1 envLoader | MissingEnvError / InvalidProfileError / InvalidEnvError | 78 | 多行错误（含 missingFields 列表 + Hint + 字段表参考） | ❌ 必须人工修 env |
| 2 preflight | PreflightFailedError | 11/12/13/14/15 | `[E2E setup] preflight failed (exit=<rc>)` + 原始 stderr 末 20 行 | ✅ 起对应服务后重试 |
| 3 seed | SeedFailedError | 21/22/23/24 | `[E2E setup] seed failed (exit=<rc>)` + stderr 末 20 行 | ✅ 看具体错（profile/JWT_SECRET/SQL） |
| 4 writeProcessEnv | TypeError | 1 | 实现 bug，提交 issue | ❌ |
| 5 写 runtime json | EACCES/EROFS | 1 | 文件系统错 | ✅ 检查权限 |

#### 2.4.3 子进程调用规范

- **统一封装**为 `runShell(cmd, args, opts)`：基于 `spawn`，`stdio: ['ignore', 'inherit', 'inherit']`，超时 60s（preflight 自有 2s/项；seed 大约 10s）；
- **环境变量**：以 `{ ...process.env, E2E_PROFILE: env.profile }` 透传，确保子进程读到 envLoader 解析后的 profile；
- **CWD**：固定 `process.cwd()`（即仓库根），与 `playwright.config.ts` 同级；
- **CI 静默**：`process.env.CI === 'true'` 时去掉 `[E2E setup]` 装饰前缀（由 `[E2E setup ci]` 替换），避免 reporter 日志膨胀。

### 2.5 globalTeardown 流程

```typescript
// tests/scripts/support/globalTeardown.ts
export default async function globalTeardown() {
  const env = loadE2EEnv();      // 重读，避免 setup 阶段 process.env 被 worker 改坏
  if (env.profile !== 'local') {
    console.log(`[E2E teardown] skip reset (profile=${env.profile}, remote profiles never auto-reset)`);
    return;
  }
  const resetFlag = process.env.E2E_RESET ?? '1';
  if (resetFlag === '0') {
    console.log('[E2E teardown] skip reset (E2E_RESET=0)');
    return;
  }
  try {
    await runShell('bash', ['scripts/dev/reset-e2e.sh', '--yes'], { timeoutMs: 30_000 });
    console.log('[E2E teardown] reset OK');
  } catch (err) {
    // 容忍：teardown 失败不应影响测试结果，只 warn
    console.warn('[E2E teardown] reset failed (non-fatal):', err);
  }
}
```

**约束**：
1. profile=staging/prod 永远 skip（与 T-0000E §2.7 第 4 行红线一致）；
2. 失败仅 `console.warn`，**不**改变 playwright 测试结果，避免「测试都过了 teardown 挂导致 CI 红」；
3. 不读 `tests/scripts/.e2e-runtime.json`（避免 setup 阶段失败时该文件不存在）。

### 2.6 prod 写操作 skip 机制

#### 2.6.1 三层防御（与 T-0000E §2.9 R1 / T-0000F F-R3 联动构成五道防线）

| 层 | 实现位置 | 触发条件 | 行为 |
|----|----------|----------|------|
| L1 配置层 | `tests/scripts/env/.env.prod.example` | `E2E_ALLOW_WRITES=0`（默认） | 用户须手动改为 1 才允许写，且 example 顶部有 ⚠️ 注释 |
| L2 envLoader | `loadE2EEnv` | profile=prod 且 `E2E_ALLOW_WRITES=1` | console.warn 醒目橙色提示，但**不**抛错（用户已显式同意） |
| L3 fixture 层（**本 Task 主战场**） | `tests/scripts/support/fixtures.ts` | profile=prod 且未打 `@prod-safe` 标签 | `test.skip(true, 'prod profile only runs @prod-safe tests')` |
| L4 fixture 层 | 写操作类 fixture（`seededRoom` / `apiWriteRequest` / `adminMutationClient`） | `allowWrites=false` | `test.skip(true, 'requires E2E_ALLOW_WRITES=1')` |
| L5 用例标签层 | T-0000J | 写操作类用例不打 `@prod-safe`；只读 smoke 才打 | 与 L3 协同，确保未打标的写用例进 prod 必跳 |

#### 2.6.2 fixture 实现骨架（本 Task 落地）

```typescript
// tests/scripts/support/fixtures.ts
import { test as base, expect } from '@playwright/test';
import type { E2EEnv } from './types';
import { loadE2EEnv } from './envLoader';

type Fixtures = {
  e2eEnv: E2EEnv;
  prodSafeGuard: void;        // 自动 use，所有用例
  apiWriteRequest: APIRequestContext;  // 写操作 fixture，allowWrites=0 时 skip
};

export const test = base.extend<Fixtures>({
  e2eEnv: [async ({}, use) => { await use(loadE2EEnv()); }, { scope: 'worker' }],

  prodSafeGuard: [async ({ e2eEnv }, use, testInfo) => {
    if (e2eEnv.profile === 'prod') {
      const tags = testInfo.tags ?? [];
      if (!tags.includes('@prod-safe')) {
        testInfo.skip(true, 'prod profile only runs tests tagged @prod-safe');
      }
    }
    await use();
  }, { auto: true }],

  apiWriteRequest: async ({ e2eEnv, playwright }, use, testInfo) => {
    if (!e2eEnv.allowWrites) {
      testInfo.skip(true, 'fixture requires E2E_ALLOW_WRITES=1');
    }
    const ctx = await playwright.request.newContext({ baseURL: e2eEnv.appServerBaseUrl });
    await use(ctx);
    await ctx.dispose();
  },
});
export { expect };
```

#### 2.6.3 与 T-0000J `@prod-safe` 标签的协同

- 本 Task 提供「**机制**」：L3 自动 skip + L4 写操作 fixture skip；
- T-0000J 提供「**标签**」：在具体用例上打 `{ tag: '@prod-safe' }`（read-only smoke），并将硬编码改为 `e2eEnv` fixture 读值；
- 二者同时合入后，`E2E_PROFILE=prod npx playwright test` 行为：未打标用例 → skip；打标用例 → 跑；写操作 fixture 用例 → skip（即便打了标，因为 `allowWrites=0`）。
