> **归属端**: Android
> **模块**: UI/Gift

## 需求描述

实现房间内礼物赠送核心入口：`GiftPanelBottomSheet`。替换 T-30026 中 🎁 按钮的 Toast 占位，点击后弹出从底部滑入的礼物面板。

## 交互逻辑

| 操作 | 行为 |
|------|------|
| 底栏 🎁 点击 | Bottom Sheet 从底部滑入（300ms ease-out） |
| 点击礼物格 | 该项金色边框 + 底部按钮更新总价 |
| 点击数量按钮 | 选中高亮 + 总价 = gift.price * count |
| 点击"充值" | 跳转 WalletScreen |
| 点击"送出" | 交给 T-30030 处理（发 WS + loading） |
| 余额 < 总价 | "送出"按钮置灰 + 显示"余额不足" |
| 返回键 / × / 外部点击 | Bottom Sheet 滑出关闭 |
| WS `BalanceUpdated` | 顶部余额条实时更新 |

## 异常流程

| 场景 | 处理 |
|------|------|
| 礼物列表加载失败 | 网格区显示骨架屏 + 重试按钮 |
| 无麦位用户 | "送出"按钮禁用 + 提示"当前无人在麦" |
| 礼物被下架（WS 推送） | 列表自动刷新 + 若选中项已下架，清除选中 |

## 通用组件

- `GiftCard`（新）：礼物格，含选中态金色边框
- `CountSelector`（新）：数量档位按钮行
- `BalanceBar`（新）：顶部余额显示条（钱包页与礼物面板共用）

## TDD 验收标准

1. 房间底栏 🎁 点击后 Bottom Sheet 滑入显示
2. 礼物列表从 `/api/v1/gifts/list` 拉取并渲染 4 列网格
3. 选中礼物后 `btn_send_gift` 文字更新为 `送出 {total} 💎`
4. 余额不足时 `btn_send_gift` disabled + 文字"余额不足"
5. 点击"充值"跳转 WalletScreen
6. WS 推送 `BalanceUpdated` 后余额条即时更新
7. Bottom Sheet 滑出关闭时清除选中状态
