# TDS: Admin Server 用户列表接口 (Task ID: T-10007)

## 一、背景与目标

- **关联需求**：来自 `doc/product.md` 用户管理模块——运营后台需要对 App 端用户进行检索与管理
- **本 Task 的目标**：实现 `GET /api/v1/admin/users` 接口，支持多条件过滤（手机号/用户ID/昵称/状态）与分页，返回用户基本信息及资产（coin_balance）

---

## 二、方案设计

### §2.1 接口规格

```
GET /api/v1/admin/users
Authorization: Bearer <admin-jwt>
```

#### 查询参数

| 参数 | 类型 | 必须 | 说明 |
|------|------|------|------|
| `phone` | string | 否 | 手机号精确匹配 |
| `nickname` | string | 否 | 昵称模糊匹配（`LIKE %xxx%`，大小写不敏感） |
| `user_id` | string (UUID) | 否 | 用户 ID 精确匹配 |
| `status` | `"normal"` \| `"banned"` | 否 | 按封禁状态过滤 |
| `page` | integer | 否 | 页码，默认 1，最小 1 |
| `size` | integer | 否 | 每页条数，默认 20，最大 100 |

#### 成功响应（200）

```json
{
  "code": 0,
  "message": "ok",
  "request_id": "xxx",
  "data": {
    "total": 42,
    "page": 1,
    "size": 20,
    "items": [
      {
        "id": "uuid-string",
        "phone": "13800138000",
        "nickname": "用户昵称",
        "avatar": "https://cdn.example.com/avatar.jpg",
        "coin_balance": 1000,
        "vip_level": 1,
        "status": "normal",
        "created_at": "2025-01-01T00:00:00Z"
      }
    ]
  }
}
```

> `status` 字段从 `UserModel.is_banned` 映射：`false` → `"normal"`，`true` → `"banned"`

#### 错误码

| HTTP | code  | 说明 |
|------|-------|------|
| 401  | 40101 | 无 Token / Token 无效 |
| 401  | 40102 | Token 已过期 |
| 403  | 40301 | 角色无 `UserRead` 权限（finance 角色） |

### §2.2 权限设计

使用已有 `Permission::UserRead`，RBAC 矩阵如下：

| 角色 | UserRead |
|------|---------|
| super_admin | ✅ |
| operator | ✅ |
| cs | ✅ |
| finance | ❌ |

### §2.3 SQL 设计

**计数 SQL**（用于 total）：

```sql
SELECT COUNT(*)
FROM users
WHERE deleted_at IS NULL
  AND ($1::text IS NULL OR phone = $1)
  AND ($2::uuid IS NULL OR id = $2)
  AND ($3::text IS NULL OR LOWER(nickname) LIKE '%' || LOWER($3) || '%')
  AND ($4::boolean IS NULL OR is_banned = $4)
```

**列表 SQL**（分页查询）：

```sql
SELECT id, phone, nickname, avatar, coin_balance, vip_level, is_banned, created_at
FROM users
WHERE deleted_at IS NULL
  AND ($1::text IS NULL OR phone = $1)
  AND ($2::uuid IS NULL OR id = $2)
  AND ($3::text IS NULL OR LOWER(nickname) LIKE '%' || LOWER($3) || '%')
  AND ($4::boolean IS NULL OR is_banned = $4)
ORDER BY created_at DESC
LIMIT $5 OFFSET $6
```

**参数绑定说明**：
- `$1` → `filter.phone`（精确匹配，`None` 跳过）
- `$2` → `filter.user_id`（UUID 精确匹配，`None` 跳过）
- `$3` → `filter.nickname`（LIKE 模糊，`None` 跳过）
- `$4` → `filter.is_banned`（布尔，`None` 跳过）
- `$5` → `limit: i64`
- `$6` → `offset: i64`

### §2.4 核心数据流

```
GET /api/v1/admin/users?phone=138...&page=2&size=10
        │
        ▼
  jwt_auth 中间件（AdminAuthContext 注入）
        │
        ▼
  list_users_handler (controller.rs)
    ├─ AdminAuthContext::require_permission(UserRead) → 403 if fail
    ├─ parse AdminUserListQuery (serde Query extractor)
    ├─ 构造 AdminUserFilter { phone, user_id, nickname, is_banned }
    ├─ page/size 校验 + clamp (size ≤ 100)
    └─ user_service.list_users(filter, page, size).await
              │
              ▼
        AdminUserService (service.rs)
          ├─ user_repo.count_users(&filter).await → total: i64
          ├─ user_repo.find_users(&filter, offset, limit).await → Vec<AdminUserListRow>
          └─ 组装 AdminUserListResponse { total, page, size, items }
                    │
                    ▼
              AdminUserRepository (repository.rs)
                ├─ PgAdminUserRepository  (生产 DB)
                └─ FakeAdminUserRepository (测试 HashMap)
```

### §2.6 分页设计

- `page` 默认 1，最小 1（小于 1 自动 clamp 为 1）
- `size` 默认 20，最大 100（大于 100 自动 clamp 为 100）
- `offset = (page - 1) * size`

### §2.7 DTO 结构体设计

```rust
/// GET /api/v1/admin/users 查询参数
#[derive(Debug, Deserialize)]
pub struct AdminUserListQuery {
    pub phone: Option<String>,
    pub nickname: Option<String>,
    pub user_id: Option<Uuid>,
    /// "normal" | "banned"
    pub status: Option<String>,
    pub page: Option<u32>,
    pub size: Option<u32>,
}

/// 传递给 AdminUserRepository 的过滤条件
#[derive(Debug, Clone, Default)]
pub struct AdminUserFilter {
    pub phone: Option<String>,
    pub user_id: Option<Uuid>,
    pub nickname: Option<String>,
    /// None = 全部；Some(false) = normal；Some(true) = banned
    pub is_banned: Option<bool>,
}

/// users 表查询的单行结果（FromRow）
#[derive(Debug, Clone, sqlx::FromRow)]
pub struct AdminUserListRow {
    pub id: Uuid,
    pub phone: String,
    pub nickname: String,
    pub avatar: Option<String>,
    pub coin_balance: i64,
    pub vip_level: i16,
    pub is_banned: bool,
    pub created_at: DateTime<Utc>,
}

/// 响应中每个用户的 JSON 结构
#[derive(Debug, Clone, Serialize)]
pub struct AdminUserItem {
    pub id: String,
    pub phone: String,
    pub nickname: String,
    pub avatar: Option<String>,
    pub coin_balance: i64,
    pub vip_level: i16,
    pub status: String,       // "normal" | "banned"
    pub created_at: String,   // RFC 3339
}

/// GET /api/v1/admin/users 成功响应 data 部分
#[derive(Debug, Serialize)]
pub struct AdminUserListResponse {
    pub total: i64,
    pub page: u32,
    pub size: u32,
    pub items: Vec<AdminUserItem>,
}
```

## 三、TDD 验收用例

### §3.1 Repository 单元测试（repository.rs）

| ID | 描述 |
|----|------|
| R-01 | `FakeAdminUserRepository`：无过滤条件，`count_users` 返回全部非软删除用户数，`find_users` 按 created_at DESC 排序 |
| R-02 | `phone` 精确过滤：`count_users` 和 `find_users` 只返回手机号完全匹配的记录 |
| R-03 | `nickname` 模糊过滤：包含子串（大小写不敏感）的记录均被匹配 |
| R-04 | `is_banned=true`：只返回封禁用户；`is_banned=false` 只返回正常用户 |

| R-06 | 软删除用户（`deleted_at IS NOT NULL`）不计入结果 |
| R-07 | 分页：`offset=5, limit=5`，只返回第 6-10 条 |
