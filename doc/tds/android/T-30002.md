# TDS: 登录 ViewModel (Task ID: T-30002)

## 一、背景与目标
- **关联需求**：product.md - 手机号一步登录
- **本 Task 的目标**：实现登录 ViewModel，通过 Retrofit 调用 App Server 登录接口，管理完整的登录状态流转（发送验证码、倒计时、登录、JWT 存储）。
- **前置依赖**：T-00003 (login 接口), T-30001

## 二、方案设计
### 核心数据流
```
LoginScreen → LoginViewModel.sendCode(phone) → AuthRepository → Retrofit POST /auth/send-code → 成功/失败状态
LoginScreen → LoginViewModel.login(phone, code) → AuthRepository → Retrofit POST /auth/login → JWT → DataStore 存储 → NavigateTo(Hall)
```

#### 🔴 [HIGH] tokenManager.saveToken 未包裹错误处理 — UI 可陷入永久 Loading 死锁

**文件：** `feature/auth/LoginViewModel.kt:132`

`tokenManager.saveToken(result.token)` 对 DataStore 发起 I/O 写操作，在磁盘满/权限撤销时会抛出 `IOException`，而该调用位于 `.onSuccess { }` 块内，**无任何 try-catch/runCatching 保护**。异常抛出后协程崩溃，`isLoading` 永久卡在 `true`，用户陷入无反馈的 Loading 死锁，且无法恢复。

**修复方案：**
```kotlin
.onSuccess { result ->
    runCatching { tokenManager.saveToken(result.token) }
        .onSuccess {
            _uiState.update { it.copy(isLoading = false, isLoginSuccess = true, isNewUser = result.isNew) }
            _navEvent.emit(NavEvent.NavigateToHall)
        }
        .onFailure {
            _uiState.update { it.copy(isLoading = false, error = "登录失败，Token 存储异常，请重试") }
        }
}
```
