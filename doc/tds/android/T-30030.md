### 连击聚合器
```kotlin
class ComboAggregator(private val windowMs: Long = 3000) {
    private var current: Combo? = null
    data class Combo(val giftId: String, val recipientId: String,
                     val msgId: String, var count: Int, var lastTs: Long)

    fun press(giftId: String, recipientId: String, unitCount: Int = 1): Combo {
        val now = System.currentTimeMillis()
        val c = current
        return if (c != null && c.giftId==giftId && c.recipientId==recipientId
                   && now - c.lastTs < windowMs) {
            c.count += unitCount; c.lastTs = now; c
        } else {
            Combo(giftId, recipientId, UUID.randomUUID().toString(), unitCount, now).also { current = it }
        }
    }
    fun flush() { current = null }
}
```
- 连击窗口内所有点击累加 `count`，但 **只发一次 WS**（发送时机：最后一次 press 后 windowMs 到或 flush 或离开窗口）
- MVP 策略：用户按"送出"立刻 flush 并发送（避免设计太复杂）；连击窗口仅作用于 UI 反馈

### 发送流程
```
state.canSend → 按钮可点
onSend() {
  val job = SendGiftJob(msgId=UUID, giftId, recipientId, count)
  _state.update { it.copy(sending=true) }
  wsClient.sendGift(job)
  val result = withTimeoutOrNull(5_000) { resultFlow.first { it.msgId == job.msgId } }
  when {
    result == null -> showError("请求超时，请重试")
    result.code == 0 -> { _state.update { it.copy(sending=false) } /* 保持面板 */ }
    result.code == 40290 -> showInsufficientDialog() // T-30032
    result.code == 40403 -> toast("接收者已下麦") /* 面板不关 */
    else -> toast("赠送失败，请重试")
  }
}
```

### 关键实现要点

1. **幂等 msg_id**：`ComboAggregator.press()` 每次生成 `UUID.randomUUID().toString()`；MVP 策略立即 `flush()` 后发送，每次"送出"均独立。
2. **5s 超时**：`withTimeoutOrNull(5_000L)` + `_sendGiftResultFlow.first { it.msgId == job.msgId }` 实现精确匹配与超时。
3. **sending 状态**：`canSend` 含 `&& !sending`，按钮自动禁用防重复发送（S30-12 验证）。
4. **WS 解析**：`handleWsMessage` 增加 `SendGiftResult` 分支，通过 `tryEmit`（non-suspend）写入 `_sendGiftResultFlow`（extraBufferCapacity=16）。
5. **TestScope 扩展函数**：`buildReadyViewModel` 作为 `TestScope` 扩展以调用 `advanceUntilIdle()`，正确处理异步 `loadGifts` + WS 余额更新的测试顺序。

#### [HIGH] GiftReceivedEvent 缺少 `receiverAvatar` 字段，与协议不一致

**文件**：`core/ws/event/GiftEvents.kt`

**问题**：任务描述明确要求新增"完整广播字段"的 `GiftReceivedEvent`，但对照 §6.4.3 协议，接收者对象包含 `avatar` 字段：
```json
"receiver": { "user_id":"uuid", "nickname":"Bob", "avatar":null }
```
当前实现只有 `receiverUserId` + `receiverNickname`，缺少 `receiverAvatar: String?`。
虽然当前 `handleWsMessage` 尚未解析 `GiftReceived`（T-30031 接入），但该数据类已声称为"完整广播字段"的协议代表，在 T-30031 实现时将不得不补加该字段并修改所有调用点，增加回归风险。

**修复**：
```kotlin
// 修复后（补充缺失字段）
data class GiftReceivedEvent(
    val receiverUserId: String,
    val receiverNickname: String,
    val receiverAvatar: String?,   // ✅ 补充，对应协议 receiver.avatar（可为 null）
    ...
)
```

#### [MEDIUM-1] `buildSendGiftJson()` 使用字符串插值构造 JSON，存在注入风险

**文件**：`feature/gift/GiftPanelViewModel.kt`，第 291 行

**问题**：直接用 Kotlin 字符串模板拼接 JSON，若 `giftId`、`recipientId`、`roomId` 中含有 `"`、`\` 或换行符，会导致 JSON 格式错误甚至内容注入。项目已引入 Gson（`GiftPanelViewModel` import 了 `JsonParser`），应统一使用。

**修复**：
```kotlin
private fun buildSendGiftJson(job: SendGiftJob): String {
    val payload = com.google.gson.JsonObject().apply {
        addProperty("room_id", job.roomId)
        addProperty("gift_id", job.giftId)
        addProperty("receiver_id", job.recipientId)
        addProperty("count", job.count)
    }
    return com.google.gson.JsonObject().apply {
        addProperty("type", "SendGift")
        addProperty("msg_id", job.msgId)
        add("payload", payload)
    }.toString()
}
```

#### [MEDIUM-2] `Combo` data class 使用可变 `var` 字段，违反不可变性原则

**文件**：`feature/gift/ComboAggregator.kt`，第 36-42 行

**问题**：`count` 和 `lastTs` 声明为 `var`，导致 `data class` 的 `equals()`/`hashCode()`/`copy()` 语义不稳定；若未来在非 Main 线程调用，`c.count += unitCount` 存在竞态；与项目不可变架构约定不符。

**修复**：
```kotlin
data class Combo(
    val giftId: String,
    val recipientId: String,
    val msgId: String,
    val count: Int,   // ✅ val
    val lastTs: Long, // ✅ val
)
// press() 中改为：current = c.copy(count = c.count + unitCount, lastTs = now)
```
