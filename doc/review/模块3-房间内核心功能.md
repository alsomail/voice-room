- [ ] **缺陷 1**：[级别 P0] **Android ⇄ App Server WebSocket 消息协议不兼容（C2S 全线断裂）**
  - **文件与行号**：`app/android/app/src/main/java/com/voice/room/android/feature/room/RoomViewModel.kt:197-198, 227, 261, 322-324, 475-476, 618` ⇄ `doc/protocol/websocket_signals.md:17-26` ⇄ `app/server/src/ws/connection.rs` (IncomingMessage) + `app/server/src/room/handler.rs` (handle_take_mic / handle_send_message 等)
  - **问题说明**：协议 §6.3 规定通用格式为 `{ "type", "msg_id", "payload": { ... }, "timestamp" }`，服务端 `IncomingMessage` 也按此结构解析（`payload.mic_index` / `payload.content` / `msg_id` snake_case）。但 Android 客户端发送的 JSON 是 **顶层平铺 + camelCase**：
    - `JoinRoom`: `{"type":"JoinRoom","roomId":"...","msgId":"..."}` —— 缺少 `payload` 包裹、`msgId` 不是 `msg_id`，`access_token` 也直接拼在顶层。
    - `TakeMic`: `{"type":"TakeMic","roomId":"...","slotIndex":N}` —— 服务端读 `payload.mic_index`，**永远拿不到下标 → 上麦请求会报 40002 MISSING_PARAMS**。
    - `LeaveMic` / `ForceTakeMic` / `SendMessage` 同样缺 `payload` 包裹。`SendMessage` 服务端读 `payload.content`，客户端给的是顶层 `content` → **聊天消息发不出去**。
    - `LeaveRoom` 服务端使用连接上下文，可侥幸跑通，掩盖了协议差异。
  - **影响范围**：T-30010 / T-30013 / T-30015 全部核心房间内交互（上麦、下麦、聊天、强制上麦、加入密码房）在真实环境下均无法工作。当前测试通过完全是因为 `FakeWebSocketClient` 没有走真协议解析。
  - **修复建议**：（任选其一，但必须四端一致）
    1. 把 Android 的所有 `wsClient.send("...")` 调用改为按协议 §6.3 构造：使用 Gson/Moshi 序列化 `data class C2SEnvelope(val type:String, val msg_id:String, val payload: Any, val timestamp: Long)`。字段一律 snake_case。
    2. 严禁字符串拼接 JSON——参见缺陷 2。
    3. 增加一个端到端 Pact / contract test：Android 序列化的消息必须能被 `serde_json::from_str::<IncomingMessage>` 成功反序列化并取到关键字段。
  - **TDD 修复记录**：
    - 在 `app/android/app/src/main/java/com/voice/room/android/core/ws/WsEnvelope.kt` 新增 `WsEnvelope.build(type, payload, msgId)` + `IWebSocketClient.sendEnvelope` 扩展，统一封装为 `{type, msg_id, payload, timestamp}`（snake_case + Gson 序列化）。
    - 重写 `RoomViewModel.kt` 全部 9 个 `wsClient.send(...)` 站点：JoinRoom 含 `room_id/access_token?/last_msg_id?`；TakeMic `mic_index`；LeaveMic/LeaveRoom 空 payload（服务端用连接上下文）；SendMessage `content`；TransferAdmin `room_id/target_user_id/action`；ForceTakeMic `target_user_id/slot_index`；ForceLeaveMic `target_user_id`；MuteUser `target_user_id/type/duration_sec`（key 为 `type` 而非 `muteType`）；KickUser `target_user_id/reason`（Gson 转义）。
    - 测试：`RoomViewModelTest.kt` TM-01/TM-04/SM-02/VM-12 + UA40-09/09b-confirm/09c + KR41-07/KR41-07b×3 共 11 条全部按新协议改写并 GREEN；`GovernanceUiTest.kt` G44-05 LeaveMic 改为断言空 payload。
