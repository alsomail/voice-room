# TDS: WS ReportEvent 信令 + 写入服务 (Task ID: T-00023)

## 一、背景与目标
- **关联需求**：E-07.5 WS 通道复用上报 ([phase1_observability §2.3 决策 2](../../product/phase1_observability.md))
- **本 Task 的目标**：复用 T-00022 的 `EventWriter`，新增 WS `ReportEvent` 信令，客户端优先走 WS（省电省流量）。

## 二、方案设计

### WS 信令
**请求 C→S**
```json
{
  "type": "ReportEvent",
  "msg_id": "uuid",
  "payload": {
    "events": [ /* 同 HTTP batch 结构，单次 ≤100 */ ]
  }
}
```

**响应 S→C**
```json
{
  "type": "EventReportAck",
  "msg_id": "uuid",
  "code": 0,
  "payload": { "received": 98, "rejected_indices": [12, 45] }
}
```
错误码：`40204` BATCH_TOO_LARGE（超 100 时仍写前 100）、`40003` payload 非法。

### 处理逻辑（ws/handler.rs）
```rust
ReportEvent => {
    let events = payload.events;
    let user_id_from_jwt = conn_ctx.user_id;
    // 以 WS 连接携带的 JWT user_id 为准覆盖 client 上报
    let batch = events.into_iter().map(|e| EventInput {
        user_id: Some(user_id_from_jwt),  // WS 必然登录
        device_id: e.device_id,
        event_name: e.event_name,
        properties: e.properties,
        client_ts: e.client_ts,
        // server_ts 由 writer 覆盖
        ..
    }).collect();
    let result = event_writer.persist(batch, Some(user_id_from_jwt)).await?;
    send_ack(msg_id, result).await;
}
```

### 涉及文件
| 文件 | 类型 | 说明 |

| `app/server/src/ws/connection.rs` | 修改 | 新增 `ReportEvent` 路由分支 |

| `app/server/src/core/analytics/writer.rs` | 不改 | 已由 T-00022 抽象共用 |
| `app/server/tests/report_event_ws_test.rs` | 新增 | RE01~RE08 |

### 复用 HTTP 写入层
`handle_report_event` 与 HTTP handler 共调用 `EventWriter::persist`，代码零重复。

### 约束
- 单次 >100 返回 `BATCH_TOO_LARGE` 但仍写前 100
- `server_ts` 由 writer 统一覆盖
- 客户端给的 `user_id` 与 JWT 不一致时，log warn 以 JWT 为准（与 T-00022 一致）

## 三、TDD 验收用例
- [ ] **RE01** WS 连接认证后发 ReportEvent 1 event，DB 新增 1 条
- [ ] **RE02** 50 events 一次上报，DB 新增 50
- [ ] **RE03** 101 events → 返回 `BATCH_TOO_LARGE` 但 DB 新增 100，rejected_indices=[100]
- [ ] **RE04** server_ts 被覆盖为服务端时间（而非 client_ts）
- [ ] **RE05** client 伪造 user_id，DB 存 JWT user_id
- [ ] **RE06** 未认证 WS 不应能触达此 handler（认证中间件在前置）
- [ ] **RE07** payload.events 为空数组 → 40003
- [ ] **RE08** 与 HTTP 通道并行写入 1000 条无丢失
