# TDS: 魅力/财富榜单 API (Task ID: T-00021)

## 一、背景与目标
- **关联需求**：E-07 榜单页（T-30033）、驱动竞争氛围
- **本 Task 的目标**：读取 T-00020 维护的 Redis ZSet，返回日/周 Top N + 当前用户排名；支持跨日自动切换 + 旧榜归档。

## 二、方案设计

### 核心数据流
```
Android GET /ranking?type=charm&period=day&limit=50
   ↓
handler
   ├── key = ranking:{type}:{period}:{date|week}
   ├── top = ZREVRANGE key 0 limit-1 WITHSCORES
   ├── me_rank = ZREVRANK key {user_id}  / me_score = ZSCORE
   └── 批量 SELECT users WHERE id IN (top user_ids) 补头像昵称
  返回

定时任务（tokio-cron-scheduler，Asia/Riyadh TZ）
   - 每日 00:00  归档昨日日榜：ZUNIONSTORE ranking_archive:day:{yyyy-mm-dd} ← 昨日 key
   - 每周日 00:00（ISO Week 切换时刻）：归档上周 week key
   - 归档后保留原 key 48h（客户端过渡期仍可读），到期由 Redis TTL 自然清理
```

### HTTP 接口
**GET /api/v1/ranking?type=charm|wealth&period=day|week&limit=50** （需 JWT）
```json
{
  "code":0,
  "data":{
    "type":"charm", "period":"day",
    "period_key":"2026-04-21",   // 方便客户端判断是否跨日
    "items":[
      { "rank":1, "user_id":"uuid", "nickname":"...", "avatar":"...",
        "score":123456, "medal":"gold" }  // gold/silver/bronze/null
    ],
    "me": { "rank": 42, "score": 6800 } // 未入榜 rank=null
  }
}
```
- `type` 枚举：charm / wealth，其他返回 40003
- `period` 枚举：day / week
- `limit` 默认 50，范围 1-100
- 期望响应 <100ms（仅 1~2 次 Redis + 1 次 batch PG）

### 关键服务接口
```rust
impl RankingService {
    pub async fn top(&self, ty: RankingType, period: Period, limit: usize, viewer: Option<Uuid>)
        -> Result<RankingResponse>;
}
```
- 复用 T-00020 的 `ranking::key(...)` 生成工具
- 当前 `period_key` 使用 `chrono_tz::Asia::Riyadh` 本地化

### 归档与补偿
- `scheduler.rs` 使用 `tokio-cron-scheduler`，Cron：
  - `0 0 0 * * *`（每日 00:00 Riyadh）→ 归档前一日日榜
  - `0 0 0 * * 0`（ISO 周一 00:00 Riyadh）→ 归档上周周榜
- 启动时对比上次归档时间（存 Redis `ranking:last_archive:{type}:{period}`），若有落差则循环补偿到当前

### 关键实现决策

1. **测试隔离**：`top_by_key(key, limit, viewer)` 接受自定义 Redis key，集成测试使用 UUID 隔离的测试 key，不污染生产数据。
2. **UTC 时区一致性**：Redis key 日期使用 UTC（与 T-00020 一致，MVP 简化），避免跨模块不一致。`period_key` 响应字段同样使用 UTC 日期。
3. **归档幂等**：`ZUNIONSTORE` 天然幂等（重复执行会覆盖相同数据）；`ranking:last_archive_{type}:day` 记录进度防止重复归档。
4. **FakeRankingService**：返回空榜单，供 HTTP 参数校验测试使用（无需 Redis/PG）。
5. **批量用户查询**：使用 `WHERE id = ANY($1)` 一次性查询所有榜单用户信息，避免 N+1 问题。
6. **Scheduler 无外部 Cron 依赖**：使用 `tokio::time::sleep(3600s)` + `tokio::select!` 实现每小时检查，避免引入 `tokio-cron-scheduler` 外部依赖。
