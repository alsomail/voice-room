## 3. 三 Tab 行为差异

| Tab | 关系 | 主行动按钮 | ⋯ 菜单 |
|-----|-----|----------|--------|
| 关注 | 我 → TA | 进 TA 房间 / 私聊 | 取消关注、加好友、屏蔽、举报 |
| 粉丝 | TA → 我 | 回关 / 私聊 | 移除粉丝（双向）、屏蔽、举报 |
| 好友 | 互关 | 私聊 / 进房间 | 删除好友、屏蔽、举报 |

> **好友定义**: 双向关注 = 好友。互关动作发生时双方好友数 +1，自动同步。

## 4. 在线状态

| 状态 | 标识 | 来源 |
|------|------|------|
| 在房间中 | 🟢 + 房间名 + "进入TA房间" | WS `PresenceUpdate { user_id, room_id }` |
| 在线（未开房间） | 🟢 在线 | Presence service |
| 1 天 / 7 天 / 30 天前 | ⚪ + 时间戳 | last_seen 字段 |
| 隐身（贵族 LV5+） | 显示 ⚪ + last_seen 但永远 ≥ 30 分钟前 | server 强制脱敏 |

## 5. 操作行为

| 操作 | 行为 |
|------|------|
| 点头像 | 弹 `UserProfileSheet`（T-30090）|
| 取消关注 | 二次确认 + API + 本地数字 -1 + 列表移除 |
| 回关 | 立即 API + 该行 Tab 自动进入"好友" |
| 删除好友 | 提示"将同时取消互相关注" + 双向取消 |
| 屏蔽 | 进入屏蔽列表（Settings 可管理）+ 互相不可私聊 / 不可送礼 / 不可看公开主页 |
| 举报 | 跳举报页 + 上下文携带 user_id |
| 私聊 | 跳 T-30092（1v1 私聊页）|
| 进房间 | 切房（如已在房二次确认）|

## 6. 异常流程

| 场景 | 处理 |
|------|------|
| 网络失败 | 整页保留 + 重试 |
| 列表为空 | 各 Tab 不同空态文案：关注 → "去搜索感兴趣的人" / 粉丝 → "暂无粉丝，多互动吧" / 好友 → "添加好友 BTN" |
| 关注上限 (1000) | Toast "关注数已达上限，请先取消部分关注" |
| 对方注销 | 行变灰 + "用户已注销" + 仅可移除 |
| 对方屏蔽我 | 私聊按钮置灰 + Toast "对方设置不接受私聊" |
| 搜索无结果 | "未找到该用户" + 空态 |
| 实时关系变更 | WS `RelationChanged { actor, target, type, value }` 推送 → 列表局部刷新（不全量重拉） |

## 9. 协议入口索引

| 操作 | 协议 |
|------|------|
| 列表 | GET `/api/v1/users/me/relations?type=following\|followers\|friends&cursor=&limit=20` |
| 关注 / 取关 | POST `/api/v1/users/:id/follow` / DELETE 同 URL |
| 屏蔽 / 取消屏蔽 | POST `/api/v1/users/:id/block` / DELETE |
| 删除好友 | DELETE `/api/v1/users/me/friends/:id` |
| 搜索 | GET `/api/v1/users/search?q=` |
| 实时同步 | WS `RelationChanged` |
| 在线状态 | WS `PresenceUpdate` |

## 10. RTL / 埋点

- RTL：在线点在 end 侧（贴名字尾部），数字 LTR
- 埋点：`relation_view`(tab/count)、`relation_action`(action/target)、`relation_search`(query_len, result_count)、`relation_join_room`(target/from=relation)
