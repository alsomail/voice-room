# T-30090: Android 用户资料卡 (UserProfileSheet)

> **归属端**: Android
> **模块**: Social (E-12)
> **前置依赖**: T-00091/93 (Follow API)，T-30005 (NoblePort)，T-30043 (RoleBadge)
> **覆盖 Tasks**: T-30090（核心设计）；为 T-30091/92 提供入口规范

## 1. 背景

资料卡是关系链的"中央十字路口"——任何用户头像点击都应弹出统一的 `UserProfileSheet`。
统一可降低重复实现 + 保证关系状态全局一致。

## 2. 入口（必须统一）

| 来源 | 入口 | from 埋点值 |
|------|------|-------------|
| 房间内麦位 / 观众席头像 | 长按 / 单击 | `room_seat` / `room_audience` |
| 房间公屏聊天用户气泡 | 点击头像 | `room_chat` |
| 榜单 / 财富榜 | 点击行 | `rank` |

| 好友/粉丝/关注列表 | 点击行 | `relations_list` |
| 家族成员列表 | 点击行 | `family_members` |
| 系统通知 deep link | URL | `notification` |

## 6. 异常流程

| 场景 | 处理 |
|------|------|
| 网络失败 | Skeleton + 重试按钮 |
| 用户已注销 | 全屏占位"用户不存在或已注销" |
| 关注超日上限 200 | Toast "今日关注次数已用完" |
| 特别关注超 50 | Toast "特别关注最多 50 人，请先取消" + 跳列表 |
| 陌生人 1v1 当日超 5 条 | 私聊页顶部黄条 + 发送按钮置灰 |
| 被对方拉黑后点关注 | 静默 403 + 通用文案"暂时无法关注该用户" |
| 自己点关注（异常路径） | 客户端拦截 |
| 关注按钮多次点击 | 1s 去抖 + 服务端幂等 |

## 9. ViewModel

```kotlin
data class UserProfileUiState(
    val userId: Long,
    val from: ProfileEntrySource,
    val profile: UserProfileVO? = null,
    val relation: RelationState = RelationState.Unknown,  // Self/NotFollowed/FollowedOneWay/MutualFriend/IBlocked/BlockedByPeer
    val isStarred: Boolean = false,
    val canDm: Boolean = false,
    val canDmReason: String? = null,           // 陌生人限流 / 拉黑
    val phase: LoadPhase = LoadPhase.Loading,
    val errorMessage: String? = null,
    val pendingAction: PendingAction? = null    // Follow/Unfollow/Star/Block 进行中
)

sealed class UserProfileEvent {
    data class Toast(val msg: String) : UserProfileEvent()
    data class JumpToRoom(val roomId: Long) : UserProfileEvent()
    data class JumpToDm(val peerId: Long) : UserProfileEvent()
    data class JumpToRelations(val type: String) : UserProfileEvent()
    object Dismissed : UserProfileEvent()
}
```

## 10. 协议入口索引

| 操作 | 协议 |
|------|------|
| 拉资料 | GET `/api/v1/users/:id/profile` |
| 拉关系状态 | GET `/api/v1/users/:id/relation` |
| 关注 | POST `/api/v1/follows/:id` |
| 取关 | DELETE `/api/v1/follows/:id` |
| 特别关注 | PUT `/api/v1/follows/:id/star` |
| 拉黑 | POST `/api/v1/users/:id/block` |
| 解除拉黑 | DELETE `/api/v1/users/:id/block` |
| 关系变更广播 | WS `FollowChanged { follower_id, followed_id, action }` |
| 好友升级 | WS `FriendCreated { peer_id }` |
| 拉黑变更 | WS `BlockChanged` |

## 11. RTL 兼容

- 数据条胶囊布局自动镜像
- 主操作栏排序保持视觉"主→次"（关注 → 特别关注 → 私聊）的视觉权重，RTL 反转后仍主在右
- 在线绿点位置 `start` 替代 `end`

## 12. 埋点

| 事件 | 时机 | 字段 |
|------|------|------|
| `profile_view` | Sheet 打开成功 | `target_user_id`, `from`, `relation`, `is_self` |
| `profile_follow` | 关注成功 | `target_user_id`, `from`, `to_friend`(bool) |
| `profile_unfollow` | 取关成功 | `target_user_id`, `was_friend`(bool) |
| `profile_star_toggle` | 特别关注切换 | `target_user_id`, `value`(bool) |
| `profile_dm_click` | 点击私聊 | `target_user_id`, `is_friend`(bool), `blocked_by_limit`(bool) |
| `profile_block` | 拉黑 | `target_user_id`, `reason`(optional) |
| `profile_room_entry` | 进 TA 的房间 | `target_user_id`, `room_id` |
| `profile_share` | 分享 | `target_user_id`, `channel` |

## 13. 性能

- Sheet 首次展开预拉 profile + relation 并行
- 列表/搜索结果场景 prefetch 上下两屏 profile（避免点开等待）
- WS `FollowChanged` 推送下，所有打开中的 Sheet 自动刷新对应 userId
