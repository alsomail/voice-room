# T-10006 — Admin Server 强制关闭房间接口

## §1 接口规格

```
DELETE /api/v1/admin/rooms/:id
Authorization: Bearer <admin-jwt>
```

### 路径参数

| 参数 | 类型 | 说明 |
|------|------|------|
| id | string (UUID) | 房间 ID |

### 成功响应（200）

```json
{
  "code": 0,
  "message": "ok",
  "request_id": "xxx",
  "data": null
}
```

### 错误码

| HTTP | code  | 说明 |
|------|-------|------|
| 400  | 40003 | 路径参数 `id` 不是合法 UUID |
| 401  | 40101 | 无 Token / Token 无效 |
| 401  | 40102 | Token 已过期 |
| 403  | 40301 | 角色无 `RoomForceClose` 权限（cs / finance） |
| 404  | 40400 | 房间不存在或已软删除 |
| 409  | 40901 | 房间已处于 closed 状态 |

## §2 技术设计

### §2.1 HTTP Method

使用 `DELETE` 语义：对资源施加不可逆操作（关闭房间）。

### §2.2 路由注册

```
/api/v1/admin/rooms/{id}
  GET    → get_room_detail_handler   (T-10005)
  DELETE → force_close_room_handler  (T-10006)
```

### §2.3 SQL（Postgres）

```sql
UPDATE rooms
SET status = 'closed', updated_at = NOW()
WHERE id = $1
  AND deleted_at IS NULL
```

- 不做状态前置过滤（`closed` 状态的幂等写由 Service 层拦截）
- 软删除房间通过 `deleted_at IS NULL` 自动忽略

### §2.4 Service 逻辑

```rust
pub async fn force_close_room(&self, room_id: Uuid) -> Result<(), AppError> {
    // 1. 查询（find_room_by_id_any_status，不做 owner 检查）
    let room = self.room_repo.find_room_by_id_any_status(room_id).await?
        .ok_or_else(|| AppError::NotFound(format!("room {room_id}")))?;
    // 2. 状态前置校验
    if room.status == "closed" {
        return Err(AppError::RoomAlreadyClosed);
    }
    // 3. 持久化
    self.room_repo.set_room_closed(room_id).await
}
```

### §2.5 权限设计（RoomForceClose）

新增 `Permission::RoomForceClose` 枚举变体，RBAC 矩阵如下：

| 角色        | RoomForceClose |
|-------------|---------------|
| super_admin | ✅            |
| operator    | ✅            |
| cs          | ❌            |
| finance     | ❌            |

**设计原因**：cs 角色定位为客服，只能查看，不能施加不可逆的强制操作；
finance 角色无房间管理权限。
