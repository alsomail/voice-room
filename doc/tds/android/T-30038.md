# TDS: 密码房进房弹窗 (Task ID: T-30038)

### 弹窗交互
```kotlin
@Composable
fun PasswordInputDialog(
    state: PasswordDialogState,   // idle / verifying / error / locked
    onSubmit: (String) -> Unit,
    onDismiss: () -> Unit
)
```
- 6 个 OutlinedTextField 单字符分格，输入自动跳下一格；最后一格输完自动 submit
- `state=error`：底部红字"密码错误，剩余 N 次"
- `state=locked`：弹窗变只读 + "已被锁定，{remaining_min} 分钟后重试"
- 返回键 `onDismiss` 关闭

### Flow
```
Hall: 点击 has_password=true 的房 card
  ↓
HallViewModel.openPasswordDialog(roomId)
  ↓ 用户输 6 位
verifyPassword(roomId, {password}) → 成功：获 access_token
  ↓
Navigate to RoomScreen(roomId, accessToken)
  ↓
RoomViewModel 发 JoinRoom { access_token } 经 WS
```

### 错误处理
| HTTP code | UI 反馈 |
|-----------|--------|
| 40103 | 红字 + 剩余次数；清空输入 |
| 42910 | 切 locked 状态，显示剩余分钟 |
| 40400 | Toast "房间不存在" + 关闭 |
| 其它 | Toast "网络错误，请重试" |

## 五、Reviewer意见
### Round 1
- **状态**：❌ 未通过（TDD）
- **审查意见**：
  1. **[HIGH] Locked 状态弹窗无法关闭**：`isReadOnly` 同时包含 `Verifying` 和 `Locked`，导致 `onDismissRequest` 被拦截、取消按钮置灰，用户在锁定状态下无法返回大厅，违反 TDS "返回键 `onDismiss` 关闭" 约定及 Android 导航规范。
  2. **[MEDIUM] LaunchedEffect 使用 Boolean key**：`state is PasswordDialogState.Error` 作为 key，依赖"必经 Verifying 中间态"保证 key 翻转；若未来 Error→Error 直接转换，清空逻辑静默失效，应改用 `state` 本身作 key。
  3. **[MEDIUM] 错误码解析 fallback 为 0**：`remaining_attempts`/`remaining_minutes` 缺失时默认 `?: 0`，显示"剩余 0 次"/"0 分钟后重试"，产生误导；应使用安全 fallback（如 1 次/30 分钟）。
  4. **[LOW] 死代码**：`if (char.length > 1)` 永远为 false（`lastOrNull().toString()` 长度 ≤ 1），注释"过滤粘贴多字符"名实不符。
- **改进建议**：
  - 将 `isDismissBlocked` 仅限于 `Verifying`，`Locked` 时保持取消按钮可用并响应返回键
  - `LaunchedEffect` 改用 `state` 作 key，用单个 `when` 处理 Idle/Error 清空逻辑
  - 错误解析 fallback：`remaining_attempts ?: 1`、`remaining_minutes ?: 30`
  - 删除 `if (char.length > 1)` 死代码行
