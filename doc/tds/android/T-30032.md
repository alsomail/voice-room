# TDS: 余额不足引导弹窗 (Task ID: T-30032)

## 一、背景与目标
- **关联需求**：E-07 余额不足转化 → 未来充值入口 ([设计文档](../../design/android/T-30032.md))
- **本 Task 的目标**：点送出时服务端返回 40290 或本地判 balance<total 时弹 `InsufficientBalanceDialog`，引导进入 WalletScreen。

## 二、方案设计

### Dialog 设计
- 标题："钻石不足" / "رصيدك غير كافٍ"
- 正文：
  - "当前余额：💎 {current}"
  - "所需：💎 {required}"
  - "差：💎 {required - current}"
- 按钮：[取消] [去充值 GoldButton]
- `dismissOnClickOutside=false`, `dismissOnBackPress=true`
- 点"去充值" → `navController.navigate("wallet")` + 关闭礼物面板
- 点"取消" → 关闭 Dialog，礼物面板保留选中态

### 触发场景
1. Server 返回 `code=40290` (INSUFFICIENT_BALANCE)
2. 本地按钮点送出前已判断 balance<total（按钮本应 disabled，保守双重拦截）

### testTag
`Key('dialog_insufficient_balance')`, `Key('btn_goto_wallet')`, `Key('btn_insufficient_cancel')`
