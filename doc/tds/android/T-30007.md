### 关键接口定义

#### `IRoomRepository` 扩展

```kotlin
suspend fun createRoom(title: String, type: String, password: String?): Result<String>
```

#### `CreateRoomUiState`

```kotlin
sealed interface CreateRoomUiState {
    object Idle : CreateRoomUiState
    object Loading : CreateRoomUiState
    data class Success(val roomId: String) : CreateRoomUiState
    data class Error(val message: String) : CreateRoomUiState
}
```

#### `CreateRoomViewModel` 核心逻辑

```kotlin
fun createRoom(title: String, type: String, password: String? = null) {
    if (_uiState.value is CreateRoomUiState.Loading) return   // 幂等保护
    val validationError = validate(title, type, password)
    if (validationError != null) {
        _uiState.value = CreateRoomUiState.Error(validationError)
        return
    }
    viewModelScope.launch {
        _uiState.value = CreateRoomUiState.Loading
        repository.createRoom(title.trim(), type, password)
            .onSuccess { roomId -> _uiState.value = CreateRoomUiState.Success(roomId) }
            .onFailure { error -> _uiState.value = CreateRoomUiState.Error(error.message ?: "创建失败") }
    }
}
```

| 标题非空 | `title.isBlank()` | "房间标题不能为空" |
| 标题 ≤ 30 Unicode 字符 | `codePointCount > 30` | "房间标题不能超过 30 个字符" |
| 密码房必须有密码 | `type=="password" && password.isNullOrBlank()` | "密码房间必须设置密码" |

### 设计选择说明

- **`@SerializedName`（Gson）**：与项目现有 DTO（`RoomItemDto.kt`、`ApiResponse.kt` 等）保持一致，未使用 Jackson `@JsonProperty`
- **`codePointCount`**：按 Unicode 码点计算字符数，保证中文 / emoji 等多字节字符计数正确（与 protocol.md §3.1 "1 个中文字符 = 1 个字符"规范匹配）
- **校验在 ViewModel 主线程执行**：`validate()` 是纯函数，无 IO，无需 suspend；先校验再 `launch`，确保校验失败时不产生协程
- **Loading 幂等检查在 `launch` 之前**：避免连续点击在协程启动前绕过检查
- **`viewModelFactory {}` DSL**：与 `lifecycle-viewmodel-ktx:2.8.6` 配套，用于 Fragment/Activity 通过 `by viewModels { CreateRoomViewModel.Factory }` 获取 ViewModel

### Round 1：🔴 未通过

**审查时间**：2025-01-22  
**审查结论**：发现 **2 个 HIGH 问题**，阻断合并，须修复后重新提交 Review。

---

#### [HIGH] H-01：`AppContainer` 中 `POST /api/v1/rooms` 使用无鉴权 Retrofit 客户端，所有创建请求将返回 401

- **位置**：`AppContainer.kt:55-61`
- **问题**：`roomRetrofit` 构建时使用 `AppHttpClientFactory.create()`（无 `authInterceptor`），注释遗留自 T-30006（GET /rooms 公开接口）。T-30007 新增的 `POST /rooms` 需要 JWT 鉴权（`protocol.md §3.1`），但 `AuthInterceptor` 未注入，运行时所有 `createRoom()` 调用将返回 `HTTP 401`。
- **修复**：为 `roomRetrofit` 注入 `AuthInterceptor`（`AppHttpClientFactory.create(authInterceptor = ...)`)，或将公开与认证 API 拆分为两个 Retrofit 实例。

#### [HIGH] H-02：`CreateRoomViewModel` 缺少 `resetState()` — BottomSheet 重新打开时 `LaunchedEffect` 触发旧 `Success` 回调，导致重复导航

- **位置**：`CreateRoomViewModel.kt`（全文）、`CreateRoomBottomSheet.kt:66-71`
- **问题**：成功后 `uiState` 永久停留在 `Success(roomId)` 状态，没有重置路径。当用户关闭 BottomSheet 后再次打开时，`LaunchedEffect(uiState)` 在新 Composition 中再次执行，`onSuccess(roomId)` 被重复调用，触发重复导航至旧房间，且用户无法创建第二个房间。同样问题存在于 Error 状态重开时显示残留错误。
- **修复**：① ViewModel 添加 `fun resetState() { _uiState.value = CreateRoomUiState.Idle }`；② `CreateRoomBottomSheet` 中 `LaunchedEffect(Unit)` 进入时调用 `viewModel.resetState()`；③ 在 `onSuccess` 回调前先 reset；④ 补充 `resetState` 测试用例。

---

#### [MEDIUM] M-01：`MAX_TITLE_LENGTH` 常量在 ViewModel 和 UI 层各定义一份，存在漂移风险
- **位置**：`CreateRoomViewModel.kt:107`、`CreateRoomBottomSheet.kt:96`
- **修复**：提升为 `internal const val`，UI 层直接引用 ViewModel 中的定义。

#### [MEDIUM] M-02：`onValueChange` 中 `MAX_TITLE_LENGTH + 1` 是无注释魔数
- **位置**：`CreateRoomBottomSheet.kt:129`
- **修复**：添加注释说明 `+1` 的 UX 意图（允许超出 1 个字符以显示 isError 提示），或去掉硬截断完全依赖 `isError` 和 ViewModel 校验兜底。

#### [MEDIUM] M-03：`LaunchedEffect(uiState)` 对相同错误消息不重新触发，连续相同错误无刷新
- **位置**：`CreateRoomBottomSheet.kt:66`
- **修复**：对导航类一次性事件，推荐改用 `Channel<Event>` / `SharedFlow`（one-shot event 模式）替代 `StateFlow` + `LaunchedEffect(state)`。

---

#### [LOW] L-01：fallback 错误文案硬编码在 ViewModel，不利于 i18n
- **位置**：`CreateRoomViewModel.kt:70`，建议提升为 `internal const`。

#### [LOW] L-02：RadioButton 组缺少无障碍语义标注（TalkBack 无法关联"房间类型"上下文）
- **位置**：`CreateRoomBottomSheet.kt:146-180`，建议添加 `semantics { contentDescription = "房间类型" }`。

#### [LOW] L-03：`FakeRoomRepository.shouldFail` 单一 flag 无法区分各方法的独立失败场景
- **位置**：`FakeRoomRepository.kt:20-21`，建议拆分为 `getRoomsShouldFail` / `createRoomShouldFail`。
