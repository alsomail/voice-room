## 一、背景与目标

### 关联需求
- 模块 9：E2E 测试基建。修复 T-0000E 体检结论 **C4：AdminServer 缺 `config/*.toml`**——当前 `app/adminServer/src/main.rs` 完全裸 `std::env::var`（line 40~46/56~67），无 toml 分层、无 profile 切换、缺字段裸 panic（`expect("DATABASE_URL must be set")`），与 AppServer 严重不对称，违反 T-0000E §4.3 第 5 行契约。

## 二、方案设计

### 2.1 核心数据流（启动期 config 加载链，与 T-00040 §2.1 同构）

```
                    cargo run / docker run
                            │
                            ▼
         AdminSettings::load()  ──┐
                            │     │ 1. dotenvy::from_path(.env)            （沿用 main.rs:29 行为）
                            │     │ 2. 解析 ADMIN_PROFILE（新）
                            │     │      ├─ env: ADMIN_PROFILE > ADMIN_ENV > ADMIN__ENVIRONMENT
                            │     │      └─ 默认 "dev"，白名单 {dev,test,staging,prod}
                            │     │ 3. 校验 profile 合法性 → 不合法 exit 78
                            │     │ 4. settings = default_for(profile)
                            │     │ 5. apply(load_config_file("default.toml"))
                            │     │ 6. apply(load_config_file("{profile}.toml"))
                            │     │ 7. apply_env_overrides()
                            │     │     （ADMIN__SERVER__*, ADMIN__LOG__*, ADMIN__JWT__*,
                            │     │      ADMIN__DATABASE__*, ADMIN__STORAGE__*；
                            │     │      兼容别名：PORT → server.port、GIFT_UPLOAD_DIR → storage.gift_upload_dir）
                            │     │ 8. 注入敏感字段：
                            │     │      ├─ DATABASE_URL → settings.database.url（必填，缺则 exit 78）
                            │     │      ├─ JWT_SECRET 或 ADMIN_JWT_SECRET → require_admin_jwt_secret()
                            │     │      │   （后者优先；缺/空/占位符 → exit 78）
                            │     │      └─ REDIS_URL    → settings.redis_url
                            │     │           ├─ profile=dev：允许缺失 → 保留 None；main.rs 装 NoopEventPublisher + WARN（0 回归）
                            │     │           └─ profile≠dev：缺失 → exit 78
                            │     │ 9. log_summary()：tracing::info!(profile, host, port, jwt_secret_len,
                            │     │       redis_url_redacted/none, db_url_redacted, storage.gift_upload_dir, ...)
                            ▼     │
                  AdminSettings   ▲
                            │     │ 任一步骤失败：
                            ▼     │   eprintln!("CONFIG ERROR: …") + std::process::exit(78)
                       main.rs    │   （EX_CONFIG, sysexits.h；与 T-00040 §2.5 完全一致）
                            │     │
                            ▼
                  pool / redis / event_publisher / state / serve …
```

**关键不变量**（与 T-00040 §2.1 文字镜像一致）：
- 加载链顺序 **与 T-0000E §2.2 严格一致**：`default → {profile}.toml → ENV`，env 优先级最高。
- 任何字段在 `default.toml` 都有合理默认（即使会被 profile 或 env 覆盖）。
- 敏感字段（`DATABASE_URL` `REDIS_URL` `JWT_SECRET` / `ADMIN_JWT_SECRET`）**永不写入 TOML**，仅由 ENV 注入；TOML 中相应位置以注释标注「由 ENV 注入」。
- 启动期 fail-fast 路径**统一退出码 78**（EX_CONFIG），错误前缀 `CONFIG ERROR:`，与 T-00040 / preflight.sh / RUNBOOK 形成一致 grep 锚点。
- `dev` profile 允许 `REDIS_URL` 缺失保留 `NoopEventPublisher`，是为了 0 回归当前裸 `cargo run`/单测路径（详见 §2.8 D-A1 与现状对照）。

### 2.4 加载链（详细步骤草图，与 T-00040 §2.4 同构）

```rust
// app/adminServer/src/infrastructure/config.rs（新签名草图，仅供说明）
impl AdminSettings {
    pub fn load() -> anyhow::Result<Self> {
        // Step 1: dotenv（沿用现有 main.rs:29 行为）
        load_dotenv();

        // Step 2: 解析 profile 入口
        let profile = Profile::from_env()?; // ADMIN_PROFILE > ADMIN_ENV > ADMIN__ENVIRONMENT > "dev"

        // Step 3: 白名单校验（dev/test/staging/prod）
        // Profile::from_env 内部 fail，错误信息：
        //   CONFIG ERROR: invalid ADMIN_PROFILE='xxx'; expected one of [dev,test,staging,prod]

        // Step 4: 默认值
        let mut settings = Self::default_for(profile);

        // Step 5/6: TOML 分层
        let dir = config_dir();
        settings.apply(load_config_file(dir.join("default.toml"))?);
        settings.apply(load_config_file(dir.join(format!("{profile}.toml")))?);

        // Step 6.5: 强制 profile 入口覆盖 toml 的 app.environment（D4 镜像）
        settings.app.environment = profile.as_str().to_owned();

        // Step 7: ENV override（非敏感字段 + 兼容别名 PORT / GIFT_UPLOAD_DIR）
        settings.apply_env_overrides();

        // Step 8: 注入敏感字段（fail-fast）
        settings.database.url = Some(require_env(
            "DATABASE_URL",
            &format!("DATABASE_URL must be set; refused to start (profile={profile})"),
        )?);
        settings.jwt_secret = require_admin_jwt_secret(
            env::var("ADMIN_JWT_SECRET").ok(),
            env::var("JWT_SECRET").ok(),
        )?;
        settings.redis_url = match env::var("REDIS_URL") {
            Ok(v) if !v.trim().is_empty() => Some(v),
            _ if profile.allow_redis_optional() => {
                tracing::warn!(
                    "REDIS_URL not set on profile={profile}; admin server will fall back to NoopEventPublisher"
                );
                None
            }
            _ => anyhow::bail!(
                "REDIS_URL must be set for non-dev profile (profile={profile})"
            ),
        };

        // Step 9: 启动摘要日志（脱敏）
        settings.log_summary();
        Ok(settings)
    }
}
```

`main.rs` 在调用 `AdminSettings::load()` 之后，对 `anyhow::Error` 做统一拦截：

```rust
fn fatal_config(err: anyhow::Error) -> ! {
    eprintln!("CONFIG ERROR: {err:#}");
    std::process::exit(78);
}

let settings = AdminSettings::load().unwrap_or_else(fatal_config);
```

随后 `let database_url = settings.database.url.expect("injected by load()");`、`let port = settings.server.port;`、`let jwt_secret = settings.jwt_secret;`、`let event_publisher = match settings.redis_url { Some(url) => Real, None => Noop };`——`expect()` 仅作内部不变量断言（已被 load() fail-fast 兜底），与 T-00040 LOW-L1 相同模式。

### 2.5 错误契约（fail-fast 错误信息表，与 T-00040 §2.5 形状一致）

| 触发条件 | stderr 前缀 | 错误正文（≤ 100 字） | 退出码 |
|---|---|---|---|
| `ADMIN_PROFILE` 不在白名单 | `CONFIG ERROR: ` | `invalid ADMIN_PROFILE='xxx'; expected one of [dev,test,staging,prod]` | 78 |
| `DATABASE_URL` 缺失 / 空 | `CONFIG ERROR: ` | `DATABASE_URL must be set; refused to start (profile=staging)` | 78 |
| `ADMIN_JWT_SECRET` 与 `JWT_SECRET` 均缺失 | `CONFIG ERROR: ` | `ADMIN_JWT_SECRET (or JWT_SECRET fallback) must be set; refusing to start` | 78 |
