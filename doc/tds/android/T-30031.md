### GiftEffectController 设计
```kotlin
class GiftEffectController @Inject constructor(
    private val wsBus: WebSocketEventBus,
    private val lottieCache: LottieCache
) {
    val fullscreenEffect: StateFlow<FullscreenAnim?> = MutableStateFlow(null)
    private val queue = Channel<GiftReceivedEvent>(Channel.BUFFERED)

    init {
        // 同时只播 1 个 L3，其他排队或丢弃（MVP 最多排 3 个）
        CoroutineScope(Dispatchers.Main).launch {
            wsBus.events.filterIsInstance<GiftReceivedEvent>().collect { evt ->
                if (evt.effectLevel >= 4) queue.trySend(evt)
            }
        }
        CoroutineScope(Dispatchers.Main).launch {
            for (evt in queue) {
                fullscreenEffect.value = FullscreenAnim(evt.animationUrl!!, evt.duration)
                delay(evt.duration)
                fullscreenEffect.value = null
            }
        }
    }
}
```

### L1 GiftChatMessage 样式
- 金色昵称（MenaTheme.Gold）
- 头像小图 16dp
- 礼物图标 24dp
- 文字："送给 {receiver} × {count}"
- 粗体（effect_level >= 3）
- 连击不新增弹幕，累加 count 更新同一 msg_id 的弹幕（用 msg_id 作 LazyColumn key 去重）

### L2 麦位光圈
- 在 MicSlot 组件外套 Modifier `.giftGlow(active=true, durationMs=2000)`
- 金色光圈 Scale 1.0→1.2→1.0 循环 2 次，然后 `active=false`
- 接收 GiftReceivedEvent 后根据 `receiver.user_id` 找对应 MicSlot 触发

### L3 全屏 Lottie
- 全屏 Box，半透明黑色背景(0.3)，中央 LottieAnimation
- 用户点击跳过 → 立即置 null
- Lottie JSON 由 animation_url 下载，首次从 CDN 拉（超时 3s），失败 fallback 本地动画

### 补偿消息（重连后）
- 重连时通过 `GetRoomSnapshot`（未定义？后续补）拿最近礼物记录；或由 Server 在 JoinRoom response 带 `recent_gifts[]`
- 补偿消息在 L1 弹幕里以普通样式驻列，**L2/L3 不回放**

`Key('fullscreen_gift_overlay')`, `Key('gift_msg_{msgId}')`, `Key('btn_skip_fullscreen_gift')`

### 架构要点
- **防腐层**：Lottie SDK 隔离在 `ILottiePlayer` 接口后，业务层零 Lottie 依赖
- **testability**：`GiftEffectController(scope)` 构造参数注入，测试用 `backgroundScope` 消除 `UncompletedCoroutinesError`
- **L1/L2 同步**：`onGiftReceived()` 内同步更新 StateFlow，emit 后即可 assert，无需 advance
- **L3 队列**：`Channel(capacity=3)` + `for (evt in l3Queue)` 顺序播放，`skipFullscreen()` cancel delay job 实现跳过

#### HIGH 问题（必须修复后重新提审）

**[HIGH-1] GiftReceived JSON 字段名不匹配 — 所有 GiftReceived 事件将被静默丢弃**
- **文件**：`feature/room/RoomViewModel.kt` L462-464
- **问题**：`giftObj.get("giftId")` 和 `giftObj.get("giftCode")` 与协议 §6.4.3 及 GiftEvents.kt 注释中的 `"id"` / `"code"` 不一致。`id`→`giftId` 并非 snake_case→camelCase 变换，而是添加了 `gift` 前缀的非标准变换。若服务端按协议发送 `"id"`，`giftObj.get("giftId")` 返回 null，`?: return` 导致所有 GiftReceived 事件被静默丢弃，整个 T-30031 特效在生产环境完全失效。
- **修复**：将 `giftObj.get("giftId")` 改为 `giftObj.get("id")`，`giftObj.get("giftCode")` 改为 `giftObj.get("code")`；并与服务端确认实际字段名，更新 GiftEvents.kt 注释保持三端一致。

**[HIGH-2] GiftFullscreenOverlay 和 GiftChatMessage 缺少 TDS 规定的 testTag**
- **文件**：`feature/room/components/GiftEffectComponents.kt` L108-128、L41-86
- **问题**：TDS §testTag 规定 `Key('fullscreen_gift_overlay')`、`Key('btn_skip_fullscreen_gift')`、`Key('gift_msg_{msgId}')`，三个 testTag 均未实现。UI 自动化测试无法定位这些组件，E31-03/E31-05 的 UI 级验收无法进行。
- **修复**：在 `GiftFullscreenOverlay` 外层 Box 添加 `Modifier.semantics { testTag = "fullscreen_gift_overlay" }`，在点击 Box 添加 `Modifier.semantics { testTag = "btn_skip_fullscreen_gift" }`，在 GiftChatMessage Box 添加 `Modifier.semantics { testTag = "gift_msg_${msg.msgId}" }`。

#### MEDIUM 问题（建议在本次迭代修复）

**[MEDIUM-1] ILottiePlayer 接口未注入任何组件，防腐层仅在文件层面存在**
- `GiftEffectController` 和 `GiftFullscreenOverlay` 均未使用 `ILottiePlayer`，`preload()`/`isCached()` 无调用点。TDS 描述的"CDN 拉取超时 3s→fallback 本地动画"逻辑缺失。建议将 `ILottiePlayer` 注入 `GiftEffectController` 构造参数，在 `playL3()` 内实现 preload 逻辑。

**[MEDIUM-2] L2 麦位动画效果与 TDS 规格不符**
- TDS 要求"Scale 1.0→1.2→1.0 循环 2 次"，实现为 alpha 透明度边框脉冲（`infiniteRepeatable`，无缩放）。视觉效果与规格不同。建议改用 scale 动画或至少限制循环次数。

**[MEDIUM-3] GiftChatMessage 未渲染 senderAvatar 和 giftIconUrl，头像尺寸不符规格**
- TDS：头像 16dp + 礼物图标 24dp。实现：24dp 静态白色占位（未使用 `msg.senderAvatar`），无礼物图标（`msg.giftIconUrl` 存于模型但未渲染）。

#### LOW 问题（下次迭代可修复）

**[LOW-1]** `GiftChatMessage` 使用硬编码 `12.sp`/`16.sp`，应改用 `MenaTheme.typography.*`。
**[LOW-2]** 颜色常量如 `Color(0xCC_FF_AA_00)`、`Color(0x99_00_00_00)`、`Color(0xFF_FF_D7_00)` 内联多处，应收敛到 `MenaColors` 主题常量，`0xFF_FF_D7_00` 在 MicGlowModifier 和 GiftEffectComponents 各定义一次，存在重复。
