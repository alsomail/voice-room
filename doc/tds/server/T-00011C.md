## 一、功能描述

實時維護 Redis 在線統計數據，供 Admin Server 讀取展示。

核心能力：
1. **用戶上線**：`PFADD stats:online_users {user_id}` — HyperLogLog 估算在線人數
2. **用戶下線**：HyperLogLog append-only，no-op + debug log
3. **進入房間**：`SADD stats:active_rooms {room_id}` — Set 精確統計活躍房間
4. **離開房間**：`SREM stats:active_rooms {room_id}`
5. **每分鐘快照**：`HSET stats:snapshot:{date}:{HH:MM}` + `EXPIRE 604800`（7 天）

## 二、技術設計

### 文件結構

```
src/stats/
├── mod.rs           — 模塊入口，re-export StatsPort / StatsService / FakeStatsService
├── service.rs       — StatsPort trait + StatsService (Redis) + FakeStatsService (Fake)
└── snapshot_task.rs — 定時快照 tokio task
```

### Redis Key 設計

| Key 模式 | 數據結構 | 說明 |
|----------|----------|------|
| `stats:online_users` | HyperLogLog | 估算在線用戶數，PFADD/PFCOUNT |
| `stats:active_rooms` | Set | 精確活躍房間 ID，SADD/SREM/SCARD |
| `stats:snapshot:{date}:{HH:MM}` | Hash | 快照字段：`online_users`, `active_rooms`；TTL 7 天 |

### StatsPort trait

```rust
#[async_trait::async_trait]
pub trait StatsPort: Send + Sync {
    async fn user_online(&self, user_id: uuid::Uuid) -> Result<(), AppError>;
    async fn user_offline(&self, user_id: uuid::Uuid) -> Result<(), AppError>;
    async fn user_join_room(&self, room_id: uuid::Uuid) -> Result<(), AppError>;
    async fn user_leave_room(&self, room_id: uuid::Uuid) -> Result<(), AppError>;
    async fn get_online_count(&self) -> Result<u64, AppError>;
    async fn get_active_room_count(&self) -> Result<u64, AppError>;
    async fn take_snapshot(&self) -> Result<(), AppError>;
}
```

### StatsService（真實 Redis 實現）

- `user_online`：`PFADD stats:online_users {user_id}`
- `user_offline`：no-op + debug log（HLL append-only）
- `user_join_room`：`SADD stats:active_rooms {room_id}`
- `user_leave_room`：`SREM stats:active_rooms {room_id}`
- `get_online_count`：`PFCOUNT stats:online_users` → u64
- `get_active_room_count`：`SCARD stats:active_rooms` → u64
- `take_snapshot`：讀取兩計數 → pipeline `HSET + EXPIRE`

### FakeStatsService（測試用）

```rust
#[derive(Default)]
pub struct FakeStatsService {
    pub online_users: std::sync::Mutex<std::collections::HashSet<uuid::Uuid>>,
    pub active_rooms: std::sync::Mutex<std::collections::HashSet<uuid::Uuid>>,
    pub snapshot_calls: std::sync::atomic::AtomicU32,
}
```

- `user_offline` 同樣為 no-op（對齊 HLL append-only 語義）
- `take_snapshot` 僅遞增 `snapshot_calls` 計數器

### snapshot_task

```rust
pub async fn snapshot_task(
    stats: Arc<dyn StatsPort>,
    interval_duration: Duration,
    mut shutdown: tokio::sync::watch::Receiver<bool>,
)
```

- 使用 `tokio::time::interval` + `tokio::select!`
- 快照失敗僅記 warn 日誌，不退出
- 收到 `shutdown=true` 後優雅退出

### AppState 更新

新增字段：
```rust
pub stats_service: Arc<dyn StatsPort>
```

`for_test()` / `for_test_with_room_repo()` 均注入 `Arc::new(FakeStatsService::default())`。

### WS 集成

`handle_socket` 新增參數 `stats: Arc<dyn StatsPort>`：
- **入口**：`stats.user_online(user_id).await.ok()` — 失敗不影響主流程
- **退出**：`stats.user_offline(user_id).await.ok()` — 同上

`ws_handler` 從 `state.stats_service.clone()` 取 stats 並傳入 `handle_socket`。

### TDD 過程記錄

- **RED**：先創建 `src/stats/` 目錄及三個文件骨架，`StatsPort` trait 和 `FakeStatsService` 結構同步定義。  
  先於實現存在的 9 個測試（ST01–ST09）作為設計規格，驗證全部能編譯。
- **GREEN**：
  - `FakeStatsService` 用 `Mutex<HashSet>` 模擬 HLL（insert 去重）和 Redis Set（insert/remove）；
  - `user_offline` 為 no-op 對齊 HLL append-only 語義；
  - `take_snapshot` 遞增 `AtomicU32` 計數器；
  - `snapshot_task` 使用 `tokio::select!` 同時監聽 tick 和 shutdown 信號；
  - 集成進 `AppState`、`handle_socket`、`ws_handler`；
  - 142 個測試一次通過，0 失敗。
- **REFACTOR**：
  - `take_snapshot` 使用 `redis::pipe()` 將 HSET + EXPIRE 合為一次 pipeline 調用；
  - 所有 stats 調用使用 `.await.ok()` 確保失敗不影響 WS 主流程；
  - `snapshot_task` 快照失敗記 warn 日誌後繼續執行（不 panic、不退出）。

### Round 1 Review 修復記錄

| 等級 | 問題 | 修復方案 | 文件 |

| HIGH | `snapshot_task` 從未被 `tokio::spawn` | 新增 `start_snapshot_task` 便捷函數；`main.rs` 在 `AppState::new()` 後 spawn，並在 `axum::serve()` 返回後發送 shutdown=true | `src/stats/snapshot_task.rs`、`src/main.rs` |
