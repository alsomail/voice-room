### 本 Task 的目标
1. 输出 `doc/tests/MIDSCENE_SETUP.md`，冻结**三种部署形态**字段映射 + **CI Secret 注入**示例 + **缺 Key skip** 策略；

### 一句话目标

### 2.1 数据流图

```
   ┌────────────── 本地 ──────────────┐        ┌────────────── CI ──────────────┐
   │ tests/scripts/env/.env.<profile> │        │ GitHub Actions secrets         │

   │   MIDSCENE_MODEL_NAME=gpt-4o     │        │   (引用 ${{ secrets.* }})      │
   │   MIDSCENE_OPENAI_BASE_URL=...   │        │   step env: 注入到 process.env │
   │   MIDSCENE_USE_AZURE_OPENAI=...  │        └────────────────────────────────┘
   └──────────────┬───────────────────┘                       │
                  │                                            │

        │ envLoader.loadE2EEnv()  →  env.midscene.{apiKey,         │
```

### 2.4 CI Secret 注入示例（GitHub Actions）

```yaml
# .github/workflows/e2e.yml（节选 — 严禁明文 Key）
name: E2E
on: [push, pull_request]

jobs:
  e2e-staging:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: 20 }
      - run: npm ci
      - run: npx playwright install --with-deps chromium
      - name: Run E2E (staging)
        env:
          E2E_PROFILE: staging
          # ⚠️ 仅引用 Secret，严禁明文
          MIDSCENE_MODEL_API_KEY: ${{ secrets.MIDSCENE_MODEL_API_KEY }}
          MIDSCENE_MODEL_NAME: gpt-4o
          MIDSCENE_OPENAI_BASE_URL: ${{ vars.MIDSCENE_OPENAI_BASE_URL }}
          MIDSCENE_CACHE: '1'
          # 其他 staging 凭据同样从 secrets 注入
          E2E_VALID_TOKEN: ${{ secrets.E2E_VALID_TOKEN }}
          # ...
        run: npm run e2e:staging
```

**安全规约**：
- ✅ 必须用 `${{ secrets.MIDSCENE_MODEL_API_KEY }}` 引用；
- ❌ 禁止 `MIDSCENE_MODEL_API_KEY: sk-xxx`（明文）；
- ❌ 禁止 `echo $MIDSCENE_MODEL_API_KEY`（GitHub Actions 默认 mask，但 `set -x` 或自定义 echo 会泄漏）；
- ✅ Azure 形态的 `AZURE_OPENAI_API_VERSION` 等非敏感字段可走 `${{ vars.* }}`，仅 Key 走 `${{ secrets.* }}`。

### 2.5 缺 Key skip 策略

#### 2.5.1 fixture 层（新增 `midsceneReady` auto fixture）

```ts
// tests/scripts/support/fixtures.ts（新增片段）
export function midsceneReadyImpl(env: E2EEnv, testInfo: MinimalTestInfo & { file?: string }): boolean {
  // 仅对 WEB spec 起效（路径包含 /tests/scripts/WEB/）
  const file = testInfo.file ?? '';
  const isWebSpec = file.includes(`${path.sep}tests${path.sep}scripts${path.sep}WEB${path.sep}`)
                 || file.includes('/tests/scripts/WEB/');
  if (!isWebSpec) return false;
  if (env.midscene.apiKey && env.midscene.apiKey.trim() !== '') return false;
  testInfo.skip(true, '[MIDSCENE] api key missing — skipped');
  return true;
}
```

注册为 `auto: true` worker fixture，与现有 `prodSafeGuard` 并列；执行顺序：`prodSafeGuard` → `midsceneReady`（prod profile + 未打 @prod-safe 已 skip 的用例不再走 midscene 检查）。

#### 2.5.2 与 T-0000H fixtures 协同矩阵

| profile | @prod-safe 标签 | 是 WEB spec | apiKey 空 | 行为 |
|---|---|---|---|---|
| local | — | 是 | 是 | **skip**（缺 Key）|
| local | — | 是 | 否 | 跑 |
| local | — | 否（API/E2E）| 是 | 跑（不依赖 Midscene）|
| staging | — | 是 | 是 | **skip**（缺 Key）|
| staging | — | 是 | 否 | 跑 |
| prod | 否 | 是 | 任意 | **skip**（prod-safe 防线先行）|
| prod | 是 | 是 | 是 | **skip**（midsceneReady）|
| prod | 是 | 是 | 否 | 跑 |

#### 2.5.3 用户提示

skip reason 严格为 `'[MIDSCENE] api key missing — skipped'`，便于 Playwright HTML 报告 grep 与 RUNBOOK FAQ 引用。

### 2.6 限流与回退

| 场景 | 表现 | 处理策略 |
|------|------|----------|
| 401 Unauthorized | Midscene SDK 抛错 | 文档 FAQ §4 指向「Key 失效 / 用错 Azure deployment 名」 |
| 429 Rate Limit | Midscene SDK 抛错或重试 | 启用 `MIDSCENE_CACHE=1`（默认）；CI 串行执行 WEB 套件（不并行 worker）|
| 网关超时 | 测试超时（默认 60s）| 单条用例 retry 1 次（playwright.config.ts `retries: process.env.CI ? 1 : 0`）；连续失败 3 条则 globalTeardown 输出 hint |
| 中转网关挂 | 连续 401/5xx | 文档建议：临时切回 OpenAI 直连（删掉 `MIDSCENE_OPENAI_BASE_URL` 即可）|
| Key 配额耗尽 | 持续 429 | **不**自动 skip（避免静默通过）；要求 RUNBOOK 手工触发「set MIDSCENE_MODEL_API_KEY=空 → 重跑 → 全部 skip」逃生通道 |

### 2.7 安全红线

1. **明文禁止**：`grep -rE '(sk-[A-Za-z0-9]{20,}|sk-proj-)' .github/ doc/ tests/scripts/env/*.example` 必须 0 命中（仅 `.env.{local,staging,prod}` 实文件可含真实 Key，且已在 `.gitignore`）；
2. **不入 runtime json**：envLoader `writeProcessEnv` 的 `.e2e-runtime.json` 持久化路径**不**写入 `MIDSCENE_MODEL_API_KEY`（当前 fixtures.ts `readE2EEnvForWorker` 优先读 runtime json，TDD 阶段需在 globalSetup 写 runtime json 时显式排除 apiKey 字段，或对 midscene.apiKey 单独从 process.env 取）；
3. **日志脱敏**：envLoader 的所有 error message（`MissingEnvError` / `InvalidEnvError`）严禁回显 `MIDSCENE_MODEL_API_KEY` 实值；类型为 secret 的字段在错误格式器中以 `***` 替代；
4. **CI 日志**：禁止 `echo`/`printenv MIDSCENE_*`；GitHub Actions 默认 mask Secret，但显式 `set -x` 会绕过，文档警示。
