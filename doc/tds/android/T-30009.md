# TDS: 房间页 UI (Task ID: T-30009)

## 一、背景与目标

- **关联需求**：来自 `product.md` 房间核心体验模块，玩家进入语聊房后看到的主界面。
- **本 Task 的目标**：纯 UI 层，用 Jetpack Compose 实现房间页完整骨架布局（顶部房间信息 + 麦位 Grid + 聊天列表 + 底部输入栏），通过 Preview 假数据验证渲染，不包含 ViewModel / WebSocket 逻辑（由 T-30010 接入）。

---

## 二、方案设计

### 核心数据流

```
RoomUiState（纯 data class，Preview 假数据填充）
        │
        ▼
RoomScreen（顶层 Composable）
   ├── RoomTopBar        ← roomName, onlineCount, onBack
   ├── MicSlotsGrid      ← List<MicSlotUi>（9 宫格，3 列 × 3 行）
   ├── ChatMessageList   ← List<ChatMessageUi>（LazyColumn，自动滚到底部）
   └── BottomInputBar    ← onSend: (String) -> Unit
```

T-30010 负责将 ViewModel.uiState 注入 `RoomScreen`，本 Task 只交付静态 UI 骨架。

---

### 数据模型（UI 层，存于 `RoomUiState.kt`）

```kotlin
/** 房间页整体 UI 状态 */
data class RoomUiState(
    val roomId: String = "",
    val roomName: String = "",
    val onlineCount: Int = 0,
    /** 固定 9 个麦位（index 0–8），由服务端 mic_slots 映射填充，未占用保持默认空值 */
    val micSlots: List<MicSlotUi> = List(9) { MicSlotUi(index = it) },
    val messages: List<ChatMessageUi> = emptyList(),
    val isLoading: Boolean = false,
)

/** 单个麦位 UI 状态 */
data class MicSlotUi(
    val index: Int,                   // 0–8，对应 9 宫格位置
    val userId: String? = null,       // null = 空麦
    val nickname: String? = null,
    val avatarUrl: String? = null,
    val isMuted: Boolean = false,
)

/** 聊天消息 UI 模型 */
data class ChatMessageUi(
    val messageId: String,
    val senderNickname: String,
    val content: String,
    val timestamp: Long,
)
```

> **麦位数量依据**：`GET /api/v1/rooms/:id` 响应体中 `mic_slots` 在 MVP 阶段为空数组，MVP 默认渲染 9 个位置（3×3）；后续 ViewModel 接入时按服务端实际返回覆盖。

### Composable 组件规格

#### `RoomScreen`
```kotlin
@Composable
fun RoomScreen(
    uiState: RoomUiState,
    onBack: () -> Unit,
    onSendMessage: (String) -> Unit,
    modifier: Modifier = Modifier,
)
```
布局：`Column(fillMaxSize)` 垂直分区：`RoomTopBar` → `MicSlotsGrid`（固定高度）→ `ChatMessageList`（`weight(1f)`）→ `BottomInputBar`

#### `RoomTopBar`
```kotlin
@Composable
fun RoomTopBar(
    roomName: String,
    onlineCount: Int,
    onBack: () -> Unit,
    modifier: Modifier = Modifier,
)
```
使用 Material3 `TopAppBar`，`testTag("room_top_bar")`；显示 `testTag("room_name")` 文本和 `testTag("room_online_count")` 文本；返回按钮 `testTag("room_back_button")`。

#### `MicSlotsGrid`
```kotlin
@Composable
fun MicSlotsGrid(
    slots: List<MicSlotUi>,
    modifier: Modifier = Modifier,
)
```
使用 `LazyVerticalGrid(columns = GridCells.Fixed(3))`，`testTag("mic_slots_grid")`。  
每项调用 `MicSlotItem(slot)`。

#### `MicSlotItem`
```kotlin
@Composable
fun MicSlotItem(
    slot: MicSlotUi,
    modifier: Modifier = Modifier,
)
```
两种状态：
- **空麦**（`userId == null`）：展示麦克风占位图标，`testTag("mic_slot_empty_${slot.index}")`
- **有人**（`userId != null`）：`AsyncImage` 加载头像（Coil，fallback = `ic_placeholder`）+ 昵称文本 + 若 `isMuted` 显示静音图标，`testTag("mic_slot_occupied_${slot.index}")`

#### `ChatMessageList`
```kotlin
@Composable
fun ChatMessageList(
    messages: List<ChatMessageUi>,
    modifier: Modifier = Modifier,
)
```
`LazyColumn`，`testTag("chat_message_list")`。  
通过 `LaunchedEffect(messages.size) { listState.animateScrollToItem(lastIndex) }` 自动滚底。  
每项调用 `ChatMessageItem(message)`，`testTag("chat_message_${message.messageId}")`。

#### `BottomInputBar`
```kotlin
@Composable
fun BottomInputBar(
    onSend: (String) -> Unit,
    modifier: Modifier = Modifier,
)
```
`Row`：`TextField`（`testTag("message_input")`，`fillMaxWidth().weight(1f)`）+ 发送 `Button`（`testTag("send_button")`，消息为空时 `enabled = false`）。点击发送后清空输入框。

## 五、Reviewer意见

> 状态枚举：🟢 通过 | 🔴 未通过
> 每轮 Review 追加一条记录，不要覆盖历史。

### Round 1
- **状态**：🟢 通过
- **审查意见**：
  - **HIGH (0/0)**：UI 层纯净性 ✅、状态提升 ✅、LazyColumn 高度约束 ✅——三项全部合格。
  - **MEDIUM (1/4)**：
    - 🟡 **M-1 MicSlotsGrid 未设固定高度（潜在运行时崩溃）**
      - `RoomScreen.kt:52` 调用 `MicSlotsGrid` 时未传入任何高度 modifier；`MicSlotsGrid.kt:42` 的 `LazyVerticalGrid` 也无 `height()` 修饰。
      - 原因：`Column` 内存在 `weight(1f)` 子节点（`ChatMessageList`），Compose 测量协议会以 `maxHeight = Infinity` 先测量 non-weighted 子节点，`LazyVerticalGrid` 收到无界高度约束时在运行时抛出 `IllegalStateException`，与 TDS §二"MicSlotsGrid（固定高度）"规格不符。
      - **修复**：在 `RoomScreen.kt` 调用时加入 `modifier = Modifier.height(240.dp)`（或其他合适固定高度）。
    - ✅ 消息自动滚底、发送后清空输入框、testTag 完整性——三项均合格。
  - **LOW (1)**：UI-11（100 条消息自动滚底边界用例）未在 `RoomScreenTest.kt` 中实现，可在 T-30010 或后续补充，不阻塞本次通过。
- **改进建议**：
  - **必改（M-1）**：在 `RoomScreen.kt` 中为 `MicSlotsGrid` 调用添加固定高度 modifier（建议 `Modifier.height(240.dp)`），防止在真实设备上运行时崩溃，应在 T-30010 接入 ViewModel 前完成修复。
  - **选做（L-3）**：补充 `UI11_autoScrollToBottomWith100Messages` 测试用例。
