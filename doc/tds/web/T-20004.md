# T-20004 — Web 端房间管理页面

## 一、需求概述

管理员在浏览器端需要一个可搜索、可筛选、分页展示的房间列表，并可对活跃房间执行"强制关闭"操作。

### 2.2 API — `adminCloseRoom`

```typescript
export async function adminCloseRoom(roomId: string): Promise<void> {
  await adminFetch<null>(`/rooms/${encodeURIComponent(roomId)}`, {
    method: 'DELETE',
  });
}
```

### 2.3 Hook — `useRoomsPage`

```
state:  items, total, loading, error
        page(default 1), pageSize(default 20)
        filters: { status?, keyword? }
        debouncedKeyword (internal, 300ms debounce)
        closingId: string | null
        selectedRoomId: string | null   # T-20005 占位
        refreshKey: number              # 用于强制刷新

debounce effect:  filters.keyword → 300ms → setDebouncedKeyword + setPage(1)
fetch effect:     deps=[debouncedKeyword, filters.status, page, pageSize, refreshKey]
                  每次建新 AbortController，cleanup 时 abort

setFilters(patch):  status 变化时立即 setPage(1)
closeRoom(id):      setClosingId → adminCloseRoom → success: refresh | fail: setError → setClosingId(null)
refresh():          setRefreshKey(prev+1)
```

### 2.4 组件 — `RoomsTable` Props

```typescript
interface RoomsTableProps {
  items: AdminRoomItem[];
  total: number;
  page: number;
  pageSize: number;
  filters: RoomsPageFilters;
  loading: boolean;
  closingId: string | null;
  onPageChange: (page: number, pageSize: number) => void;
  onFiltersChange: (patch: Partial<RoomsPageFilters>) => void;
  onCloseRoom: (roomId: string) => void;
  onRowClick: (roomId: string) => void;
  onRefresh: () => void;
}
```

列定义：roomId、title、room_type（typeNormal/typePassword/typePaid）、status（RoomStatusTag）、
member_count/max_members、owner_nickname、created_at、操作（Popconfirm + 关闭按钮）

关键逻辑：
- `status === 'closed'` → 关闭按钮 `disabled`
- `closingId === row.room_id` → 关闭按钮 `loading`
- Table 设置 `data-testid="rooms-table"`
- `onRow: (record) => ({ onClick: () => onRowClick(record.room_id) })`
- close 按钮区域 `stopPropagation`

### 2.5 组件 — `RoomStatusTag`

- `status='active'`：green Tag，`data-testid="status-tag-active"`
- `status='closed'`：default Tag，`data-testid="status-tag-closed"`

### HIGH（阻塞发布）

#### [HIGH-TS01] `useRoomsPage.ts:80` — 不安全类型断言绕过类型检查

**描述**：fetch effect 中将 `params` 先声明为宽泛的 `Record<string, string | number>`，再用 `as Parameters<typeof adminGetRooms>[0]` 强转，导致 TypeScript 无法检查字段合法性。例如 `status: 'inactive'` 或拼写错误的字段名，编译期不会报错，运行时才会产生错误请求。

```ts
// ❌ 当前写法 — 绕过 AdminGetRoomsParams 的 status 字面量类型限制
const params: Record<string, string | number> = { page, page_size: pageSize };
if (filters.status) params.status = filters.status;
const data = await adminGetRooms(
  params as Parameters<typeof adminGetRooms>[0],  // ← 不安全断言
  controller.signal,
);

// ✅ 修改建议 — 直接构造 AdminGetRoomsParams，零断言
const data = await adminGetRooms(
  {
    page,
    page_size: pageSize,
    ...(filters.status   && { status:  filters.status }),
    ...(debouncedKeyword && { keyword: debouncedKeyword }),
  },
  controller.signal,
);
```

### MEDIUM（需改进，可在本迭代一并修复后重新 Review）

#### [MEDIUM-01] `RoomsTable.tsx:141` — `Input.Search` 使用非受控 `defaultValue`

**描述**：`<Input.Search defaultValue={filters.keyword ?? ''} .../>` 使用非受控模式。当外部（如 T-20005 完成后可能的「重置筛选器」逻辑）调用 `setFilters({keyword: undefined})` 时，DOM 输入框不会同步更新。整个工具栏的 `Select` 已使用受控 `value`，两者应保持一致。

```ts
// ✅ 修改建议
<Input.Search
  value={filters.keyword ?? ''}
  onChange={(e) => onFiltersChange({ keyword: e.target.value || undefined })}
  ...
/>
```

#### [MEDIUM-02] `useRoomsPage.ts:87, 130` — `err as Error` 缺少类型守卫

**描述**：JS 允许 `throw` 任意值（字符串、普通对象等），直接 `err as Error` 会将非 Error 对象赋给 `error` state，导致调用方访问 `error.message` 时得到 `undefined`。

```ts
// ✅ 修改建议（两处相同模式）
} catch (err) {
  const e = err instanceof Error ? err : new Error(String(err));
  if (e.name !== 'AbortError') {
    setError(e);
    setItems([]);
  }
}
```

#### [MEDIUM-03] `RoomStatusTag.tsx:18, 24` — 状态文本未走 i18n

**描述**：Tag 内容直接渲染原始字符串 `{status}`（"active" / "closed"），而同文件其他标签均使用 `useTranslation`。破坏本地化一致性，多语言环境下会显示英文原始值。

```tsx
// ✅ 修改建议
import { useTranslation } from 'react-i18next';

export function RoomStatusTag({ status }: RoomStatusTagProps) {
  const { t } = useTranslation();
  if (status === 'active') {
    return <Tag color="success" data-testid="status-tag-active">{t('rooms.statusActive')}</Tag>;
  }
  return <Tag data-testid="status-tag-closed">{t('rooms.statusClosed')}</Tag>;
}
```

### LOW（建议优化，不阻塞）

#### [LOW-01] `useRoomsPage.ts:100` — `eslint-disable` 注释缺少说明

eslint-disable-next-line 注释正确且必要（有意用 `filters.status` 而非整个 `filters` 对象，避免 object reference 变化导致多余 refetch），但缺少原因说明，后续维护者可能误删。建议改为：

```ts
// eslint-disable-next-line react-hooks/exhaustive-deps
// 故意订阅 filters.status 而非 filters 整体——filters 对象引用在每次 setFilters 时都会更新，
// 订阅整体会导致多余 refetch。status/keyword 变化已通过各自依赖项精确捕获。
```

#### [LOW-02] `RoomsTable.tsx` — `columns` / `statusOptions` 未 `useMemo` 化

`columns`（含内联渲染函数）和 `statusOptions` 在每次渲染时重新创建。目前因 `RoomsTable` 未被 `React.memo` 包裹，性能影响有限；但若后续对其 memo 化，应同步用 `useMemo` 稳定这两个引用。

#### [LOW-03] `apiClient.ts:190` — `adminCloseRoom` 不支持 `AbortSignal`

与 `adminGetRooms` 不同，`adminCloseRoom` 无 `signal` 参数。组件卸载时 `closeRoom` 的 `finally` 块仍会执行 `setClosingId(null)` / `setError`。React 18 下不会报错，TDS 也未强制要求，但建议与 `adminGetRooms` 保持一致以支持取消。
