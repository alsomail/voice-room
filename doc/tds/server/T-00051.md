# TDS: 创建订单 API + 风控 (Task ID: T-00051)

## 一、背景与目标
- **关联需求**：[phase1_payment_billing.md §6.2](../../product/phase1_payment_billing.md) — 用户点击 SKU → 创建 internal order，作 obfuscatedAccountId 透传给 Google Play。
- **本 Task 的目标**：实现 `POST /api/v1/payments/orders` 写 PENDING 订单 + 返回 `order_id`，叠加风控（日失败 > 10 / 设备黑名单）拒绝高风险用户。

## 二、方案设计

### 核心数据流
```
Android 充值页点击 SKU
   ↓
POST /api/v1/payments/orders { sku_id }     [JWT auth]
   ↓
PaymentOrderService::create_order
   ├─ SKU 校验（is_active=true 否则 40902）
   ├─ RiskCheckService::evaluate(user_id, device_id)
   │   ├─ 24h FAILED 订单数 > 10 → 40903
   │   └─ device_id 命中 risk_blacklist → 40903
   ├─ INSERT payment_orders (state=PENDING, msg_id=request_id)
   └─ 返回 { order_id, sku: {...} }
```

## 三、TDD 验收用例
- [ ] **正向**：合法 sku_id + 合法 JWT → 200，返回 `order_id` UUID + 完整 sku 字段，DB 新增 PENDING 行
- [ ] **正向**：同一用户并发 5 次 → 5 个 PENDING 订单全部成功（无锁竞争）
- [ ] **异常**：sku_id 不存在 → 404 + code=40902
- [ ] **异常**：sku.is_active=false → 404 + code=40902
- [ ] **异常**：未带 Authorization → 401
- [ ] **异常**：风控触发（mock 24h 内 11 个 FAILED 订单）→ 429 + code=40903
- [ ] **异常**：device_id 命中黑名单 → 429 + code=40903
- [ ] **响应字段**：order_id 必须是 UUID v4 格式（用于 obfuscatedAccountId）
