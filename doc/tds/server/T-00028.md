# TDS: WS KickUser 信令 + 10min 冷却 (Task ID: T-00028)

## 一、背景与目标
- **关联需求**：E-10 房主/管理员踢人 ([phase1_room_governance §3.5](../../product/phase1_room_governance.md))
- **本 Task 的目标**：实现 `KickUser` WS 信令：权限校验 → Redis 冷却 → DB 审计 → 清理成员 → 广播。

## 二、方案设计

### 信令
**请求 C→S `KickUser`**
```json
{ "type":"KickUser", "msg_id":"uuid",
  "payload":{ "room_id":"uuid", "target_user_id":"uuid", "reason":"harassment" } }
```

**响应 S→C `KickUserResult`**
```json
{ "type":"KickUserResult", "msg_id":"uuid", "code":0 }
```
错误码：
| code | 含义 |
|------|------|
| 40301 | PERMISSION_DENIED（操作者非 owner/admin） |
| 40302 | CANNOT_KICK_OWNER |
| 40400 | 房间不存在或 target 不在房间 |
| 40003 | reason 缺失或格式非法 |

**广播 S→目标 `UserKicked`**
```json
{ "type":"UserKicked", "payload":{ "room_id":"uuid", "reason":"harassment",
  "cooldown_sec": 600, "operator_nickname":"..." } }
```

**广播 S→房间其他人 `UserLeft`**（已存在信令）增加 `reason:"kicked_by_admin"` 字段：
```json
{ "type":"UserLeft", "payload":{ "user_id":"uuid", "reason":"kicked_by_admin", "operator_id":"uuid" } }
```

### 处理流程（强事务）
```rust
pub async fn handle_kick(ctx, payload) {
    // 1. 权限校验：ctx.user_id ∈ {owner, admin}
    let room = room_service.load(room_id).await?;
    let is_owner = room.owner_id == ctx.user_id;
    let is_admin = room.admin_user_id == Some(ctx.user_id);
    if !is_owner && !is_admin { return err(40301); }
    if payload.target_user_id == room.owner_id { return err(40302); }
    // 管理员不能踢管理员（自己）但可以踢普通 — 简化：target 必须是 member/非 owner

    // 2. 找到 target 连接
    let connections = registry.get_connections_for_user(target_user_id);
    if !room_manager.is_in_room(room_id, target_user_id) { return err(40400); }

    // 3. Redis 冷却
    redis.setex(&format!("kicked:{room_id}:{target_user_id}"), 600, reason).await?;

    // 4. DB INSERT room_kick_records
    sqlx::query!("INSERT INTO room_kick_records(...) VALUES ...").execute(&pool).await?;

    // 5. RoomManager 移除 + 若在麦自动下麦
    if let Some(slot) = room_manager.mic_slot_of(room_id, target_user_id) {
        room_manager.leave_mic(room_id, slot).await;
        broadcast(room_id, MicLeft { slot, user_id: target_user_id, forced: true }).await;
    }
    room_manager.remove_member(room_id, target_user_id).await;

    // 6. 广播
    for conn in connections { conn.send(UserKicked{...}).await; }
    broadcast_except(room_id, target_user_id, UserLeft{user_id, reason:"kicked_by_admin", operator_id}).await;

    // 7. 主动关闭被踢者的 WS 连接
    for conn in connections { conn.close(Reason::Kicked).await; }
}
```

### JoinRoom 冷却拦截
- JoinRoom 前置检查：`GET kicked:{room_id}:{user_id}`，存在则返回 `KICKED_COOLDOWN(42911)` + remaining_sec

### 并发保护
- 多个管理员同时踢同一人：通过 `RoomManager` 内的 `DashMap.remove()` 原子性；Redis SETEX 覆盖无副作用；room_kick_records 可能插入多条（接受 —— 真实记录多位管理员操作）
- 但要求"仅成功一次踢出"：用 `DashMap.remove()` 返回值判断，只有第一次返回 Some 的走完完整流程
