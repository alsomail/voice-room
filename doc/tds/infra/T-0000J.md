# TDS: E2E 用例 baseURL/密码 typo 修复 + @prod-safe 标签 (Task ID: T-0000J)

> **主设计**：[T-0000E](./T-0000E.md)（§2.8 用例标签 / §2.9 安全红线）

## 一、背景与目标
- 修复体检缺口 C3/C14：DB 密码 typo + `playwright.config.ts` baseURL 注释。
- 全量去硬编码，引入 `@prod-safe` 标签体系。

## 二、方案设计

### 涉及文件清单
| 文件路径 | 变更类型 | 说明 |
|---|---|---|
| `tests/scripts/E2E/*.spec.ts` | 修改 | 删除硬编码 `app_server_pwd` 等，改读 `process.env.DATABASE_URL`（通过 envLoader 已规范化） |
| `tests/scripts/API/*.spec.ts` | 修改 | 同上 |
| `tests/scripts/WEB/*.spec.ts` | 修改 | 用相对路径 `page.goto('/login')`，依赖 baseURL |
| `playwright.config.ts` | 修改 | 取消 `baseURL` 注释，从 envLoader 注入；profile=prod 时设置 `grep: /@prod-safe/` |
| `tests/scripts/E2E/TC-AUTH.spec.ts` 等 | 修改 | read-only 用例（如 health check、token 内省）打 `{ tag: '@prod-safe' }` |

### 标签策略
至少打 `@prod-safe` 的 5 类只读用例：
- 各 server 的 `/health` 探活
- Token 内省（GET /me）
- 房间列表只读查询
- 礼物目录只读查询
- 排行榜只读查询

### 硬编码扫描清单
执行 `grep -rE 'app_server_pwd|localhost:3000|localhost:3001|localhost:5173' tests/scripts/`，命中点全部改读 env。

## 三、TDD 验收用例

- [ ] `grep -rE 'localhost:(3000|3001|5173)' tests/scripts/` 0 命中（除 envLoader 默认值外）
- [ ] 至少 5 个用例打 `@prod-safe` 标签
- [ ] WEB 用例可用 `page.goto('/login')` 相对路径运行（Chromium 通过）
- [ ] `E2E_PROFILE=prod` 跑 playwright，未打标用例 100% skip
