## 三、TDD 验收用例
- [ ] **隐身**：king（invisibility=all）出现在观众席查询时被剔除；admin/房主仍可见；MemberSnapshot 同步过滤
- [ ] **优先抢麦**：king（weight=10.0）vs 普通用户（weight=1.0）100ms 内并发抢同一坑位 100 次 → king 成功率 ≥ 85%（softmax 抽签）
- [ ] **礼物折扣**：duke（10%）送 1000 钻礼物 → 实际扣款 900 钻；gift_transactions.discount_pct=10, original_price=1000；房间 GiftReceived.payload 仍显示原价（产品规则）
- [ ] **折扣 floor 至千位**：duke 送 1234 钻 → discount=123 → 实付 1111；测断言 floor((1234*90)/100/1000)*1000=1000... → 实际产品规则按 phase1_nobility §3.5.8 校核；测试与 phase1_nobility 完全一致
- [ ] **免密**：duke 进入密码房 + 房主开关 respect_owner_switch=false → 200 直接入；房主开 respect_owner_switch=true → 400 PASSWORD_REQUIRED
- [ ] **配置驱动**：admin 修改 `privileges.gift_discount.percent` → 缓存 invalidation 后下一次送礼立即生效（< 60s）

### 🔴 协议路径绑定

- **打回理由**：

#### [HIGH] bypass_password 校验调用了语义错误的函数

**文件**：`app/server/src/room/handler/lifecycle.rs:130`

```rust
// 当前错误实现
crate::modules::nobility::privileges::can_trigger_global_broadcast(level)
// ↑ 该函数语义是「判断是否可以触发全服 NobleEntranceGlobal（§10.4.6）」，与密码房免密无关
```

两个函数当前恰好都检查 `level >= 5`（duke/king），结果等价，但属于**语义混用**：
- 若未来 earl (LV4) 或 viscount (LV3) 在种子数据中开启 `bypass_password.enabled = true`，此处代码会静默失效（earl 有免密权限但 `can_trigger_global_broadcast(4)` 返回 false）
- 若 `can_trigger_global_broadcast` 调整为 `level >= 4`（业务变更），duke 将意外获得 bypass_password 权限

**正确修复**：改为调用 `can_bypass_password(&privileges)` 或专用的 `can_bypass_password_by_level(level)` 函数：

```rust
// 方案 A：从 UserNobleDto 扩展 bypass_password 字段，读取实际特权
can_bypass = noble.bypass_password_enabled; // UserNobleDto 新增字段

// 方案 B：专用 level 判断（若种子约定 LV5+ = bypass）
crate::modules::nobility::privileges::can_bypass_password_by_level(level)
// 函数名语义清晰，避免与 §10.4.6 全服广播条件混淆
```

## 七、Round 3 修复记录（E2R3）

**问题**：bypass_password 使用语义错误的 `can_trigger_global_broadcast(level)` 函数（Round 2 Review 拒绝 [HIGH]）。

### 根本原因

`lifecycle.rs` 中 bypass 判断调用：
```rust
// 旧（错误）：语义上是"是否可触发全服广播"，与密码绕过无关
crate::modules::nobility::privileges::can_trigger_global_broadcast(level)
```

这是功能上的巧合（两者都检查 level ≥ 5），但违反单一职责原则。当未来种子数据将 earl (LV4) 加入 bypass_password 白名单时，旧代码需要改两处；而新代码只需更新 `UserNobleDto.bypass_password_enabled`。

### 修复内容

| 文件 | 修改 |
|------|------|
| `app/server/src/modules/nobility/dto.rs` | `UserNobleDto` 新增 `bypass_password_enabled: bool` 字段（`#[serde(default)]`）|
| `app/server/src/room/handler/lifecycle.rs` | bypass 判断改为 `noble.bypass_password_enabled` |
| `app/server/src/room/handler/mod.rs` | `FakeNobleService` 填充 `bypass_password_enabled = level >= 5`；新增 `t70_earl_cannot_bypass_password_room` 集成测试 |

### 新增测试

- **`t70_earl_cannot_bypass_password_room`**：earl (LV4, `bypass_password_enabled=false`) 无 token 进入密码房 → code 40104 ✅
