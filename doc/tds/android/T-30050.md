## 二、方案设计

### 2.1 数据流图（gradle flavor → BuildConfig → DI → Retrofit/OkHttp/Analytics）

```
   ./gradlew assemble{Local|Staging|Prod}{Debug|Release}
                              │
                              ▼
   ┌──────────────────────────────────────────────────────────┐
   │ android.productFlavors.{flavor}.buildConfigField(...)    │
   │   API_BASE_URL / WS_URL / ANALYTICS_ENDPOINT             │
   │   APP_ENVIRONMENT                                        │
   │ (local.properties 与 ENV 仍可 override，最高优先)        │
   └──────────────────────────────────────────────────────────┘
                              │
                  ┌───────────┴────────────┐
                  ▼                        ▼
   ┌────────────────────────┐   ┌────────────────────────┐
   │ generated BuildConfig  │   │ manifestPlaceholders   │
   │ .API_BASE_URL          │   │ ["usesCleartextTraffic"]│
   │ .WS_URL                │   │   = "true"|"false"     │
   │ .ANALYTICS_ENDPOINT    │   │ + src/{flavor}/res/xml/│
   │ .APP_ENVIRONMENT       │   │   network_security_    │
   │ .SENTRY_DSN            │   │   config.xml           │
   └────────────────────────┘   └────────────────────────┘
                  │                        │
                  ▼                        ▼
   ┌────────────────────────────────────────────────────┐
   │ AppContainer / AppEnvironment.fromBuildConfig(...) │
   │   ├─ Retrofit(baseUrl=API_BASE_URL)                │
   │   ├─ OkHttpWebSocketClient(WS_URL)                 │
   │   ├─ HttpTransport(ANALYTICS_ENDPOINT)             │
   │   └─ SentryAnalytics(SENTRY_DSN)                   │
   └────────────────────────────────────────────────────┘
                              │
                              ▼ 运行时强制
   ┌────────────────────────────────────────────────────┐
   │ Android NetworkSecurityConfig（系统层）             │
   │   staging/prod APK 发起 http:// 请求 → IOException │
   │   （不依赖业务代码自检）                             │
   └────────────────────────────────────────────────────┘
```

**关键不变量**：
1. **flavor 是默认值层**：`local.properties` / 环境变量仍可 override（保 0 回归与 DX）
2. **NetworkSecurityConfig 是系统级强约束**：staging/prod 即使代码里漏写校验，明文请求也被系统拦截
3. **APP_ENVIRONMENT 字段是真值源**：业务代码（如 Sentry tag、日志摘要）依赖此字段而非 applicationId 后缀解析

### 2.4 productFlavors gradle DSL 草图（设计示意，TDD 阶段实现）

```kotlin
android {
    // ① 环境维度
    flavorDimensions += "env"

    // ② defaultConfig 保留现有 buildConfigField 作为 typo 兜底（D-2）
    defaultConfig {
        // 现有 API_BASE_URL / WS_URL / ANALYTICS_ENDPOINT / APP_ENVIRONMENT / SENTRY_DSN 不动
        // 新增 flavor 字段值会覆盖此处（gradle 行为）
    }

    productFlavors {
        create("local") {
            dimension = "env"
            isDefault = true                                 // 关键不变量 §2.11.3
            applicationIdSuffix = ".local"
            versionNameSuffix = "-local"

            // local 仍允许 local.properties / ENV 覆盖（D-1）
            val apiBaseUrl = resolveConfigValue(
                localProperties, "voiceRoomApiBaseUrl", "VOICE_ROOM_API_BASE_URL",
                "http://10.0.2.2:3000/api"
            )
            buildConfigField("String", "API_BASE_URL", "\"$apiBaseUrl\"")
            buildConfigField("String", "WS_URL",
                "\"${resolveConfigValue(localProperties, "voiceRoomWsUrl", "VOICE_ROOM_WS_URL", "ws://10.0.2.2:3000/ws")}\"")
            buildConfigField("String", "ANALYTICS_ENDPOINT",
                "\"${resolveConfigValue(localProperties, "voiceRoomAnalyticsEndpoint", "VOICE_ROOM_ANALYTICS_ENDPOINT", "$apiBaseUrl/v1/events/batch")}\"")
            buildConfigField("String", "APP_ENVIRONMENT", "\"local\"")
            manifestPlaceholders["usesCleartextTraffic"] = "true"
        }

        create("staging") {
            dimension = "env"
            applicationIdSuffix = ".stg"
            versionNameSuffix = "-stg"
            buildConfigField("String", "API_BASE_URL", "\"https://stg-api.example.com/api\"")
            buildConfigField("String", "WS_URL",        "\"wss://stg-api.example.com/ws\"")
            buildConfigField("String", "ANALYTICS_ENDPOINT", "\"https://stg-api.example.com/api/v1/events/batch\"")
            buildConfigField("String", "APP_ENVIRONMENT", "\"staging\"")
            manifestPlaceholders["usesCleartextTraffic"] = "false"
            // staging/prod 保留 ENV override 能力（CI 注入真域名）
        }

        create("prod") {
            dimension = "env"
            // 无 applicationIdSuffix → 包名为 com.voice.room.android（与商店一致）
            buildConfigField("String", "API_BASE_URL", "\"https://api.example.com/api\"")
            buildConfigField("String", "WS_URL",        "\"wss://api.example.com/ws\"")
            buildConfigField("String", "ANALYTICS_ENDPOINT", "\"https://api.example.com/api/v1/events/batch\"")
            buildConfigField("String", "APP_ENVIRONMENT", "\"prod\"")
            manifestPlaceholders["usesCleartextTraffic"] = "false"
        }
    }

    buildTypes {
        debug {
            applicationIdSuffix = ".debug"
            versionNameSuffix   = "-debug"
            // 移除 manifestPlaceholders["usesCleartextTraffic"]（迁移到 flavor）
        }
        release {
            isMinifyEnabled = false
            // 同上移除
            proguardFiles(getDefaultProguardFile("proguard-android-optimize.txt"), "proguard-rules.pro")
        }
    }
}
```

**设计决策**：
- **D-1（local override 链路保留）**：local flavor 仍走 `resolveConfigValue()` 三级 fallback；DX 不退化，物理机切到 LAN IP 仍走 `local.properties`
- **D-2（defaultConfig 作为 typo 兜底）**：保留 defaultConfig 的 buildConfigField 实现，避免 flavor 块字段名拼写错误时编译期不报错运行期 NPE（gradle 不强制 flavor 必须包含全部字段）
- **D-3（NetworkSecurityConfig 走 flavor sourceset 而非 manifest 占位）**：单纯依赖 `usesCleartextTraffic` 占位符不够——`networkSecurityConfig` xml 在 staging/prod 也需关 cleartext，避免误配；buildType 层 xml 删除避免 `staging+debug` 时 buildType 资源被合并误开（R1）

### 2.5 网络安全配置（staging/prod 编译期 + 运行时双重锁）

#### 2.5.1 `src/local/res/xml/network_security_config.xml`
```xml
<?xml version="1.0" encoding="utf-8"?>
<network-security-config>
    <base-config cleartextTrafficPermitted="true" />
    <!-- 局域网友好：明示常见本机/LAN 网段，便于排障 -->
    <domain-config cleartextTrafficPermitted="true">
        <domain includeSubdomains="false">10.0.2.2</domain>      <!-- 模拟器→宿主 -->
        <domain includeSubdomains="false">127.0.0.1</domain>
        <domain includeSubdomains="false">localhost</domain>
        <!-- 192.168.0.0/16 与 10.0.0.0/8 由 base-config=true 兜底，xml 不支持 CIDR -->
    </domain-config>
</network-security-config>
```

#### 2.5.2 `src/staging/res/xml/network_security_config.xml` 与 `src/prod/...`
```xml
<?xml version="1.0" encoding="utf-8"?>
<network-security-config>
    <base-config cleartextTrafficPermitted="false" />
</network-security-config>
```

**双重锁机制**：
1. **编译期（manifestPlaceholder）**：`android:usesCleartextTraffic="${usesCleartextTraffic}"` 在 staging/prod 渲染为 `false`
2. **运行时（NetworkSecurityConfig）**：xml `cleartextTrafficPermitted="false"` 强制系统层拦截 http:// 请求 → `java.io.IOException: Cleartext HTTP traffic to xxx not permitted`
3. 任一防线失效，另一道仍能拦截 → 满足 §2.9.4「明文流量违反则编译失败/运行失败」

### 2.9 与现有 buildConfig override 链路的关系

| 优先级 | 来源 | 适用 flavor | 说明 |
|---|---|---|---|
| 1（最高）| `local.properties` 中 `voiceRoom*` 键 | local（设计），staging/prod（如开发者本地需要）| 现有 `resolveConfigValue()` 第一级 |
| 2 | 环境变量 `VOICE_ROOM_*` | 所有 flavor | CI 注入真实 staging/prod 域名时使用 |
| 3 | flavor `buildConfigField` 默认值 | 各自 flavor | 本任务新增层 |
| 4 | `defaultConfig.buildConfigField` | 兜底 | 仅在 flavor 字段 typo 时回落（D-2） |

> staging/prod flavor 的字段写死了占位 URL（不走 `resolveConfigValue`），是为了**让 staging/prod 有可重现的默认行为**——CI 通过 ENV override，本地开发也不会意外走到错误环境。
