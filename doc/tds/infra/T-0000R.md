  4. ✅ **TC-ROOM-00003 locator 修复**：改用 `.ant-modal-wrap.last()` + `getByRole('button')` 唯一锁定确认按钮（line 102-105），技术有效但偏离 TDS 方案（原建议 `.ant-modal-confirm-title` scope）
  5. 🔴 **TC-ROOM-00002 分页测试形同虚设**（阻断）：
     - **问题**：宽松断言 "页码变化、或当前只有一页而未发生跳转均属正常"（line 55）**等同于必过**，无论是否真的翻页都通过
     - **根因**：seed-e2e.sql 只新增了 3 个房间（User A/B/C），未满足 TDS 验收用例 "≥12 房间数据下真实验证分页"
     - **commit message 误导**：称 "数据库已有 81 个房间" 指整个开发环境，而非 seed 幂等保证的数据
     - **TDS 方案 A 要求**："在 `globalSetup` 或专用 fixture 内通过 AppServer 接口幂等创建 ≥ 12 个 E2E 房间"
  6. 🟠 **TC-ROOM-00003 / TC-USER-00003 自愈模式反模式**（高风险）：
     - **问题**：测试中动态修改数据库（Docker psql 激活房间 / API 解封用户），违反测试隔离原则
     - **隐患**：全新数据库中无已关闭房间/已封禁用户时自愈失败；并发执行时相互干扰；测试变成"修数据"而非"验功能"
     - **修复方向**：将自愈逻辑移至 `globalSetup` 或 seed-e2e.sql 幂等保证前置数据
  7. 🟡 **TC-USER-00002 try-catch 吞掉真实失败**（中等）：二次确认框的 AI 操作失败被静默吞掉（line 79-86），可能掩盖 UI 问题
  8. 🟡 **known-issues.md 未登记 webkit 时序模式**（可选）：TDS 第二节提到登记通用 wait-pattern，实际内联在代码中，降低复用性
- **改进建议**：
  1. **TC-ROOM-00002 修复**（必须）：
     - 在 seed-e2e.sql 或 globalSetup 中幂等创建 12+ 房间（建议循环 INSERT，分配给不同 owner 避免 idx_rooms_owner_active 冲突）
     - 恢复真实断言：`await agent.aiAssert('当前页码显示为第 2 页，或分页控件高亮"2"')`
  2. **自愈逻辑重构**（必须）：
     - TC-ROOM-00003：将 "激活 3 个已关闭房间" 移至 globalSetup，确保测试前至少 3 个 active 房间
     - TC-USER-00003：将 "封禁一个用户" 移至 globalSetup，确保测试前至少 1 个 banned 用户
  3. **TC-USER-00002 try-catch 优化**（建议）：如果二次确认框非必现，在 catch 中打印 WARN 日志（当前完全静默）
  4. **known-issues.md 登记**（可选）：webkit 抽屉时序模式（waitForLoadState + drawer detached wait）登记为通用模式，供其他用例复用
