## TC-WS-00001：WS 握手 - JWT 正确/错误

- **归属模块**：`WS`
- **测试类型**：`Security`
- **回归级别**：`P0`

**【前置条件】**
1. U1 登录获得 `TOKEN_U1`。

**【执行步骤与断言】**
| 步骤序号 | 目标端 | 操作动作 (Action) | 预期结果 (Assertion) |
| :------: | :----- | :---------------- | :------------------- |
| 1 | `AppServer` | WS 连接 `ws://localhost:3000/ws?token={TOKEN_U1}` | HTTP 101 Switching Protocols，连接建立 |
| 2 | `AppServer` | WS 连接 `?token=invalid_token` | 连接被服务端立即关闭，close code=4001 或 1008 |

**【数据清理】**
- 关闭连接。

## TC-WS-00003：断线重连 - 携带 last_msg_id 补偿
**【元数据】**
- **归属模块**：`WS`
- **测试类型**：`Integration`
- **回归级别**：`P1`

**【前置条件】**
1. U1 已进入房间 R1，最后接收到的 msg_id=`M100`。
2. 服务端房间内发生 5 条广播消息 M101~M105。

**【执行步骤与断言】**
| 步骤序号 | 目标端 | 操作动作 (Action) | 预期结果 (Assertion) |
| :------: | :----- | :---------------- | :------------------- |
| 1 | `AppServer` | U1 连接断开（模拟 TCP RST） | 服务端检测并在 30s 内清理 |
| 2 | `AppServer` | U1 重新连接 WS，携带 `?last_msg_id=M100` 或发送 Resume 消息 | 服务端回放 M101~M105 到客户端 |
| 3 | `AppServer` | 客户端 `last_msg_id` 过旧（超出缓冲窗口） | 收到 `ResumeFailed` 指示需重新拉取房间状态快照 |

- **回归级别**：`P1`

**【前置条件】**
