## 需求描述

新建 `RankingScreen` 排行榜页，展示日榜/周榜的魅力值（收礼）与财富值（送礼）排名。

## 入口

| 入口位置 | 交互 |
|---------|------|
| 大厅（HallScreen）顶部栏 🏆 图标 | 点击进入 |
| 房间页 TopBar 溢出菜单"榜单" | 点击进入（弹出房间内专用浮窗；Phase 1 MVP 可直接复用 RankingScreen） |

## 交互逻辑

| 操作 | 行为 |
|------|------|
| 切换主 Tab | 重新加载对应榜单（魅力/财富） |
| 切换子 Tab | 重新加载对应周期（日/周） |
| 下拉刷新 | 重新拉取当前榜单 |
| 点击榜单项 | Phase 1 MVP：Toast"个人资料页即将上线"（Phase 2 跳个人资料页） |
| 未入榜 | 底部栏显示"你还未上榜，继续加油！" + 进入房间送礼入口按钮 |

## 异常流程

| 场景 | 处理 |
|------|------|
| 榜单为空（新榜刚切换） | 占位图 + "榜单刚刚刷新，快去送礼占据榜首吧！" |
| 网络失败 | Skeleton → 错误占位 + 重试 |
| 时区切换中（任务滞后） | 显示旧榜 + Snackbar"榜单即将刷新" |

## 通用组件

- `AvatarWithFrame`（T-30018）
- `TopThreePodium`（新）：前三名展示台（Top1 居中大 + Top2 左 + Top3 右）
- `RankingItem`（新）：常规榜单项
- `MyRankingBar`（新）：底部固定自身排名条

## TDD 验收标准

1. 默认展示"魅力榜/日榜"
2. 四组 Tab 组合独立加载数据（切换时显示 loading）
3. Top1 头像带王冠 + 金色光圈
4. Top2 头像银色光圈；Top3 头像铜色光圈
5. 当前用户在 Top50 内时底部栏高亮"我: No.N"
6. 当前用户未入榜时底部栏显示"你还未上榜"
7. 下拉刷新正常工作
8. 榜单空时显示占位图
9. 大厅顶部 🏆 图标 + 房间页"榜单"菜单可进入此页
