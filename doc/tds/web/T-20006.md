# TDS: 用户管理页面 (Task ID: T-20006)

## 一、背景与目标
- **关联需求**：来自 `doc/product.md` 用户管理模块——运营后台需要对 App 端用户进行检索与管理
- **本 Task 的目标**：实现 `/users` 路由页面，通过 Ant Design Table 展示用户列表，支持按手机号/ID/昵称搜索、状态筛选（全部/正常/封禁）和分页加载，对接 T-10007 提供的 `GET /api/v1/admin/users` 接口

## 二、方案设计

### §2.1 核心数据流

```
URL Query String (?phone=&nickname=&status=&page=&size=)
        │  ← 初始化时从 URL 读取参数
        ↓
useUsersPage (Hook)
  ├── 搜索表单状态（phone / nickname / userId / status）
  ├── 分页状态（page, pageSize=20）
  ├── 防竞态：AbortController（每次 fetch 新建，cleanup 时 abort）
  ├── 调用 adminGetUsers(params, signal) → apiClient.ts
  └── 返回 { items, total, loading, error, filters, page, pageSize,
              setFilters, setPage, refresh }
        │
        ↓
UsersPage (页面入口)
  ├── UserSearchForm（搜索表单：手机号 / 用户ID / 昵称 / 状态）
  └── UsersTable（Ant Design Table：列表 + 分页）
```

**URL 同步策略**：搜索参数（phone/nickname/userId/status/page/size）双向同步到 URL Query String，支持刷新保持搜索状态，便于运营分享链接。

### §2.3 接口规格（来自 T-10007）

```
GET /api/v1/admin/users
Authorization: Bearer <admin-jwt>
```

#### 查询参数

| 参数 | 类型 | 必须 | 说明 |
|------|------|------|------|
| `phone` | string | 否 | 手机号精确匹配 |
| `nickname` | string | 否 | 昵称模糊匹配 |
| `user_id` | string (UUID) | 否 | 用户 ID 精确匹配 |
| `status` | `"normal"` \| `"banned"` | 否 | 封禁状态过滤 |
| `page` | integer | 否 | 页码，默认 1 |
| `size` | integer | 否 | 每页条数，默认 20，最大 100 |

#### 成功响应（200）

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "total": 42,
    "page": 1,
    "size": 20,
    "items": [
      {
        "id": "uuid-string",
        "phone": "13800138000",
        "nickname": "用户昵称",
        "avatar": "https://cdn.example.com/avatar.jpg",
        "coin_balance": 1000,
        "vip_level": 1,
        "status": "normal",
        "created_at": "2025-01-01T00:00:00Z"
      }
    ]
  }
}
```

### §2.4 组件设计

#### `useUsersPage` Hook

```typescript
export interface UsersPageFilters {
  phone?: string;
  nickname?: string;
  userId?: string;
  status?: 'normal' | 'banned';
}

export interface UseUsersPageReturn {
  items: AdminUserItem[];
  total: number;
  loading: boolean;
  error: Error | null;
  page: number;
  pageSize: number;
  filters: UsersPageFilters;
  setPage: (page: number, pageSize: number) => void;
  setFilters: (filters: UsersPageFilters) => void;
  refresh: () => void;
}
```

核心逻辑与 `useRoomsPage` 保持一致：
- 搜索参数变化时重置 `page=1`
- 每次 fetch 新建 `AbortController`，cleanup 时 abort 防竞态
- `refreshKey` 自增触发强制刷新
- 搜索参数双向同步 URL Query String（使用 `react-router-dom` 的 `useSearchParams`）

## 三、TDD 验收用例

### 正向用例

- [ ] **P01** 初次加载：页面挂载时自动发起 `GET /admin/users?page=1&size=20`，loading 期间表格显示骨架屏（`loading={true}`），数据返回后正确渲染 N 行
- [ ] **P02** 搜索：填写手机号 `13800138000` 后点击"搜索"，请求携带 `phone=13800138000&page=1&size=20`
- [ ] **P03** 搜索昵称：填写昵称后点击"搜索"，请求携带 `nickname=xxx`
- [ ] **P04** 搜索用户 ID：填写 UUID 后点击"搜索"，请求携带 `user_id=xxx`
- [ ] **P05** 状态筛选：选择"封禁"后点击"搜索"，请求携带 `status=banned`
- [ ] **P06** 重置：点击"重置"按钮，表单清空，URL 参数清除，重新发起无参数请求
- [ ] **P07** 分页：点击第 2 页，请求携带 `page=2`；切换每页条数到 50，请求携带 `size=50`
- [ ] **P08** 刷新按钮：点击刷新，以当前 filters 重新发起请求
- [ ] **P09** URL 状态持久化：带参数 `?phone=138&status=banned&page=2` 刷新页面，搜索框/下拉框/分页自动恢复对应值，并自动发起对应查询
- [ ] **P10** `UserStatusTag` 渲染：`status="normal"` 显示绿色"正常"；`status="banned"` 显示红色"封禁"
- [ ] **P11** 空列表：API 返回 `total=0, items=[]` 时，表格显示"暂无数据"空态

### 异常用例

- [ ] **E01** 网络失败：fetch 抛出 `TypeError: Failed to fetch`，页面显示错误 `Alert`，不渲染旧数据
- [ ] **E02** 401 未授权：API 返回 `HTTP 401`（无现有 token），`useAuthStore.logout()` 被调用，页面跳转 `/login`
- [ ] **E03** 组件卸载防竞态：快速切换路由（卸载 UsersPage），进行中的 fetch 被 abort，不触发 `setState`（无 "Can't perform a React state update on an unmounted component" 警告）
- [ ] **E04** 搜索参数变化重置分页：处于第 3 页时修改搜索条件并提交，`page` 自动重置为 1
- [ ] **E05** 操作列按钮：本期"查看详情"按钮处于 disabled 状态（T-20007 占位），点击无响应
