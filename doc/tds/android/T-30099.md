## 二、方案设计

### 核心数据流

本 Task 不改变运行时数据流，只调整：
1. **Compose 语义树**：组件最外层 Box 增加 `Modifier.semantics(mergeDescendants = true) {}`，并以稳定 `testTag` 暴露给测试，使语义查询不依赖文案 / 子节点 index。
2. **测试查询路径**：`onNodeWithText` / `onAllNodes(hasText())[i]` → `onNodeWithTag(...)`；查询合并节点内部子项使用 `useUnmergedTree = true`；长内容用 `performScrollTo()` 兜底；substring 文本断言用 `substring = true`。
3. **TextField 单向数据流**：`GoldOutlinedTextField` 内部不得截断 `onValueChange`，输入事件必须透传到调用方的 state holder。

### BUG-ANDROID-002 修复策略（语义树查询失效）

**业务侧**：
- `GoldButton` / `GoldOutlinedTextField`：外层 Box 加 `Modifier.semantics(mergeDescendants = true) {}`，并保留稳定 `testTag` 入参；
- 给 LoginScreen / ProfileContent / PlaceholderScreen / RoomBottomBar / HostMicSlot / MicSlotsGrid / ChatBackground 等视觉升级页面的关键交互元素补 `Modifier.testTag(...)`；
- 命名约定：`<feature>_<element>` 全小写下划线（例：`login_phone_input`、`profile_avatar`、`placeholder_subtitle`、`room_bottom_chat_btn`、`mic_slot_host`、`mic_slots_grid_item_<index>`）。

**测试侧**：
- 替换所有 `onNodeWithText("...")` / `onAllNodes(hasText("..."))[i]` 为 `onNodeWithTag("<tag>")`；
- 查询合并节点内部 child 时改用 `composeTestRule.onNodeWithTag("xxx", useUnmergedTree = true)`；
- 长列表 / 长文案先 `performScrollTo()` 再断言；
- 文案 substring 断言统一加 `substring = true`。

### BUG-ANDROID-003 修复策略（setContent 重入）

定位 `PlaceholderScreenTest.PH09`（RTL 验证）：当前实现先 setContent 默认方向，再 setContent RTL 方向以做对比，触发 `Cannot call setContent twice per test!`。

**修复**：改为单次 `setContent`，在 `CompositionLocalProvider(LocalLayoutDirection provides LayoutDirection.Rtl)` 内渲染目标 Composable，需对比两种方向的用例拆分为独立 `@Test` 方法（每个方法只有一次 setContent）。

```kotlin
@Test fun PH09_rtl_layout() {
    composeTestRule.setContent {
        CompositionLocalProvider(LocalLayoutDirection provides LayoutDirection.Rtl) {
            PlaceholderScreen(...)
        }
    }
    composeTestRule.onNodeWithTag("placeholder_subtitle").assertIsDisplayed()
}
```

### BUG-ANDROID-004 修复策略（onValueChange 回调断链）

审查 `core/theme/GoldOutlinedTextField.kt`，常见原因：
1. 内部 `remember { mutableStateOf("") }` 截断了外层 onValueChange；
2. `onValueChange = { internalState.value = it }` 只更新内部 state，没回调外层。

**修复**：保持单一 source of truth：内部不持有 `var text by remember { mutableStateOf(...) }`，或若内部需要本地缓存格式化后的展示，必须在 `onValueChange` 中先调用外层回调再更新本地 state：

```kotlin
OutlinedTextField(
    value = value,                       // 来自调用方
    onValueChange = { newValue ->
        onValueChange(newValue)          // 必须回调外层
    },
    ...
)
```

如果内部确实需要 visual transformation，使用 `VisualTransformation` 而非内部 state 截断输入。
