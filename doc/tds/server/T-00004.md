# TDS: JWT 中间件 (Task ID: T-00004)

## 一、背景与目标
- **关联需求**：所有需要认证的 C 端接口
- **前置依赖**：T-00003 (手机号一键登录接口)
- **本 Task 的目标**：实现 Axum JWT 校验中间件，校验 JWT 有效性并注入 user_id 到请求上下文

## 二、方案设计
### 核心数据流
Request → Authorization Header → JWT 中间件 → shared::jwt::decode_jwt() → 提取 sub (user_id) → 注入 AuthContext → next handler

### 涉及文件清单
| 文件路径 | 变更类型 | 说明 |
|----------|----------|------|
| `app/server/src/common/middleware/jwt_auth.rs` | 新增 | JWT 校验中间件 |
| `app/server/src/common/auth/context.rs` | 新增 | AuthContext 结构体 |
| `app/server/src/common/auth/claims.rs` | 新增 | AppClaims { sub, iat, exp, iss } |
| `app/shared/src/jwt/decode.rs` | 引用 | JWT 解码 |

### 接口 / 信令定义
无新增接口，作为 Axum 中间件层透明校验 Authorization Header

## 三、TDD 验收用例
- [ ] 正向：合法 token → 注入 user_id 到 AuthContext → 后续 handler 可取到
- [ ] 异常：无 Authorization Header → 401 (40101)
- [ ] 异常：格式错误（非 "Bearer xxx"）→ 401 (40101)
- [ ] 异常：签名无效 → 401 (40101)
- [ ] 异常：token 过期 → 401 (40102)
- [ ] 异常：iss 不是 "voiceroom" → 401 (40101)
