## 3. 审批操作

| 动作 | 行为 |
|------|------|
| 点 ✅ 通过 | 二次确认（仅当家族 ≥ 90% 满员时） + 调用 API |
| 点 ❌ 拒绝 | 弹 BottomSheet 选择拒绝原因（成员已满 / 等级不够 / 重复申请 / 其他+自定义） |
| 点头像 / 资料卡 | 弹 `UserProfileSheet`（T-30090） |
| 批量通过 | 仅当剩余席位 ≥ 选中数才允许，否则 Toast |
| 批量拒绝 | 二次确认 + 选择统一原因 |

## 4. 状态机

| 状态 | UI |
|------|---|
| `Loading` | Skeleton |
| `Empty` | 空态 "暂无待审批申请" |
| `Idle` | 列表展示 |
| `BatchSelecting` | 行前出现 checkbox |
| `Submitting (single)` | 该行按钮 loading |
| `Submitting (batch)` | 全屏 loading + 进度 "3/10" |
| `Error` | Snackbar |

## 5. 异常流程

| 场景 | 处理 |
|------|------|
| 申请并发处理（其他 Admin 已审）| 列表实时移除（WS `FamilyApplicationProcessed` 推送），Toast "该申请已被 法蒂玛 处理" |
| 通过时家族刚满员 | 后端返 40945 FAMILY_FULL → 列表保留行 + 红字提示 |
| 申请人已加入其他家族 | 后端返 40946 → 行变灰 + "该用户已加入其他家族" |
| Admin 在审批中被罢免 | 后端返 401 → 全屏占位"您不再是家族管理员" + 返回 |
| 家族被解散 | 401 占位 |
| 网络失败 | 整页保留 + 重试按钮 |

## 6. 通用组件

- `ApplicationRow`（新）：头像 + 等级 + 贵族 + 申请理由 + 时间戳 + 操作三按钮
- `RejectReasonBottomSheet`（新）：4 项预设 + 自定义输入（≤100 字）
- `BatchActionBar`（复用 T-20013 思路）

## 8. 协议入口索引

| 操作 | 协议 |
|------|------|
| 待审批列表 | GET `/api/v1/families/:id/applications?state=pending&cursor=` |
| 历史 | GET `/api/v1/families/:id/applications?state=approved\|rejected` |
| 通过 | POST `/api/v1/families/:id/applications/:appId/approve` |
| 拒绝 | POST `/api/v1/families/:id/applications/:appId/reject { reason }` |
| 批量 | POST `/api/v1/families/:id/applications/batch { ids, action, reason? }` |
| 实时移除 | WS `FamilyApplicationProcessed { app_id, processor_id }` |
