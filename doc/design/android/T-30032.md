# T-30032: 余额不足引导弹窗

> **归属端**: Android
> **模块**: UI/Wallet
> **前置依赖**: T-30028 (礼物面板)
> **参考**: [business_flows.md §2.7.4 A](../../product/business_flows.md)

---

## 需求描述

当用户尝试送礼但钻石不足时弹出 `InsufficientBalanceDialog`，引导跳转钱包页（即充值引导）。

## 触发时机

1. 礼物面板本地预校验：点击"送出"前检查余额 < 总价
2. Server 返回 `INSUFFICIENT_BALANCE` 错误（防御：余额被并发扣减）

## 弹窗布局

```
┌────────────────────────────┐
│       💎 钻石不足           │  ← 标题, titleLarge, 金色
│                            │
│    当前余额：   12 💎        │  ← bodyMedium
│    所需余额：  520 💎        │
│    还差：     508 💎        │  ← 红色强调
│                            │
│  [  取消  ]  [  去充值  ]   │  ← 右侧金色主按钮
└────────────────────────────┘
```

## 交互逻辑

| 操作 | 行为 |
|------|------|
| "去充值" | 跳转 WalletScreen（不关闭底层礼物面板，返回时面板保持选中状态） |
| "取消" | 关闭弹窗，礼物面板保留 |
| 点击外部 | 不关闭（防误触） |
| 返回键 | 关闭弹窗 |

## 通用组件

- 标准 Material3 `AlertDialog`
- 按钮使用 `GoldButton`（主）+ TextButton（次）

## 自动化测试锚点

| testTag | 元素 | 断言 |
|---------|------|------|
| `insufficient_balance_dialog` | 弹窗容器 | 余额不足时可见 |
| `dialog_current_balance` | 当前余额 | 等于 user.diamond_balance |
| `dialog_required_amount` | 所需金额 | 等于 gift.price * count |
| `dialog_shortfall` | 差额 | 等于 required - current（红色） |
| `btn_go_recharge` | 去充值按钮 | 点击后跳转 WalletScreen |
| `btn_cancel_dialog` | 取消按钮 | 点击后关闭 |

## TDD 验收标准

1. 余额不足时弹窗自动显示
2. 显示当前余额、所需金额、差额（红色）
3. 点击"去充值"跳转 WalletScreen
4. 跳转后返回礼物面板仍保留选中礼物+接收者+数量
5. 点击外部不关闭
6. 点击"取消"关闭弹窗
7. Server 返回 INSUFFICIENT_BALANCE 时也能正确触发
