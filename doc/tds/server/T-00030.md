# TDS: WS TransferAdmin + ForceTakeMic/ForceLeaveMic (Task ID: T-00030)

## 一、背景与目标
- **关联需求**：E-10 管理员任命 + 抱麦 ([phase1_room_governance §2.1.3](../../product/phase1_room_governance.md))
- **本 Task 的目标**：房主任命/撤销管理员 + 管理员抱上/抱下麦；原子切换；广播 AdminChanged/MicTaken/MicLeft。

## 二、方案设计

### 信令
**`TransferAdmin` C→S**（仅房主）
```json
{ "type":"TransferAdmin", "msg_id":"uuid",
  "payload":{ "room_id":"uuid", "target_user_id":"uuid", "action":"assign"|"revoke" } }
```

**广播 `AdminChanged` S→所有成员**
```json
{ "type":"AdminChanged", "payload":{
    "room_id":"uuid", "admin_user_id":"uuid|null",
    "previous_admin_id":"uuid|null", "operator_id":"uuid" } }
```

**`ForceTakeMic` C→S**（owner/admin）
```json
{ "type":"ForceTakeMic", "msg_id":"uuid",
  "payload":{ "room_id":"uuid", "target_user_id":"uuid", "slot_index":2 } }
```

**`ForceLeaveMic` C→S**（owner/admin）
```json
{ "type":"ForceLeaveMic", "msg_id":"uuid",
  "payload":{ "room_id":"uuid", "target_user_id":"uuid" } }
```

### 业务规则
**TransferAdmin assign**:
- 仅房主（ctx.user_id == room.owner_id）可发，否则 40301
- target_user_id 必须在房间
- target 不能是房主（40302）
- 若已有其它管理员（room.admin_user_id is Some）→ 先隐式 revoke 旧管理员 previous_admin_id
- UPDATE rooms SET admin_user_id = target
- INSERT admin_logs (跨服务不走，仅 App Server 本地 log)
- 广播 AdminChanged

**TransferAdmin revoke**:
- target_user_id 必须是当前管理员
- UPDATE rooms SET admin_user_id = NULL
- 广播 AdminChanged admin_user_id=null

**ForceTakeMic**:
- 权限：owner 或 admin
- target 不能为被禁麦（mic_muted:* 存在 → 40306）
- 麦位空闲检查：`slot_index` 未被占
- 广播 `MicTaken { forced_by: operator_id }` 给房间
- **target 端 RTC 权限处理**：客户端收到后自己决定是否请求麦克风权限（本 Task 仅通知）

**ForceLeaveMic**:
- 权限：owner 或 admin（但管理员不能抱下房主）
- target 必须在麦（否则 40404 MIC_NOT_FOUND）
- `room_manager.leave_mic(slot)` + 广播 `MicLeft { forced_by: operator_id }`
