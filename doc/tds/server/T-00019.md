# TDS: 礼物配置表 + 列表 API (Task ID: T-00019)

## 一、背景与目标
- **关联需求**：E-07 礼物闭环基础配置 ([phase1_gift_economy.md §3 档位表](../../product/phase1_gift_economy.md))
- **本 Task 的目标**：提供礼物配置的数据库存储、种子数据、C 端列表接口（含多语言 + 缓存）。

## 二、方案设计

### 核心数据流
```
Android GET /gifts/list?lang=ar ─► handler
    ├── Redis GET gifts:list:ar       (hit)→ 返回
    └── Miss → SELECT FROM gifts WHERE is_active=true AND is_deleted=false
              ORDER BY tier, sort_order
              → 缓存 60s + 返回
```

### 迁移脚本 (005_create_gifts.sql)
```sql
CREATE TABLE IF NOT EXISTS gifts (
    id             UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    code           VARCHAR(32) NOT NULL UNIQUE, -- 稳定标识，如 'rose_01'
    name_en        VARCHAR(64) NOT NULL,
    name_ar        VARCHAR(64) NOT NULL,
    icon_url       TEXT NOT NULL,
    price          BIGINT NOT NULL CHECK (price >= 1),
    tier           SMALLINT NOT NULL CHECK (tier BETWEEN 1 AND 5),
    effect_level   SMALLINT NOT NULL DEFAULT 1, -- 1:none 2:slot 3:bottom 4:fullscreen 5:fullscreen+border
    animation_url  TEXT,
    sort_order     INT NOT NULL DEFAULT 0,
    is_active      BOOLEAN NOT NULL DEFAULT true,
    is_deleted     BOOLEAN NOT NULL DEFAULT false,
    created_at     TIMESTAMPTZ NOT NULL DEFAULT now(),
    updated_at     TIMESTAMPTZ NOT NULL DEFAULT now()
);
CREATE INDEX idx_gifts_active_order ON gifts(tier, sort_order) WHERE is_active AND NOT is_deleted;
```

### HTTP 接口：GET /api/v1/gifts/list
- **鉴权**：可选（登录/未登录皆可读取）
- **Query**：无
- **Header**：`Accept-Language: ar|en` 默认 `ar`
- **响应**：
```json
{
  "code":0,
  "data": {
    "items":[
      { "id":"uuid", "code":"rose_01", "name":"وردة", "icon_url":"...", "price":1,
        "tier":1, "effect_level":1, "animation_url":null, "sort_order":10 }
    ],
    "version": "1713312000"  // 缓存版本，用于客户端做增量
  }
}
```

### 缓存策略
- Redis key: `gifts:list:{lang}` TTL 60s
- 后台（T-10014）CRUD 写入后 `DEL gifts:list:ar gifts:list:en`，保证前端秒级刷新

- **改进建议**：
  1. **[MEDIUM] 缓存偏差**：实现使用进程内存缓存，TDS 设计为 Redis 缓存。MVP 阶段单实例可接受，建议 T-10014 Admin CRUD 落地时同步评估切换 Redis，确保多实例部署时 `invalidate_all()` 能跨实例统一失效。
  2. **[MEDIUM] G04 测试覆盖弱点**：G04 通过 FakeGiftService（预置数据全为 active）验证，断言（name非空/price>0）不能真正证明过滤逻辑。建议在 service 单元测试中补充含 `is_active=false` / `is_deleted=true` 礼物的用例，验证其不出现在返回列表中（参考 GS06 的 FakeGiftRepo 模式）。
  3. **[LOW] version 不同步**：ar/en 各自 miss 缓存时生成不同 `version` 时间戳，同一批数据可能有两个不同 version 值。建议后续将 version 改为基于数据最新 `updated_at` 或统一生成，避免客户端增量刷新误判。
  4. **[LOW] FakeGiftService UUID 不稳定**：每次 `list_active` 调用均 `Uuid::new_v4()`，与真实 GiftService 缓存命中后返回相同 id 的行为不一致，可在需要依赖 id 稳定性的测试中注意。
