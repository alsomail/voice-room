# TDS: WEB E2E 9-FAIL 修复（TC-ROOM/GIFT/USER 测试代码/数据/时序硬化） (Task ID: T-0000R)

## 一、背景与目标

- **关联战报**：[report-20260428-154125 SUMMARY](../../../tests/report-20260428-154125/SUMMARY.md)、[TC-ROOM Report](../../../tests/report-20260428-154125/WEB/TC-ROOM/Report.md)、[TC-GIFT Report](../../../tests/report-20260428-154125/WEB/TC-GIFT/Report.md)、[TC-USER Report](../../../tests/report-20260428-154125/WEB/TC-USER/Report.md)

- **本 Task 的目标**：把 v3 回归 WEB 套件最后 9 个 FAIL（4 个 case × 多浏览器）全部修绿，使 WEB 套件 54/54 PASS，**不涉及任何业务代码（仅测试脚本/Seed/选择器/时序）**

> **归属判定**：经审阅 3 份 Report.md，9 个 FAIL 根因 100% 为**测试侧问题**（测试数据不足 / 选择器歧义 / Midscene 提示词偏差 / webkit 时序），与房间/礼物/用户业务实现无关。因此挂在**模块 9（E2E 测试基建）** 而非业务模块 3/6。Master-coordinator 委派文案中 "真业务/前端 bug" 的判断与战报根因冲突，以战报为准。

## 二、方案设计

### 9 个 FAIL 根因分组

| Bug ID | 用例 | 浏览器 | 根因 | 修复方向 |
|--------|------|--------|------|---------|
| BUG-WEB-001a | TC-GIFT-00002 | chromium | 180s test timeout（Midscene 动作链过长） | ① `test.setTimeout(300_000)` 单 case 提升；② 拆分为 "新增 + 编辑" / "上下架 + 删除" 两个子 case |
| BUG-WEB-001b | TC-GIFT-00002 | firefox | AI 断言文字 "已下架" 不存在（实为 toggle 开关） | 改 Midscene prompt → "礼物行的状态开关处于关闭（灰色/未激活）状态" |
| BUG-WEB-002 | TC-ROOM-00002 | chromium/firefox/webkit | 测试数据不足 10 条，分页第 2 页不存在 | 方案 A：在 `globalSetup` 或专用 fixture 内通过 AppServer 接口幂等创建 ≥ 12 个 E2E 房间；方案 B：断言改为 "点击下一页按钮后页码 ≠ 之前" 而非硬断言 "第 2 页"。优先 A（更真实） |
| BUG-WEB-003 | TC-ROOM-00003 | chromium/firefox/webkit | Playwright strict mode：`getByText('确认强制关闭')` 命中 2 元素（`.ant-modal-title` + `.ant-modal-confirm-title`） | `page.locator('.ant-modal-confirm-title').getByText('确认强制关闭')` 或 `.first()`（保留可读性首选 scope locator） |
| BUG-WEB-004 | TC-USER-00002 | webkit only | 封禁后抽屉关闭动画未完成时 Midscene 已截图 | 操作后串行：`await page.waitForLoadState('networkidle'); await page.locator('.ant-drawer-open').waitFor({ state: 'detached', timeout: 5_000 });` 再交给 AI 断言 |
