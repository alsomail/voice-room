# T-30019: Splash 启动页

> **归属端**: Android  
> **模块**: UI/Splash  
> **前置依赖**: T-30018 (MenaTheme)  
> **参考**: [Android App 界面设计规范 §2.1](../../product/android_app_design.md)

---

## 需求描述

品牌 Splash 启动页，展示 App Logo + 动画，自动判断登录态后导航。

## 页面布局

```
┌──────────────────────────────┐
│                              │
│        (深色背景 #1A1A2E)     │
│                              │
│                              │
│         [App Logo]           │  ← 居中，金色，120dp
│          光效动画             │  ← scale 0.5→1.0 + alpha 0→1 (800ms)
│                              │
│                              │
│                              │
│         v1.0.0               │  ← 底部居中，白色 bodySmall
│                              │
└──────────────────────────────┘
```

## 交互逻辑

1. 进入 Splash → 播放 Logo 动画（800ms）
2. 动画结束后延迟 500ms（总停留 ~1.3s）
3. 检查 `TokenManager` 中 JWT 有效性：
   - **有效** → 导航到 `MainScreen`（替换栈，不可返回）
   - **无效/过期** → 导航到 `LoginScreen`（替换栈）
4. 导航使用 `popUpTo(splash) { inclusive = true }` 防止回退到 Splash

## 异常流程

| 场景 | 处理 |
|------|------|
| JWT 解析异常 | 视为无效，跳转 LoginScreen |
| 本地存储读取失败 | 视为未登录，跳转 LoginScreen |

## 通用组件提取

无新组件（使用 MenaTheme 包裹）

## 自动化测试锚点

| testTag | 元素 | 点击/状态断言 |
|---------|------|--------------|
| `splash_screen` | 整个 Splash 容器 | 可见性：进入后立即可见 |
| `splash_logo` | Logo 图片 | 可见性：动画后可见 |
| `splash_version` | 版本号文字 | 文本内容包含版本号 |

## TDD 验收标准

1. Splash 页面在 `MenaTheme` 深色背景下渲染
2. Logo 显示并有缩放+淡入动画
3. 有有效 JWT 时，1.5s 内自动导航到 MainScreen
4. 无有效 JWT 时，1.5s 内自动导航到 LoginScreen
5. 导航后无法通过返回键回到 Splash
