# TDS: 禁麦/禁言 UI 反馈 + 抱麦集成 (Task ID: T-30044)

## 一、背景与目标
- **关联需求**：E-10 本地 UI 对禁/抱的硬约束 ([设计文档](../../design/android/T-30044.md))
- **本 Task 的目标**：禁麦禁言本地置灰防发请求；ForceTakeMic/ForceLeaveMic 自动化处理。

### State
```kotlin
data class SelfGovernanceState(
    val micMutedUntil: Instant? = null,
    val chatMutedUntil: Instant? = null
) {
    val micMuted: Boolean get() = micMutedUntil?.isAfter(now()) == true
    val chatMuted: Boolean get() = chatMutedUntil?.isAfter(now()) == true
}
```

### ChatInput
```kotlin
OutlinedTextField(
    enabled = !state.chatMuted,
    placeholder = {
        Text(if (state.chatMuted)
            stringResource(R.string.chat_muted_placeholder, remainingMin)
            else stringResource(R.string.chat_placeholder))
    }
)
// 发送按钮同步置灰
```

### MicSlot "+" 按钮
```kotlin
IconButton(
    onClick = {
        if (state.micMuted) {
            Toast("你已被禁麦，暂不能上麦")
            return@IconButton
        }
        viewModel.takeMic(slotIndex)
    },
    enabled = !state.micMuted    // 视觉灰
)
```

### 被 ForceTakeMic 流程
```kotlin
WsEvent.MicTaken (target_user_id=self, forced_by != null) ->
    if (!permissionManager.hasMic()) {
        permissionManager.request().onResult { granted ->
            if (granted) rtcEngine.startPublishing(slot)
            else wsClient.send(MicLeave(slot))   // 自动下麦
        }
    } else {
        rtcEngine.startPublishing(slot)
    }
```

### 被 ForceLeaveMic 流程
```kotlin
WsEvent.MicLeft (target_user_id=self, forced_by != null) ->
    rtcEngine.stopPublishing()
    Toast("你已被抱下麦")
    // UI 状态由 roomState.onMicSelf = false 驱动
```

### testTag
`Key('chat_input')` + `enabled/disabled` 属性断言、`Key('mic_slot_$idx')`

## 四、实现结果
- **实际修改的文件**：
  - `feature/room/governance/SelfGovernanceState.kt`（新建）：`micMutedUntil/chatMutedUntil: Long?` + `isMicMuted(nowMs)/isChatMuted(nowMs)` 计算函数；用 `Long` 而非 `Instant` 便于 FakeClock 注入
  - `feature/room/IMicPermissionChecker.kt`（新建）：`hasMicPermission()` + `requestMicPermission(callback)` 接口 + `AlwaysGrantedMicPermissionChecker` 默认实现
  - `feature/room/RoomViewModel.kt`（修改）：
    - 新增 `micPermissionChecker: IMicPermissionChecker` 构造参数（默认 `AlwaysGrantedMicPermissionChecker`）
    - 新增 `_selfGovernanceState: MutableStateFlow<SelfGovernanceState>` + 对外暴露 `selfGovernanceState: StateFlow`
    - `UserMuted` handler 新增同步更新 `_selfGovernanceState`（与 MuteCountdownViewModel 互补）
    - `onMicPermissionGranted()` 新增禁麦守卫（禁麦中 → ShowToast，不发 WS）
    - `sendMessage()` 新增禁言守卫（禁言中 → 直接 return，不发 WS）
    - `MicTaken` handler 新增 `forcedBy` 检测：无权限时调用 `requestMicPermission`，拒绝则发 `LeaveMic`；有权限时直接推流
    - `MicLeft` handler 新增 `forcedBy` 检测：被强制下麦时发出 `ShowToast("你已被抱下麦")`
    - 提取私有 `startPublishingInternal(roomId, userId)` suspend 函数消除重复
  - `test/.../FakeMicPermissionChecker.kt`（新建）：`hasPermission` 控制字段 + `requestCallCount` 计数 + `grantPermission()/denyPermission()` 回调触发方法
  - `test/.../GovernanceUiTest.kt`（新建）：G44-01~G44-08 共 8 个单元测试全部通过
- **测试覆盖率**：8/8 验收用例通过；全套 659 个测试，9 个预存颜色常量失败（与 T-30044 无关）
- **完成时间**：2025-07-14
