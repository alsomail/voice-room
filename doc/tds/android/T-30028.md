### 数据结构
```kotlin
data class GiftPanelUiState(
    val gifts: List<GiftVO> = emptyList(),
    val loading: Boolean = true,
    val error: String? = null,
    val selectedGiftId: String? = null,
    val selectedCount: Int = 1,
    val balance: Long = 0,
    val recipients: List<MicUserVO> = emptyList(), // 来自房间状态
    val selectedRecipientId: String? = null,       // 默认主麦
    val activeTab: GiftTab = GiftTab.Hot,
) {
    val selectedGift get() = gifts.firstOrNull { it.id == selectedGiftId }
    val totalPrice get() = (selectedGift?.price ?: 0L) * selectedCount
    val canSend get() = selectedGift != null && selectedRecipientId != null
                         && balance >= totalPrice && recipients.isNotEmpty()
}

enum class GiftTab { Hot, All, Backpack /* Phase2 占位 */ }
```

### 核心交互
| 动作 | 行为 |
|------|------|
| 🎁 按钮点击 | 先检查连接 & 房间状态 → 弹 BottomSheet（`ModalBottomSheet`，高 55%） |
| 礼物列表 | `LazyVerticalGrid(columns=Fixed(4))`，选中项金色边框 |
| 数量选择器 | 档位 1/10/66/520/786/1314；选中高亮；总价实时计算 |
| 余额条 | 左侧💎数字，右侧"充值"→ Toast（T-30032 后期替换为 InsufficientBalanceDialog） |
| 外部点击 / × / 返回键 | 关闭 BottomSheet，清除选中态 |
| WS `BalanceUpdated` | 更新 balance，若 `totalPrice > balance` 则 `canSend=false` |
| WS `GiftUnlisted` (T-10014 触发) 或 list 刷新 | 若选中被下架，清 selectedGiftId |

### 数据拉取
- 打开面板时 `repo.listGifts()`：若内存缓存 <60s 直接返回，否则 `GET /api/v1/gifts/list` + Accept-Language（根据 `LocalConfiguration.locale`）
- 展示顺序按后端 `sort_order`；Hot Tab 取 tier∈[2,3]，All Tab 全部
