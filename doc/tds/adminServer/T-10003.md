# TDS: 管理员 JWT 中间件 + RBAC (Task ID: T-10003)

## 一、背景与目标
- **关联需求**：所有需要认证的 Admin 接口
- **前置依赖**：T-10002 (管理员登录接口)
- **本 Task 的目标**：实现 Axum 中间件，校验 Admin JWT 并基于 role 进行 RBAC 权限控制

## 二、方案设计
### 核心数据流
Request → Authorization Header → JWT 中间件 → shared::jwt::decode_jwt() → 验证 iss="voiceroom-admin" → 提取 {admin_id, role} → RBAC 中间件 → 检查 role 是否有权访问该 endpoint → 注入 AdminAuthContext → next handler

## 三、TDD 验收用例
- [ ] 正向：合法 Admin JWT → 注入 AdminAuthContext → 后续 handler 可取到 admin_id 和 role
- [ ] 正向：super_admin 可访问所有接口
- [ ] 正向：operator 可访问用户管理和房间管理
- [ ] 正向：cs 只能读取用户信息，不能执行封禁
- [ ] 正向：finance 只能访问统计和财务接口
- [ ] 异常：无 token → 401 (40101)
- [ ] 异常：C 端 JWT（iss="voiceroom"）尝试访问 Admin 接口 → 401 (40101)
- [ ] 异常：token 过期 → 401 (40102)
- [ ] 异常：角色权限不足 → 403 (40301)
- [ ] 正向：敏感操作（ban/close/update）自动写入 admin_logs
