# T-00046 WS 广播可观测性增强（BUG-CHAT-WS-BROADCAST-SILENT）

## 一、需求背景

Round 16 E2E 实证：Android 客户端走 WS `SendMessage` 信令发消息，WS 连接 ✅、JoinRoom ✅、INSERT ✅、连接保活 5 分钟+，但 Android 公屏空白。

根因：`app/server/src/ws/broadcaster.rs` 第 80 行 `let _ = sender.send(json.clone())` 静默吞掉所有发送错误，无法判断 send_loop 是否存活，stale connection 永久驻留 registry。

## 二、技术方案

修改 `broadcast_to_room_inner`（`app/server/src/ws/broadcaster.rs`）：

1. **广播开始**：`tracing::info!(room_id=%room_id, total_connections=N, "broadcast: starting")`
2. **循环 `get_connections_in_room` 返回的 Vec**，对每个 `(conn_id, sender)` 执行 `match sender.send(json.clone())`：
   - `Ok(_)` → `tracing::debug!(connection_id=%conn_id, room_id=%room_id, "broadcast: sent")`；ok_count += 1
   - `Err(_)` → `tracing::warn!(connection_id=%conn_id, room_id=%room_id, "broadcast: receiver dropped, removing stale connection")`；收集 `stale_ids`；fail_count += 1
3. **单连接失败不阻断其他**（continue，不 break/return）
4. **循环结束后**：遍历 `stale_ids`，调用 `registry.unregister(conn_id)`（锁外批量清理，避免死锁）
5. **广播总结**：`tracing::info!(room_id=%room_id, sent=ok_count, failed=fail_count, "broadcast: done")`
6. **不打印消息正文**（PII 保护）

注意：`broadcast_to_room_inner` 接收 `registry: &ConnectionRegistry`（不可变引用），`unregister` 只需 `&self`（DashMap 内部可变性），无需改函数签名。

## 三、TDD 验收用例

- **BR-08**: 向正常连接广播 → ok_count=1, fail_count=0, connection 仍在 registry
- **BR-09**: receiver drop 后广播 → fail_count=1, WARN 日志（通过 registry.count 间接验证）, stale connection 被移出 registry
- **BR-10**: 2 正常 + 1 stale 混合广播 → 正常连接全部收到消息, stale 被清理, registry 剩余 count=2

## 四、实现结果

- 修改 `app/server/src/ws/broadcaster.rs` `broadcast_to_room_inner` 第 79-83 行，重构发送循环：
  - 广播开始打 `INFO room_id total_connections "broadcast: starting"`
  - 每条连接 `match sender.send(json.clone())`：成功 `DEBUG`/失败 `WARN`，分别累加 `ok_count`/`fail_count`，失败者收集到 `stale_ids`
  - 循环外批量 `registry.unregister(conn_id)`，避免 DashMap 持锁期间二次写入
  - 广播结束打 `INFO room_id sent failed "broadcast: done"`
  - 全程不打印消息正文（PII 保护）
- 新增 BR-08 / BR-09 / BR-10 三条 `#[tokio::test]`，先确认 RED（BR-09/10 因 stale 未清理失败）→ 实现后全 GREEN。
- `cargo test -p voice-room-server --lib`：**471 passed; 0 failed**，无既有用例回归。
