# TDS: Seed/Reset/Preflight 三件套脚本 (Task ID: T-0000G)

> **主设计**：[T-0000E](./T-0000E.md)（§2.5 Seed 契约 / §2.6 preflight 契约）

## 一、背景与目标
- 修复体检缺口 C2/C12：`scripts/dev/seed-e2e.sql` 不存在，无法生成 E2E Token/ID。
- 输出 3 个脚本，让 local E2E 真正可冷启动跑通。

## 二、方案设计

### 涉及文件清单
| 文件路径 | 变更类型 | 说明 |
|---|---|---|
| `scripts/dev/seed-e2e.sql` | 新增 | 幂等创建 E2E 数据；末尾 `\echo` 输出回填字段 |
| `scripts/dev/reset-e2e.sh` | 新增 | 幂等清空 E2E 数据，不 DROP TABLE |
| `scripts/dev/preflight.sh` | 新增 | 5 端健康检查，彩色输出 + 修复 hint |
| `scripts/dev/init-db.sh` | 修改 | 启动时同时创建 `e2e_runner` PG role（最小权限） |

### Seed 约束（详见 T-0000E §2.5）
- 使用确定性 UUID：`uuid_generate_v5('e2e.namespace', 'user_a')` 等
- 全部 `ON CONFLICT (id) DO UPDATE`
- JWT 在 SQL 中无法签发：seed 末尾通过 `psql --variable` 接收 `JWT_SECRET`，调一个 Rust CLI（`shared/bin/sign-jwt.rs`）生成 token，写到 `scripts/dev/.seed-output.env`，由 `globalSetup` 读取并 export

### Preflight 输出格式
```
[1/5] Postgres   ✅ ready (5432)
[2/5] Redis      ✅ ready (6379)
[3/5] AppServer  ❌ FAIL  curl: (7) Failed to connect to 127.0.0.1:3000
        ↳ Hint: cargo run -p voice-room-server -- --profile local
```

### Reset 安全
- 仅删除 `phone LIKE '+96650000090%'` 或 `id IN (E2E ID 集合)` 的行
- 启动前打印将删除的行数，要求 `--yes` 跳过确认（CI 用）
- profile=staging/prod 时**直接退出**，禁止误删

## 三、TDD 验收用例
- [ ] `psql -f seed-e2e.sql` 重复运行 3 次，最终行数一致
- [ ] `seed-e2e.sql` 后 `.seed-output.env` 包含全部 7 个 E2E_*_TOKEN/E2E_*_ID 字段
- [ ] `preflight.sh` 在 PG 关闭时 1 秒内退出码 11
- [ ] `preflight.sh` 在 AppServer 未起时 2 秒内退出码 13，输出含 `cargo run` hint
- [ ] `reset-e2e.sh` 在 `E2E_PROFILE=staging` 时立即退出，不执行 DELETE
- [ ] `reset-e2e.sh` 后业务表（如 `users`、`rooms`）的非 E2E 行数不变
