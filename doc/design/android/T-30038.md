# T-30038: 密码房进房弹窗

> **归属端**: Android
> **模块**: UI/Room
> **前置依赖**: T-00026 (密码校验 API), T-30007 (大厅房间卡片)
> **参考**: [business_flows.md §2.8.2](../../product/business_flows.md)

---

## 需求描述

用户点击带锁 🔒 的房间卡片时弹出 `PasswordInputDialog`，输入 6 位数字密码，验证通过后携带 short-live token 执行 JoinRoom。

## 触发时机

- 大厅房间卡片有 `has_password=true` 标志
- 用户点击该卡片（未进入该房间前）
- 已进入该房间的用户切后台再前台，**不需要**重新验证

## 布局

```
┌────────────────────────────────┐
│      🔒 请输入房间密码          │  ← titleLarge, 金色
│                                 │
│   ┌──┬──┬──┬──┬──┬──┐          │  ← SixDigitPinField (复用 T-30036)
│   │  │  │  │  │  │  │          │
│   └──┴──┴──┴──┴──┴──┘          │
│                                 │
│   (密码错误，剩 3 次机会)        │  ← bodyMedium 红色（仅错误时显示）
│                                 │
│   [  取消  ]  [  进入  ]        │
└────────────────────────────────┘
```

## 交互规则

| 情景 | 行为 |
|------|------|
| 输入满 6 位 | 自动提交（无需点进入） |
| 密码错误 | 底部红字显示"密码错误，剩 N 次机会"，输入框清空 |
| 错误 5 次被锁 | 底部红字"已锁定，30 分钟后重试"，按钮全灰 |
| 成功 | 关闭弹窗 + WS JoinRoom { password_token } |
| 返回键 | 关闭弹窗，不进房 |
| 取消 | 关闭弹窗 |

## 通用组件

- `SixDigitPinField`（T-30036 共用）
- `LockedCountdownText`：显示剩余锁定时间（60s 刷新一次）

## TDD 验收

1. 大厅点密码房卡片时弹窗可见
2. 非密码房不弹此弹窗直接进房
3. 输入满 6 位自动提交
4. 连续 5 次错误后按钮全灰，锁定提示可见
5. 成功后 JoinRoom 携带 token，失败不携带
6. 锁定倒计时每分钟更新
