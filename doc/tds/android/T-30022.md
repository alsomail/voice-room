# TDS: 大厅页视觉升级 (Task ID: T-30022)

## 一、背景与目标
- **关联需求**：product.md Phase 0.5 — 交互壳体与基础体验 / 大厅页黑金风格
- **前置依赖**：T-30018（MenaTheme 主题系统 ✅）、T-30020（MainScreen 三Tab框架 ✅）
- **已有基础**：T-30005（HallScreen UI）、T-30006（Paging3 分页）、T-30007（CreateRoomBottomSheet）
- **本 Task 的目标**：将 HallScreen 从 Material3 默认白色主题改造为中东黑金风格——深色 RoomCard + OnlineCountBadge + 顶部栏 + 分类横滑(占位) + 金色 FAB，**Paging3 分页逻辑完全不变**。

## 二、方案设计

### 核心设计原则

1. **纯 UI 改造**：仅修改 Composable 渲染层，不触碰 `RoomListViewModel`、`RoomPagingSource`、`IRoomRepository` 等数据层
2. **复用 MenaTheme**：所有颜色取 `MenaColors.*`，形状取 `MenaShapes`，排版取 `MenaTypography`
3. **组件拆分**：将 `RoomCard` 从 `HallScreen.kt` 独立为 `RoomCard.kt`，新增 `OnlineCountBadge.kt`
4. **testTag 兼容**：保留所有现有 testTag（`room_card_{roomId}`、`hall_loading`、`hall_error_text`、`hall_retry_button`、`hall_empty_state`、`hall_loading_more`、`room_type_icon_password`），新增标签不冲突

### 核心数据流

```
RoomListViewModel.pagingFlow (不变)
       │
       ▼
collectAsLazyPagingItems() (不变)
       │
       ▼
HallScreen(pagingItems, onNavigateToRoom, onCreateRoom)  ← 新增参数
       │
       ├── HallTopBar (新增)                    ← 金色标题 + 图标
       ├── CategoryTabRow (新增, 占位)           ← ScrollableTabRow
       ├── LazyVerticalGrid + PullToRefreshBox  ← 结构不变，颜色改造
       │     └── RoomCard (改造)                ← 深色底 + 渐变色块 + OnlineCountBadge
       └── FloatingActionButton (新增)          ← 金色 FAB
```

### 页面布局结构（升级后）

```
Scaffold(
  containerColor = MenaColors.Background,
  topBar = { HallTopBar() },
  floatingActionButton = { CreateRoomFab(onClick = onCreateRoom) }
) {
  Column {
    CategoryTabRow()                           // 分类横滑占位
    PullToRefreshBox(Paging3逻辑不变) {
      when (loadState.refresh) {
        Loading  → hall_loading (改用 MenaColors.Primary 色)
        Error    → hall_error_text + hall_retry_button (深色风格)
        NotLoading → {
          if (empty) → hall_empty_state (深色占位)
          else → LazyVerticalGrid(GridCells.Fixed(2)) {
            items(...) { RoomCard(改造后) }
            append loading → hall_loading_more
          }
        }
      }
    }
  }
}
```

### 接口 / 信令定义

本 Task 为纯 UI 改造，**不涉及**任何 API 或 WebSocket 信令变更。数据来源与格式完全沿用 T-30006 的 `RoomItem` 领域模型。

### 关键实现约束

1. **Paging3 逻辑绝对不变**：`LazyPagingItems` 的 `loadState` 分支判断、`pagingItems.refresh()`/`retry()` 调用、`items(count, key)` 渲染方式——全部原样保留
2. **Grid 列数**：从 `GridCells.Adaptive(160.dp)` 改为 `GridCells.Fixed(2)`，保证两列等宽
3. **RoomCard 尺寸**：由 Grid 自动分配宽度，卡片间距 `horizontalArrangement = spacedBy(12.dp)`、`verticalArrangement = spacedBy(12.dp)`，外层 `padding(horizontal=16.dp)`
4. **CreateRoomBottomSheet 集成**：在 `MainScreen.kt` 中用 `var showCreateRoom by remember { mutableStateOf(false) }` 控制 Sheet 显隐，`HallScreen(onCreateRoom = { showCreateRoom = true })`
