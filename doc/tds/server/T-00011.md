## 一、功能描述

建立 WebSocket 长连接，承载后续所有实时信令（房间事件、心跳、消息广播等）。

核心能力：
1. JWT 鉴权（握手阶段，通过 `?token=` query 参数传递）
2. 连接注册到全局 `ConnectionRegistry`（DashMap 无锁并发）
3. 心跳检测：30 秒无 ping 自动断开连接
4. 双向信令：ping/pong、未来扩展的业务消息

## 三、技术设计

### 文件结构

```
app/server/src/ws/
├── mod.rs          — 模块声明 + pub use
├── handler.rs      — WS 路由处理器（握手 + JWT 鉴权）
├── connection.rs   — 单连接生命周期（tokio::select! 双向读写）
├── registry.rs     — ConnectionRegistry（DashMap<Uuid, ConnectionHandle>）
└── heartbeat.rs    — 心跳检测 task（后台循环）
```

### ConnectionRegistry

```rust
pub struct ConnectionHandle {
    pub sender: mpsc::UnboundedSender<String>,
    pub user_id: Uuid,
    pub last_heartbeat: Arc<RwLock<Instant>>,
}

pub struct ConnectionRegistry {
    connections: DashMap<Uuid, ConnectionHandle>,
}
```

- **线程安全**：DashMap 分片锁，无需外部 Mutex
- **广播**：`broadcast_to_all` 使用 `DashMap::retain`，自动清除断开的连接
- **注销**：连接断开时 `handle_socket` 调用 `registry.unregister`

### 心跳检测

- 每 10 秒扫描所有连接
- `last_heartbeat` 由 ping 消息处理时更新（`handle_text_message`）
- 超过 30 秒无更新 → `DashMap::retain` 返回 `false` → 自动移除

### AppState 扩展

```rust

    // 已有字段...

}
```

新增 `ws_registry` 字段，`for_test()` 自动初始化（无需额外参数）。

## 五、Review 意见

### Round 1：🔴 未通过

**审查日期**：2026-01-27

#### [HIGH] 同一 user_id 重复连接导致 Registry 条目被错误注销

**位置**：`src/ws/connection.rs:93–138`（`handle_socket`）

**问题**：当同一 user_id 建立第二个 WS 连接时，`registry.register` 会覆盖旧 handle，导致旧连接的 `tx` 被 drop，旧连接 `rx.recv()` 返回 `None` → `break` → `registry.unregister(user_id)` 将**新连接的 handle 从 Registry 中删除**。新连接继续运行，但已无法通过 Registry 接收任何消息。

**修复方向**：引入独立的 `connection_id: Uuid`（与 `user_id` 解耦）作为 Registry 的 key，注销时仅删除自己的条目，不影响后续连接。

---

#### [MEDIUM] 其他问题（需一并修复）

1. **`heartbeat_task` 缺乏取消机制**（`heartbeat.rs:44`）：无 shutdown 信号，无法优雅停机，建议接受 `CancellationToken` 或 `watch::Receiver`。
2. **`registry.send()` 返回值被丢弃**（`connection.rs:126`）：pong 发送失败时无日志，应 `if !registry.send(...) { tracing::warn!(...) }`。
3. **`broadcast_to_all` 静默移除连接**（`registry.rs:69`）：retain 移除失败连接时应打印 tracing 日志。
4. **handler.rs 死代码**（`handler.rs:47–56`）：if/else 两分支均返回 `UNAUTHORIZED`，`status` 变量冗余，应合并。

---

**结论**：将负责人改回 `TDD`，修复 HIGH 问题及上述 MEDIUM 问题后重新提交 Review。
