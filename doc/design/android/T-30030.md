# T-30030: SendGift 客户端 + 幂等 + 连击

> **归属端**: Android
> **模块**: Logic/Gift
> **前置依赖**: T-30028, T-30029, T-00020 (Server SendGift 事务)
> **参考**: [business_flows.md §2.7.3](../../product/business_flows.md), [competitors.md §A.1 连击](../../product/competitors.md)

---

## 需求描述

实现礼物"送出"动作的客户端逻辑：生成幂等 msg_id、发送 WS 信令、处理响应/错误、实现"同礼物 3s 内连击合并"。

## 核心数据流

```
用户点击"送出"
   │
   ├─ 校验：余额 >= 总价？→ 否：触发 T-30032 弹窗，return
   │
   ├─ 生成 msg_id (UUID v4)
   │
   ├─ 检查连击窗口：
   │   ├─ 上次同礼物+同接收者 sendTime < 3s
   │   │   └─ count 累加到待发请求 + 重置定时器（不立即发送）
   │   └─ 否：立即发送 + 启动 3s 延迟定时器
   │
   ├─ WS 发送: SendGift { gift_id, receiver_id, count, msg_id, room_id }
   ├─ 按钮置 loading
   │
   ├─ 超时 5s 未响应 → 失败处理，还原按钮
   │
   └─ 收到响应：
       ├─ 成功：按钮还原 + 面板保留状态（不自动关闭，方便连送）
       ├─ INSUFFICIENT_BALANCE → T-30032 弹窗
       ├─ RECEIVER_UNAVAILABLE → Toast"对方已离开"
       └─ 其他错误 → Toast 错误消息
```

## 连击合并规则

- **窗口期**: 3 秒
- **同一性判定**: `gift_id` + `receiver_id` 相同
- **行为**:
  - 窗口内继续点击：累加本地 `pendingCount`，刷新"送出 xN 💎"按钮文字
  - 窗口结束或面板关闭：flush 发送，清空 pending
  - 用户期间切换礼物/接收者：立即 flush 上一组，开启新组

## 异常流程

| 场景 | 处理 |
|------|------|
| WS 断开 | flush pending；显示"网络异常，请稍后重试"；按钮还原 |
| Server 未响应超时（5s） | 按钮还原；Toast"送礼超时" |
| 重复 msg_id（重试） | Server 幂等返回，客户端正常处理 |
| 用户快速关闭面板 | flush pending 请求 |

## TDD 验收标准

1. 点击"送出"后生成唯一 msg_id（每次调用 UUID 不同）
2. 3s 内同礼物+同接收者连击：count 本地累加，只发送一次 WS
3. 超出 3s 或切换礼物/接收者：立即 flush
4. 余额不足时不发送 WS，直接触发 T-30032 弹窗
5. 收到 GiftReceived 广播时不重复处理（基于 msg_id 去重）
6. WS 断开时 flush pending + 错误提示 + 按钮还原
7. 超时 5s 自动失败（不留僵尸 pending）
