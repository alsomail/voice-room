# TDS: AppServer config 补全 + staging.toml 新增 (Task ID: T-00040)

> **主设计**：[T-0000E](./T-0000E.md)（§2.4.2 AppServer config 字段冻结）

## 一、背景与目标
- 修复体检缺口 C5：`config/dev.toml` 等仅 1-3 行，多环境隔离形同虚设。
- 输出补全后的 5 个 toml 文件 + 启动期强校验。

## 二、方案设计

### 涉及文件清单
| 文件路径 | 变更类型 | 说明 |
|---|---|---|
| `app/server/config/default.toml` | 修改 | 补全 `[server][database][redis][jwt][log]` 五大章节 |
| `app/server/config/dev.toml` | 修改 | 仅覆盖与 default 不同字段（`environment="dev"`、log level=debug 等） |
| `app/server/config/test.toml` | 修改 | port=3001，等 |
| `app/server/config/staging.toml` | 新增 | `environment="staging"`，host/port 与默认一致，依赖 env 注入 |
| `app/server/config/prod.toml` | 修改 | `environment="prod"`，log format=json |
| `app/server/src/config.rs` 或 `bootstrap` | 修改 | 加载链 `default → {APP_PROFILE}.toml → env override`；启动期校验 `JWT_SECRET`/`DATABASE_URL` 等 6 个关键字段 |
| `app/server/Cargo.toml` | 修改（如未引入） | 引入 `config = "0.14"`（与 adminServer 同版本） |

### 字段冻结
见 T-0000E §2.4.2，禁止个性化扩字段。

### 启动期校验
缺关键字段时：
```rust
eprintln!("CONFIG ERROR: missing field `{field}` (set via env {ENV})");
std::process::exit(78); // EX_CONFIG (sysexits.h)
```

### Profile 切换
环境变量 `APP_PROFILE`（默认 `dev`），`config::Config::builder().add_source(File::with_name(&format!("config/{profile}")))`。

## 三、TDD 验收用例
- [ ] 现有 server 测试 0 回归
- [ ] `APP_PROFILE=staging cargo run` 加载 `staging.toml` 成功
- [ ] 缺 `DATABASE_URL` 时退出码 78，stderr 含 `missing field 'database_url'`
- [ ] `APP_PROFILE=invalid_xxx` 时退出码 78
- [ ] 单元测试：config 加载链优先级 default < profile < env
