# TDS: 续费/过期 cron (Task ID: T-00068)

## 一、背景与目标
- **关联需求**：[phase1_nobility.md §3.4 续费 / §3.5.12 过期降级](../../product/phase1_nobility.md)。
- **目标**：Tokio cron 每小时：① 自动续费扫描 ② 过期清理 + grace period；连续 3 次续费失败关闭 auto_renew。

## 二、方案设计
### 核心数据流
```
每小时 tick:
  Phase A 自动续费:
    SELECT user_id FROM user_nobles WHERE auto_renew=true AND expire_at < now() + 24h LIMIT 500
    for each: 调用 PurchaseService::renew(user_id, current_tier_id) (复用 T-00067 逻辑)
      ├─ 成功 → WS NobleRenewSuccess + reset failed_renew_count=0
      └─ 失败 (40912 INSUFFICIENT_BALANCE) → failed_renew_count += 1
                if failed_renew_count >= 3: auto_renew=false, WS NobleRenewFailed reason=AUTO_RENEW_DISABLED
                else: WS NobleRenewFailed
  Phase B 过期清理:
    SELECT user_id FROM user_nobles WHERE expire_at + grace_days × 1d < now() LIMIT 500
    for each: 删除 user_nobles + INSERT noble_history(event=expire) + WS NobleExpired
  Phase C 过期前提醒:
    SELECT user_id FROM user_nobles WHERE expire_at - now() <= 3d AND last_warn_at IS NULL OR last_warn_at < now()-1d
    WS NobleRenewFailed kind=expire_warn (复用信令但 reason=WARN)（实际产品上是 NobleExpired in_grace_period=false 客户端识别）
```

## 三、TDD 验收用例
- [ ] auto_renew=true & 余额够 → 推送 NobleRenewSuccess + expire_at += 30d
- [ ] auto_renew=true & 余额不足 → NobleRenewFailed reason=INSUFFICIENT_BALANCE，failed_renew_count=1
- [ ] 连续 3 次失败 → auto_renew=false，NobleRenewFailed payload.auto_renew_disabled_after=3
- [ ] expire_at + grace_days < now → 删除 user_nobles + NobleExpired in_grace_period=false
- [ ] expire_at < now < expire_at + grace → 不删但 WS NobleExpired in_grace_period=true（保留特权 grace_days 天）
- [ ] cron 5min 内完成（500 行 LIMIT + SKIP LOCKED）
- [ ] failed_renew_count 写入 user_nobles
- [ ] 同一用户同一 tick 不重复处理（advisory lock）

### 🔴 协议路径绑定
- [ ] **PROTO-1**：NobleRenewFailed.payload.{tier_id, reason, failed_count, auto_renew_disabled_after, next_retry_at} 字段严格
- [ ] **PROTO-2**：NobleExpired.payload.{tier_id, expired_at, in_grace_period, grace_until} 字段严格

#### Cron 注册代码

```rust
// main.rs — 每小时触发 Phase A + Phase B
let cron_registry = Arc::clone(&registry);
tokio::spawn(async move {
    let mut interval = tokio::time::interval(Duration::from_secs(3600));
    loop {
        interval.tick().await;
        nobility_cron::run_renew_phase(&cron_registry).await;
        nobility_cron::run_expire_phase(&cron_registry).await;
    }
});
```

> **注意**：`run_renew_phase` / `run_expire_phase` 当前为 async stub（打印 debug log，无真实 DB 查询）。生产 DB 逻辑需在后续迭代中传入 `&sqlx::PgPool` 并实现扫描逻辑。
