## 3. 订单状态映射（与 server 7 状态对齐）

| Server 状态 | UI 标签 | 颜色 | 主行动 |
|-------------|--------|-----|------|
| `PENDING` | "待支付" | 灰 | 跳 Play / 取消 |
| `VERIFYING` | "验证中" | 蓝 | 等待轮询 |
| `VERIFIED` | "已支付待入账" | 蓝 | - |
| `CREDITED` | "✅ 成功" | 绿 | 查看 |
| `ACKED` | "✅ 成功" | 绿 | 查看（与 CREDITED UI 同） |
| `FAILED` | "❌ 失败" | 红 | 重新充值 |
| `REFUNDED` | "已退款" | 橙 | 客服 |

> 客户端**不**关心 ACKED 与 CREDITED 的差异（用户层无感），只显示"成功"。

## 5. 异常流程

| 场景 | 处理 |
|------|------|
| 订单超 30 分钟仍 VERIFYING | 详情页提示"如未到账请联系客服并提供订单号" |
| FAILED 显示"重新充值"按钮 | 跳充值页并预选同 SKU |
| REFUNDED | 红字"该订单已退款，钻石已扣回 -1200💎" + 联系客服 |
| 网络失败 | 顶部黄条 + 重试 |
| 列表为空 | 空态插画 + "您还没有充值记录" + 跳充值页 CTA |
| 跨设备订单（其他设备产生） | 一致显示，详情顶部小字"于 Pixel 7 完成支付" |
