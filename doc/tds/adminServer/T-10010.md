# TDS: Admin Server 数据统计接口 (Task ID: T-10010)

## 一、背景与目标

- **关联需求**：来自 `doc/product.md` 数据统计模块——运营后台需要查看平台核心运营指标（DAU、新增用户、活跃房间、在线人数）
- **本 Task 的目标**：实现 `GET /api/v1/admin/stats/overview` 接口，支持按日期范围查询，返回平台概览统计数据；在线人数与活跃房间数 MVP 阶段降级为 mock 值（Redis/内存状态由 App Server 维护，T-10011 后再接入）

---

## 二、方案设计

### §2.1 接口规格

```
GET /api/v1/admin/stats/overview?start_date=2024-01-01&end_date=2024-01-31
Authorization: Bearer <admin-jwt>
```

#### 查询参数

| 参数 | 类型 | 必须 | 说明 |
|------|------|------|------|
| `start_date` | string (YYYY-MM-DD) | 否 | 统计起始日期，缺省为今天 |
| `end_date` | string (YYYY-MM-DD) | 否 | 统计截止日期，缺省为今天 |

> 默认值：`start_date = end_date = today`（UTC 日期）
> 若仅传一个参数，另一个补全为当天。
> 若 `start_date > end_date`，返回 400/40003。

#### 成功响应（200）

```json
{
  "code": 0,
  "message": "ok",
  "request_id": "xxx",
  "data": {
    "dau": 1234,
    "new_users": 56,
    "active_rooms": 12,
    "online_users": 89,
    "date_range": {
      "start": "2024-01-01",
      "end": "2024-01-31"
    }
  }
}
```

| 字段 | 类型 | 来源 | 说明 |
|------|------|------|------|
| `dau` | integer | DB users 表 | `updated_at::date` 在范围内的未删除用户数（近似 DAU） |
| `new_users` | integer | DB users 表 | `created_at::date` 在范围内的未删除用户数 |
| `active_rooms` | integer | **MVP mock = 0** | rooms 状态由 App Server 内存维护，T-10011 接入 Redis 后替换 |
| `online_users` | integer | **MVP mock = 0** | 在线人数存于 App Server Redis，T-10011 接入后替换 |
| `date_range` | object | 请求参数 | 回显实际查询的日期范围 |

#### 错误码

| HTTP | code  | 场景 |
|------|-------|------|
| 400  | 40003 | `start_date` / `end_date` 格式非法（非 YYYY-MM-DD）；`start_date > end_date` |
| 401  | 40101 | 无 Token / Token 无效 |
| 401  | 40102 | Token 已过期 |
| 403  | 40301 | 角色无 `StatsRead` 权限（cs 角色） |
| 500  | 50000 | 数据库内部错误 |

### §2.3 数据层设计

#### users 表可用时间字段（来自 `app/server/migrations/001_create_users.sql`）

| 字段 | 类型 | 含义 | 用途 |
|------|------|------|------|
| `created_at` | TIMESTAMPTZ | 账号创建时间 | `new_users` 统计 |
| `updated_at` | TIMESTAMPTZ | 最近更新时间 | `dau` 近似统计（无专门 session 表） |
| `deleted_at` | TIMESTAMPTZ | 软删除时间 | 过滤已删除用户 |

> **DAU 说明**：当前无 `user_sessions` 表，使用 `updated_at` 近似 DAU（用户发生任何操作时 App Server 会更新 `updated_at`）。此为 MVP 方案，若需精确 DAU 需在 App Server 侧写入 session 记录。

#### SQL 设计（仅 users 表，参数化占位符）

**new_users**（新增用户数）：
```sql
SELECT COUNT(*)
FROM users
WHERE created_at::date >= $1
  AND created_at::date <= $2
  AND deleted_at IS NULL
```

**dau**（日活跃用户，用 updated_at 近似）：
```sql
SELECT COUNT(*)
FROM users
WHERE updated_at::date >= $1
  AND updated_at::date <= $2
  AND deleted_at IS NULL
```

> 参数绑定：`$1` → `start_date: NaiveDate`，`$2` → `end_date: NaiveDate`（通过 SQLx `chrono` feature 直接绑定）

### §2.5 DTO 结构体设计

```rust
/// GET /api/v1/admin/stats/overview 查询参数
#[derive(Debug, Deserialize, Default)]
pub struct StatsOverviewQuery {
    /// 统计起始日期（YYYY-MM-DD），缺省今天
    pub start_date: Option<String>,
    /// 统计截止日期（YYYY-MM-DD），缺省今天
    pub end_date: Option<String>,
}

/// 日期范围（回显用）
#[derive(Debug, Serialize)]
pub struct DateRange {
    pub start: String, // YYYY-MM-DD
    pub end: String,   // YYYY-MM-DD
}

/// GET /api/v1/admin/stats/overview 成功响应的 data 部分
#[derive(Debug, Serialize)]
pub struct StatsOverviewResponse {
    pub dau: i64,
    pub new_users: i64,
    pub active_rooms: i64,
    pub online_users: i64,
    pub date_range: DateRange,
}
```

### §2.7 Service 设计

```rust
pub struct AdminStatsService {
    repo: Arc<dyn AdminStatsRepository>,
}

impl AdminStatsService {
    pub fn new(repo: Arc<dyn AdminStatsRepository>) -> Self {
        Self { repo }
    }

    /// 获取统计概览。
    /// - 解析并校验日期参数（缺省今天，start > end 返回 AppError::InvalidParam）
    /// - 并发查询 new_users 和 dau（可用 tokio::try_join!）
    /// - active_rooms / online_users MVP 固定为 0
    pub async fn get_overview(
        &self,
        query: StatsOverviewQuery,
    ) -> Result<StatsOverviewResponse, AppError>;
}
```

日期解析逻辑（在 service.rs 中实现）：

```rust
fn parse_date(s: &str) -> Result<NaiveDate, AppError> {
    NaiveDate::parse_from_str(s, "%Y-%m-%d")
        .map_err(|_| AppError::InvalidParam("日期格式须为 YYYY-MM-DD".into()))
}
```

### §2.8 核心数据流

```
GET /api/v1/admin/stats/overview?start_date=2024-01-01&end_date=2024-01-31
        │
        ▼
  jwt_auth 中间件（AdminAuthContext 注入）
        │
        ▼
  stats_overview_handler (controller.rs)
    ├─ AdminAuthContext::require_permission(StatsRead) → 403 if fail
    ├─ Query<StatsOverviewQuery> 提取
    └─ stats_service.get_overview(query).await
              │
              ▼
        AdminStatsService (service.rs)
          ├─ 解析 start_date / end_date（缺省今天，validate start <= end）
          ├─ tokio::try_join!(
          │     repo.count_new_users(start, end),
          │     repo.count_dau(start, end)
          │   )
          ├─ active_rooms = 0  // TODO(T-10011): Redis SCARD rooms:active
          ├─ online_users = 0  // TODO(T-10011): Redis SCARD online:users
          └─ 组装 StatsOverviewResponse
                    │
                    ▼
              AdminStatsRepository (repository.rs)
                ├─ PgAdminStatsRepository (生产 DB)
                └─ FakeAdminStatsRepository (测试 HashMap)
```
