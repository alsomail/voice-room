# TDS: 房间页视觉升级 (Task ID: T-30025)

## 一、背景与目标

- **关联需求**：product.md Phase 0.5 — 交互壳体与基础体验 / 房间页黑金风格
- **前置依赖**：T-30018（MenaTheme 主题系统 ✅）
- **已有基础**：T-30009（RoomScreen UI）、T-30011（MicSlotCard 三态）、T-30013（上麦/下麦 WS 逻辑）、T-30014（ChatMessageList）
- **本 Task 的目标**：将 RoomScreen 从 Material3 默认主题改造为中东黑金风格——深色背景 + 主麦突出（80dp 金色光圈）+ 副麦 4 列（60dp AvatarWithFrame）+ 空麦位虚线"+"+ 弹幕金色昵称 + 系统消息金黄色居中。**WS/上下麦逻辑完全不变**。

## 二、方案设计

### 核心设计原则

1. **纯 UI 改造**：仅修改 Composable 渲染层，不触碰 `RoomViewModel`、`IWebSocketClient`、`IMediaService`、`MicSlotUi` 数据结构及任何 WS 信令逻辑
2. **复用 MenaTheme**：所有颜色取 `MenaColors.*`，头像使用现有 `AvatarWithFrame` 组件
3. **主麦/副麦分离**：`slots[0]` 为主麦（HostMicSlot），`slots[1..8]` 为副麦（4 列 LazyVerticalGrid）
4. **testTag 兼容**：保留所有现有 testTag（`mic_slot_empty_${index}`、`mic_slot_occupied_${index}`、`mic_slot_muted_icon_${index}`、`chat_message_list`、`user_message_{index}`、`system_message_{index}`），新增标签不冲突

### 核心数据流

```
RoomViewModel.uiState (不变)
       │  uiState.micSlots[0]     → HostMicSlot  (80dp 金色光圈)
       │  uiState.micSlots[1..8]  → GuestMicSlotsGrid (60dp 4列)
       │  uiState.messages        → ChatMessageList (金色昵称 / 金黄系统消息)
       ▼
RoomScreen (纯 UI 改造，Scaffold 结构不变)
```

### 布局结构

```
RoomScreen (Scaffold, modifier.imePadding)
├── topBar: RoomTopBar (不变)
├── bottomBar: ChatInputBar (不变)
└── content: Column
    ├── Box(深色背景 MenaColors.Background)
    │   ├── HostMicSlot(slots[0])          // 主麦：居中, 80dp + 金色光圈
    │   └── GuestMicSlotsGrid(slots[1..8]) // 副麦：4列, 每格 60dp
    └── ChatMessageList(messages)           // 弹幕列表 (weight=1f)
```

### 组件设计细节

#### 1. HostMicSlot（主麦 — 新文件）

```kotlin
// HostMicSlot.kt
@Composable
fun HostMicSlot(
    slot: MicSlotUi,
    modifier: Modifier = Modifier,
    onClick: (index: Int) -> Unit = {},
) {
    // 布局：居中 Column，padding vertical 12dp
    // 头像区：Box 叠加层
    //   层 1: Canvas → drawCircle 金色光圈（主麦专属发光效果）
    //         - color = MenaColors.Primary.copy(alpha = 0.35f)，radius = size/2 + 12dp (px)
    //         - 若 slot.isOccupied，追加 infiniteRepeatable 脉冲动画（scale 0.9→1.1，1200ms）
    //   层 2: AvatarWithFrame(imageUrl=slot.avatarUrl, size=80.dp, showFrame=slot.isOccupied)
    //         - 空麦时 showFrame=false，显示虚线圆圈 + "+" （见 EmptyHostSlot 子组件）
    // 昵称：slot.nickname（非空时显示），MenaColors.Primary，bodyMedium，居中
    // testTag: 复用 "mic_slot_empty_0" / "mic_slot_occupied_0"（保持兼容）
}
```

**实现要点**：
- Canvas 绘制金色光圈：`drawCircle(color = goldGlow, radius = avatarRadius + glowPx, style = Stroke(width = 6.dp.toPx()))`
- 动画：`rememberInfiniteTransition` + `animateFloat(0.92f → 1.08f, 1200ms, RepeatMode.Reverse)`，用 `graphicsLayer(scaleX, scaleY)` 应用到 Canvas 层
- 空麦时不显示光圈，改为虚线边框（见下）

#### 4. MicSlotsGrid（布局重组 — 修改）

```kotlin
@Composable
fun MicSlotsGrid(
    slots: List<MicSlotUi>,        // 仍接受 9 个麦位（兼容现有调用方）
    modifier: Modifier = Modifier,
    onMicSlotClick: (index: Int) -> Unit = {},
) {
    val hostSlot = slots.getOrNull(0)
    val guestSlots = slots.drop(1)   // slots[1..8]，最多 8 个

    Column(modifier = modifier.background(MenaColors.Background)) {
        // 主麦行：Box 水平居中
        if (hostSlot != null) {
            Box(Modifier.fillMaxWidth(), contentAlignment = Alignment.Center) {
                HostMicSlot(slot = hostSlot, onClick = onMicSlotClick)
            }
        }
        // 副麦区：4 列 LazyVerticalGrid，userScrollEnabled=false
        LazyVerticalGrid(
            columns = GridCells.Fixed(4),
            modifier = Modifier
                .fillMaxWidth()
                .testTag("mic_slots_grid"),   // 保留原 testTag 供测试定位
            userScrollEnabled = false,
        ) {
            items(items = guestSlots, key = { it.index }) { slot ->
                MicSlotCard(slot = slot, onClick = onMicSlotClick)
            }
        }
    }
}
```

**高度策略**：`RoomScreen` 中移除 `height(240.dp)` 硬编码，改为 `wrapContentHeight()`（主麦约 112dp + 副麦 2 行约 80dp×2 = ~272dp 自适应）
