# TDS: App Server 麦位下麦接口 (Task ID: T-00015)

## 一、背景与目标

- **关联需求**：语音房间核心功能——麦位管理（T-00014 上麦接口的对称操作）
- **前置依赖**：T-00014（上麦接口 ✅ Done）、T-00013（离开房间逻辑 ✅ Done）
- **本 Task 的目标**：处理 `LeaveMic` WS 信令，通过内存写锁保证并发安全，校验用户是否在麦上，成功后原子释放麦位并广播 `MicLeft` 事件；同时修复被动下麦（用户离开房间时）缺少广播的问题

## 二、方案设计

### 信令格式（对齐 protocol.md）

**请求（客户端 → 服务端）**：
```json
{
  "type": "LeaveMic",
  "msg_id": "uuid"
}
```
> 无 payload，服务端从 `mic_slots` 中查找用户当前所在麦位

**成功响应（服务端 → 请求方）**：
```json
{
  "type": "LeaveMicResult",
  "msg_id": "uuid",
  "code": 0,
  "payload": { "mic_index": 2 },
  "timestamp": 1713312000
}
```

**错误码**：
| code | 含义 | 触发条件 |
|------|------|---------|
| `40400` | 用户不在房间 | `registry.get_room_id(connection_id)` 返回 None |
| `40304` | 用户不在麦上 | `leave_mic_slot(user_id)` 返回 None |

**广播（服务端 → 房间所有成员，含请求方）**：
```json
{
  "type": "MicLeft",
  "payload": { "mic_index": 2, "user_id": "uuid" },
  "timestamp": 1713312000
}
```

### 核心逻辑：leave_mic_slot（原子操作）

```rust
/// 原子查找用户所在麦位并清除。持写锁期间完成"查找 + 清除"，保证并发安全。
/// 返回 Some(mic_index) 表示成功下麦，None 表示用户不在任何麦位。
pub fn leave_mic_slot(&self, user_id: Uuid) -> Option<usize> {
    let mut slots = self.mic_slots.write().unwrap_or_else(|e| e.into_inner());
    for (i, slot) in slots.iter_mut().enumerate() {
        if *slot == Some(user_id) {
            *slot = None;
            return Some(i);
        }
    }
    None
}
```

**关键**：`leave_mic_slot` 是同步函数，写锁在函数返回时立即释放，不跨越任何 `.await` 点。与 `take_mic_slot` 保持对称设计。

> **注意**：T-00013 中 `remove_from_mic_slots` 返回 `bool`，不含麦位索引，无法广播 `MicLeft`。本 Task 新增 `leave_mic_slot` 返回 `Option<usize>`，并修改 `do_leave_room` 调用 `leave_mic_slot` 替代 `remove_from_mic_slots`，以支持被动下麦广播。

### handle_leave_mic 处理流程（5 步）

| 步骤 | 操作 | 失败行为 |
|------|------|---------|
| 1 | `registry.get_room_id(connection_id)` | → code:40400 |
| 2 | `room_manager.get_room(room_id)` | → code:40400（防御性） |
| 3 | `room_state.leave_mic_slot(user_id)` | None → code:40304 |
| 4 | 广播 `MicLeft` 给 `registry.get_connections_in_room(room_id)` | 失败不阻断 |
| 5 | 返回 `LeaveMicResult { code:0, payload:{ mic_index } }` | — |

### 被动下麦修复（do_leave_room）

修改 `do_leave_room` 中对 `remove_from_mic_slots` 的调用：

```rust
// 修改前（T-00013 实现，无广播）
if room_state.remove_from_mic_slots(user_id) {
    // 无广播
}

// 修改后
if let Some(mic_index) = room_state.leave_mic_slot(user_id) {
    // 广播 MicLeft 给房间内所有成员
    broadcast_mic_left(&registry, room_id, mic_index, user_id).await;
}
```

### 并发安全分析

`leave_mic_slot` 与 `take_mic_slot` 均使用 `std::sync::RwLock` 写锁，在单节点进程内保证原子性。由于写锁不跨 `.await`，不存在死锁风险。
