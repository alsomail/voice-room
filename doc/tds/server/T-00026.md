# TDS: 密码房进房校验 + 锁定机制 (Task ID: T-00026)

## 一、背景与目标
- **关联需求**：E-10 密码房进房流程 ([phase1_room_governance §2.1.1 + §3.2](../../product/phase1_room_governance.md))
- **本 Task 的目标**：提供 POST `/rooms/:id/verify-password`，签发 60s 短期 token；失败 5 次锁定 30min；WS JoinRoom 读 token 才允许进入。

## 二、方案设计

### 接口：POST /api/v1/rooms/:id/verify-password
Request: `{ "password": "123456" }`

成功 200:
```json
{ "code":0, "data": { "access_token":"<jwt-60s>" } }
```
token claim：
```json
{ "sub": user_id, "room_id": xxx, "iat": now, "exp": now+60, "iss":"voiceroom-room-access" }
```

错误：
| HTTP | code | 说明 |
|------|------|------|
| 400 | 40003 | password 非 6 位数字 |
| 404 | 40400 | 房间不存在/已关闭 |
| 400 | 40014 | 非密码房 |
| 401 | 40103 | 密码错误（未达锁定） |
| 429 | 42910 | 已锁定 (payload `{ locked_remaining_sec }`) |

### Redis Key
| Key | 类型 | TTL | 用途 |
|-----|------|-----|------|
| `pwd_fail:{user_id}:{room_id}` | Int | 1800s | 失败计数 |
| `pwd_lock:{user_id}:{room_id}` | String | 1800s | 锁定标记 |

流程：
```
1. 若 pwd_lock 存在：返回 42910 + TTL
2. bcrypt 验证 password
   - 成功 → DEL pwd_fail → 签发 room_access token
   - 失败 → INCR pwd_fail; 若 ==5 → SETEX pwd_lock 1800 + INCR 保留
           返回 40103 payload `{ remaining_attempts: 5-count }`
```

### WS JoinRoom 读 token
- `JoinRoom` payload 增 `access_token?: string`
- 对密码房：
  - 无 token → 返回 `PASSWORD_REQUIRED(40104)`
  - 有 token：解析验证 iss='voiceroom-room-access' && sub==user_id && room_id 匹配 && 未过期
  - token 过期 → `TOKEN_EXPIRED(40105)`

### 并发安全
失败计数 `INCR` 是原子的，锁定 key 的 SETEX 用 `SET ... NX EX 1800` 防止已锁定后重复 set，失败计数到 5 时同一批并发仅一条触发 SETEX。
