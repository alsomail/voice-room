# TDS: 接收者选择器 (Task ID: T-30029)

## 一、背景与目标
- **关联需求**：E-07 送礼前选择麦位接收者 ([设计文档](../../design/android/T-30029.md))
- **本 Task 的目标**：在 GiftPanelBottomSheet 顶部嵌入横向滚动的接收者头像条；默认选主麦；麦位变化实时刷新。

## 二、方案设计

### 涉及文件
| 文件 | 类型 | 说明 |
|------|------|------|
| `gift/components/RecipientSelector.kt` | 新增 | Composable |
| `gift/GiftPanelViewModel.kt` | 修改 | 订阅 RoomStateFlow 中 micSlots |
| `gift/GiftPanelBottomSheet.kt` | 修改 | 插入 RecipientSelector |
| `androidTest/.../gift/RecipientSelectorTest.kt` | 新增 | UI 测试 |

### 组件 API
```kotlin
@Composable
fun RecipientSelector(
    recipients: List<MicUserVO>,  // 仅 on-mic（slot_index != null）
    selectedId: String?,
    onSelect: (String) -> Unit,
    modifier: Modifier = Modifier
)
```

### 交互
- LazyRow 横向滚动，每项 AvatarWithFrame(48dp) + 昵称省略号
- 主麦（slot=0）置首；其他麦位按 slot_index 升序
- 选中项金色 2dp 光圈边框 + 底部金色点
- 空列表时显示 "当前无人在麦"（居中灰字），`selectedRecipientId` 为 null
- 首次渲染若 selected 为 null 且有列表，默认选第一个（主麦）

### 数据来源
- 复用 `RoomViewModel.state.micSlots`（T-30010），通过 Hilt 或父 VM 转发
- Slot 变化 → `GiftPanelViewModel` 重新计算 `recipients`；若选中用户下麦 → 自动切主麦

### testTag
`Key('recipient_selector')`, `Key('recipient_item_{userId})')`, `Key('recipient_empty')`
