# T-30020: MainScreen 底部三Tab框架

> **归属端**: Android  
> **模块**: UI/Navigation  
> **前置依赖**: T-30018 (MenaTheme), T-30019 (Splash)  
> **参考**: [Android App 界面设计规范 §2.3](../../product/android_app_design.md)

---

## 需求描述

实现 App 主页框架，包含底部三 Tab 导航：房间大厅 / 消息 / 我的。

## 页面布局

```
┌──────────────────────────────┐
│                              │
│    [当前 Tab 的内容区域]       │  ← 占满剩余空间
│    (HallScreen /             │
│     MessagesScreen /         │
│     ProfileScreen)           │
│                              │
│                              │
├──────────────────────────────┤
│  🏠 广场   💬 消息   👤 我的   │  ← 底部导航栏 56dp
│  (金色)   (灰色)   (灰色)    │     选中项金色，未选中灰色
└──────────────────────────────┘
```

## 导航栏设计

| 属性 | 值 |
|------|------|
| 背景色 | `#1A1A2E` (MenaColors.Background) |
| 选中图标+文字 | `#D4AF37` (MenaColors.Primary) |
| 未选中图标+文字 | `#6C6C6C` (MenaColors.OnBackgroundTertiary) |
| 高度 | 56dp |
| 图标尺寸 | 24dp |
| 文字 | labelSmall |

### Tab 定义

| Tab | 图标 (Material Icons) | 阿语标签 | 英语标签 | 路由 |
|-----|----------------------|----------|----------|------|
| 房间 | `Icons.Default.Home` | الغرف | Rooms | `main/rooms` |
| 消息 | `Icons.Default.Chat` | الرسائل | Messages | `main/messages` |
| 我的 | `Icons.Default.Person` | حسابي | Me | `main/profile` |

## 交互逻辑

1. 默认选中"房间"Tab
2. Tab 切换时保持各页面状态（使用 `NavHost` + `saveState`/`restoreState`）
3. 二次点击同一 Tab 滚动到顶部（房间列表场景）
4. 消息 Tab 未来支持未读数红点（Phase 0.5 不实现）

## 通用组件提取

| 组件 | 文件 | 说明 |
|------|------|------|
| `MenaBottomNavigation` | `common/ui/MenaBottomNavigation.kt` | 封装底部导航栏的样式和行为 |

## 自动化测试锚点

| testTag | 元素 | 点击/状态断言 |
|---------|------|--------------|
| `main_screen` | 主页容器 | 登录后可见 |
| `bottom_nav` | 底部导航栏 | 始终可见 |
| `tab_rooms` | 房间 Tab | 默认选中（金色）；点击后显示 HallScreen |
| `tab_messages` | 消息 Tab | 点击后显示 MessagesScreen |
| `tab_profile` | 我的 Tab | 点击后显示 ProfileScreen |

## TDD 验收标准

1. 登录成功后导航到 MainScreen，默认显示房间 Tab
2. 底部三个 Tab 可点击切换，选中项为金色
3. Tab 切换保持各页面状态（如房间列表滚动位置）
4. `MenaTheme` 深色背景正确应用
5. RTL 布局下 Tab 顺序不变（图标+文字不需要镜像）
