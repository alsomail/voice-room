# TDS: 登录逻辑与路由守卫 (Task ID: T-20002)

## 一、背景与目标
- **关联需求**：product.md - Admin Web 权限控制
- **本 Task 的目标**：调用 Admin Server 登录接口，通过 Zustand 管理 JWT 状态，实现路由鉴权守卫，确保未登录用户无法访问受保护页面。
- **前置依赖**：T-10002 (Admin 登录接口), T-20001 (管理员登录页 UI)

## 二、方案设计
### 核心数据流
App 启动 → 检查 localStorage JWT → 有效？→ 跳转目标页 / 跳转 /login
路由切换 → AuthGuard 组件 → useAuthStore.isAuthenticated → true → 渲染子路由 / false → Navigate to /login

### 接口 / 信令定义

**状态管理设计**：
```typescript
interface AuthStore {
  token: string | null;
  admin: Admin | null;
  isAuthenticated: boolean;
  login: (username: string, password: string) => Promise<void>;
  logout: () => void;
  checkAuth: () => boolean;  // 检查 token 是否过期
}
```

**JWT 拦截器逻辑**：
1. 请求拦截：自动从 useAuthStore 取 token，附加到 `Authorization: Bearer xxx`
2. 响应拦截：收到 401 → useAuthStore.logout() → Navigate to /login

## 五、Reviewer意见
> 状态枚举：🟢 通过 | 🔴 未通过
### Review理由：
**状态：🔴 未通过**

**审查日期**：2025-05-10

#### [HIGH-01] 401 响应拦截器缺失（TDS §二 明确要求）

TDS 设计方案要求：「响应拦截：收到 401 → `useAuthStore.logout()` → Navigate to /login」，且 TDD 验收用例列出「异常：token 过期（401 响应）→ 自动清除 token → 跳转 /login」。  
但 `app/web/src/core/network/apiClient.ts` 中 401 响应仅抛出 `Error`，**未调用 logout、未跳转 /login、无对应测试用例**。  
TDS「实现结果」章节亦未提及对 `apiClient.ts` 的修改，说明此变更被遗漏。  
**修复**：在 `adminFetch` 中检测 `response.status === 401`，调用 `useAuthStore.getState().logout()`，并跳转 `/login`；补充对应 `apiClient.test.ts` 测试用例。

#### [HIGH-02] AuthGuard 运行时不调用 checkAuth()，过期 token 不触发退出

`AuthGuard` 仅读取 `isAuthenticated`（仅在模块初始化时计算一次），未调用已实现的 `checkAuth()` 方法。用户进入 dashboard 后若 token 在使用中到期，守卫不会重新校验，结合 HIGH-01 造成用户滞留在受保护页面且 API 调用不会自动 logout。  
**修复**：`AuthGuard` 改为调用 `const checkAuth = useAuthStore(s => s.checkAuth); if (!checkAuth()) { return <Navigate to="/login" replace />; }`。

#### [MEDIUM-01] decodeJwtPayload 缺少 Base64 补位

`atob()` 处理 Base64url 转 Base64 时未补 `=` padding，部分浏览器（Safari）会抛异常，被 `try/catch` 吞掉后导致合法 token 误判为无效。  
**修复**：在 `atob()` 前补位：`` base64 + '=='.slice(0, (4 - (base64.length % 4)) % 4) ``

#### [LOW-01] ADMIN_TOKEN_KEY 常量重复定义

`apiClient.ts`（私有）和 `useAuthStore.ts`（已导出）各自定义 `'adminToken'`，有不一致风险。  
**修复**：`apiClient.ts` 从 `useAuthStore.ts` import 该常量。

#### [LOW-02] localStorage XSS 风险未文档化

TDS 审查重点要求此风险须在代码中注释说明，当前代码无任何相关注释。  
**修复**：在 `ADMIN_TOKEN_KEY` 常量处添加 XSS 风险说明注释及缓解措施（严格 CSP）。
