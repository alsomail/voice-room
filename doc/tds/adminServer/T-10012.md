# TDS: Admin Server 操作审计日志 (Task ID: T-10012)

## 一、背景与目标

- **关联需求**：来自 `doc/product.md` 运营管理模块——需要记录所有敏感管理操作（封禁/解封/关闭房间/充值），以便事后追溯与合规审查

### §2.3 方案选型：Controller 层显式调用（MVP）

**方案对比：**

| 方案 | 优点 | 缺点 |
|------|------|------|
| Axum 中间件 | 自动拦截所有请求，零侵入业务层 | 只能记录 HTTP 语义（路径/方法），无法获取 action 语义和 target_id |
| Service 层调用 | 可获取 action 语义和 target_id | 需向 service 注入 HTTP 层的 IP 信息，污染分层边界 |
| **Controller 层调用（MVP 采用）** | 可获取 IP（HTTP 层），业务成功后显式调用；service 不感知 HTTP | 需在各敏感 controller 手动调用，有遗漏风险 |

**MVP 决策**：Controller 层显式调用。service 负责业务，controller 在业务成功后调用 `state.audit_logger.log_action()`，fire-and-forget（失败仅 warn，不影响主业务响应）。

**后续迭代**：可引入 Axum 中间件记录基础 HTTP 日志（所有请求），与 Controller 层语义日志并存。

### §2.4 IP 地址提取与存储

`ip_address` 列类型为 `INET`（PostgreSQL）。由于 `sqlx` workspace 依赖未开启 `ipnetwork` feature，Rust 侧使用 `Option<String>` 存储，SQL 侧用 `::inet` 转换：

```rust
// INSERT 时
// ip_address = $5::inet   -- 若 ip 为 None，传入 NULL，NULLIF 无需额外处理
// SELECT 时  
// ip_address::text as ip_address  -- 从 INET 转回字符串

// IP 提取逻辑（controller 层）
fn extract_ip(headers: &HeaderMap) -> Option<String> {
    // X-Forwarded-For: ip1, ip2, ... → 取第一个
    if let Some(xff) = headers.get("x-forwarded-for") {
        if let Ok(s) = xff.to_str() {
            let first = s.split(',').next()?.trim().to_string();
            if !first.is_empty() { return Some(first); }
        }
    }
    // X-Real-IP 备选
    if let Some(xri) = headers.get("x-real-ip") {
        if let Ok(s) = xri.to_str() {
            return Some(s.trim().to_string());
        }
    }
    None
}
```

> **注意**：若 IP 字符串无法被 PostgreSQL 解析为 INET（如空字符串、非法格式），INSERT 会返回 DB 错误。Controller 层使用 fire-and-forget，`log_action` 失败只打 warn 日志，不影响主业务。

### §2.5 AuditLogger 设计

```rust
// src/modules/audit/service.rs

pub struct AuditLogger {
    repo: Arc<dyn AuditRepository>,
}

impl AuditLogger {
    pub fn new(repo: Arc<dyn AuditRepository>) -> Self {
        Self { repo }
    }

    /// fire-and-forget：失败仅记录 warn，不返回 Error。
    pub async fn log_action(
        &self,
        admin_id: Uuid,
        action: &str,
        target_type: Option<&str>,
        target_id: Option<Uuid>,
        ip: Option<String>,
        detail: Option<serde_json::Value>,
    ) {
        let entry = CreateAuditLog {
            admin_id,
            action: action.to_string(),
            target_type: target_type.map(str::to_string),
            target_id,
            ip,
            detail,
        };
        if let Err(e) = self.repo.insert(entry).await {
            tracing::warn!(error = %e, admin_id = %admin_id, action = action, "audit log write failed");
        }
    }
}
```

### §2.6 AuditRepository Trait

```rust
// src/modules/audit/repository.rs

#[derive(Debug, Clone)]
pub struct CreateAuditLog {
    pub admin_id: Uuid,
    pub action: String,
    pub target_type: Option<String>,
    pub target_id: Option<Uuid>,
    pub ip: Option<String>,
    pub detail: Option<serde_json::Value>,
}

#[derive(Debug, Clone, serde::Serialize, serde::Deserialize, sqlx::FromRow)]
pub struct AdminLogRow {
    pub id: Uuid,
    pub admin_id: Uuid,
    pub action: String,
    pub target_type: Option<String>,
    pub target_id: Option<Uuid>,
    pub ip_address: Option<String>,   // 从 INET 转 text
    pub detail: Option<serde_json::Value>,
    pub created_at: chrono::DateTime<chrono::Utc>,
}

#[derive(Debug, Default)]
pub struct AdminLogFilter {
    pub admin_id: Option<Uuid>,
    pub action: Option<String>,
    pub start_date: Option<chrono::DateTime<chrono::Utc>>,
    pub end_date: Option<chrono::DateTime<chrono::Utc>>,
    pub page: i64,   // 0-based
    pub size: i64,   // 默认 20，上限 100
}

#[async_trait::async_trait]
pub trait AuditRepository: Send + Sync {
    async fn insert(&self, log: CreateAuditLog) -> Result<(), sqlx::Error>;
    async fn list(&self, filter: AdminLogFilter) -> Result<(i64, Vec<AdminLogRow>), sqlx::Error>;
}
```

**PgAuditRepository** 的 `list` 查询使用动态过滤：

```sql
-- insert（ip 用 ::inet 转换，NULL 安全）
INSERT INTO admin_logs (admin_id, action, target_type, target_id, detail, ip_address)
VALUES ($1, $2, $3, $4, $5, $6::inet)

-- count（总数用于分页）
SELECT COUNT(*) FROM admin_logs
WHERE ($1::uuid IS NULL OR admin_id = $1)
  AND ($2::text IS NULL OR action = $2)
  AND ($3::timestamptz IS NULL OR created_at >= $3)
  AND ($4::timestamptz IS NULL OR created_at <= $4)

-- list（分页）
SELECT id, admin_id, action, target_type, target_id,
       ip_address::text AS ip_address, detail, created_at
FROM admin_logs
WHERE ($1::uuid IS NULL OR admin_id = $1)
  AND ($2::text IS NULL OR action = $2)
  AND ($3::timestamptz IS NULL OR created_at >= $3)
  AND ($4::timestamptz IS NULL OR created_at <= $4)
ORDER BY created_at DESC
LIMIT $5 OFFSET $6
```

> 参考 T-10007 分页模式（COUNT + SELECT，page 从 1 开始，DTO 层转为 0-based offset）。

### §2.9 核心数据流

```
POST /api/v1/admin/users/:id/ban
        │
        ▼
  ban_user_handler (user/controller.rs)
    ├─ AdminAuthContext → admin_id: Uuid
    ├─ extract_ip(headers) → ip: Option<String>
    ├─ user_service.ban_user(admin_id, user_id, req).await  → Ok(resp)
    │
    └─ state.audit_logger.log_action(
           admin_id, "ban_user"/"unban_user",
           Some("user"), Some(user_id),
           ip, Some(json!({...detail}))
       ).await  ← fire-and-forget，失败仅 warn

DELETE /api/v1/admin/rooms/:id
        │
        ▼
  force_close_room_handler (room/controller.rs)
    ├─ AdminAuthContext → admin_id: Uuid
    ├─ extract_ip(headers) → ip: Option<String>
    ├─ room_service.force_close_room(admin_id, room_id).await → Ok(())
    └─ state.audit_logger.log_action(admin_id, "close_room", Some("room"), Some(room_id), ip, None).await

GET /api/v1/admin/logs
        │
        ▼
  list_logs_handler (audit/controller.rs)
    ├─ require_permission(LogRead)
    ├─ ListLogsQuery::from_query_params
    └─ audit_service.list_logs(filter).await → ListLogsResponse
              │
              ▼
        AuditService::list_logs
          └─ audit_repo.list(AdminLogFilter).await → (total, items)
```
