- [ ] **缺陷 1**：[级别 P0] **Android Transport 与 App Server 上报契约严重不匹配（WS + HTTP 双通道全部失败）**
  - **文件与行号**：
    - `app/android/app/src/main/java/com/voice/room/android/core/analytics/transport/WsTransport.kt:30-40`
    - `app/android/app/src/main/java/com/voice/room/android/core/analytics/transport/HttpTransport.kt:28-36`
    - 对照：`app/server/src/core/analytics/writer.rs::EventInput`（L33-52）+ `app/server/src/modules/events/ws.rs::handle_report_event`（L80）
  - **问题说明**：客户端 Transport 在序列化时只输出 `event_name / properties / session_id / client_ts` 四个字段，且 `properties` 直接传入 `entity.propertiesJson`（一个 **字符串**，非对象）。但服务端 `EventInput` 要求顶层独立字段 `device_id`（必填）、`user_id`、`app_version`、`os_version`、`locale`、`network_type`，以及 `properties: serde_json::Value`（对象）。该不匹配带来三处真实破坏：
    1. **HTTP 通道**：`device_id` 默认为 `""`，写入服务在 `writer.rs:168` 立即返回 `AppError::ParameterMissing("device_id")` → HTTP 400 / code=40002，**100% 拒收**。
    2. **WS 通道**：协议契约违反 `doc/protocol/index.md §6.3`。Android 发送 `{ "type":"ReportEvent", "payload":[ {...} ] }`（payload 为数组）；但服务端 `ws.rs:80` 调用 `payload_val.get("events").and_then(|v| v.as_array())`，要求 `payload = { "events": [...] }`。Android 也未携带 `msg_id`。结果：`code=40003 VALIDATION_ERROR`，**100% 失败**。
    3. **JSONB 类型污染**：即便 device_id 通过，`properties` 被反序列化为 `Value::String("{...}")` 而非 `Value::Object`，PostgreSQL 写入后 JSONB 列存为字符串标量，`admin_server` `properties->>'gift_id'` 全部失效（T-10015 GET 接口字段提取无效）；T-20013 前端 JSON 折叠展示也将变成纯字符串，丧失结构化能力。
  - **复现路径**：以 Android 真机调用 `EventReportClient.track("login_success", mapOf(...))` → flush → 服务端日志连续报 `40002` 或 `40003`；Web 行为流 Tab 中无法看到任何 Android 上报数据。
  - **修复建议**：
    1. 重新定义 `EventQueueEntity`：将公共字段 `deviceId / userId / appVersion / osVersion / locale / networkType` 升级为独立列（或在 `propertiesJson` 之外单独保存），随事件一起入队（**入队时刻**抓取一次，不能 flush 时再注入，否则 session/network 漂移）。
    2. WS Transport 输出严格遵守 `payload = { "events": [...] }`，并写入 `msg_id`（UUID）；事件对象至少包含 `event_name / device_id / user_id / session_id / client_ts / properties(对象) / app_version / os_version / locale / network_type`。
    3. HTTP Transport 输出顶层结构 `{ "events": [...] }`，事件对象同上。
    4. `properties` 必须以 **对象** 而非字符串 序列化（用 `gson.fromJson(propertiesJson, Map::class.java)` 反序列化后再嵌入 outer map）。
    5. 强烈建议在 `voice-room-shared` crate 或 Android `core/analytics/wire/` 增加一份单一事实源的 wire schema（与服务端 `EventInput` 一一对应），并补充契约测试。
  - **影响面**：T-30035 全部 7 项验收（队列/节流/Crash 模式/核心事件/敏感字段过滤）在生产链路中均无法验证；T-00022 / T-00023 实际未被任何线上客户端命中；T-10015 / T-20013 即便完工也只能看到 Web 自己造的数据。
  - **TDD 修复记录**：[等待 TDD 填写修复逻辑与 Commit ID]
