# TDS: 手动补单 / 退款 API (Task ID: T-10026)

## 一、背景与目标

## 三、TDD 验收用例
- [ ] 仅 super_admin 通过；其他角色 403
- [ ] reason 为空 → 400
- [ ] FAILED 订单 recredit → state=CREDITED + 余额 + + 写 audit
- [ ] ACKED 订单尝试 recredit → 409 + code=40904
- [ ] refund 成功扣余额（即使余额不足允许变负，但 wallet_transactions 标 `negative_allowed=true`）
- [ ] 重复 recredit（同 order_id）→ 幂等返回首次结果
- [ ] WS BalanceUpdated 准时推送

### 🔴 协议路径绑定的强制验收
- [ ] **PROTO-1**：reason 字符串透传写入 admin_logs
- [ ] **PROTO-2**：`BalanceUpdated.reason` 严格为 `admin_recredit` 或 `admin_refund`
