# TDS: 治理日志查询 API (Task ID: T-10016)

## 一、背景与目标
- **关联需求**：E-10 运营客服查证 ([phase1_room_governance §2.2](../../product/phase1_room_governance.md))
- **本 Task 的目标**：Admin Server 提供踢人/禁言审计日志查询接口，支持 room_id / target_user_id / operator_user_id / type / 时间范围筛选。

## 二、方案设计

### 接口
**GET /api/v1/admin/governance/kicks**
**GET /api/v1/admin/governance/mutes**
（需 Admin JWT；super_admin / operator / cs 可查；finance 禁止）

Query：
- `room_id?`, `target_user_id?`, `operator_user_id?`
- `from?`, `to?`（默认 7 天内，最大 90 天）
- mutes 专属：`type?=mic|chat`
- `page=1`, `limit=20` (max 100)

响应（kicks）：
```json
{ "code":0, "data":{
  "total":12,"page":1,"limit":20,
  "items":[
    { "id":"uuid","room_id":"uuid","room_title":"...",
      "target_user_id":"uuid","target_nickname":"...",
      "operator_user_id":"uuid","operator_nickname":"...",
      "reason":"harassment","created_at":"RFC3339" }
  ]
}}
```
响应（mutes）多一个 `type` 与 `duration_sec`。

### SQL
```sql
SELECT k.*, r.title as room_title,
       tu.nickname as target_nickname, ou.nickname as operator_nickname
  FROM room_kick_records k
  JOIN rooms r ON r.id = k.room_id
  JOIN users tu ON tu.id = k.target_user_id
  JOIN users ou ON ou.id = k.operator_user_id
 WHERE ($1::uuid IS NULL OR k.room_id = $1)
   AND ($2::uuid IS NULL OR k.target_user_id = $2)
   AND ($3::uuid IS NULL OR k.operator_user_id = $3)
   AND k.created_at BETWEEN $4 AND $5
 ORDER BY k.created_at DESC
 LIMIT $6 OFFSET $7;
```

## 四-A、Review 修复记录（Round 1）

### [HIGH] page 负值校验修复
- **问题**：`service.rs:104` 仅检查 `page == 0`，传入 `page=-1` 时 offset = -40，PgGovernanceRepo 执行 `OFFSET -40` 触发 PostgreSQL 错误导致 HTTP 500
- **修复**：将 `if page == 0` 改为 `if page < 1`，覆盖所有非正整数（负数 + 零）
- **新增测试**：
  - `SV-09`：`page=-1` → ValidationError（G16-08 补充）
  - `SV-10`：`page=-100` 极端负值 → ValidationError
- **位置**：`service.rs:104`

### [MEDIUM] mute_type 枚举白名单校验修复
- **问题**：传入 `type=xyz` 等非法值时静默返回空结果，调用方无法区分参数错误与无数据
- **修复**：在 `resolve_params` 中新增白名单校验，`mute_type` 仅允许 `"mic"` / `"chat"`（None 则跳过校验），非法值返回 `ValidationError`
- **新增测试**：
  - `SV-11`：`mute_type="xyz"` → ValidationError
  - `SV-12`：`mute_type="MIC"`（大写）→ ValidationError（大小写敏感）
  - `SV-13`：`mute_type=""`（空字符串）→ ValidationError
  - `SV-14`：`mute_type="mic"` → 正常
  - `SV-15`：`mute_type="chat"` → 正常
  - `SV-16`：`mute_type=None` → 正常
- **位置**：`service.rs:113-119`

### [LOW] resolve_params 返回值简化
- **问题**：返回 5-元组 `(filter, page, limit, offset, limit)`，末位 `limit` 重复，调用方均以 `_` 丢弃
- **修复**：简化为 4-元组 `(filter, page, limit, offset)`，同步更新两处调用方解构
- **位置**：`service.rs:73, 133, 145, 181`
