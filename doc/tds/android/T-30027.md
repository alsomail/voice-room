## 五、Reviewer意见
### Round 1

- **审查意见**：

#### [CRITICAL] WS `BalanceUpdated` payload 字段与协议不符

**文件**：`WalletViewModel.kt:142,160` / `BalanceUpdatedEvent.kt` / `WalletViewModelTest.kt:171`

协议 `doc/protocol/websocket_signals.md §6.4.1` 规定余额字段位于 `payload.diamond_balance`（嵌套结构），但代码和注释均使用顶层 `new_balance`：
```kotlin
// ❌ 当前实现（与协议不符）
val newBalance = json.get("new_balance")?.asLong ?: return
// ✅ 应按协议读取
val payload = json.getAsJsonObject("payload") ?: return
val newBalance = payload.get("diamond_balance")?.asLong ?: return
```
测试 W27-04 同样使用错误格式，导致测试通过但验证的是非标准行为。生产环境 WS 余额将永远无法实时更新。

#### [HIGH-1] `testTag("wallet_txn_list")` 挂载位置错误

**文件**：`WalletScreen.kt:262`

TDS 协议：`Key('wallet_txn_list') → 流水 LazyColumn`。但该 tag 被打到 `WalletTxnItem` 内每个 `Card` 上，导致 LazyColumn 无标签、每条记录有重复相同标签。应将 `.testTag("wallet_txn_list")` 移至 `LazyColumn` 的 `modifier` 上。

#### [HIGH-2] `refresh()` 缺少 401 错误处理

**文件**：`WalletViewModel.kt:121-125`

`loadBalance()` 正确处理了 401 → `NavigateToLogin`，但 `refresh()` 的 `onFailure` 路径无同等处理，token 失效时下拉刷新只显示错误提示而不跳转登录，行为不一致，需补充 401 分支。

#### [HIGH-3] `BalanceUpdatedEvent` 是未使用的死代码

**文件**：`core/ws/event/BalanceUpdatedEvent.kt`

全局无引用，KDoc 注释中的消息格式描述（`"new_balance":xxx`）与协议矛盾。请删除或修正后实际接入使用。

#### [MEDIUM-1] 导航路由 `"wallet"` 为魔法字符串

**文件**：`MainScreen.kt:102,106`

TDS 要求在 `Destinations.kt` 中定义 `WALLET = "wallet"` 常量，两处硬编码字符串字面量未落实。

#### [MEDIUM-2] 缺失 `WalletScreenTest`（androidTest）

W27-06（流水颜色）、W27-07（空状态占位）、W27-09（Profile 跳转导航）三个验收用例无 Compose UI 自动化测试覆盖，TDS 计划中的 `WalletScreenTest.kt` 未创建。

- **改进建议**：
  1. 优先修复 CRITICAL：将 `handleWsMessage` 中的字段路径改为 `payload.diamond_balance`，同步修正 W27-04 测试用例的模拟消息格式和 `BalanceUpdatedEvent` 的 KDoc。
  2. 修复 HIGH-1：将 `.testTag("wallet_txn_list")` 移至 `LazyColumn`，从 `WalletTxnItem` Card 中移除。
