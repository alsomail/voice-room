## 二、方案设计

### 2.1 数据流图（npm script → env 注入 → 脚本/Playwright → 输出）

```
                    ┌─────────────────────────────────────────────┐
                    │            开发者/CI shell                  │
                    │   npm run <script>                          │
                    └───────────────────┬─────────────────────────┘
                                        │
                ┌───────────────────────┼───────────────────────┐
                ▼                       ▼                       ▼
       ┌────────────────┐      ┌────────────────┐      ┌──────────────────┐
       │ e2e:{profile}  │      │ db:seed/reset  │      │ preflight        │
       │ cross-env      │      │ cross-env      │      │ cross-env        │
       │ E2E_PROFILE=X  │      │ E2E_PROFILE=   │      │ E2E_PROFILE=     │
       │ [E2E_ALLOW_W]  │      │   local        │      │   ${E2E_PROFILE} │
       └───────┬────────┘      └───────┬────────┘      └─────────┬────────┘
               ▼                       ▼                         ▼
   ┌──────────────────────┐  ┌──────────────────────┐  ┌──────────────────┐
   │ playwright test      │  │ bash scripts/dev/    │  │ bash scripts/dev/│
   │ [--grep @prod-safe]  │  │  seed-e2e.sh /       │  │  preflight.sh    │
   │  ↓                   │  │  reset-e2e.sh        │  │                  │
   │ globalSetup:         │  │ (T-0000G 退出码      │  │ (T-0000G 退出码  │
   │  envLoader →         │  │   21/22/23/24)       │  │   11/12/13/14/15)│
   │  preflight →         │  └──────────┬───────────┘  └─────────┬────────┘
   │  [seed]              │             │                        │
   └──────────┬───────────┘             │                        │
              ▼                         ▼                        ▼
   ┌──────────────────────────────────────────────────────────────────┐
   │  退出码透传到 npm（npm run 默认透传子进程退出码）→ CI fail-fast │
   └──────────────────────────────────────────────────────────────────┘
```

**关键点**：
1. 所有 script 都不在 npm 层做业务逻辑，仅做「env 前缀 + 调用」二件事；
2. `cross-env` 仅负责跨平台 env 注入（macOS/Linux/Windows cmd），不做语义改写；
3. 退出码 100% 透传：`npm run` 默认会以子进程 exit code 作为自身 exit code（无 `--silent` / `; exit 0` 等吞码模式）；
4. `e2e:prod-smoke` 在 CLI 层就用 `--grep '@prod-safe'` 兜底，与 globalSetup 的 prod profile 二次过滤构成双保险（即使 globalSetup 失效也不会跑写操作用例）。

### 2.3 6 个 script 命令冻结表

> **不变量**：以下 6 个 script 的 `key` 名称与语义由本 TDS + T-0000E §4.3 共同冻结，**任何下游任务（RUNBOOK / CI workflow / 教程）只能引用，不得改名/删除/合并**。

| # | script key | 冻结命令（cross-env 形态） | env 注入 | 调用对象 | 期望退出码 | 备注 |
|---|---|---|---|---|---|---|
| 1 | `e2e:local` | `cross-env E2E_PROFILE=local playwright test` | `E2E_PROFILE=local` | `@playwright/test` CLI | 0 全绿 / 1 用例失败 / 11~15 preflight 透传（globalSetup 调 preflight 失败时 Playwright 以非 0 退出） | 默认行为 = 现有手工 `E2E_PROFILE=local npx playwright test`，零行为漂移 |
| 2 | `e2e:staging` | `cross-env E2E_PROFILE=staging playwright test` | `E2E_PROFILE=staging` | `@playwright/test` CLI | 0 / 1 / 13~15（远端 5 端 HTTP 检查） | 启动期 envLoader 缺关键字段抛 `MissingEnvError`（T-0000H 契约），表现为退出码 1 + 明确错误信息 |
| 3 | `e2e:prod-smoke` | `cross-env E2E_PROFILE=prod playwright test --grep "@prod-safe"` | `E2E_PROFILE=prod`（隐式 `E2E_ALLOW_WRITES=0`） | `@playwright/test` CLI | 0 / 1 / 13~15 | **CLI `--grep` 与 globalSetup grep 双重保险**；任何未打 `@prod-safe` 的用例必被 skip；`--grep` 后参数必须用**双引号**（Windows cmd 不识别单引号） |
| 4 | `db:seed` | `cross-env E2E_PROFILE=local E2E_ALLOW_WRITES=1 bash scripts/dev/seed-e2e.sh` | `E2E_PROFILE=local`、`E2E_ALLOW_WRITES=1` | T-0000G `seed-e2e.sh` | 0 / 21 profile 拒绝 / 22 JWT / 23 SQL | 强制只在 local 跑（脚本自身也会拒绝 staging/prod，构成第二道防线） |
| 5 | `db:reset` | `cross-env E2E_PROFILE=local bash scripts/dev/reset-e2e.sh` | `E2E_PROFILE=local` | T-0000G `reset-e2e.sh` | 0 / 21 profile 拒绝 / 24 无连接 | 永不自动 reset 远端，与 T-0000E §2.7 globalTeardown 契约一致 |
| 6 | `preflight` | `bash scripts/dev/preflight.sh` | 透传当前 shell（不强制覆盖 `E2E_PROFILE`，由 preflight.sh 自身默认 `local`） | T-0000G `preflight.sh` | 0 / 11 PG / 12 Redis / 13 AppServer / 14 AdminServer / 15 Web | **不**用 cross-env：preflight.sh 是 bash 脚本，无 env 前缀需求；Windows 用户走 WSL 跑（见 §2.6） |

**双引号 vs 单引号决策**：所有 script 内的字符串参数（如 `@prod-safe`）一律使用 **双引号包裹**。json 文件中需要 `\"@prod-safe\"` 转义。原因：Windows cmd 把单引号当作普通字符，会导致 grep pattern 实际匹配 `'@prod-safe'`（含单引号）→ 0 命中 → 用例全 skip → **CI 假绿**（见 §六 R3）。

### 2.4 与 envLoader / preflight 的行为契约

| 契约项 | 期望行为 | 来源 |
|---|---|---|
| 退出码透传 | `npm run preflight` 在 PG 挂时退出码 = 11；CI 据此 fail-fast | T-0000E §2.6（11~15 退出码） + npm 默认行为 |
| envLoader 启动期 fail-fast | `npm run e2e:staging` 缺 `STAGING_DB_URL` 等关键字段 → globalSetup 内 `loadE2EEnv()` 抛 `MissingEnvError` → playwright exit 1 + 在控制台首屏打印缺失字段名 | T-0000E §2.7 + T-0000H |
| seed 幂等性 | `npm run db:seed && npm run db:seed` 二次执行不报错（ON CONFLICT DO UPDATE） | T-0000G §2.5 |
| preflight 时间预算 | `npm run preflight` 在 5 端全绿时 **壁时 ≤ 1s**（因 5 个 HTTP/CLI 检查并发或快速顺序，总壁时上限脚本自身约束 10s） | T-0000E §三 M1 + T-0000G preflight.sh §2.6 |
| prod 写禁止双保险 | `npm run e2e:prod-smoke` 即便用户在 shell 里强行 `export E2E_ALLOW_WRITES=1`，CLI `--grep "@prod-safe"` 仍只跑只读用例 | T-0000E §2.8 + §2.9 |

### 2.5 与 T-0000J `@prod-safe` 标签协同

- T-0000J 负责给 ≥5 条 read-only smoke 用例打 `@prod-safe` 标签（`test('...', { tag: '@prod-safe' }, ...)`）；
- 本 Task 的 `e2e:prod-smoke` 在 CLI `--grep "@prod-safe"` 直接消费该标签；
- **执行顺序约束**：T-0000I 可与 T-0000J 并行 Plan/TDD，但 T-0000I 的 `e2e:prod-smoke` TDD 验收（§三 U-7）需要等 T-0000J 至少打过 1 条 `@prod-safe` 标签后才能跑「非空命中集」断言。本 Task TDD Agent 应在 §三 U-7 中允许「断言至少 0 条用例命中且无错误退出」作为退化形态（如 T-0000J 尚未合入），完整断言由 T-0000L RUNBOOK 验证。

### 2.6 跨平台兼容性（macOS/Linux/Windows + WSL）

| 平台 | `e2e:*` | `db:seed` / `db:reset` | `preflight` |
|---|---|---|---|
| macOS（开发主力） | ✅ cross-env 工作 | ✅ bash 工作 | ✅ bash 工作 |
| Linux（CI） | ✅ cross-env 工作 | ✅ bash 工作 | ✅ bash 工作 |
| Windows cmd / PowerShell | ✅ cross-env 工作（关键依赖） | ⚠️ 需要 Git Bash 或 WSL（`bash` 命令） | ⚠️ 需要 Git Bash 或 WSL |
| WSL2（Windows 推荐路径） | ✅ 全工作 | ✅ 全工作 | ✅ 全工作 |

**决策**：本仓库 5 端冷启动依赖 `docker-compose` + `cargo`，Windows 用户事实上必走 WSL2；本 TDS 不为纯 Windows cmd 包装 `*.cmd` 脚本（YAGNI），但 `cross-env` 保证 e2e:* 三个 script 在裸 cmd 也能跑 staging/prod（远端服务，无需本地 docker）。

**cross-env 依赖处理**：
- 检查 `package.json` 现有 `devDependencies`：当前无 cross-env；
- 落地时执行 `npm install --save-dev cross-env@^7.0.3`，将其加入 devDependencies；
- 若未来工具链统一切到 pnpm/yarn，仅改 lockfile，scripts 内容不变。

## 三、TDD 验收用例

> **执行原则**：每条用例须给出可复现的 shell 命令；TDD Agent 实跑后在 §四 4.1 表中贴退出码与壁时证据。

### 集成级（真跑 6 个 script，要求 5 端 + DB 已就绪）
- [ ] **U-4（preflight 全绿，时间预算）**：5 端启动后 `time npm run preflight` 退出码 0；壁时 ≤ 1s（real time）；stdout 含 `[5/5]` 或 5 个 ✅ 行。

- [ ] **U-8（e2e:staging fail-fast）**：临时清空 `STAGING_*` 关键 env，`npm run e2e:staging` 退出码非 0，stdout/stderr 首屏出现 `MissingEnvError` 字样并定位缺失字段名。

- [ ] **U-10（db:reset profile 隔离）**：`E2E_PROFILE=staging npm run db:reset; echo $?` 输出 `21`（脚本拒绝），不触碰任何远端数据。
