> **归属端**: Android
> **模块**: UI/Room

## 需求描述

三合一：公告弹窗 / 常驻公告入口 / 全局角色徽章。覆盖"房间个性化"与"身份识别"两个主要 UX 诉求。

## A. AnnouncementPopup（首次进房自动弹出）

### 触发条件

- 进房后 JoinRoom 成功回执有 `announcement` 非空
- 本地 `last_shown_announcement:{room_id}` 与 server 内容哈希不同（公告变化重新弹）
- 或 24 小时内未弹过

### 布局

```
┌────────────────────────────────┐
│       📄 房间公告               │
│                                 │
│   欢迎来到 Leila 的房间~         │
│   请保持友善，禁止广告~          │
│                                 │
│   [  不再提示  ] [  知道了  ]   │
└────────────────────────────────┘
```

- "不再提示"：SharedPreferences 记住 hash，24h 内不再弹
- "知道了"：关闭

## B. 常驻公告入口

房间顶部（房间名右侧）显示 📄 图标，点击再次弹出公告。若公告为空则不显示图标。

## C. 角色徽章（全局复用）

| 场景 | 渲染位置 |
|------|---------|
| 麦位上用户昵称旁 | 👑（房主）/ 🛡️（管理员） |
| 观众席列表项 | 同上 |
| 聊天区消息昵称前 | 同上 |
| 个人信息卡（T-30040 弹窗顶部） | 同上 |

## 核心数据流

```
Server AdminChanged { room_id, new_admin_user_id }
   ↓ WsBus.dispatch
RoomViewModel.onAdminChanged
   ↓ 更新内存 roomState.adminUserId
Compose State 驱动所有徽章重渲染
```

## 通用组件（关键：徽章必须抽取独立）

- `RoleBadge(role: Role)` — **唯一入口**，4 处渲染点都用它
- `RoleResolver(userId, roomState)` — 纯函数，从 RoomState 推出 Role（owner/admin/member）
- `AnnouncementDialog(text, onDismissForever, onDismiss)` — 公告弹窗

## 测试锚点

| Key | 说明 |
|-----|------|
| `Key('announcement_popup')` | 公告弹窗根 |
| `Key('btn_show_announcement')` | 房间顶部常驻 📄 图标 |
| `Key('btn_announcement_mute_24h')` | 不再提示 |
| `Key('btn_announcement_dismiss')` | 知道了 |
| `Key('role_badge_$userId')` | 每个徽章（根据 userId 定位） |

## TDD 验收

1. 进房 announcement 非空则弹
2. 24h 内再次进同房且公告未变则不弹
3. 公告内容变更后再次进房立即弹（哈希不同）
4. 公告为空时顶部 📄 图标不显示
5. 房主徽章金色皇冠 ✓ / 管理员金色盾牌 ✓ / 普通无
6. AdminChanged 到达后 500ms 内所有徽章重渲染
7. `RoleResolver` 纯函数单测覆盖 (owner, admin, member, unknown_user)
8. 麦位/观众席/聊天三处徽章渲染一致（同一 Composable）
