# TDS: 待 ACK 订单后台对账 cron (Task ID: T-00054)

## 一、背景与目标
- **关联需求**：[phase1_payment_billing.md §6.4 / §0 acknowledgePurchase 必须 3 天内完成](../../product/phase1_payment_billing.md) — Google 规定未 ack 的购买 3 日后自动退款，必须有兜底 cron。
- **本 Task 的目标**：Tokio cron 每 5min 扫 `state IN (VERIFYING, CREDITED) AND age > 阈值` 订单，重新查 Google 状态强制推进至 ACKED 或 FAILED。

## 二、方案设计

### 核心数据流
```
tokio_cron_scheduler 每 5min
  ↓
SELECT id FROM payment_orders WHERE
  (state='VERIFYING' AND created_at < now()-INTERVAL '10min')
  OR (state='CREDITED' AND acked_at IS NULL AND credited_at < now()-INTERVAL '1h')
  LIMIT 100
  ↓
逐单调 GooglePlayBillingPort::get_product_purchase 重新校验
  ├─ purchaseState=0 + 未 acked → adapter.acknowledge → state=ACKED
  ├─ purchaseState=0 + 已 acked → state=ACKED（仅更新本地状态）
  ├─ purchaseState=1 (CANCELED) → state=FAILED + 反向 wallet（仅当 CREDITED）
  └─ Google 5xx → 写 Sentry + 下次 cron 重试（最多 24h）
```

## 三、TDD 验收用例
- [ ] **VERIFYING 超时**：mock 一笔 12 分钟前 VERIFYING 订单 → cron 推进至 ACKED 或 FAILED
- [ ] **CREDITED 未 ACK**：mock 一笔 1.5h 前 CREDITED 未 ack 订单 → 调 acknowledge → state=ACKED
- [ ] **Google 5xx**：mock 临时错误 → 订单状态保留，下次 tick 重试；3 次失败写 Sentry
- [ ] **不影响主流程**：cron 运行期间正常 verify 请求 P95 不退化（用 SELECT FOR UPDATE SKIP LOCKED）
- [ ] **写 admin_logs**：每次推进结果落审计
