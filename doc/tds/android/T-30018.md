# TDS: MenaTheme 中东黑金主题系统 (Task ID: T-30018)

## 一、背景与目标
- **关联需求**：Phase 0.5 所有 Android UI 任务的基础依赖（T-30019 ~ T-30026 均依赖本 Task）
- **前置依赖**：无
- **本 Task 的目标**：封装 Material3 黑金主题系统（Colors / Typography / Shapes）+ RTL Provider + 三个通用组件（GoldButton / GoldOutlinedTextField / AvatarWithFrame），为全局 UI 提供统一视觉基座

## 二、方案设计

### 核心数据流
```
MenaTheme (Composable 入口)
  ├── darkColorScheme(MenaColors 映射)
  ├── MenaTypography (Roboto + Noto Sans Arabic)
  ├── MenaShapes (16dp / 24dp / 12dp / Circle)
  └── CompositionLocalProvider(LocalLayoutDirection)
        └── MaterialTheme(colorScheme, typography, shapes) { content() }
```
MenaTheme 作为全局主题入口包裹所有 Composable。内部封装 `darkColorScheme`、`Typography`、`Shapes`，并通过 `CompositionLocalProvider` 注入 RTL 布局方向。所有子组件（GoldButton / GoldOutlinedTextField / AvatarWithFrame）在 MenaTheme 内使用 `MaterialTheme.colorScheme` 获取色值，无需额外传参。

### 色彩定义（黑金主题 — MenaColors）

来源：`doc/design/android/T-30018.md` + `doc/product/android_app_design.md` § 4 颜色令牌

```kotlin
object MenaColors {
    val Background         = Color(0xFF1A1A2E)  // 页面背景（深紫黑）
    val Surface            = Color(0xFF16213E)  // 卡片/容器
    val SurfaceVariant     = Color(0xFF0F3460)  // 次级容器
    val Primary            = Color(0xFFD4AF37)  // 金色主色（按钮/强调）
    val PrimaryBright      = Color(0xFFFFD700)  // 亮金色（VIP/高亮/渐变终点）
    val OnBackground       = Color(0xFFFFFFFF)  // 主文字（白色）
    val OnBackgroundSecondary = Color(0xFFB0B0B0) // 次要文字
    val OnBackgroundTertiary  = Color(0xFF6C6C6C) // 三级文字/未选中
    val Error              = Color(0xFFE74C3C)  // 错误/封禁
    val Success            = Color(0xFF2ECC71)  // 成功/在线
    val SystemMessage      = Color(0xFFF39C12)  // 系统消息黄色
}
```

### 组件规格

**GoldButton**：
- 金色水平渐变背景（`Primary` → `PrimaryBright`，`Brush.horizontalGradient`）
- 白色文字（`OnBackground`）
- 24dp 圆角（胶囊 capsule shape）
- `enabled=false` 时透明度降至 38%，不可点击
- 参数：`text: String`, `onClick: () -> Unit`, `modifier: Modifier = Modifier`, `enabled: Boolean = true`
- testTag：由调用方通过 modifier 注入

**GoldOutlinedTextField**：
- 深色背景（`Surface`）
- 金色边框 1dp（unfocused: `Primary` 50% 透明度，focused: `PrimaryBright` 100%）
- 白色输入文字（`OnBackground`）
- 金色 label/placeholder（`Primary`）
- 12dp 圆角
- 参数：`value: String`, `onValueChange: (String) -> Unit`, `label: String = ""`, `modifier: Modifier = Modifier`, `placeholder: String = ""`, `singleLine: Boolean = true`, `keyboardOptions: KeyboardOptions = KeyboardOptions.Default`, `keyboardActions: KeyboardActions = KeyboardActions.Default`

**AvatarWithFrame**：
- 圆形头像（Coil `AsyncImage` + `CircleShape` clip）
- `showFrame=true` 时外圈金色描边 2dp（`Primary` 色 + `CircleShape` border）
- 默认占位图（`Icons.Default.Person` 或 drawable 占位）
- 参数：`imageUrl: String?`, `size: Dp = 60.dp`, `showFrame: Boolean = true`, `modifier: Modifier = Modifier`

### RTL 支持
- `MenaTheme` 内部通过 `LocalConfiguration.current.layoutDirection` 检测系统 locale
- 使用 `CompositionLocalProvider(LocalLayoutDirection provides layoutDirection)` 注入
- 所有组件使用 `start/end` padding 而非 `left/right`
- 参考设计规范：`Row` / `LazyRow` 自动镜像，数字和英文保持 LTR

### Round 2 修复（BUG-ANDROID-001 — QA 回流）

- **触发**：QA Gate 报告 `tests/report-20260429-081255` — 57/57 androidTest 全部 `ArrayIndexOutOfBoundsException: length=18; index=55/46/62...`，P0 阻塞。
- **根因**：`MenaColors.kt` 11 处 `Color(*_VALUE)` 误调到 `Color(value: ULong)` 重载，把低 6 位当作 colorspace ID（`0x37=55`、`0x2E=46`、`0x60=96`…），Android 13 仅 18 个合法 colorspace（ID 0–17），整体越界 → 类初始化即崩。
- **修复**：全部 11 处改为 `Color(*_VALUE.toInt())`，强制走 `Color(color: Int)` sRGB ARGB 重载。`0xFFxxxxxxxx.toInt()` 在 Kotlin 中得到带符号 32-bit ARGB，色值数值不变；保留 `const val *_VALUE: ULong` 供 JVM 单测继续复用。
