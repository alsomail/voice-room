## 一、背景与目标
- **关联需求**：[phase1_payment_billing.md §6.5](../../product/phase1_payment_billing.md) — 接入 Google Real-Time Developer Notifications，处理异步退款 / 撤销。
- **本 Task 的目标**：实现 RTDN Pub/Sub 推送端点，幂等接收 + 解析 ONE_TIME_PRODUCT 消息 + REFUNDED 事件回扣余额（强事务）。

## 二、方案设计

### 核心数据流
```
Google Play → Pub/Sub topic → POST /api/v1/payments/google/rtdn
   ↓
RtdnService::handle
  ├─ 验证 X-Goog-* 签名（OIDC token / IAP）
  ├─ INSERT payment_rtdn_events ON CONFLICT DO NOTHING（message_id 幂等去重）
  ├─ 解析 message.data (base64 → DeveloperNotification JSON)
  ├─ oneTimeProductNotification.notificationType:
  │   ├─ ONE_TIME_PRODUCT_PURCHASED (1) → ack only（已由 verify 处理）
  │   ├─ ONE_TIME_PRODUCT_CANCELED  (2) → 强事务回扣 + state=REFUNDED + WS BalanceUpdated reason=payment_refund
  └─ 未识别 → log warn + 返回 200（避免 Google 重投影响其他事件）
```

## 三、TDD 验收用例
- [ ] **正向**：mock Google OIDC token + ONE_TIME_PRODUCT_CANCELED → 200，余额回扣，state=REFUNDED，写 wallet_transactions(type='refund')
- [ ] **签名失败**：伪造 token → 401 + code=40906；不入幂等表
- [ ] **重复 message_id**：同一 message_id 第二次到达 → 200 幂等，无副作用
- [ ] **状态机**：ACKED→REFUNDED 合法；其他 state→REFUNDED 直接 40905
- [ ] **未识别 notificationType**：返回 200 + warn log，不报错
- [ ] **强事务**：余额扣完后 REFUNDED 写入失败 → 全回滚

### 🔴 协议路径绑定的强制验收
- [ ] **PROTO-1**：集成断言 RTDN 端点 URL 严格 `/payments/google/rtdn`，签名头 `Authorization: Bearer <OIDC>` 校验通过
- [ ] **PROTO-2**：REFUNDED 后 `BalanceUpdated.reason === "payment_refund"`，`delta < 0`

### Round 2
- **状态**：❌ 未通过（2026-05-10，Reviewer: everything:code-reviewer）
- **审查意见**：
  - ✅ P0-1 OIDC 签名验证：`controller.rs` 已提取 `Authorization: Bearer` token 并调用 `validate_rtdn_oidc_token()`；验证失败时通过 `PaymentError::RtdnSignatureInvalid`（HTTP 401 UNAUTHORIZED）拒绝请求，无签名请求不再能触达余额修改路径。
  - ✅ P0-3 幂等逻辑顺序：`rtdn_service.rs::handle_rtdn` 已调整为 `check_rtdn_processed` → 分发业务 → `insert_rtdn_processed`（仅成功后写入），不再存在先写幂等记录再处理的反转问题。
  - ✅ P0-4 eventTimeMillis 来源：已从 `notification.event_time_millis` 取值（`DeveloperNotification` 体内），不再使用 `publishTime`。
  - ✅ 协议路径：`POST /webhook/google/rtdn` → `rtdn_webhook_handler` 函数名、URL 完全一致。
  - ⚠️ HIGH（非阻塞，下轮修复）：`validate_rtdn_oidc_token` 使用 `Algorithm::HS256`（共享密钥），而 Google Pub/Sub 实际签发的 OIDC token 为 RS256（Google 私钥签名）。当 `rtdn_oidc_secret` 非空时，生产环境下所有真实 Google RTDN 推送将被拒绝（RS256 token 无法通过 HS256 验证）。需改用 RS256 + Google JWKS 公钥验证。
  - ⚠️ INFO：`RtdnSignatureInvalid` 的错误码为 40911，不在协议 `payment_api.md §9.6` 公开表（§9.6 止步于 40910），属内部 webhook 鉴权码，可接受；原 P0-1 所述"code=40906"系当时草稿值，40906 现已正确对应 TOKEN_REPLAY。
  - ❌ P0-6（T-00055 关联）：模块级联动打回，整体轮次 ❌。
- **🔴 修复要点**：见 T-00055 Round 2（编译失败）；下轮连带将 HS256→RS256+JWKS。

> [Round2补充] RS256+JWKS验签已修复：`validate_rtdn_oidc_token()` 改为 `async`，接受 `Option<DecodingKey>` 参数注入（测试传 RSA 公钥，生产传 None 从 Google JWKS 获取）；算法从 `HS256` 改为 `RS256`；controller 已更新为 `.await` 异步调用；单元测试 T53-06..14 全覆盖（空audience跳过、RS256有效token通过、HS256 token被拒、过期被拒、audience/issuer不匹配被拒），不依赖真实 Google JWKS 网络。
