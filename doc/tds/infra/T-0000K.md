# TDS: Midscene LLM 配置接入文档 + CI Secret 流程 (Task ID: T-0000K)

> **主设计**：[T-0000E](./T-0000E.md)

## 一、背景与目标
修复体检缺口 C11：Midscene LLM 配置无文档。用户已自备 LLM Key，输出标准化文档让其他成员/CI 能复现。

## 二、方案设计

### 涉及文件清单
| 文件路径 | 变更类型 | 说明 |
|---|---|---|
| `doc/tests/MIDSCENE_SETUP.md` | 新增 | 配置接入文档 |
| `.github/workflows/e2e.yml`（如存在） | 修改 | 引用 GitHub Secrets |

### 文档章节
1. **支持的部署形态**
   - OpenAI 直连（`MIDSCENE_MODEL_NAME=gpt-4o`，无 base URL）
   - Azure OpenAI（特殊 endpoint + api-version）
   - 中转/自托管（如 OneAPI/LiteLLM，`MIDSCENE_OPENAI_BASE_URL=https://...`）
2. **本地配置**：将 Key 写入 `.env`（gitignored），不要写入 `.env.example`
3. **CI Secret 配置**：GitHub `Settings → Secrets → Actions` 添加 `MIDSCENE_MODEL_API_KEY`
4. **限流与成本**
   - 启用 `MIDSCENE_CACHE=1`
   - 仅 WEB 用例使用 LLM（API/AND 用例不依赖）
5. **回退策略**：缺 Key 时 WEB 用例 `test.skip(true, 'MIDSCENE key missing')`，不 fail
6. **验证 Key 是否生效**：单条 smoke `tests/scripts/WEB/TC-AUTH.spec.ts` 的 `should login via Midscene` 用例

## 三、TDD 验收用例
- [ ] 文档包含 3 种部署形态的完整 env 示例
- [ ] WEB 用例 globalSetup 在缺 `MIDSCENE_MODEL_API_KEY` 时 skip 而非 fail
- [ ] CI workflow 不出现明文 Key（`grep -i 'sk-' .github/` 0 命中）
- [ ] 文档含「如何排查 401 / 429 / 限流」FAQ
