# T-20010: 解封用户确认弹窗

> **归属端**: Web (Admin)  
> **模块**: User  
> **前置依赖**: T-20007 (UserDetailDrawer), T-10009 (封禁/解封接口)  
> **已有基础**: T-20008 (BanModal 封禁弹窗已完成)

---

## 需求描述

当前 `UserDetailDrawer` 的 [解封] 按钮缺少独立的确认交互。需增加解封确认弹窗，与封禁操作对称，防止运营误操作。

## 页面设计

```
┌─────────────────────────────────┐
│  确认解封用户                     │
│                                 │
│  确定要解封用户 @阿里巴巴 吗？     │
│  解封后该用户将恢复正常访问权限。    │
│                                 │
│  解封原因:                       │
│  ┌───────────────────────────┐  │
│  │ 处罚到期 / 误封 / 其他     │  │  ← Select
│  └───────────────────────────┘  │
│                                 │
│  备注 (可选):                    │
│  ┌───────────────────────────┐  │
│  │                           │  │  ← TextArea
│  └───────────────────────────┘  │
│                                 │
│          [取消]   [确认解封]     │
└─────────────────────────────────┘
```

## 交互逻辑

1. 点击 UserDetailDrawer 中的 [解封] → 弹出 UnbanModal
2. 选择解封原因（必填）+ 可选备注
3. 点击 [确认解封] → `Modal.confirm` 二次确认 → 调用 `adminUnbanUser(userId, params)` API
4. 成功后：关闭弹窗 + 刷新用户详情 + `message.success("解封成功")`
5. 失败后：弹窗保持打开 + `message.error(errorMsg)`

## API 调用

```typescript
// apiClient.ts 扩展
export async function adminUnbanUser(
  userId: string,
  params: { reason: string; remark?: string }
): Promise<void>
// PUT /api/v1/admin/users/:id/unban
```
