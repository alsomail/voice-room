# TDS: 钻石购买/续费 API（强事务）(Task ID: T-00067)

## 一、背景与目标
- **关联需求**：[phase1_nobility.md §3.5.10 升级补差 / §3.4 续费规则](../../product/phase1_nobility.md)。
- **目标**：`POST /nobles/purchase`，强事务扣钻 + upsert user_nobles。同档续费 `expire_at += duration_days`；升级按剩余天数补差扣款；降级 40911 拒绝；余额不足 40912。

## 二、方案设计
### 核心数据流
```
POST /api/v1/nobles/purchase { tier_id, msg_id, auto_renew, duration_days }
   ↓
NobilityPurchaseService::handle (单一 SQLx Tx)
  ├─ INSERT idempotency_keys (msg_id, ...) ON CONFLICT → 返回历史结果
  ├─ SELECT FOR UPDATE users WHERE id=? AND noble_lock=true (advisory lock)
  ├─ SELECT new_tier_row, current_user_noble_row
  ├─ 决策操作类型：
  │   - current=null → operation=purchase
  │   - current.tier_id=new.tier_id → operation=renew
  │   - current.level < new.level → operation=upgrade（补差公式 §10.3.3）
  │   - current.level > new.level → 40911 DOWNGRADE_NOT_ALLOWED
  │   - operation=renew 且 expire_at - now > 30d → 40913 SAME_TIER_RENEWAL_OVERLAP
  ├─ 计算 charge_diamonds（upgrade 时 = max(0, new × ratio - refund)）
  ├─ users.diamond_balance < charge_diamonds → 40912 INSUFFICIENT_BALANCE
  ├─ UPDATE users.diamond_balance -= charge_diamonds
  ├─ INSERT wallet_transactions (type='noble_purchase'/'renew'/'upgrade_proration')
  ├─ UPSERT user_nobles
  ├─ INSERT noble_history (event=...)
  ├─ 月津贴立即发放（pay_immediately=true）→ users.diamond_balance += stipend；wallet_transactions(type='noble_stipend')
  └─ COMMIT
   ↓
WS 推送 BalanceUpdated（reason=`noble_purchase|renew|upgrade_proration|noble_stipend`）+ NobleChanged
   ↓
若用户在房间 → 房间广播 NobleChanged（更新观众列表/麦位徽章）
```

## 三、TDD 验收用例
- [ ] 同 msg_id 重复请求 → 200 幂等返回首次结果
- [ ] purchase（首次）：余额扣 monthly_diamonds × duration/30，user_nobles 新增，expire_at = now + duration
- [ ] renew（同档）：expire_at += duration_days
- [ ] upgrade：补差公式正确（`refund = floor(old.monthly × remaining / 30)`，`charge = max(0, new.monthly × duration/30 - refund)`）；单测覆盖 5 个边界
- [ ] downgrade：返回 409 + code=40911；DB 不变
- [ ] 余额不足：返回 422 + code=40912；事务回滚
- [ ] tier 已下架（is_active=false）：返回 404 + code=40914
- [ ] 续费叠加上限：当前 expire_at - now > 30d 时返 409 + code=40913
- [ ] 月津贴立即发放（duke 60000，king 200000）：测断言 wallet_transactions 双行（debit + stipend credit）
- [ ] 强事务断电：mock COMMIT 前 panic → 回滚后再次调用一切正常
- [ ] WS 广播：CREDITED 后 1s 内房间内成员 + 本人收到 NobleChanged

### 🔴 协议路径绑定
- [ ] **PROTO-1**：URL `/api/v1/nobles/purchase`；请求体字段 `tier_id/msg_id/auto_renew/duration_days` 严格
- [ ] **PROTO-2**：`NobleChanged.payload.from_tier_id/to_tier_id/expire_at/reason` 严格按 [§10.4.1](../../protocol/nobility_api.md)
- [ ] **PROTO-3**：`BalanceUpdated.reason` 取值在枚举集合内
