# TDS: 订单与 SKU Schema 与迁移 (Task ID: T-00050)

## 一、背景与目标
- **关联需求**：[phase1_payment_billing.md §2 / §6.1](../../product/phase1_payment_billing.md) — Google Play 真支付数据模型奠基。
- **本 Task 的目标**：建立 `payment_skus` / `payment_orders` 表与 5 档 SKU 种子，确保 `purchaseToken` 唯一约束与 (provider, provider_order_id) 唯一约束兜底防止重复入账，作为 E-08 全模块的数据基石。

## 二、方案设计

### 核心数据流
```
SQLx Migration → 创建 payment_skus / payment_orders / payment_rtdn_events 三表 + 索引 + 唯一约束
            ↓
插入 5 档生产种子 SKU（diamond_60 / 300 / 600 / 1980 / 6480）+ 1 档 noble_pack 占位（E-09 用）
            ↓
为后续所有 E-08 Tasks 提供数据契约与种子可复现性
```

| 文件路径 | 变更类型 | 说明 |
