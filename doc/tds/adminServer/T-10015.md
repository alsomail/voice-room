# TDS: 用户行为查询 API (Task ID: T-10015)

## 一、背景与目标
- **关联需求**：E-07.5 Admin 看板"行为流"Tab 数据源
- **本 Task 的目标**：Admin Server 提供按 user_id + 事件名 + 时间范围查询 events 的接口，30 天内分区命中优化。

## 二、方案设计

### HTTP 接口
**GET /api/v1/admin/users/:id/events** （需 Admin JWT）
Query：
- `event_name` 可多值（逗号分隔）
- `from` ISO8601（默认 24h 前）
- `to` ISO8601（默认 now）
- `page` 默认 1，`limit` 默认 20（max 100）

响应：
```json
{
  "code":0,
  "data": {
    "total": 123, "page":1, "limit":20,
    "items":[
      { "id":"uuid","event_name":"gift_send_success","server_ts":"...",
        "client_ts":"...","session_id":"...","device_id":"...",
        "properties":{...},"app_version":"1.2.0","os_version":"Android 14",
        "locale":"ar-SA","network_type":"wifi" }
    ]
  }
}
```

错误码：
| HTTP | code | 说明 |
|------|------|------|
| 400 | 40003 | 时间窗 >30 天 / page/limit 非法 / event_name 格式非法 |
| 401 | 40101/40102 | 未认证/过期 |
| 403 | 40301 | 权限不足（仅 super_admin/operator 可查，cs 可查但 `event_name` 不含 `admin_*`） |
| 404 | 40400 | 用户不存在 |

### 权限
- `super_admin` / `operator`：全量可查
- `cs`（只读客服）：可查但过滤掉 `admin_*` 前缀事件
- `finance`：禁止（403）

### SQL
```sql
SELECT * FROM events
 WHERE user_id = $1
   AND server_ts BETWEEN $2 AND $3
   AND ($4::text[] IS NULL OR event_name = ANY($4))
 ORDER BY server_ts DESC
 LIMIT $5 OFFSET $6;
```
- 时间窗超 30 天拒绝（400）
- 分区裁剪：`server_ts` 条件命中分区键

## 五、Reviewer意见
### Round 1
- **状态**：❌ 未通过（2 HIGH 问题，需返回 TDD 修复）
- **审查意见**：
  1. **[HIGH] operator 角色权限错误**（`query_handler.rs:61`）  
     `let filter_admin_events = ctx.role != "super_admin"` 导致 operator 的 admin_* 事件被过滤。  
     TDS § 权限规定 `super_admin` / `operator` 均为全量可查，只有 `cs` 需过滤 `admin_*`。  
     修复：将条件改为 `ctx.role == "cs"`。
  2. **[HIGH] page=0 / limit=0 未返回 400/40003**（`query_service.rs:85–86`）  
     当前用 `.max(1)` 静默截断为 1，违反 TDS 错误码表（page/limit 非法 → 40003）。  
     修复：在 `.max(1)` 之前显式校验 `== 0` 并返回 `ValidationError`。
  3. **[MEDIUM] BETWEEN 闭区间影响分区剪枝**（`query_repo.rs:51,79`）  
     分区边界为半开区间 `[from, to)`，SQL 使用 BETWEEN 为闭区间 `[from, to]`，  
     当 `to` 精确落在分区边界时会多扫一个分区。建议改为 `>= $2 AND < $3`。
  4. **[MEDIUM] 审计日志 filters 字段不完整**（`query_handler.rs:70–86`）  
     只记录了 `filter_admin_events` 标志，缺少 `event_name`/`from`/`to`/`page`/`limit`，  
     无法从审计日志还原具体查询行为。
- **改进建议**：  
  - 优先修复 HIGH#1（权限）和 HIGH#2（参数校验），补充对应测试用例后重新提交 Review。
