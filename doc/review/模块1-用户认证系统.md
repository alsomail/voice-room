  - **关联任务**：T-10001 / T-10002
  - **文件与行号**：`app/adminServer/src/modules/auth/repository.rs:60-67` 与 `app/adminServer/migrations/001_create_admins.sql:10-20`
  - **问题说明**：`PgAdminRepository::find_by_username` 执行 `SELECT ... FROM admins WHERE username = $1 AND deleted_at IS NULL`，但 `001_create_admins.sql` 创建的 `admins` 表只有 `id / username / password_hash / role / display_name / is_active / last_login_at / created_at / updated_at` 共 9 个字段，**没有 `deleted_at` 列**，且 `migrations/` 目录中也没有任何后续 ALTER TABLE 补加该列。生产环境对该 SQL 调用时 PostgreSQL 必然返回 `42703 column "admins.deleted_at" does not exist`，由 `From<sqlx::Error>` 转为 `AppError::DatabaseError` → HTTP 500 / 50000。**Admin 登录链路 100% 不可用，整个 B 端管控阻塞**。当前所有单测通过仅因 `FakeAdminRepository` 不走 SQL，掩盖了该回归。
