# TDS: App Server 进入房间逻辑 (Task ID: T-00012)

## 一、背景与目标

- **关联需求**：语音房间核心功能——用户通过 WebSocket 进入房间
- **前置依赖**：T-00011（WebSocket 连接管理 ✅ Done）、T-00011C（在线统计 ✅ Done）
- **本 Task 的目标**：处理 `JoinRoom` WS 信令，校验房间存在性，将用户加入房间内存状态，广播 `UserJoined` 事件，返回房间快照

## 二、方案设计

### 信令格式（对齐 protocol.md）

**请求（客户端 → 服务端）**：
```json
{
  "type": "JoinRoom",
  "msg_id": "uuid",
  "payload": { "room_id": "uuid" }
}
```

**成功响应（服务端 → 请求方）**：
```json
{
  "type": "JoinRoomResult",
  "msg_id": "uuid",
  "code": 0,
  "payload": {
    "room": {
      "room_id": "uuid",
      "title": "房间名",
      "owner_id": "uuid",
      "member_count": 3,
      "mic_slots": [null, null, null, null, null, null, null, null, null]
    }
  },
  "timestamp": 1713312000
}
```

**错误码**：
- `code: 40002`：payload 缺失或 room_id 格式非法
- `code: 40400`：房间不存在或已关闭

**广播（服务端 → 房间所有成员，含自己）**：
```json
{
  "type": "UserJoined",
  "payload": { "user_id": "uuid", "nickname": "xxx", "avatar": "url" },
  "timestamp": 1713312000
}
```

### 数据结构

- mic_slots: RwLock<Vec<Option<Uuid>>>（9 个麦位）

RoomManager（src/room/manager.rs）：
- rooms: DashMap<Uuid, Arc<RoomState>>
- get_or_create_room / get_room / remove_room / room_count

### handle_join_room 处理流程

1. 解析 payload → 提取 room_id（失败 → code:40002）
2. room_service.get_active_room_detail(room_id) → None → code:40400
3. auth_service.get_user_by_id(user_id) → 获取 nickname/avatar
4. room_manager.get_or_create_room(room_id)
5. room_state.members.insert(user_id, MemberInfo)
6. registry.set_room_id(connection_id, room_id)
7. stats.user_join_room(room_id).await.ok()
8. 广播 UserJoined 给 registry.get_connections_in_room(room_id)
9. 返回 JoinRoomResult { code:0, payload: room_snapshot }

### AppState 扩展

新增 room_manager: Arc<RoomManager>（for_test() 直接 Arc::new(RoomManager::new())）

### Round 1
- 状态：❌ 未通过（Round 1）
- 审查时间：2025-01
- 审查人：Review Agent

**问题清单：**

| ID | 级别 | 文件:行 | 问题描述 | 修改建议 |
|----|------|---------|---------|---------|
| I-01 | 🔴 HIGH（阻断） | `src/room/state.rs:47` | `mic_slots.read().unwrap()` 存在毒化锁（Poison Lock）panic 风险。当任意线程持有写锁期间 panic，后续所有 `.read()` 均抛出 `Err(PoisonError)`，`.unwrap()` 级联 panic，导致整个 WebSocket 连接崩溃。 | 改为 `self.mic_slots.read().unwrap_or_else(|e| e.into_inner()).clone()`，或将字段类型改为 `parking_lot::RwLock`（与 DashMap 内部锁一致，无毒化机制）。 |
| I-02 | 🟡 MEDIUM | `src/room/handler.rs:69-72` | `get_user_by_id` 错误路径用 `_ =>` 通配符，将 DB 故障（`Err(e)`）和用户不存在（`Ok(None)`）一并静默吞掉，无任何日志记录，排障困难。 | 拆分 arm：`Err(e) => { tracing::warn!("get_user_by_id failed for {user_id}: {e}"); ("Unknown".to_string(), None) }` |
| I-03 | 🟡 MEDIUM | `src/room/handler.rs:31-42` | `#[allow(clippy::too_many_arguments)]` 屏蔽 9 参数告警，5 个依赖项可封装为 `JoinRoomDeps` struct，消除 lint 压制并提升可扩展性。 | 提取依赖 struct，函数签名精简至 ≤ 4 个参数。 |
| I-04 | 🟢 LOW | `src/modules/room/service.rs:199-204` | `get_active_room_detail` 未在 service 层校验 active 状态，完全依赖 repository 实现过滤，存在防御缺失。 | 增加 service 层 active 状态注释或防御性断言。 |
| I-05 | 🟢 LOW | `src/room/manager.rs:46` | `r.clone()` 对 Arc 的惯用写法建议改为 `Arc::clone(&r)` 以明确语义。 | 改为 `Arc::clone(&r)` |

**结论：** 因 I-01（HIGH）存在真实的生产 panic 风险，本轮 Review 不通过，退回 TDD 修复。
