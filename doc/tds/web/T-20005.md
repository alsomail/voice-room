# TDS: Web 端房间详情弹窗 (Task ID: T-20005)

## 一、背景与目标

- **关联需求**：T-20004 已实现房间管理列表，点击行时写入 `selectedRoomId` 占位 state，T-20005 负责接收该信号、拉取房间详情并在 Modal 中展示完整信息，同时提供 [强制关闭] 操作入口。
- **前置依赖**：T-10005（后端 `GET /api/v1/admin/rooms/:id` 接口）、T-20004（`selectedRoomId` / `closeRoom` 接口契约）
- **本 Task 的目标**：在不破坏 T-20004 已有逻辑的前提下，新增 `RoomDetailModal` 组件和 `useRoomDetail` Hook，使管理员可在同一页面内快速查看房间详情并执行强制关闭操作。

## 二、方案设计

### 2.1 核心数据流

```
RoomsPage
  │
  ├─ useRoomsPage（已有）
  │    ├─ selectedRoomId: string | null   ← 点击表格行写入
  │    ├─ setSelectedRoomId(null)          ← Modal 关闭时调用
  │    └─ closeRoom(roomId)               ← [强制关闭] 透传
  │
  └─ <RoomDetailModal
         selectedRoomId={selectedRoomId}
         onClose={() => setSelectedRoomId(null)}
         onCloseRoom={closeRoom}
         closingId={closingId}
       />
         │
         └─ useRoomDetail(selectedRoomId)
               │  roomId 变化 → 创建 AbortController → adminGetRoomDetail
               └─ { detail, loading, error }
```

**设计原则**：
- `selectedRoomId` 是唯一事实源，由 `useRoomsPage` 持有，`RoomDetailModal` 只读取、不写入
- `closeRoom` 和 `closingId` 从 `RoomsPage` 透传，复用 T-20004 的单一实现，避免重复逻辑
- Modal 设置 `destroyOnClose={true}`，确保切换房间时不展示旧 detail 数据

### 2.3 接口定义 — `adminGetRoomDetail`

> 对应 `doc/protocol.md §4.5 获取房间详情（后台）`

**⚠️ 注意**：protocol.md §4.5 中 `owner` 为**嵌套对象**，非扁平字段。TDS 以 protocol.md 为准。

```typescript
// 新增至 src/core/network/apiClient.ts

export interface AdminRoomDetailOwner {
  user_id: string;
  nickname: string;
  avatar: string | null;
}

export interface AdminRoomDetail {
  room_id: string;
  title: string;
  status: 'active' | 'closed';
  room_type: 'normal' | 'password' | 'paid';
  member_count: number;
  max_members: number;
  owner: AdminRoomDetailOwner;
  mic_slots: unknown[];
  created_at: string;
  updated_at: string;
}

export async function adminGetRoomDetail(
  roomId: string,
  signal?: AbortSignal,
): Promise<AdminRoomDetail> {
  const res = await adminFetch<AdminRoomDetail>(
    `/rooms/${encodeURIComponent(roomId)}`,
    { signal },
  );
  return res.data;
}
```

### 2.4 Hook — `useRoomDetail`

```
文件：src/pages/rooms/useRoomDetail.ts

state:
  detail: AdminRoomDetail | null   (初始 null)
  loading: boolean                 (初始 false)
  error: Error | null              (初始 null)

行为：
  - roomId = null  → 清空 detail/error，不发起 fetch
  - roomId 变化    → abort 上一次请求，创建新 AbortController，发起 adminGetRoomDetail
  - 成功           → setDetail(data), setLoading(false)
  - AbortError     → 静默忽略（error 保持 null）
  - 其他错误       → setError(e), setDetail(null), setLoading(false)
  - cleanup        → controller.abort()

签名：
  export function useRoomDetail(roomId: string | null): {
    detail: AdminRoomDetail | null;
    loading: boolean;
    error: Error | null;
  }
```

### 2.5 组件 — `RoomDetailModal` Props

```typescript
interface RoomDetailModalProps {
  selectedRoomId: string | null;
  onClose: () => void;
  onCloseRoom: (roomId: string) => Promise<void>;
  closingId: string | null;
}
```

**内部结构**：
- `open = selectedRoomId !== null`
- `destroyOnClose={true}`
- `onCancel={onClose}`
- 内部调用 `useRoomDetail(selectedRoomId)`

**布局区域**：

| 区域 | 内容 | data-testid |
|------|------|-------------|
| 基本信息 | title、room_type（文字）、status（RoomStatusTag）、owner.nickname、member_count/max_members、created_at | `detail-basic-info` |
| 成员列表 | 占位符文字 | `members-placeholder` |
| 历史聊天记录 | 占位符文字 | `chat-placeholder` |
| 底部操作 | [强制关闭] Button（danger，status=active 才可点） | `close-room-btn` |
| 加载态 | Spin | `detail-loading` |
| 错误态 | Alert type="error" | `detail-error` |

**[强制关闭] 按钮逻辑**：
- `disabled={detail?.status !== 'active' || closingId !== null}`
- `loading={closingId === selectedRoomId}`
- 点击 → `Modal.confirm` 二次确认
- 确认 → `await onCloseRoom(selectedRoomId)` → 成功后调用 `onClose()`

### 关键实现决策

1. **`destroyOnHidden` 替代 `destroyOnClose`**：antd v6 中 `destroyOnClose` 已废弃，改用 `destroyOnHidden`，确保切换房间时旧数据被清除。
2. **AbortError 检测**：jsdom 中 `DOMException` 不继承 `Error`，故通过 `(err as {name?:string}).name === 'AbortError'` 判断，而非 `instanceof Error`。
3. **`closeRoom` re-throw**：令 `closeRoom` 在失败时 re-throw，使 `Modal.confirm` 的 `onOk` 可通过 reject 保持弹窗打开（antd 规范：onOk rejected → confirm stays open）。Table 侧通过 `.catch(()=>{})` 抑制 unhandled rejection，不影响原有行为。
4. **Close 按钮多元素问题**：antd Modal 的 `<button aria-label="Close">` 内部嵌套 `<span aria-label="Close">`，`getByLabelText('Close')` 会匹配多个元素，改为 `document.querySelector('.ant-modal-close')` 精确定位。
