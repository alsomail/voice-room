> **归属端**: Android
> **模块**: UI/Room
> **前置依赖**: T-00025 (创建房间 API 升级), T-30007 (原创建房间入口)
> **参考**: [phase1_room_governance.md §2.1.1](../../product/phase1_room_governance.md), [business_flows.md §2.8.1](../../product/business_flows.md)

## 需求描述

重构 `CreateRoomScreen`，新增封面选择、分类、公告、密码锁四项能力，一步完成房主期望的房间个性化配置。

## 通用组件提取

| 组件 | 说明 |
|------|------|
| `CoverPickerBottomSheet` | 独立，详见 T-30037 |
| `SixDigitPinField` | 6 位数字分格输入框（密码输入复用，T-30038 也用） |
| `CharCountTextArea` | 带计数器的 TextArea |
| `CategoryDropdown` | 分类下拉，枚举从 `RoomCategory` enum 取 |

## 交互规则

| 操作 | 行为 |
|------|------|
| 点封面 | 打开 `CoverPickerBottomSheet`，选中后回填 |
| 密码 Switch 关闭 | 密码输入框隐藏，提交时 `password=null` |
| 密码 Switch 开启 | 输入框显示；未满 6 位时"创建"按钮灰 |
| 公告字数 ≥ 200 | 计数器红色，超出字符禁止输入 |
| 房间名为空 | "创建"按钮灰 |
| 提交失败（409 已有活跃房间） | Toast "你已有活跃房间，请先关闭" + 停留当前页 |

## TDD 测试锚点

| Key | 元素 | 断言 |
|-----|------|------|
| `Key('input_room_name')` | 房间名 | — |
| `Key('dropdown_category')` | 分类下拉 | 初始值 "chat" |
| `Key('textarea_announcement')` | 公告框 | 字符数响应 |
| `Key('switch_password_enable')` | 密码开关 | 切换时密码框可见性切换 |
| `Key('input_password')` | 密码输入 | 仅数字 6 位 |
| `Key('btn_submit_create_room')` | 提交 | 所有必填满足时启用 |

## TDD 验收

1. 未填房间名按钮置灰
2. 密码 Switch 关闭时密码框隐藏且不提交 password
3. 密码非 6 位数字时提交按钮置灰
4. 公告计数 ≥200 置红
5. 创建成功后自动 JoinRoom 进入
6. 提交前若分类为 null 按钮置灰
