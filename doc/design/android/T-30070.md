## 状态机

| 状态 | UI 表现 |
|------|---------|
| `Loading` | Pager Skeleton |
| `Idle (notMember)` | 当前贵族卡片显示"您还不是贵族"+引导购买 |
| `Idle (member)` | 显示当前贵族卡片 + 倒计时 + 续费/升级按钮 |
| `PurchaseDialog` | 底部弹起 BottomSheet（30天/90天/年；钻石/真金切换） |
| `Purchasing` | 主按钮 loading |
| `PurchaseSuccess` | 全屏 Lottie 庆祝（贵族升级动画 5s） + 自动播放新进场特效预览 |
| `PurchaseError` | Snackbar errorCode + 保留弹窗状态 |

## 交互逻辑

| 操作 | 行为 |
|------|------|
| Pager 横滑 | 切换贵族档位（带 fling 动画 + haptic） |
| 点击预览 Lottie | 播放完整进场特效（含 BGM 试听 5s） |
| 点击"立即开通" | 弹起 PurchaseSheet：天数 1/3/12 月切换 + 通道（钻石/真金） |
| 通道切钻石 | 不足跳 RechargeScreen（T-30060） |
| 通道切真金 | 调 E-08 创建 noble_pack 订单 |
| 升级（高于当前） | 显示"补差 X 钻石"换算结果 + 二次确认 |
| 降级 | 立即返回错误"您已是 X，无需购买更低等级" |
| 关闭自动续费 | 二次确认 + PUT `/nobles/me { auto_renew=false }` |
| 续费成功 | 卡片倒计时刷新 + 庆祝动画 |
| 真金支付返回（成功）| BalanceUpdated + NobleChanged → 中心页自动刷新 |

## 异常流程

| 场景 | 处理 |
|------|------|
| 余额不足（钻石通道） | 弹窗"钻石不足，去充值"→ T-30060 |
| 网络失败 | 顶部黄条"加载失败" + 重试 |
| 升级补差计算失败 | 二次确认弹窗显示"补差金额异常，请联系客服" |
| 续费 cron 失败推送 | 当前卡片显示红字"续费失败，余额不足" + 一键充值 |
| 已过期重新进入 | 卡片显示"贵族已过期 X 天" + "重新开通" |

## ViewModel 状态

```kotlin
data class NobleCenterUiState(
    val tiers: List<NobleTier> = emptyList(),
    val currentNoble: UserNoble? = null,                    // null = 未开通
    val selectedTierLevel: Int = 1,                         // pager 当前档位
    val purchaseSheetVisible: Boolean = false,
    val purchaseSelection: PurchaseSelection? = null,       // 天数 + 通道
    val purchasePhase: PurchasePhase = PurchasePhase.Idle,
    val previewPlayingLevel: Int? = null,
    val errorMessage: String? = null
)

sealed class NobleEvent {
    object NavigateToRecharge : NobleEvent()
    object NavigateToGooglePlay : NobleEvent()
    data class ShowSuccess(val newTier: NobleTier) : NobleEvent()
    data class ShowError(val code: String) : NobleEvent()
}
```

## RTL 兼容

- Pager 滑动方向反向（阿语从右到左浏览）
- 倒计时数字保留 LTR（不翻转）
- 特权清单 ✅/❌ 图标位置 `start` 替代 `end`
- BGM 试听波形动画反向
