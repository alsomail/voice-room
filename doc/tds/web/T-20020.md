# TDS: Web 多 profile env + VITE_ADMIN_API_BASE_URL 收口 (Task ID: T-20020)

> **主设计**：[T-0000E](./T-0000E.md)（§2.4.3 Web env 字段冻结）

## 一、背景与目标
- 修复体检缺口 C8：Web 缺 `.env.test/staging`；`VITE_ADMIN_API_BASE_URL` 在代码使用但未列入 example。
- 删除 apiClient 默认值 fallback，强制走 env。

## 二、方案设计

### 涉及文件清单
| 文件路径 | 变更类型 | 说明 |
|---|---|---|
| `app/web/.env.example` | 修改 | 增加 `VITE_ADMIN_API_BASE_URL=` |
| `app/web/.env.development` | 修改 | 增加 `VITE_ADMIN_API_BASE_URL=http://127.0.0.1:3001/api/v1/admin` |
| `app/web/.env.test` | 新增 | 指向 staging 远端（占位） |
| `app/web/.env.staging` | 新增 | 占位 URL |
| `app/web/.env.production` | 修改 | 增加 admin API URL |
| `app/web/src/core/network/apiClient.ts` | 修改 | 删除第 50 行默认值 `??  'http://localhost:3001/api/v1/admin'`，缺值抛错 |
| `app/web/src/vite-env.d.ts` | 修改 | 加 `VITE_ADMIN_API_BASE_URL` 类型声明 |
| `app/web/src/core/config/env.ts` | 修改 | 暴露 `adminApiBaseUrl`，启动期校验 |

### Profile 切换
Vite 原生 `--mode {profile}`：
- `vite` → `.env.development`
- `vite --mode test` → `.env.test`
- `vite --mode staging` → `.env.staging`
- `vite build --mode production` → `.env.production`

### 删除默认值的兼容性
对单元测试：在 `vitest.config.ts` 或 `setup.ts` 中默认注入测试值，避免现有测试因缺 env 失败。
