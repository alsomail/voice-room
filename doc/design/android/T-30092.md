## 3. 消息类型

| 类型 | 渲染 | 备注 |
|------|------|------|
| 文本 | 气泡（己右他左 LTR / RTL 镜像） | ≤500 字 |
| 表情 | 系统 + 自定义贴纸 | - |
| 图片 | 缩略图 + 全屏 | ≤5MB |
| 语音 | 波形条 + 时长 | ≤60s，长按发送 |
| 礼物 | 卡片 + 心动值 +N | 调用送礼面板 |
| 系统消息 | 灰条 | 关注 / 取关 / 屏蔽 / 解除 |
| 房间邀请卡 | 卡片"邀你进房" + 进房按钮 | 仅当邀请方在房间 |
| 链接 | 自动检测 + 风险提示 | 防诈骗 |

## 4. 已读 / 输入中 (Typing)

- 已读：发出方旁边显示"已读 HH:mm"
- 输入中：对方输入时下方"对方正在输入…"灰文字（throttle 2s 推送一次）
- 撤回：本人 2 分钟内可撤；撤回后双方显示"消息已撤回"

## 5. 状态机

| 状态 | UI |
|------|---|
| `Loading` | Skeleton |
| `Idle` | 列表 + 输入 |
| `Sending` | 该消息行 ⏳ |
| `Sent` | ✓ |
| `Read` | ✓✓ + 时间 |
| `Failed` | ❗ + 重试 |
| `Banned` | 输入栏黄条 + 禁用 |
| `Blocked` | 输入栏灰条"对方设置不接受私聊" + 禁用，仅可看历史 |
| `UserDeleted` | 全屏占位"用户已注销" |

## 6. 异常 / 安全

| 场景 | 处理 |
|------|------|
| 网络断开 | 顶部黄条 + 自动重连 |
| 发送频率限制 | 1 秒 1 条；超出本地 Toast |
| 关注门槛（产品规则） | 双方非互关 → 仅允许 1 条问候消息 / 24h；超出"对方未关注，需建立关系后继续聊天" |
| 内容审核（敏感词） | server 拦截 → 红字"消息含敏感内容，未发出" |
| 链接风险 | 检测后红色警告"该链接可能为诈骗" + 用户确认后才打开 |
| 截屏检测（高敏感档）| 弹 Toast "提醒：当前对话已被截屏" |
| 举报 | 长按消息 → 举报 + 截图证据 |

## 9. 协议入口索引

| 操作 | 协议 |
|------|------|
| 历史 | GET `/api/v1/dm/:peerId/messages?cursor=&limit=50` |
| 发送 | WS `SendDm { to_user, msg_id, type, content }` |
| 接收 | WS `Dm { from_user, msg_id, type, content, ts }` |
| 已读 | WS `MarkDmRead { peer_id, last_msg_id }` |
| 输入中 | WS `DmTyping { peer_id }`（throttle 2s） |
| 撤回 | WS `RecallDm { peer_id, msg_id }` |
| 关注校验 | 由 server 在 SendDm 处理时拦截 |
| 屏蔽状态查询 | GET `/api/v1/users/:id/relation_status` |

## 10. 性能 / RTL / 埋点

- LazyColumn 反向排版（最新在底），首次进入滚到底
- 50 条窗口；上滑加载历史；下滑距底 < 100dp 自动滚最新，否则显示"⬇ 新消息"按钮
- RTL：己消息靠左、他消息靠右（与 LTR 镜像）；时间戳/已读字符串使用 Bidi LTR 包裹
- 埋点：`dm_view`、`dm_send`(type/length)、`dm_gift_send`(gift_id, count)、`dm_blocked_show`、`dm_room_invite_click`、`dm_report`
