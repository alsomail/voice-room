### 与现状偏差（驱动本设计）

`playwright.config.ts` 第 41 行当前形态：

```ts
baseURL: process.env.ADMIN_WEB_URL || undefined,
```

存在 **2 项残余风险**：
1. **求值时序**：Playwright 在 `defineConfig()` **同步求值** `use.baseURL`，此时 globalSetup 还未运行，`process.env.ADMIN_WEB_URL` 可能为空（仅依赖 shell `export` 或根 `.env`）。T-0000H §2 残余风险 R1 显式承诺由 T-0000J 通过 fixture 解决。
2. **隐式回退 undefined**：`||  undefined` 让 WEB 用例 `page.goto('/login')` 静默拼成 `about:blank/login`，调试困难。

`tests/scripts/{API,E2E}/*.spec.ts` 多处出现：

```ts
const APP_BASE = process.env.APP_SERVER_BASE_URL ?? 'http://localhost:3000';
process.env.DATABASE_URL ?? 'postgres://app_server_user:app_server_pwd@localhost:5432/voice_room'
```

存在 **3 项问题**：
1. **DB 密码 typo `app_server_pwd`** 与根 `.env.example`（T-0000F §2.3 已修正为 `app_server_pass`）不一致，本机 psql 直连 100% 失败；
2. **localhost:3000/3001/5173 硬编码**：staging/prod profile 下回退默认值会打到错误地址；
3. **`'dotenv/config'` import** 与 T-0000H §2.5「envLoader 是唯一 env 加载源、worker 内禁止再 `dotenv.config()`」规约冲突。

```ts
const ADMIN_WEB_URL = process.env.ADMIN_WEB_URL ?? 'http://localhost:5173';
await page.goto(`${ADMIN_WEB_URL}/login`);
```

期望改为相对路径 `await page.goto('/login')`，依赖 `use.baseURL`。

## 二、方案设计

### 2.1 核心数据流

```
┌──────────────── playwright 主线程（启动期） ────────────────┐
│  playwright.config.ts                                      │
│    │ globalSetup: tests/scripts/support/globalSetup.ts     │
│    │ use.baseURL = (lazy) process.env._E2E_RUNTIME_         │
│    │              ADMIN_WEB_URL ?? process.env.ADMIN_WEB_URL│
│    ▼                                                       │
│  globalSetup.ts (T-0000H)                                  │
│    Step1: env = loadE2EEnv()                               │
│    Step2: runPreflight(env)                                │
│    Step3: runSeed(env)                                     │
│    Step4: writeProcessEnv(env)  ─┐                         │
│    Step5: persist .e2e-runtime.json + globalThis.__E2E_ENV__│
│           └─► process.env.ADMIN_WEB_URL  (T-0000H §2.5)    │
│           └─► process.env._E2E_RUNTIME_ADMIN_WEB_URL       │
│              (T-0000J 新增私有 key，避免与 shell 冲突)       │
└────────────────────────────────────────────────────────────┘
                       │
                       ▼
┌──────────────── worker 进程（用例执行期） ───────────────────┐
│  fixtures.ts (T-0000H)                                     │
│    e2eEnvWorker  = readE2EEnvForWorker()  // .e2e-runtime.json│
│    prodSafeGuard (auto) → profile=prod && !@prod-safe → skip│
│    apiWriteRequest      → !allowWrites → skip               │
│                                                            │
│  WEB 用例：                                                 │
│    page.goto('/login')  ◄── use.baseURL 来自 envLoader      │
│                                                            │
│  API/E2E 用例（去硬编码后）：                                 │
│    const { e2eEnv } = test.use(...) / fixture              │
│    psql(e2eEnv.databaseUrl, ...)   // 不再 ?? typo 默认值    │
│    request.post(`${e2eEnv.appServerBaseUrl}/api/...`)       │
└────────────────────────────────────────────────────────────┘
```

> **不变量**：
> 1. **唯一 env 源**：所有 spec 文件**禁止** `import 'dotenv/config'`；env 一律来自 `e2eEnv` fixture 或 `writeProcessEnv()` 注入后的 `process.env`。
> 2. **baseURL 求值时序消解**：`use.baseURL` 不再依赖 shell 预 export，因为 globalSetup Step 4 在任何 worker `test()` 之前已写入 `process.env.ADMIN_WEB_URL`；Playwright 内部对 `use.baseURL` 函数式 / 字段读取均在 worker 侧惰性发生（与 globalSetup 后顺序兼容）。
> 3. **typo 零容忍**：`grep -r 'app_server_pwd' tests/ playwright.config.ts` 必须 0 命中（含注释、字符串、模板字符串）。

### 2.4 @prod-safe 标签语义与注入方式

#### 2.4.1 read-only 含义（与 T-0000E §2.6 五道防线 L4/L5 协同）

打 `@prod-safe` 的用例必须**满足全部 5 条**：

1. **HTTP 方法 ≤ {GET, HEAD, OPTIONS}**：不调 POST/PUT/PATCH/DELETE；
2. **不触发 DB 写**：spec 内不出现 `INSERT/UPDATE/DELETE/TRUNCATE` SQL 子串（grep 自检）；
3. **不触发 Redis 写**：仅 `GET/EXISTS/TTL`；不调 `SET/DEL/INCR/EXPIRE`；
4. **不触发支付 / 打赏 / 送礼 / 钱包扣减**：钱包余额、礼物 send、充值类用例**不可**打标；
5. **不触发管理员写操作**：admin 后台的封禁、审核通过/拒绝、配置编辑等**不可**打标。

#### 2.4.2 标签注入方式（Playwright 1.42+ 原生 tag 语法）

**优先方式（推荐）**：`test.describe` 级或 `test()` 级 `tag` 元数据：

```ts
// 方式 A：describe 块整体打标
test.describe('TC-INFRA API - 健康探针 @prod-safe', () => {
  test('TC-INFRA-00001: AppServer /health 200', async ({ request }) => { /* ... */ });
});

// 方式 B：单 case 通过 metadata 打标（Playwright 原生 tag）
test('TC-USER-00001: GET /me 当前用户 @prod-safe', { tag: '@prod-safe' }, async () => { /* ... */ });
```

**与 T-0000H/I 的双重保险**：
- **CLI 层**：`npm run e2e:prod-smoke` 运行 `playwright test --grep "@prod-safe"`（T-0000I §2.3 冻结）；
- **Config 层**：`playwright.config.ts` 第 30 行 `grep: process.env.E2E_PROFILE === 'prod' ? /@prod-safe/ : undefined`（T-0000H 已落地）；
- **Fixture 层**：T-0000H §2.6 `prodSafeGuard` auto fixture 在 prod profile 下若 testInfo.tags 不含 `@prod-safe` 则 `test.skip()`。

> **判定唯一规则**：`testInfo.tags` 数组必须包含字面字符串 `'@prod-safe'`（含 `@` 前缀）。`test.title` 包含 `@prod-safe` 子串作为 fallback grep 命中，但**不**触发 fixture L3 跳过。建议**两者都打**（describe title 含 + tag metadata 显式声明），保证 grep 命中与 fixture 守护双绿。
