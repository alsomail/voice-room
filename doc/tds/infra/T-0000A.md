# TDS: Docker Compose 开发环境 (Task ID: T-0000A)

## 一、背景与目标
- **关联需求**：product.md - 工程基建
- **本 Task 的目标**：提供一键启动的本地开发依赖环境（PostgreSQL + Redis），所有端开发者 `docker-compose up -d` 即可使用

## 二、方案设计

### 文件位置

```
/docker-compose.yml          # 仓库根目录
/scripts/dev/init-db.sh      # 数据库初始化脚本（创建两个 Role + 授权）
```

### docker-compose.yml 内容

```yaml
version: "3.9"

services:
  postgres:
    image: postgres:16-alpine
    container_name: vr-postgres
    restart: unless-stopped
    ports:
      - "5432:5432"
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: voiceroom
    volumes:
      - vr-pgdata:/var/lib/postgresql/data
      - ./scripts/dev/init-db.sh:/docker-entrypoint-initdb.d/init-db.sh
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 5s
      timeout: 3s
      retries: 5

  redis:
    image: redis:7-alpine
    container_name: vr-redis
    restart: unless-stopped
    ports:
      - "6379:6379"
    volumes:
      - vr-redisdata:/data
    command: redis-server --appendonly yes
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 3s
      retries: 5

volumes:
  vr-pgdata:
  vr-redisdata:
```

### init-db.sh 内容

```bash
#!/bin/bash
set -e

psql -v ON_ERROR_STOP=1 --username "$POSTGRES_USER" --dbname "$POSTGRES_DB" <<-EOSQL
  -- App Server 受限账号
  CREATE ROLE app_server_user WITH LOGIN PASSWORD 'app_server_pass';
  -- Admin Server 全权账号
  CREATE ROLE admin_server_user WITH LOGIN PASSWORD 'admin_server_pass';

  -- 基础授权（后续 migration 创建表后再细化 GRANT）
  GRANT CONNECT ON DATABASE voiceroom TO app_server_user;
  GRANT CONNECT ON DATABASE voiceroom TO admin_server_user;

  -- admin_server_user 拥有 public schema 全权
  GRANT ALL PRIVILEGES ON SCHEMA public TO admin_server_user;
  ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON TABLES TO admin_server_user;
  ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON SEQUENCES TO admin_server_user;
EOSQL
```
