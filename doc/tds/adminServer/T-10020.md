# TDS: AdminServer 引入 config/ 目录 (Task ID: T-10020)

> **主设计**：[T-0000E](./T-0000E.md)（§2.4.2 字段冻结，与 AppServer 完全对称）

## 一、背景与目标
- 修复体检缺口 C4：AdminServer 完全没有 `config/*.toml`，全部走环境变量，与 AppServer 不对称。
- 改造为 `default + {profile} + env` 加载链。

## 二、方案设计

### 涉及文件清单
| 文件路径 | 变更类型 | 说明 |
|---|---|---|
| `app/adminServer/config/default.toml` | 新增 | 与 AppServer `default.toml` 章节对称（port=3001） |
| `app/adminServer/config/{dev,test,staging,prod}.toml` | 新增 | 4 个 profile |
| `app/adminServer/Cargo.toml` | 修改 | 加 `config = "0.14"` |
| `app/adminServer/src/config.rs` | 新增 | 加载逻辑 + 强类型 struct |
| `app/adminServer/src/main.rs` | 修改 | 第 40-45 行 `env::var("DATABASE_URL")` 等替换为 `AppConfig::load()`；保留 env override 优先级 |

### Profile 切换
环境变量 `ADMIN_PROFILE`（默认 `dev`）。

### 字段冻结
与 AppServer 完全相同（见 T-0000E §2.4.2），仅默认 port=3001、`service_name="admin-server"`、`environment="dev"`。

### env override 兼容性
为兼容现有 docker-compose / CI，保留以下环境变量优先级最高：
- `DATABASE_URL` → `database.url`
- `REDIS_URL` → `redis.url`
- `JWT_SECRET` → `jwt.secret`（注意旧名是 `JWT_SECRET`，**保留兼容**而非改名）
- `PORT` → `server.port`
- `GIFT_UPLOAD_DIR` → `storage.gift_upload_dir`
