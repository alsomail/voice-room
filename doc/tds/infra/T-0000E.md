## 二、方案设计

### 2.1 三环境定义

| 环境 | Profile 标识 | 服务部署 | Web URL | App Server | Admin Server | DB/Redis | 写权限 |
|---|---|---|---|---|---|---|---|
| 本地开发 | `local` | 本机 docker + cargo run | `http://localhost:5173` | `http://127.0.0.1:3000` | `http://127.0.0.1:3001` | localhost | ✅ 允许 |
| 测试环境 | `staging` | 远端（凭据待填） | `https://stg-admin.example.com` | `https://stg-api.example.com` | `https://stg-admin-api.example.com` | 远端（待填） | ✅ 允许 |
| 生产环境 | `prod` | 远端（凭据待填） | `https://admin.example.com` | `https://api.example.com` | `https://admin-api.example.com` | 远端（待填） | ❌ 默认禁止，仅 `@prod-safe` 标签的只读用例 |

### 2.2 单一切换入口

```
              E2E_PROFILE=local | staging | prod
                    │
  ┌─────────────────┼─────────────────┬─────────────────┐
  ▼                 ▼                 ▼                 ▼
Playwright      Web Dev Server     Android Build     Server Run
envLoader.ts    vite --mode=$P     ./gradlew         APP_PROFILE=$P
↓               ↓                  assemble{P}Debug  ADMIN_PROFILE=$P
.env.$P         .env.$P            BuildConfig.*     config/{default+$P}.toml
```

- **Rust（AppServer + AdminServer）**：复用 `config` crate，加载链 `default.toml → {profile}.toml → 环境变量`，Profile 通过环境变量 `APP_PROFILE` / `ADMIN_PROFILE` 切换（默认 `dev`）。
- **Web（Vite）**：原生 `--mode {profile}` 自动加载 `.env.{profile}`。
- **Android**：`productFlavors { local; staging; prod }`，每 flavor 独立 `BuildConfig` 与 `applicationIdSuffix`，三档 APK 同设备共存。
- **E2E（Playwright）**：`tests/scripts/support/envLoader.ts` 根据 `process.env.E2E_PROFILE`（默认 `local`）选择 `tests/scripts/env/.env.{profile}` 加载。

#### 2.4.2 AppServer / AdminServer config 字段冻结

`default.toml` 必须包含的章节（值用 `${ENV_VAR}` 占位，由 `config` crate 通过 env override 解析）：

```toml
[app]
name = "voice-room-server"   # 或 admin-server
environment = "dev"

[server]
host = "0.0.0.0"
port = 3000                  # admin: 3001

[database]
# DSN 完全由 DATABASE_URL 注入（不在 toml 中）
max_connections = 10
connect_timeout_secs = 5

[redis]
# URL 完全由 REDIS_URL 注入

[jwt]
# 密钥由 APP_JWT_SECRET / ADMIN_JWT_SECRET 注入
expire_secs = 86400

[log]
level = "info"
format = "json"
service_name = "voice-room-server"
```

启动时校验：缺 `DATABASE_URL` / `*_JWT_SECRET` 时给 **明确错误信息+退出码 78（EX_CONFIG）**，禁止裸 panic。

### 2.5 Seed 数据契约（冻结）

`scripts/dev/seed-e2e.sql` 的输出（即 `.env` 必须回填的字段）：

| 字段 | Seed 来源 | 备注 |
|------|----------|------|
| `E2E_USER_A_ID` | `users` 表，phone=`+966500000900` | 普通用户 A |
| `E2E_USER_B_ID` | `users` 表，phone=`+966500000901` | 普通用户 B |
| `E2E_ROOM_ID` | `rooms` 表，owner=A | 测试房间 |
| `E2E_VALID_TOKEN` | A 用户 24h JWT | seed 末尾 `\echo` 输出 |
| `E2E_ADMIN_TOKEN` 等 | `admins` 表 + JWT | 4 个角色 |
| `E2E_EXPIRED_TOKEN` / `E2E_EXPIRED_ADMIN_TOKEN` | 过期 90 天的固定值 | 不通过 seed，通过文档示例值 |

**幂等性约束**：seed 全程使用 `INSERT ... ON CONFLICT DO UPDATE`；UUID/ID 使用确定性命名空间（如 `uuid_generate_v5(NAMESPACE_E2E, 'user_a')`）保证多次运行返回同一 ID。

`scripts/dev/reset-e2e.sh`：`DELETE FROM xxx WHERE phone LIKE '+96650000090%' OR id IN (E2E ID 集合)`，**不 DROP TABLE**。

### 2.6 健康预检契约（preflight.sh）

```bash
./scripts/dev/preflight.sh [--profile local]
```

5 项检查（任一失败立即退出，退出码标记失败项）：

| 序号 | 检查项 | 命令 | 失败退出码 |
|------|--------|------|-----------|
| 1 | Postgres | `pg_isready -h $PGHOST` | 11 |
| 2 | Redis | `redis-cli -u $REDIS_URL PING` | 12 |
| 3 | App Server | `curl -fsS $APP_SERVER_BASE_URL/health` | 13 |
| 4 | Admin Server | `curl -fsS $ADMIN_SERVER_BASE_URL/health` | 14 |
| 5 | Web | `curl -fsS -o /dev/null $ADMIN_WEB_URL` | 15 |

输出格式（彩色对齐）：
```
[1/5] Postgres   ✅ ready (5432)
[2/5] Redis      ✅ ready (6379)
[3/5] AppServer  ❌ FAIL  curl: (7) Failed to connect to 127.0.0.1:3000
        ↳ Hint: cargo run -p voice-room-server -- --profile local
```

### 2.7 envLoader / globalSetup / globalTeardown 契约

```typescript
// tests/scripts/support/envLoader.ts
export interface E2EEnv {
  profile: 'local' | 'staging' | 'prod';
  allowWrites: boolean;
  appServerBaseUrl: string;
  adminServerBaseUrl: string;
  adminWebUrl: string;
  appWsUrl: string;
  databaseUrl?: string;       // staging 远端可选
  redisUrl?: string;          // staging 远端可选
  androidAppId?: string;
  tokens: { valid; expired; admin; op; cs; fin; expiredAdmin };
  ids: { roomId; userAId; userBId };
  midscene: { apiKey; modelName; baseUrl?; cache };
}
export function loadE2EEnv(): E2EEnv;  // 缺关键字段抛 MissingEnvError
```

```typescript
// playwright.config.ts
import { loadE2EEnv } from './tests/scripts/support/envLoader';
const env = loadE2EEnv();
export default defineConfig({
  globalSetup: require.resolve('./tests/scripts/support/globalSetup'),
  globalTeardown: require.resolve('./tests/scripts/support/globalTeardown'),
  use: { baseURL: env.adminWebUrl, /* ... */ },
});
```

`globalSetup` 流程：
1. 调 `loadE2EEnv()`（缺值即 fail-fast）
2. 调 `preflight.sh`（profile=local 时强制；staging/prod 时仅 HTTP 5 端检查，跳过 PG/Redis 直连）
3. profile=local 且 `E2E_ALLOW_WRITES=1` 时调 `seed-e2e.sql`
4. 暴露解析后的 env 到 `process.env`（已规范化）

`globalTeardown` 流程：
1. 仅 profile=local 时调 `reset-e2e.sh`
2. staging/prod **从不**自动 reset（避免误删远端数据）

### 2.8 用例标签体系（@prod-safe）

```typescript
test('TC-AUTH-00010 read-only token introspection', { tag: '@prod-safe' }, async () => { /* ... */ });
```

`globalSetup` 中：profile=prod 时设置 Playwright 的 `grep: /@prod-safe/`，未打标的用例自动 skip。

### 2.9 安全红线

1. **prod 写禁止**：`E2E_ALLOW_WRITES=0`（prod 默认）时，`globalSetup` 注入一个 `expect.context()`，凡走非 GET HTTP 方法的请求自动 fail。
2. **凭据隔离**：`.env.prod` 永不入库；CI 仅从 GitHub Secrets / 1Password CLI 取。
3. **DB 用户最小权限**：seed 使用独立 `e2e_runner` 角色（仅测试库），不复用 `app_server_user`。
4. **明文流量**：staging/prod flavor `usesCleartextTraffic=false`，违反则编译失败。
5. **Token 寿命**：`E2E_VALID_TOKEN` 24h，CI 每日重 seed；过期 token 用固定 90d 前签发的常量。
