## 五、Reviewer意见
### Round 1（2026-05-10，Reviewer: Dod）
**状态：❌ 打回**

#### [HIGH] P1-3 `monthly_stipend` JSONB 字段名与协议不符

**文件**：`app/shared/src/models/nobility.rs:115-119`、`app/server/migrations/011_create_nobility.sql` 种子数据

协议 §10.2.3 明确定义：
```jsonc
"monthly_stipend": { "percent": 20, "pay_immediately": true }
```
但实现使用了 `diamonds: i64` 字段名（与协议冲突）：
```rust
// ❌ 当前实现
pub struct MonthlyStipendPrivilege {
    pub diamonds: i64,   // 应为 percent: i64
    pub pay_immediately: bool,
}
```
SQL 种子数据同样使用 `"diamonds"` 键，客户端解析协议字段 `percent` 时会静默拿到 null。

#### [HIGH] P1-3 种子数据月津贴数值错误

**文件**：`app/server/migrations/011_create_nobility.sql` L108-L242

产品 §3.5.11 规定各档月津贴返还比例（月费钻石数 × 比例）：

| tier | 协议 percent | 产品应发钻石 | 种子实际 diamonds | 差异 |
|------|------------|------------|-----------------|------|
| knight | 5% | 150 | 0 | ❌ 缺失 |
| baron | 8% | 800 | 0 | ❌ 缺失 |
| viscount | 10% | 3000 | 0 | ❌ 缺失 |
| earl | 12% | 12000 | 0 | ❌ 缺失 |
| duke | 15% | **45000** | **60000** | ❌ 数值错误 |
| king | 20% | 200000 | 200000 | ✅（凑巧一致）|

另，LV1-LV4 种子 `pay_immediately: false`，但产品§3.5.11 所有档位均为"续费成功立即发放"，应改为 `true`。

**修复方向**：
1. 将 `MonthlyStipendPrivilege.diamonds` 改为 `percent: i64`
2. 更新 6 档种子 `monthly_stipend.percent` 为 5/8/10/12/15/20
3. LV1-LV4 `pay_immediately` 改为 `true`

#### [MEDIUM] P1-4 `noble_privileges_validate()` 未绑定为 CHECK 约束

**文件**：`app/server/migrations/011_create_nobility.sql` L76-L93

函数已正确定义（检查 mic_priority / gift_discount / monthly_stipend 键存在），但 `noble_tiers` 建表语句未添加 CHECK 约束：
```sql
-- ❌ 缺少
ALTER TABLE noble_tiers ADD CONSTRAINT chk_privileges
    CHECK (noble_privileges_validate(privileges));
-- 或在 CREATE TABLE 内：
privileges JSONB NOT NULL CHECK (noble_privileges_validate(privileges)),
```
当前任何格式非法的 privileges JSONB 均可直接写入，validate 函数从未被调用。
