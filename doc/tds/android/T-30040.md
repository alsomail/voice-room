# TDS: 用户操作菜单 BottomSheet (Task ID: T-30040)

## 一、背景与目标
- **关联需求**：E-10 按角色动态权限的用户操作菜单 ([设计文档](../../design/android/T-30040.md), [权限矩阵 §2.3](../../product/phase1_room_governance.md))
- **本 Task 的目标**：`UserActionBottomSheet` 根据（我的角色, 目标角色）组合渲染可用操作项。

## 二、方案设计

### 权限矩阵
| 我的角色 \\ 目标角色 | owner | admin | member |
|---------------------|-------|-------|--------|
| owner (房主) | 查看资料 \| 举报 | 卸任管理员 \| 抱上麦 \| 抱下麦 \| 禁麦 \| 禁言 \| 踢出 \| 查看资料 \| 举报 | 任命管理员 \| 抱上麦 \| 抱下麦 \| 禁麦 \| 禁言 \| 踢出 \| 查看资料 \| 举报 |
| admin | 查看资料 \| 举报 | 查看资料 \| 举报 | 抱上麦 \| 抱下麦 \| 禁麦 \| 禁言 \| 踢出 \| 查看资料 \| 举报 |
| member | 查看资料 \| 举报 | 查看资料 \| 举报 | 查看资料 \| 举报（若 target=self 则无） |

`抱上麦/抱下麦` 根据目标当前是否在麦动态切换。

### 点击触发
- `AssignAdmin / RevokeAdmin`：弹 Confirm Dialog → WS TransferAdmin
- `ForceTakeMic`：如房间所有麦位满 → Toast；否则 WS ForceTakeMic { slot_index = 第一个空闲 }
- `ForceLeaveMic`：WS ForceLeaveMic
- `MuteMic / MuteChat`：弹时长选择 Dialog [5min/30min/2h/24h] → WS MuteUser
- `Kick`：跳 T-30041 KickReasonDialog
- `ViewProfile`：MVP 占位 Toast "即将上线"
- `Report`：MVP 占位 Toast
