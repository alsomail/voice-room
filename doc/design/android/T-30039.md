## 需求描述

提供房间在场用户全量列表。入口：房间顶部在线人数点击。

## 规则

| 规则 | 说明 |
|------|------|
| 高度 | 屏幕 70% |
| 分组 | "麦上"（置顶）+ "观众"（下方） |
| 排序 | 麦上按 slot_index 升序；观众按 joined_at 倒序 |
| 分页 | LazyColumn 懒加载，首次 20 条，滚动到底拉下一页 |
| 实时 | WS `UserJoined` / `UserLeft` / `MicTaken` / `MicLeft` 事件实时调整列表 |
| 徽章 | 房主 👑 金色 / 管理员 🛡️ 金色 / 普通无 |
| 时长 | 显示进房到现在的时长（"1h2m"/"35m"/"10s"） |

## 通用组件

- `RoleBadge(role)` — 房主/管理员徽章（T-30043 共用）
- `UserListItem(user, role, duration)` — 可点击的列表项
- `AudienceSection(title, items)` — 分组容器

## 测试锚点

| Key | 说明 |
|-----|------|

| `Key('audience_count_header')` | 标题"共 N 人" |

| `Key('audience_section_watch')` | 观众分组 |
| `Key('audience_item_$userId')` | 单用户条目 |

## TDD 验收

1. 100 人房间 LazyColumn 滑动帧率 ≥55fps（profile 验证）
2. WS `UserJoined` 到达后 500ms 内列表出现新用户
3. 麦上永远置顶，slot_index 变化时重排
4. 房主/管理员徽章正确
5. 空状态（刚进房 1 人）显示"房间内暂无观众"
6. 点击任意用户触发 T-30040 菜单（含传递 targetUserId）
