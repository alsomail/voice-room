# TDS: 操作日志页面 (Task ID: T-20009)

## 一、背景与目标

- **关联需求**：来自 `doc/product.md` 运营管理模块——管理后台需要查看所有管理员的敏感操作记录（封禁/解封/关闭房间），以便事后追溯与合规审查
- **本 Task 的目标**：实现 `/logs` 路由页面，通过 Ant Design Table 按时间倒序展示操作审计日志，支持按操作人 ID / 操作类型 / 时间范围筛选和分页，对接 T-10012 提供的 `GET /api/v1/admin/logs` 接口。页面为只读，无操作列。

---

## 二、方案设计

### §2.1 核心数据流

```
URL Query String (?admin_id=&action=&start_date=&end_date=&page=&size=)
        │  ← 初始化时从 URL 读取参数
        ↓
useLogsPage (Hook)
  ├── 搜索表单状态（adminId / action / startDate / endDate）
  ├── 分页状态（page, pageSize=20）
  ├── 防竞态：AbortController（每次 fetch 新建，cleanup 时 abort）
  ├── 调用 adminGetLogs(params, signal) → apiClient.ts
  └── 返回 { items, total, loading, error, filters, page, pageSize,
              setFilters, setPage, refresh }
        │
        ↓
LogsPage (页面入口)
  ├── LogSearchForm（搜索表单：操作人 ID / 操作类型 / 时间范围）
  └── LogsTable（Ant Design Table：列表 + 分页，只读）
```

**URL 同步策略**：搜索参数（admin_id/action/start_date/end_date/page/size）双向同步到 URL Query String，支持刷新保持搜索状态，便于运营分享链接。

### §2.3 接口规格（来自 T-10012）

```
GET /api/v1/admin/logs
Authorization: Bearer <admin-jwt>
```

#### 查询参数

| 参数 | 类型 | 必须 | 说明 |
|------|------|------|------|
| `admin_id` | string (UUID) | 否 | 按操作人 ID 精确过滤 |
| `action` | string | 否 | 按操作类型过滤，如 `ban_user`、`unban_user`、`close_room` |
| `start_date` | RFC3339 | 否 | 时间范围开始（含） |
| `end_date` | RFC3339 | 否 | 时间范围结束（含） |
| `page` | integer | 否 | 页码，默认 1 |
| `size` | integer | 否 | 每页条数，默认 20，最大 100 |

#### 成功响应（200）

```json
{
  "code": 0,
  "msg": "ok",
  "data": {
    "total": 42,
    "page": 1,
    "size": 20,
    "items": [
      {
        "id": "uuid",
        "admin_id": "uuid",
        "action": "ban_user",
        "target_type": "user",
        "target_id": "uuid",
        "ip_address": "192.168.1.1",
        "detail": { "ban_reason": "违规行为", "ban_type": "temp" },
        "created_at": "2025-01-13T10:00:00Z"
      }
    ]
  }
}
```

### §2.4 组件设计

#### `useLogsPage` Hook（复用 `useUsersPage` 模式）

```typescript
export interface LogsPageFilters {
  adminId?: string;
  action?: string;
  startDate?: string;  // ISO 8601
  endDate?: string;    // ISO 8601
}

export interface UseLogsPageReturn {
  items: AdminLogItem[];
  total: number;
  loading: boolean;
  error: Error | null;
  page: number;
  pageSize: number;
  filters: LogsPageFilters;
  setPage: (page: number, pageSize: number) => void;
  setFilters: (filters: LogsPageFilters) => void;
  refresh: () => void;
}
```

核心逻辑与 `useUsersPage` 保持一致：
- 搜索参数变化时重置 `page=1`
- 每次 fetch 新建 `AbortController`，cleanup 时 abort 防竞态
- `refreshKey` 自增触发强制刷新
- 搜索参数双向同步 URL Query String（使用 `react-router-dom` 的 `useSearchParams`）

URL 参数映射：`admin_id` ↔ `adminId`，`action` ↔ `action`，`start_date` ↔ `startDate`，`end_date` ↔ `endDate`

#### `LogsTable` 组件

列定义（**只读，无操作列**）：

| 列名 | 字段 | 说明 |
|------|------|------|
| 日志 ID | `id` | 省略显示（ellipsis），宽度 220px |
| 操作人 ID | `admin_id` | 省略显示（ellipsis），宽度 220px |
| 操作类型 | `action` | `<Tag>` 组件渲染（ban_user=红、unban_user=绿、close_room=橙色） |
| 目标类型 | `target_type` | 文本，可为空 `-` |
| 目标 ID | `target_id` | 省略显示（ellipsis），宽度 220px，可为空 `-` |
| IP 地址 | `ip_address` | 文本，可为空 `-` |
| 详情 | `detail` | `JSON.stringify(detail)` 缩略显示，宽度 200px，ellipsis tooltip |
| 操作时间 | `created_at` | `new Date(val).toLocaleString()`，宽度 180px |

**工具栏**：右侧刷新按钮（与 `UsersTable` 保持一致）。

#### `LogSearchForm` 组件

Ant Design `Form` 水平布局（`layout="inline"`）：
- `Input` 操作人 ID（`placeholder="操作人 ID (UUID)"`）
- `Select` 操作类型（全部 / ban_user / unban_user / close_room）
- `DatePicker.RangePicker` 时间范围（选取后转为 ISO 8601 字符串传给 Hook）
- `Button type="primary"` 搜索
- `Button` 重置

表单提交时整体更新 filters（按搜索按钮触发，非 debounce），重置时清空表单并清除 URL 参数。

#### `apiClient.ts` 新增类型与函数

```typescript
// ─────────────────────────────────────────────────────────────────────────────
// Admin Logs（T-20009，对应 T-10012 后端接口）
// ─────────────────────────────────────────────────────────────────────────────

/** GET /admin/logs 查询参数（T-10012） */
export interface AdminGetLogsParams {
  admin_id?: string;
  action?: string;
  start_date?: string;  // RFC3339
  end_date?: string;    // RFC3339
  page?: number;
  size?: number;
}

/** 单条审计日志条目（T-10012） */
export interface AdminLogItem {
  id: string;
  admin_id: string;
  action: string;
  target_type?: string;
  target_id?: string;
  ip_address?: string;
  detail?: Record<string, unknown>;
  created_at: string;
}

/** GET /admin/logs 响应 data */
export interface AdminLogsData {
  total: number;
  page: number;
  size: number;
  items: AdminLogItem[];
}

/**
 * GET /admin/logs — 管理员查询操作审计日志（T-10012）
 * T-20009: 用于操作日志页面展示审计列表
 */
export async function adminGetLogs(
  params?: AdminGetLogsParams,
  signal?: AbortSignal,
): Promise<AdminLogsData>
```

  - **[HIGH-3 ✅ 通过] 错误处理**：`catch` 块过滤 AbortError 后设置 `error` state；`LogsPage` 在 `error != null` 时渲染 `<Alert type="error" showIcon>`，用户可见错误提示完整。

  - **[MEDIUM-3 ✅ 通过] loading 状态**：`<Table loading={loading}>` 正确透传，刷新按钮亦带 `loading={loading}` 防重复点击。
  - **[MEDIUM-4 ✅ 通过] action Tag 颜色**：`ACTION_COLOR` 映射完整（`ban_user=red` / `unban_user=green` / `close_room=orange`），未知 action 降级为 `default`。
  - **[LOW-1 ✅ 通过] TypeScript 类型**：`AdminGetLogsParams` / `AdminLogItem` / `AdminLogsData` 类型定义完整，无 `any` 滥用，API 响应链路类型安全。
  - **[LOW-2 ✅ 通过] 国际化**：页面所有可见文字（标题、按钮、列名、占位符）均使用 `t('logs.*')`，中英文 key 均已补全。
- **改进建议（非阻塞，下一迭代优化）**：
  1. **[MEDIUM] `finally { setLoading(false) }` 应跳过 AbortError**：当过滤参数快速变化时，旧 fetch 被 abort → 其 `finally` 先于新 fetch 完成 → 短暂将 `loading` 置为 `false` 再重置为 `true`，造成 loading 指示器闪烁。建议改为：`if (e.name !== 'AbortError') setLoading(false)`，将 `setLoading(false)` 移出 `finally` 或加保护条件。
  2. **[LOW] `LogsPage.handleReset` 建议加 `useCallback`**：当前为每次渲染创建新引用，若后续对 `LogSearchForm` 加 `React.memo` 会失效，先行包裹可避免后患。
  3. **[LOW] `LogSearchForm.handleReset` 建议使用 `form.resetFields()` 代替手动 `setFieldsValue`**：`resetFields()` 可确保所有字段（含未来新增字段）一并清空，维护成本更低。
