# TDS: Android × Server 跨语言 WS 回环测试（8 场景）(Task ID: T-00104)

## 一、背景与目标

- **关联需求**：BUG-CHAT-WS 复盘 R3 — server 自己写 envelope 自己 parse 永远过单测，但 server 字符串与 Android 反序列化字符串脱钩。本 Task 用**真 Android 客户端 + 真 server 进程**跑 8 个核心场景的端到端回环。
- **用户决策 #3**：仅做 Android × Server，Web 不强制（留 follow-up）。
- **本 Task 的目标**：建立 `tests/cross-lang/android-server-ws/` 测试套，每个场景：
  1. 启动真 server（test profile + 临时 PG）
  2. 启动 Android Instrumented Test 触发 WS 入站消息
  3. 断言 Android UI state 与 server 数据库状态一致
  4. 断言双方 envelope 在 schema 校验下互通

## 二、方案设计

### 8 个核心场景

| # | 场景 | C→S 触发 | S→C 验证 |
|---|------|---------|---------|

| 8 | KickUser→UserLeft | Admin `KickUser` | 被踢者收 `connection_close` + 同房间收 `UserLeft` |

### 4.2 运行状态

```
npm run test:cross-lang:ws

Test Suites: 8 passed, 8 total
Tests:       19 passed, 19 total
Time:        ~1 s
```

**当前环境**：服务器地址 `ws://192.168.1.19:3000/ws`，`.env.local` 中 `E2E_VALID_TOKEN` / `E2E_ADMIN_TOKEN` 为空值。

| 状态 | 原因 | 行为 |
|------|------|------|
| Server HTTP 可达（`/api/v1/rooms` 返回 401） | Token 未配置 → createOrGetRoom 返回 401 Unauthorized | `serverAvailable=false` → 所有 test 输出 `SKIP-KNOWN: server unavailable at <url>`，测试 PASS 而非 FAIL |
| Server WS 不可达（tryConnect 返回 false） | 相同，Token 空或连接被拒 | 每个 test 内 `SKIP-KNOWN` 早返回 |

**当有效 token 填入 `.env.local` 后**，所有 test 将进入真实执行路径，对 server 响应做 JSON Schema 验证 + 字段级断言。

### 4.3 发现的协议差异（Schema vs 任务描述 vs 协议文档）

| # | 差异描述 | 事实源 | 影响 |
|---|----------|--------|------|
| D-01 | 任务描述 `TakeMic payload: {"slot":0}`，但 `TakeMic.schema.json` 要求 `{"mic_index":0}` | `TakeMic.schema.json` 为准 | CROSS-3 测试使用 `mic_index`（非 `slot`）|
| D-02 | 任务描述 `SendGift payload: {"gift_id","to_user_id","amount":1}`，但 schema 要求 `{room_id, gift_id, receiver_id, count}` | `SendGift.schema.json` 为准 | CROSS-6 测试使用 `receiver_id` + `count` + `room_id`（非 `to_user_id` / `amount`）|
| D-03 | 任务描述 `UserMuted payload.user_id` 存在，但 `UserMuted.schema.json` payload 字段为 `target_user_id` | `UserMuted.schema.json` 为准 | CROSS-7 测试断言 `payload.target_user_id`（非 `user_id`）|
| D-04 | `GiftReceived.schema.json` 文件**不存在**于 `doc/protocol/schemas/ws/`（仅在 `websocket_signals.md §6.8.1` 有协议描述）| 协议文档 §6.8.1 | CROSS-6-GIFT-02 跳过 AJV 验证，改用结构性断言；`schema-validator.ts` 的 `schemaExists()` 检测此情况 |
