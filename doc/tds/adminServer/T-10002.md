# TDS: 管理员登录接口 (Task ID: T-10002)

## 一、背景与目标
- **关联需求**：product.md - Admin Server 管理后台
- **前置依赖**：T-10001 (管理员表设计)
- **本 Task 的目标**：实现 POST `/api/v1/admin/login` 管理员账号密码登录接口

## 二、方案设计
### 核心数据流
Client → POST /api/v1/admin/login {username, password} → Controller → AdminAuthService → 查 admins 表 → shared::crypto::verify_password() → 生成 Admin JWT (iss="voiceroom-admin", 含 role) → 更新 last_login_at → 写 admin_logs (action="admin_login") → 返回 {token, admin}

### 接口 / 信令定义

**前置依赖**：T-10001

**接口定义**：参见 `doc/protocol.md` §三 3.1 POST /api/v1/admin/login

**Admin JWT Claims**：
```json
{ "sub": "admin_uuid", "role": "operator", "iat": ..., "exp": ..., "iss": "voiceroom-admin" }
```
- 有效期 7 天
- iss 固定 "voiceroom-admin"（区分 C 端 JWT）

#### [HIGH-1] main.rs 为空桩，服务无法启动

文件：`app/adminServer/src/main.rs`

当前实现仅有 `println!("voice-room admin server")`，没有调用 `build_app()`、没有绑定端口、没有初始化依赖。所有业务代码在生产环境下均为死代码，功能无法部署。

**必须修复**：实现完整的 `#[tokio::main] async fn main()`，包括读取环境变量、初始化 PgPool、注入真实 Repository、启动 Axum 服务器。

---

#### [HIGH-2] 缺少真实数据库实现（PgAdminRepository / PgAdminLogRepository）

文件：`app/adminServer/src/modules/auth/repository.rs`

`repository.rs` 只有 Trait 接口和 Fake 内存实现，没有任何 SQLx/PostgreSQL 查询代码。即使修复 main.rs，也无法注入真实 DB 适配器对接 admins / admin_logs 表。

**必须修复**：实现 `PgAdminRepository`（find_by_username、update_last_login_at）和 `PgAdminLogRepository`（insert_login_log），基于 SQLx 查询 PostgreSQL。

---

#### [MEDIUM-1] 账号不存在时未调用 bcrypt，可通过响应时序枚举有效用户名

文件：`app/adminServer/src/modules/auth/service.rs`（第 60–70 行）

代码注释声称"防止用户名枚举"，但账号不存在时立即返回（<1ms），密码错误时执行 bcrypt（~300ms，cost=12）。响应时间相差约 300ms，VPN 内部攻击者可借此枚举有效用户名。

**修复方案**：账号不存在时仍用预计算的 DUMMY_HASH 执行一次 `verify_password`，确保两种失败路径耗时一致（常量时间认证模式）。

---

#### [LOW 备注]

1. `async-trait = "0.1"` 硬编码版本，建议移至 workspace 统一管理。
2. `AppState::jwt_secret` 为 `pub` 字段，JWT Secret 应封装为私有。
3. `AdminLoginRequest` 的 `username` / `password` 字段无最大长度校验（bcrypt 72 字节截断风险）。
