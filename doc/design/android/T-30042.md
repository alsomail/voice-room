# T-30042: 被踢/被禁提示弹窗

> **归属端**: Android
> **模块**: UI/Governance
> **前置依赖**: T-00028 (UserKicked 广播), T-00029 (UserMuted 广播)
> **参考**: [business_flows.md §2.8.5 / §2.8.6](../../product/business_flows.md)

---

## 需求描述

作为被操作者的客户端反馈机制。收到 `UserKicked` 全屏弹窗 + 自动退房；收到 `UserMuted` Toast + 底部倒计时 Chip。

## A. UserKickedDialog（全屏 Dialog）

### 布局

```
┌────────────────────────────────┐
│                                 │
│          😔                      │  ← 大表情
│                                 │
│     你已被移出房间               │  ← titleLarge
│                                 │
│   原因：骚扰他人                 │
│                                 │
│   10 分钟后你可再次进入本房间     │
│                                 │
│       [   我知道了   ]          │
└────────────────────────────────┘
```

### 交互

- 不可手势关闭
- "我知道了" → 关闭 + 自动退出房间返回大厅
- 进入大厅后该房卡片"进入"按钮置灰 10 分钟（本地倒计时 + SharedPreferences 存 `kicked_until:{room_id}`）

### 已过冷却期
- 冷却期结束后按钮自动恢复
- 若本地倒计时与 Server 不一致（如时钟漂移）， Server 端 `KICKED_COOLDOWN` 错误会兜底

## B. UserMuted 反馈（非 Dialog，更轻量）

### 布局（底部 Chip）

```
╔══════════════════════════════╗
║  🔇 你已被禁麦 04:58          ║  ← FloatingChip, 房间页底部浮层
╚══════════════════════════════╝
```

### 交互

- Toast 一次："你已被禁{麦|言} N 分钟"
- 浮层 Chip 持续显示直到倒计时结束
- Chip 点击可折叠成小图标
- 解除广播 `UserMuted { duration_sec: 0 }` 到达 → Chip 淡出

## 通用组件

- `KickedDialog(reason, cooldownSec)` — 全屏弹窗
- `MuteCountdownChip(type, remainingSec)` — 倒计时 Chip
- `KickCooldownStore` — SharedPreferences 持久化冷却期（进程重启也保留）

## 测试锚点

| Key | 说明 |
|-----|------|
| `Key('dialog_kicked')` | Kicked 弹窗根 |
| `Key('text_kick_reason')` | 原因文本 |
| `Key('btn_kicked_confirm')` | 我知道了 |
| `Key('mute_countdown')` | Mute Chip |
| `Key('mute_countdown_text')` | Chip 内文本 |

## TDD 验收

1. UserKicked 到达后 Dialog 可见且不可手势关闭
2. 确认后自动退房回到大厅
3. 10min 内大厅该房卡片"进入"按钮灰
4. 应用重启后冷却期仍生效
5. Server `KICKED_COOLDOWN` 与本地不一致时 Toast 提示
6. Mute Chip 倒计时每秒刷新
7. 多条 UserMuted 到达时取最新一条（若 server 端延长）
8. 解除广播到达后 Chip 淡出消失
