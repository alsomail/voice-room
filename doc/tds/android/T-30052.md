# T-30052 TDS — ChatMessageList 气泡样式修复

## 一、需求背景
BUG-CHAT-WS-BUBBLE：`UserMessageItem`（位于 `feature/room/ChatMessageList.kt`，非 RoomScreen.kt）只渲染裸 Text，无 Surface/Card/background 容器，Midscene 视觉 AI 无法识别"气泡形状"，导致 aiWaitFor 超时。

## 二、技术方案

### 2.1 修改范围（最小化）
- **唯一改动文件（业务）**：`feature/room/ChatMessageList.kt`（含 `UserMessageItem`）
- **主题扩展**：`core/theme/MenaColors.kt` 新增 `ChatBubble` 颜色令牌
- **禁止改动**：其他 ChatMessage 类型（`SystemNoticeItem`）、业务逻辑、WS 链路、5 节点日志（节点 5 `Log.d("ChatMessageList", ...)` 保留）

### 2.2 UserMessageItem 改造目标
在内容 Text 外包一层 `Surface` 容器（圆角 + 气泡背景色 + padding），并添加 `Modifier.testTag("chat_bubble")`。
为了让 androidTest 能直接调用，将 `UserMessageItem` 的可见性从 `private` 改为 `internal`（同模块测试可访问）。

### 2.3 MenaColors 新增令牌
```kotlin
const val CHAT_BUBBLE_VALUE: ULong = 0xFF2A2A2AuL
val ChatBubble: Color = Color(CHAT_BUBBLE_VALUE.toInt())
```
遵循现有 MenaColors 的 ULong + .toInt() 双值规约（避开 BUG-ANDROID-001）。

### 2.4 RTL 支持
`UserMessageItem` 仍以 `Row` + `Column` 布局，左对齐由父级 `fillMaxWidth()` 决定；RTL 时由 Compose 自动镜像，保持现状。
