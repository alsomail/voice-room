# TDS: 解封用户确认弹窗 (Task ID: T-20010)

## 一、背景与目标

- **关联需求**：用户管理模块——运营/客服对已封禁用户执行解封操作，需要选择解封原因并填写备注，防止误操作
- **本 Task 的目标**：实现 `UnbanModal` 解封确认弹窗，替换 T-20008 中 `UsersPage` 内的内联 `Modal.confirm` 解封流程，与 `BanModal` 完全对称，调用独立解封接口

---

## 二、方案设计

### §2.1 核心数据流

```
UserDetailDrawer.onUnbanClick(userId)
        │
        ▼
UsersPage.unbanUserId = userId
        │
        ▼
UnbanModal(userId=unbanUserId)
    ├── Form（解封原因/备注）
    └── 确认 → Modal.confirm 二次确认
              └── useUnbanUser.unban(userId, req) → adminUnbanUser(userId, req)
                        └── PUT /api/v1/admin/users/:id/unban
                                    │
                                    ▼
                              onSuccess(userId)
                              → 关闭 Drawer + 刷新用户列表 + 成功 Toast
```

### §2.3 接口规格

```
PUT /api/v1/admin/users/:id/unban
Authorization: Bearer <admin-jwt>
Content-Type: application/json

{
  "reason": "处罚到期",
  "remark": "备注（可选）"
}
```

成功响应（200）：`{ code: 0, data: null }`

错误码：
- `40401`：用户不存在
- `40901`：用户当前未被封禁（幂等，已是正常状态）
- `40301`：角色无 `user:ban` 权限

### §2.5 UnbanModal 组件设计

```typescript
interface UnbanModalProps {
  userId: string | null;  // null = 关闭
  onClose: () => void;
  onSuccess: (userId: string) => void;
}
```

**表单字段：**
- `reason`：Select（必填）— 选项：处罚到期 / 误封 / 申诉核实 / 其他
- `remark`：TextArea（可选，最多 200 字）

**提交流程：**
`validateFields` → `setIsConfirming(true)` → `Modal.confirm` 二次确认 → `useUnbanUser.unban()` → `onSuccess`

**并发防护（吸取 T-20008 教训）：**
- `isConfirming` state：`Modal.confirm` 弹出期间禁用提交按钮
- `handleClose` 中**必须**调用 `setIsConfirming(false)`（防止外层 Modal 先于 confirm 关闭时 state 残留）
- `Modal.confirm.afterClose` 中亦重置 `setIsConfirming(false)`（双重保险）
- 提交按钮：`loading={loading || isConfirming}` + `disabled={loading || isConfirming}`

**错误处理：**
- 已知业务错误码 `40901` → i18n 映射友好文案（`users.unban.alreadyNormal`）
- 其余错误 → `common.requestError` 通用文案

### §2.6 useUnbanUser Hook 设计

```typescript
export interface UseUnbanUserReturn {
  loading: boolean;
  error: Error | null;
  unban: (userId: string, req: AdminUnbanUserRequest) => Promise<void>;
}

export function useUnbanUser(): UseUnbanUserReturn
```

与 `useBanUser` 完全对称：失败时 re-throw，让 `UnbanModal` 决定错误展示。

### §2.7 UsersPage 集成

**新增 state：**
```typescript
const [unbanUserId, setUnbanUserId] = useState<string | null>(null);
```

**替换原内联解封逻辑：**
原 `handleUnbanClick` 中 `Modal.confirm` 内联 → 改为 `setUnbanUserId(userId)`

**新增回调（useCallback 包裹）：**

```typescript
  message.success(t('users.unban.successMsg'));
  setUnbanUserId(null);
  setSelectedUserId(null);
  refresh();

const handleUnbanClose = useCallback(() => {
  setUnbanUserId(null);

```

**渲染：**
```tsx
<UnbanModal
  userId={unbanUserId}
  onClose={handleUnbanClose}
  onSuccess={handleUnbanSuccess}
/>
```
