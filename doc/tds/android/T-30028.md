### 数据结构
```kotlin
data class GiftPanelUiState(
    val gifts: List<GiftVO> = emptyList(),
    val loading: Boolean = true,
    val error: String? = null,
    val selectedGiftId: String? = null,
    val selectedCount: Int = 1,
    val balance: Long = 0,
    val recipients: List<MicUserVO> = emptyList(), // 来自房间状态
    val selectedRecipientId: String? = null,       // 默认主麦
    val activeTab: GiftTab = GiftTab.Hot,
) {
    val selectedGift get() = gifts.firstOrNull { it.id == selectedGiftId }
    val totalPrice get() = (selectedGift?.price ?: 0L) * selectedCount
    val canSend get() = selectedGift != null && selectedRecipientId != null
                         && balance >= totalPrice && recipients.isNotEmpty()
}

enum class GiftTab { Hot, All, Backpack /* Phase2 占位 */ }
```

- **审查意见**：

  发现 1 个 HIGH 级别问题，直接违反 TDS §三 G28-09 验收标准；另有 1 个 MEDIUM 级别（缓存线程安全）和 2 个 LOW 级别问题。

  #### [HIGH] "点击重试"按钮 onClick 为空 lambda，onRetry 参数不存在
  **文件**：`feature/gift/GiftPanelBottomSheet.kt:184`

  错误状态下的"点击重试" `TextButton.onClick` 是一个**完全空的 lambda** `{}`，且 `GiftPanelBottomSheet` 函数签名中根本没有声明 `onRetry` 参数。用户点击该按钮永远不会触发任何操作，直接违反 G28-09（"网络失败：显示骨架屏 + '点击重试'按钮"）。ViewModel 的 `retryLoad()` 方法实现是正确的，但 UI 层完全没有绑定它。

  ```kotlin
  // 当前（BROKEN）—— GiftPanelBottomSheet.kt:184
  TextButton(onClick = { /* retryLoad 由 onSelectTab 逻辑触发，外部通过 onRetry 传入 */ }) {
      Text("点击重试", color = MenaColors.Primary)
  }
  // 函数签名中没有 onRetry 参数
  ```

  修复步骤：
  1. 函数签名添加 `onRetry: () -> Unit = {}`
  2. 按钮改为 `TextButton(onClick = onRetry)`
  3. `RoomScreen.kt` 调用处传入 `onRetry = { giftViewModel.retryLoad() }`

  #### [MEDIUM] 缓存 check-then-act 存在 TOCTOU 竞态，@Volatile 不足
  **文件**：`data/gift/RetrofitGiftRepository.kt:44-57`

  `@Volatile` 只保证单次读写的可见性，无法保护"读缓存 → 判断过期 → 发请求 → 写缓存"这一复合操作。两个协程并发调用 `listGifts()` 时，均可能通过过期检查，各自发起 HTTP 请求，形成重复请求。

  修复：将整个 check-act 块用 `Mutex.withLock {}` 包裹：
  ```kotlin
  private val cacheMutex = Mutex()

  override suspend fun listGifts(locale: String): Result<List<GiftVO>> = runCatching {
      cacheMutex.withLock {
          val now = System.currentTimeMillis()
          val cached = cachedGifts
          if (cached != null && (now - cacheTimestamp) < cacheDurationMs) {
              return@runCatching cached
          }
          val gifts = parseBody(apiService.listGifts(acceptLanguage = locale)).map { it.toDomain() }
          cachedGifts = gifts
          cacheTimestamp = System.currentTimeMillis()
          gifts
      }
  }
  ```

  #### [LOW] `dismiss()` 未重置 selectedCount，"清除选中态" 不完整
  **文件**：`feature/gift/GiftPanelViewModel.kt:153-155`

  TDS §核心交互 描述"关闭 BottomSheet，**清除选中态**"，但 `dismiss()` 仅清除 `selectedGiftId`，`selectedCount` 保留上次档位（如 1314）。建议同步重置 `selectedCount = 1`，或在 TDS/代码注释中明确说明保留 count 是刻意的 UX 决策。

  #### [LOW] `CountSelector` 在小屏幕可能水平溢出
  **文件**：`feature/gift/components/CountSelector.kt:43-51`

  6 个数量档位 Chip 在 `Row` 中固定排列，360dp 以下屏幕可能裁切最后一个 Chip（1314）。建议外层 Row 加 `horizontalScroll(rememberScrollState())`。

- **改进建议**：
  1. **[必须修复]** HIGH：为 `GiftPanelBottomSheet` 添加 `onRetry` 参数并绑定到重试按钮，确保 G28-09 真正通过
  2. **[建议修复]** MEDIUM：用 `Mutex` 替换 `@Volatile` 保护缓存复合操作
  3. **[可选]** LOW：明确 `dismiss()` 是否应重置 selectedCount；CountSelector 添加横向滚动
