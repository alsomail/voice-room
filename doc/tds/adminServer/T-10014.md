# TDS: 礼物 CRUD 管理 API (Task ID: T-10014)

## 一、背景与目标
- **关联需求**：E-07 礼物配置运营化（上下架、调价、新增）
- **本 Task 的目标**：在 Admin Server 提供礼物表 CRUD 与上传图片/Lottie 接口，所有写操作入审计日志并让 App Server 清除缓存。

## 二、方案设计

### 接口清单
| Method | Path | 权限 | 说明 |
|--------|------|------|------|
| GET | `/api/v1/admin/gifts?include_inactive=true&page=&size=` | `super_admin/operator` | 列表（含未上架/已软删选项） |
| POST | `/api/v1/admin/gifts` | `super_admin/operator` | 新增 |
| PUT | `/api/v1/admin/gifts/:id` | `super_admin/operator` | 更新（含 is_active 切换） |
| DELETE | `/api/v1/admin/gifts/:id` | `super_admin` | 软删（is_deleted=true） |
| POST | `/api/v1/admin/gifts/upload` | `super_admin/operator` | multipart 上传 icon/animation |

### POST /api/v1/admin/gifts
```json
// req
{
  "code":"unicorn_01", "name_en":"Unicorn", "name_ar":"يونيكورن",
  "icon_url":"/uploads/xxx.png", "price":66, "tier":3, "effect_level":3,
  "animation_url":null, "sort_order":35, "is_active":true
}
// resp 201
{ "code":0, "data":{ "id":"uuid", ...} }
```
**校验**：
- `code` 唯一，长度 1-32，仅英文数字下划线
- `price >= 1`，`tier ∈ [1,5]`，`effect_level ∈ [1,5]`
- `name_en` `name_ar` 1-64 字符
- `icon_url` 必须指向受控目录 `/uploads/gifts/` 或 CDN 白名单前缀

### 上传接口 POST /api/v1/admin/gifts/upload
- multipart/form-data，字段 `file`、`kind(icon|animation)`
- 白名单 MIME：`image/png`, `image/jpeg`, `image/webp`, `application/json`（Lottie JSON）
- Size ≤ 1MB（图）/ 2MB（Lottie）
- 存储：本地 `app/server/static/uploads/gifts/`（MVP）或 S3 兼容存储（后续）
- 返回 `{ url: "/uploads/gifts/2026-04-21/xxxx.png" }`

### 缓存失效
CRUD 写操作后 Admin Server 通过 Redis `PUBLISH admin:events {type:"gifts_invalidate"}` 通知 App Server 清除 `gifts:list:*`（App Server T-00019 订阅）。或 Admin Server 直连 Redis `DEL gifts:list:ar gifts:list:en`（更简单，MVP 推荐）。

### admin_logs
- `action`: `gift_create` / `gift_update` / `gift_delete` / `gift_upload`
- `target_id`: gift_id（upload 无 target）
- `detail`: 变更前后差异 diff

  **HIGH（1 个）**：

  1. **[HIGH] 上传 Handler 在 Tokio 异步上下文中使用阻塞同步 I/O**（`gift/handler.rs:295-307`）  
     `upload_gift_file_handler` 是 `async fn`，但调用了 `std::fs::create_dir_all` 和 `std::fs::write`（操作系统阻塞调用）。  
     Tokio 默认工作线程数等于 CPU 核数，阻塞调用会长期占用工作线程，高并发上传时导致整个 Admin Server 的所有异步任务被饿死（event loop stall）。  
     **修复**：替换为 `tokio::fs::create_dir_all(&sub_dir).await` 和 `tokio::fs::write(&file_path, &data).await`（tokio "fs" feature 默认已启用）。

  **MEDIUM（2 个）**：

  2. **[MEDIUM] icon_url 路径/域名白名单校验缺失**（`gift/service.rs:validate_create_request / validate_update_request`）  
     TDS §POST /api/v1/admin/gifts 明确要求：`icon_url 必须指向受控目录 /uploads/gifts/ 或 CDN 白名单前缀`，但 `validate_create_request` 和 `validate_update_request` 对 `icon_url` 字段均无任何路径前缀校验。运营操作员（含被盗号者）可提交任意外部 URL，导致 App 端渲染外部可控图片，存在内容劫持风险。

     ```rust

     if !ALLOWED_PREFIXES.iter().any(|p| req.icon_url.starts_with(p)) {
         return Err(AppError::ValidationError("icon_url 必须指向受控目录或 CDN 白名单".to_string()));
     }
     ```

  3. **[MEDIUM] update/delete 的 audit `detail` 内容过于稀疏**（`gift/handler.rs:137, 174`）
