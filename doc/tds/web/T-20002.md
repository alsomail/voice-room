# TDS: 登录逻辑与路由守卫 (Task ID: T-20002)

## 一、背景与目标
- **关联需求**：product.md - Admin Web 权限控制
- **本 Task 的目标**：调用 Admin Server 登录接口，通过 Zustand 管理 JWT 状态，实现路由鉴权守卫，确保未登录用户无法访问受保护页面。
- **前置依赖**：T-10002 (Admin 登录接口), T-20001 (管理员登录页 UI)

## 二、方案设计
### 核心数据流
App 启动 → 检查 localStorage JWT → 有效？→ 跳转目标页 / 跳转 /login
路由切换 → AuthGuard 组件 → useAuthStore.isAuthenticated → true → 渲染子路由 / false → Navigate to /login

### 涉及文件清单
| 文件路径 | 变更类型 | 说明 |
|----------|----------|------|
| `app/web/src/stores/useAuthStore.ts` | 新增 | Zustand store: token, admin, login(), logout(), isAuthenticated |
| `app/web/src/components/AuthGuard.tsx` | 新增 | 路由守卫组件 |
| `app/web/src/core/network/apiClient.ts` | 修改 | JWT 拦截器（自动附加 Authorization header + 401 自动跳转登录） |
| `app/web/src/router/index.tsx` | 修改 | 配置路由，/login 为公开，其余走 AuthGuard |

### 接口 / 信令定义

**状态管理设计**：
```typescript
interface AuthStore {
  token: string | null;
  admin: Admin | null;
  isAuthenticated: boolean;
  login: (username: string, password: string) => Promise<void>;
  logout: () => void;
  checkAuth: () => boolean;  // 检查 token 是否过期
}
```

**JWT 拦截器逻辑**：
1. 请求拦截：自动从 useAuthStore 取 token，附加到 `Authorization: Bearer xxx`
2. 响应拦截：收到 401 → useAuthStore.logout() → Navigate to /login

## 三、TDD 验收用例
- [ ] 正向：登录成功后 token 存入 localStorage + Zustand store
- [ ] 正向：已登录用户访问 /login 自动重定向到 /dashboard
- [ ] 正向：未登录用户访问 /dashboard 自动重定向到 /login
- [ ] 正向：API 请求自动携带 Authorization header
- [ ] 异常：token 过期（401 响应）→ 自动清除 token → 跳转 /login
- [ ] 异常：手动清除 localStorage token → 刷新页面跳转 /login
- [ ] 正向：RBAC — 不同 role 的 admin 看到不同菜单项
