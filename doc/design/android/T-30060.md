# T-30060: Android 充值页（SKU 列表 + Billing 入口）

> **归属端**: Android
> **模块**: Wallet / Payment
> **前置依赖**: T-30027 (钱包页), T-00051 (创建订单 API), T-30061 (BillingPort)
> **覆盖 Tasks**: T-30060 / T-30062 / T-30063（同一画面下的三段流程）
> **参考**: [phase1_payment_billing.md §4](../../product/phase1_payment_billing.md), [Yalla / Mico 充值页竞品对比]

---

## 需求描述

为 Android 用户提供"自助 Google Play 充值"入口。本设计覆盖：
1. 充值首页（SKU 列表）
2. 唤起 Billing 弹窗
3. 验签等待与结果反馈
4. 充值历史入口

## 入口

| 入口位置 | 交互 |
|---------|------|
| `WalletScreen` "充值" 按钮 | 点击进入 `RechargeScreen`（替换 T-30027 的占位 Toast） |
| 余额不足弹窗（T-30032）"去充值" | 直接跳 `RechargeScreen` 并标记来源 = `gift_panel` |
| 个人中心 "我的钱包" → 充值 | 同上 |

## 状态机

| 状态 | UI 表现 | 触发 |
|------|---------|------|
| `Loading` | 余额 / SKU 列表 Skeleton | 进入页面 |
| `Idle` | 显示 SKU，主按钮置灰 | 数据加载完成，未选档位 |
| `Selected` | 选中卡片金色边框，主按钮高亮 | 点击 SKU |
| `Creating` | 主按钮转 loading，禁用所有 SKU | 调 `/payments/orders` |
| `Billing` | 全屏遮罩 "正在调起 Google Play..." | BillingClient.launchBillingFlow 已调起 |
| `Verifying` | 全屏遮罩 "正在确认充值..." + 进度提示 | 收到 purchaseToken，调 `/google/verify` |
| `Success` | 全屏成功页（金色钻石 Lottie + "+600 💎"动画 3s） → 自动返回钱包 | verify 返回 ACKED |
| `Error` | AlertDialog（"支付未成功 / 请联系客服"），保留页面状态 | 网络失败 / 验签失败 / 风控阻断 |

## 交互逻辑

| 操作 | 行为 |
|------|------|
| 点击 SKU 卡片 | 切换选中态（金色边框 + haptic feedback） |
| 点击主按钮 | 流程：`/orders` → BillingPort.launch → onPurchasesUpdated → `/verify` |
| 用户在 Google Play 弹窗取消 | 静默回到 `Selected` 态，不弹错 |
| 充值成功 | Lottie 庆祝动画 → 余额由 `BalanceUpdated` WS 推送刷新（**严禁本地预扣加**，红线 #1） |
| 充值失败 | 弹窗"未成功，订单号 xxx，可联系客服"，按钮"复制订单号" |
| 杀进程 | App 启动 `PendingPurchaseResumer` 自动重试 verify，用户感知为"打开 App 后余额自动到账" |
| 返回键 | 在 `Verifying` 态拦截返回，提示"请等待充值确认" |

## 异常流程

| 场景 | 处理 |
|------|------|
| BillingClient 不可用（无 Google Play 服务） | SKU 列表灰显，顶部黄条提示"当前设备不支持 Google Play 充值，请联系客服" |
| 网络失败创建订单 | Snackbar "网络异常，请重试" |
| Google 验签失败 40901 | 弹窗"支付校验失败"+ 写 Sentry tag `payment_verify_fail` |
| 风控阻断 40903 | 弹窗"今日充值次数过多，请稍后再试" |
| 用户已有 PENDING 订单 | 顶部黄条"您有未完成订单，是否继续？"+ "继续/放弃"按钮 |

## ViewModel 状态

```kotlin
data class RechargeUiState(
    val balance: Long = 0,
    val skus: List<SkuVO> = emptyList(),
    val selectedSku: SkuVO? = null,
    val phase: RechargePhase = RechargePhase.Idle,   // Idle/Creating/Billing/Verifying/Success/Error
    val errorMessage: String? = null,
    val pendingOrderId: String? = null,              // 持久化到 DataStore 用于恢复
    val networkAvailable: Boolean = true
)

sealed class RechargeEvent {
    object OpenHistory : RechargeEvent()
    data class CopyOrderId(val orderId: String) : RechargeEvent()
    data class ShowBillingError(val code: String) : RechargeEvent()
    object NavigateBack : RechargeEvent()
}
```

## 与服务端协议绑定

> 详见 [doc/protocol/index.md](../../protocol/index.md) 与 TDS T-30060/62/63

| 操作 | 协议入口 | 说明 |
|------|---------|------|
| 拉 SKU 列表 | GET `/api/v1/payments/skus` | T-00051 衍生 |
| 创建订单 | POST `/api/v1/payments/orders` | obfuscatedAccountId = 内部 order_id |
| 验签入账 | POST `/api/v1/payments/google/verify` | 强事务 |
| 余额刷新 | WS `BalanceUpdated` | 单一事实源，**严禁**本地推算 |
| 历史查询 | GET `/api/v1/wallet/transactions?type=recharge` | 复用 T-00018 |

## RTL 兼容

- 余额卡片金色渐变方向反向（左→右 改 右→左）
- SKU 卡片角标位置 `start` 替代 `end`
- 阿语金额格式：`٠٫٩٩ $`（用 ICU `NumberFormat` 自动处理）
- 主按钮 progress dot 反向流动

## 埋点（接 T-30035）

| 事件名 | 时机 | 关键字段 |
|--------|------|---------|
| `recharge_page_view` | 进入页面 | `from`(wallet/gift_panel/profile) |
| `recharge_sku_select` | 点 SKU | `sku_id`, `price_usd` |
| `recharge_order_create` | 调 `/orders` 成功 | `order_id`, `sku_id` |
| `recharge_billing_launched` | BillingFlow 启动 | `order_id` |
| `recharge_billing_cancelled` | 用户取消 | `order_id` |
| `recharge_verify_success` | verify 200 + ACKED | `order_id`, `latency_ms` |
| `recharge_verify_fail` | verify 失败 | `order_id`, `error_code` |
| `recharge_resume_pending` | 启动期恢复 PENDING | `order_id_count` |
