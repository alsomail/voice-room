## 二、方案设计

### §2.1 组件签名

```kotlin
@Composable
fun ChatInputBar(
    inputText: String,
    onInputTextChange: (String) -> Unit,
    onSendMessage: (String) -> Unit,
    modifier: Modifier = Modifier,
    isSending: Boolean = false,     // T-30016 会用到，本 Task 暂不使用
)
```

**设计决策**：
- 无状态（Stateless）：状态由调用方管理，便于 T-30016 将状态提升至 ViewModel
- `isSending` 参数预留：T-30016 发送中禁用逻辑已可接入，本 Task 只验证 CI-07 基础行为

### §2.2 核心数据流

```
RoomScreen (local: inputText, onInputTextChange)
        │
        ▼
ChatInputBar(inputText, onInputTextChange, onSendMessage)
   ├── TextField(testTag="chat_input_field")
   │    ├── keyboardOptions: ImeAction.Send
   │    └── keyboardActions: onSend { if (canSend) onSendMessage(inputText) }
   └── IconButton(testTag="chat_send_button", enabled=canSend)
         └── canSend = inputText.isNotBlank() && !isSending
```

### §2.3 UI 规格

| 元素 | 组件 | testTag | enabled 条件 |
|------|------|---------|-------------|
| 文字输入框 | `TextField` (`singleLine=true`) | `chat_input_field` | 始终可编辑 |
| 发送按钮 | `IconButton` (Send icon) | `chat_send_button` | `inputText.isNotBlank() && !isSending` |

**布局**：`Row` → `TextField(weight=1f)` + `Spacer(8dp)` + `IconButton`
**内边距**：`horizontal=12.dp, vertical=8.dp`
**占位文字**：`"说点什么..."`

### §2.4 软键盘布局适配

`RoomScreen.kt` 的 `Scaffold` 添加 `Modifier.imePadding()`：

```kotlin
Scaffold(
    modifier = modifier.imePadding(),  // ← 软键盘弹出时自动调整底部间距
    ...
)
```

这要求 Activity 设置 `android:windowSoftInputMode="adjustResize"` 或配合 `WindowCompat.setDecorFitsSystemWindows(window, false)`（已在 T-30009 架构中考虑）。

### §2.5 RoomScreen 集成策略（T-30015 阶段）

T-30016 将状态提升至 ViewModel 前，`RoomScreen` 暂用本地 `remember` 管理 `inputText`：

```kotlin
// T-30015 阶段临时本地状态（T-30016 将迁移至 ViewModel）
var localInputText by remember { mutableStateOf("") }

ChatInputBar(
    inputText = localInputText,
    onInputTextChange = { localInputText = it },
    onSendMessage = { text ->
        onSendMessage(text)
        localInputText = ""   // 发送后清空
    },
)
```
