- **本 Task 的目标**：将现有 `ChatInputBar`（纯输入框+发送按钮）扩展为完整底部操作栏，新增 🎤麦克风开关、🎁礼物（灰禁）、❤️表情（灰禁）、🚪退出四个功能按钮，满足一期语聊房用户操作基本诉求。

## 二、方案设计

### 核心数据流

```
RoomViewModel
  ├── uiState.isCurrentUserOnMic   ← MicTaken/MicLeft WS 事件更新
  ├── uiState.isCurrentUserMuted   ← toggleMicMute() 本地切换
  └── toggleMicMute()              ← 调用 IMediaService（不发 WS）

RoomScreen (bottomBar)
  └── RoomBottomBar
        ├── GoldOutlinedTextField (输入框，沿用 ChatInputBar 逻辑)
        ├── MicButton      ← isOnMic + isMuted → 颜色/启用
        ├── GiftButton     ← 始终禁用，点击 Toast
        ├── EmoteButton    ← 始终禁用，点击 Toast
        └── ExitButton     ← 点击弹 AlertDialog → 确认 → onLeaveRoom()
```

**2c. 新增 `toggleMicMute()` 方法**：

```kotlin
/**
 * 切换当前用户麦克风静音状态（T-30026）。
 *
 * - 仅在用户已上麦（isCurrentUserOnMic = true）时有效。
 * - 不发送任何 WS 消息，纯本地媒体操作。
 * - CancellationException 必须 re-throw。
 */
fun toggleMicMute() {
    val currentState = _uiState.value as? RoomViewState.Success ?: return
    if (!currentState.uiState.isCurrentUserOnMic) return
    val willMute = !currentState.uiState.isCurrentUserMuted
    viewModelScope.launch {
        try {
            if (willMute) {
                mediaService.stopPublishAudio()
            } else {
                mediaService.startPublishAudio()
            }
            val updated = _uiState.value as? RoomViewState.Success ?: return@launch
            _uiState.value = RoomViewState.Success(
                updated.uiState.copy(isCurrentUserMuted = willMute)
            )
        } catch (e: CancellationException) {
            throw e
        } catch (e: Exception) {
            _events.trySend(RoomEvent.ShowToast("麦克风操作失败：${e.message}"))
        }
    }
}
```

 *   - 输入框：  "chat_input_field"（沿用原 ChatInputBar 协议，不回归）

**MicButton 实现逻辑**：

```kotlin
// 不在麦上：灰色禁用
// 在麦上 + 未静音：Success 绿色 + Mic 图标
// 在麦上 + 已静音：Error 红色 + MicOff 图标
val micTint = when {
    !isOnMic  -> MenaColors.OnBackgroundTertiary
    isMicMuted -> MenaColors.Error
    else       -> MenaColors.Success
}
IconButton(
    onClick = onMicToggle,
    enabled = isOnMic,
    modifier = Modifier.testTag("btn_mic_toggle"),
) {
    Icon(
        imageVector = if (isMicMuted) Icons.Default.MicOff else Icons.Default.Mic,
        contentDescription = if (isMicMuted) "取消静音" else "静音",
        tint = micTint,
    )
}
```

**GiftButton & EmoteButton**：

```kotlin
// 始终禁用灰色，点击弹 Toast
IconButton(
    onClick = { /* Toast 由 context 弹出 */ },
    enabled = false,
    modifier = Modifier.testTag("btn_gift"),
) {
    Icon(Icons.Default.CardGiftcard, contentDescription = "礼物", tint = MenaColors.OnBackgroundTertiary)
}
```

> 注：`enabled=false` 的 `IconButton` 点击事件不会触发，需要用 `Modifier.clickable { }` 包装或先判断 `enabled` 再处理。实际实现：使用 `Box(Modifier.clickable { context.showToast("敬请期待") })` 包裹 `IconButton(enabled=false)`，或者将按钮 `enabled=true` 但在 `onClick` 中显示 Toast 并不执行实际功能。

**推荐实现**（避免 disabled 拦截点击）：

```kotlin
val context = LocalContext.current
IconButton(
    onClick = { Toast.makeText(context, "礼物功能敬请期待", Toast.LENGTH_SHORT).show() },
    enabled = true,   // 允许点击以触发 Toast
    modifier = Modifier.testTag("btn_gift"),
) {
    Icon(
        imageVector = Icons.Default.CardGiftcard,
        contentDescription = "礼物",
        tint = MenaColors.OnBackgroundTertiary,  // 视觉上灰色
    )
}
```

var showExitDialog by remember { mutableStateOf(false) }
