# TDS: Dev/Staging 开发者菜单：测试购买入口 [Dev-only] (Task ID: T-30065)

## 一、背景与目标
- **关联需求**：[phase1_payment_billing.md §9.2 Dev Mock 通道](../../product/phase1_payment_billing.md)。
- **目标**：Android `dev` / `staging` flavor 独立 sourceSet 提供 `DevToolsScreen`，含"模拟充值-成功/失败/PENDING"按钮，直接调 T-00055 mock API；production flavor 源码集**不包含**该 Composable 类。

## 二、方案设计
### Flavor sourceSet 隔离
```
app/android/app/src/dev/java/.../debug/DevToolsScreen.kt          ← dev only
app/android/app/src/staging/java/.../debug/DevToolsScreen.kt      ← staging only
app/android/app/src/main/java/.../profile/ProfileScreen.kt        ← 连击版本号 7 次入口（仅在编译期 BuildConfig.DEBUG_TOOLS_ENABLED=true 时显示）
```

`build.gradle.kts` 配置：
```kotlin
android {
  productFlavors {
    create("production") { buildConfigField("Boolean", "DEBUG_TOOLS_ENABLED", "false") }
    create("staging")    { buildConfigField("Boolean", "DEBUG_TOOLS_ENABLED", "true") }
    create("dev")        { buildConfigField("Boolean", "DEBUG_TOOLS_ENABLED", "true") }
  }
}
```

## 三、TDD 验收用例
- [ ] **production APK 反编译**：`assembleProductionRelease` 产物 `unzip + dex2jar + jadx grep DevToolsScreen` 应为空
- [ ] **dev/staging APK**：连击版本号 7 次进入 DevToolsScreen
- [ ] **success 按钮**：调 mock API → 钱包余额实际增加（通过 WS BalanceUpdated）
- [ ] **fail 按钮**：Toast 失败原因
- [ ] **pending 按钮**：状态显示"处理中"，金额未增加；再点 success → 推进至成功
- [ ] **testTag**：`debug_mock_recharge_${outcome}`（success/fail/pending）便于自动化测试
- [ ] **顶部黄条**：显示"仅限开发环境 / 不影响财务报表"

### 🔴 协议路径绑定

- [ ] **PROTO-2**：production APK grep DexClass 不含 `DevToolsScreen`
