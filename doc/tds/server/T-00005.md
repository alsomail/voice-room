# TDS: 获取用户信息接口 (Task ID: T-00005)

## 一、背景与目标
- **关联需求**：product.md - 用户资料
- **前置依赖**：T-00004 (JWT 中间件)
- **本 Task 的目标**：实现 GET `/api/v1/users/me` 接口，返回当前登录用户的完整信息

## 二、方案设计
### 核心数据流
Client → GET /api/v1/users/me (Bearer token) → JWT 中间件 → AuthContext.user_id → UserService.get_by_id() → users 表查询 → 返回 UserResponse（不含 password_hash 等敏感字段）

### 涉及文件清单
| 文件路径 | 变更类型 | 说明 |
|----------|----------|------|
| `app/server/src/modules/auth/controller.rs` | 修改 | 增加 get_me handler |
| `app/server/src/modules/auth/service.rs` | 修改 | get_user_by_id 逻辑 |
| `app/server/src/modules/auth/dto.rs` | 修改 | UserResponse（含 coin_balance, vip_level） |

### 接口 / 信令定义
参见 `doc/protocol.md` §二 2.3 GET /api/v1/users/me

## 三、TDD 验收用例
- [ ] 正向：需要 JWT 认证，返回完整用户信息（id, phone, nickname, avatar, coin_balance, vip_level）
- [ ] 正向：不返回敏感字段（如 deleted_at）
- [ ] 异常：无 token → 401
- [ ] 异常：用户已被软删除 → 404
