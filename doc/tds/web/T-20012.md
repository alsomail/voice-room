### AdjustBalanceModal 设计
- Props：`userId`, `currentBalance`, `open`, `onClose`, `onSuccess`
- 表单字段：
  - `amount`: InputNumber（可负数），必填非零，绝对值 ≤10,000,000
  - `reason`: Input.TextArea 2-200 字符，必填
- 当 `amount < 0` 时，Modal 头部显示红色警示"扣减操作"，二次 `Modal.confirm`
- 调用 `POST /api/v1/admin/users/:id/wallet/adjust`，成功后 `onSuccess(newBalance)` 让父 Drawer 刷新
- `isConfirming` 防重复提交

### GiftManagementPage 设计
- `Table` 列：图标（预览）/ code / 名称(双语) / 价格 / tier / 上架状态 Switch / 操作（编辑/删除）
- 筛选：tier 下拉 + 状态筛选（全部/上架/下架/已删）
- 顶部"新增礼物"按钮打开 `GiftEditModal`
- `Switch` 本地状态 optimistic update + 失败回滚

### GiftEditModal
- 表单字段对齐 T-10014
- 图片上传：`Upload` 组件调 `/api/v1/admin/gifts/upload`，返回 URL 回填

- 动效 URL 可选（Lottie JSON）
