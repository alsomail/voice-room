# TDS: 手动调整余额 API (Task ID: T-10013)

## 一、背景与目标
- **关联需求**：E-07 "封闭内循环"充值通道 ([phase1_gift_economy.md §1.2](../../product/phase1_gift_economy.md))
- **本 Task 的目标**：Admin 手动为任意用户加/减钻石，事务性写余额 + 流水 + admin_logs，并通过 Redis PubSub 通知 App Server 推送 `BalanceUpdated`。

## 二、方案设计

### 核心数据流
```
POST /api/v1/admin/users/:id/wallet/adjust { amount, reason }
   ↓ admin_auth middleware（要求 role ∈ super_admin|operator|finance，拒绝 cs）
   ↓
handler
   BEGIN TX
     1. SELECT users WHERE id=? FOR UPDATE
        - not found / deleted → 404
     2. new_balance = current + amount
        - new_balance < 0 → 400 (40204 INSUFFICIENT_BALANCE)
     3. UPDATE users SET diamond_balance = new_balance
     4. INSERT wallet_transactions (type='admin_adjust', amount, balance_after, operator_id, reason)
     5. INSERT admin_logs (action='wallet_adjust', target_id, detail jsonb)
   COMMIT
   ↓
Redis PUBLISH admin:events {type:"balance_updated", user_id, new_balance, delta, reason}
   ↓ App Server 订阅后推送 WS BalanceUpdated（T-00018）
   ↓
200 { new_balance }
```

### HTTP 接口
**POST /api/v1/admin/users/:id/wallet/adjust** （需 Admin JWT + `WalletAdjust` 权限）

Request:
```json
{ "amount": 1000, "reason": "运营补偿：客诉处理 #1234" }
```
- `amount`：非零整数，正/负皆可；绝对值 ≤10,000,000（防误操作）
- `reason`：必填，2-200 字符

Response 200:
```json
{ "code":0, "data":{ "user_id":"uuid", "new_balance":1234, "delta":1000 }, "request_id":"..." }
```

错误码：
| HTTP | code | 说明 |
|------|------|------|
| 400 | 40003 | amount=0 / reason 空 / amount 绝对值超限 |
| 401 | 40101/40102 | 未认证/过期 |
| 403 | 40301 | role=cs（无权限） |
| 404 | 40400 | 用户不存在或已删除 |
| 400 | 40204 | 调整后余额 <0 |

### RBAC
在 `modules/auth/rbac.rs` 中 `WalletAdjust` permission 授权 `super_admin, operator, finance`，`cs` 拒绝。

### 跨服务事件（Redis PubSub）
复用 T-10011 的 `admin:events` 频道，格式：
```json
{ "type":"balance_updated", "user_id":"uuid", "new_balance":1234, "delta":1000, "reason":"...", "admin_id":"uuid", "ts":1720000000 }
```

### admin_logs
```
action = "wallet_adjust"
target_type = "user"
target_id = user_id
detail = { amount, reason, new_balance, delta }
```
由 `audit::middleware` 或服务内显式写入（参考 T-10012 模式）。
