# T-30041: 踢人原因选择弹窗

> **归属端**: Android
> **模块**: UI/Governance
> **前置依赖**: T-30040 (用户操作菜单), T-00028 (KickUser 信令)
> **参考**: [business_flows.md §2.8.5](../../product/business_flows.md)

---

## 需求描述

房主/管理员选择"踢出"后弹出的原因选择弹窗。强制用户选择或填写原因，写入审计表。

## 布局

```
┌────────────────────────────────┐
│      ⚠️ 踢出 Yasmin             │  ← titleLarge
│                                 │
│   选择原因：                    │
│   ⚫ 骚扰他人                    │  ← RadioGroup, 默认选中
│   ⚪ 刷屏                       │
│   ⚪ 辱骂/人身攻击               │
│   ⚪ 其他（填写）                │
│                                 │
│   ┌────────────────────────┐   │  ← "其他"选中时显示
│   │ 填写原因...              │   │
│   └────────────────────────┘   │
│                                 │
│   被踢用户 10 分钟内无法进入本房 │
│                                 │
│  [  取消  ]  [  确认踢出  ]     │
└────────────────────────────────┘
```

## 交互规则

| 操作 | 行为 |
|------|------|
| 选"其他"未填写 | 确认按钮置灰 |
| 选择其他选项 | 确认按钮启用 |
| 确认 | WS `KickUser { room_id, target_user_id, reason }` |
| 成功 | 关闭 + Toast "已将 Yasmin 踢出房间" |
| 失败（PERMISSION_DENIED） | Toast "权限不足" |
| 失败（CANNOT_KICK_OWNER） | Toast "不能踢出房主" |
| 点击外部 / 返回键 | 不关闭（防误触） |

## 通用组件

- `ReasonRadioGroup(options, selected)` — 通用单选（抱下麦等将来可复用）
- reason 提交时规整为枚举值+自定义文本：`"harassment" | "spam" | "abuse" | "other:${custom}"`

## 测试锚点

| Key | 说明 |
|-----|------|
| `Key('dialog_kick_reason')` | 弹窗根 |
| `Key('kick_reason_0')` ~ `Key('kick_reason_3')` | 4 个选项 |
| `Key('kick_reason_custom_input')` | "其他"填写框 |
| `Key('btn_confirm_kick')` | 确认按钮 |
| `Key('btn_cancel_kick')` | 取消按钮 |

## TDD 验收

1. 默认选中"骚扰他人"
2. 选"其他"但未填写，确认按钮置灰
3. 选"其他"并填写后，确认按钮启用
4. 确认后 WS SendSignal 被调用 1 次，reason 符合枚举
5. 成功后 Toast 出现且弹窗关闭
6. 返回键不关闭弹窗
7. PERMISSION_DENIED 场景 Toast 显示"权限不足"
