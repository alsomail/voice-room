## 一、背景与目标

- **关联需求**：模块 9（E2E 测试基建）的 P0 起点，落地 T-0000E §2.11 Migration Path **Step 1：env 模板就绪**。
- **本 Task 的目标**：用一组 `.example` 文件锁住三档 profile（`local` / `staging` / `prod`）的环境变量契约，使新人 `cp` 后改少量字段即可启动 E2E；并修复主 `.env.example` 数据库密码 typo（C1）、补齐 `E2E_PROFILE` / `E2E_ALLOW_WRITES` / `VITE_ADMIN_API_BASE_URL` 缺字段（C3 / C13）。
- **与现状偏差**：当前 `.env.example` 字段名拼写错误（`app_server_pwd`，与 init-db.sh 注入的 `app_server_pass` 不一致），且 `tests/scripts/env/` 目录尚未建立，导致 `tests/scripts/**` 在多环境切换时无配置可读。
- **不在本 Task 范围内**：
  1. envLoader 实现 → T-0000H；
  2. Web `.env.example` 内的 `VITE_ADMIN_API_BASE_URL` 收口 → T-20020（本 Task 仅在根 `.env.example` 提示 Web 端字段，便于新人发现）；
  3. Seed 生成 token → T-0000G；
  4. CI Secret 注入 Midscene Key → T-0000K。

## 二、方案设计

### 2.1 核心数据流

```
开发者 / CI
   │  cp .env.example .env                     ← 主项目根（启动 docker-compose / 服务进程）
   │  cp tests/scripts/env/.env.local.example  tests/scripts/env/.env.local
   │  cp tests/scripts/env/.env.staging.example tests/scripts/env/.env.staging
   │  cp tests/scripts/env/.env.prod.example   tests/scripts/env/.env.prod
   ▼
真实 .env / .env.<profile>（gitignored）
   │
   ▼
T-0000H envLoader.ts ── 按 E2E_PROFILE 选择文件 ── loadE2EEnv()
   │
   ▼
playwright globalSetup → preflight / seed / 注入 process.env
```

### 2.4 与 T-0000H envLoader 的接口契约（错误格式冻结）

> 本节是「Plan 阶段」对「T-0000H 实现阶段」的**冻结契约**，T-0000H 必须按此抛错。

#### 2.4.1 异常类

```typescript
// tests/scripts/support/envLoader.ts（由 T-0000H 实现）
export class MissingEnvError extends Error {
  constructor(
    public readonly profile: 'local' | 'staging' | 'prod',
    public readonly missingFields: string[],
    public readonly envFilePath: string,        // 例：tests/scripts/env/.env.local
  ) {
    super(MissingEnvError.format(profile, missingFields, envFilePath));
    this.name = 'MissingEnvError';
  }
  static format(profile: string, fields: string[], path: string): string {
    return [
      `[E2E envLoader] MissingEnvError: profile=${profile} missing ${fields.length} required field(s):`,
      ...fields.map(f => `  - ${f}`),
      `Hint: copy ${path}.example to ${path} and fill in values.`,
      `Reference: doc/tds/infra/T-0000F.md §2.3 field table.`,
    ].join('\n');
  }
}
```

#### 2.4.2 触发时机

- 在 `loadE2EEnv()` 内部，按 `E2E_PROFILE` 取出该 profile 的「必填字段集」（即 §2.3 表格中对应列为 ✅ 的字段），逐一校验：
  - 字段未声明 → 计入 `missingFields`；
  - 字段为空字符串 → 计入 `missingFields`；
  - 字段为合法非空值 → 通过。
- 校验完毕后，若 `missingFields.length > 0`，抛 `MissingEnvError`。
- **fail-fast 原则**：错误必须发生在 `playwright` 加载用例**之前**（即 globalSetup 第 1 步），禁止允许任何用例进入 `test()` 块后再崩溃。

#### 2.4.3 退出码语义

- 抛出 `MissingEnvError` 后，`playwright` 进程以**退出码 78（EX_CONFIG）** 退出，与 T-0000E §2.4.2 的服务端启动校验对齐（globalSetup 顶层 try/catch 捕获并 `process.exit(78)`）。

#### 2.4.4 「URL 占位防误用」次级校验（接口预留，不强制 T-0000H 落地）

- staging/prod profile 下，若 `APP_SERVER_BASE_URL` / `ADMIN_SERVER_BASE_URL` / `ADMIN_WEB_URL` / `APP_WS_URL` 任一 host 仍为 `*.example.com`（即未替换占位），envLoader 应抛 `PlaceholderUrlError`（与 §2.4.1 同级风格）。该校验在 T-0000E §2.12 R4 中标为 P1，T-0000H 决议是否落地；本 TDS **仅冻结字段约束**，不强制实现。
