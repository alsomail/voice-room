#### [HIGH-01] semantics 位于 clickable 之前 — contentDescription 无法绑定到可点击节点

**文件**：`MicSlotCard.kt:62–72`

**问题**：
```kotlin
modifier
    .size(80.dp)
    .semantics { contentDescription = contentDesc }  // ← 外层节点（祖先）
    .clickable { onClick(slot.index) }               // ← clickable.mergeDescendants=true 只合并"子孙"
    .then(Modifier.testTag(...))                     // ← 内层节点（被合并进 clickable）
```
`clickable` 的 `mergeDescendants = true` 只向**内**（descendant）合并语义，不合并外层祖先节点。  
`contentDescription` 因此落在独立的祖先语义节点上，而 TalkBack 焦点聚焦于拥有 `onClick` 的 clickable 节点时，**不会朗读 contentDescription**。  
测试 MC-08/09/10 使用 `assertExists()` 只验证节点存在而不验证与 click action 同节点，故测试通过但生产无障碍实际失效。

**修复方案**（推荐）— 使用 `clickable` 的 `onClickLabel` 参数，将描述与可点击行为绑定在同一语义节点：
```kotlin
Box(
    modifier = modifier
        .size(80.dp)
        .testTag(if (!isOccupied) "mic_slot_empty_${slot.index}" else "mic_slot_occupied_${slot.index}")
        .clickable(onClickLabel = contentDesc) { onClick(slot.index) },
    contentAlignment = Alignment.Center,
)
```

#### [HIGH-02] SoundWaveAnimationPlaceholder 使用裸 `if` 而非 `AnimatedVisibility`

**文件**：`MicSlotCard.kt:98–104`

**问题**：
```kotlin
if (!isMuted) {
    SoundWaveAnimationPlaceholder(...)   // 裸 if：状态切换时直接销毁/重建
}
```
TDS §2.4 明确要求 "rememberInfiniteTransition + **AnimatedVisibility**"，但实现使用了裸 `if`。  
当 `isMuted` 在 OCCUPIED ↔ MUTED 之间切换时，`SoundWaveAnimationPlaceholder` 被立即销毁/重建：  
1. 无淡入淡出过渡，视觉突变；  
2. `rememberInfiniteTransition` 每次重新创建，动画从头开始；  
3. 与 TDS 规格不符。

**修复**：
```kotlin
AnimatedVisibility(visible = !isMuted) {
    SoundWaveAnimationPlaceholder(
        modifier = Modifier
            .matchParentSize()
            .testTag("mic_slot_sound_wave"),
    )
}
```
`AnimatedVisibility` 默认 `fadeIn + fadeOut`，与 MVP 音浪占位效果匹配。

#### [HIGH-03] `Modifier.then(if(A) B else C)` 条件式 testTag

**文件**：`MicSlotCard.kt:67–72`

**问题**：
```kotlin
.then(
    if (!isOccupied)
        Modifier.testTag("mic_slot_empty_${slot.index}")
    else
        Modifier.testTag("mic_slot_occupied_${slot.index}")
)
```
`Modifier.then(if(A) modA else modB)` 在 `isOccupied` 状态变化时，Compose diff 会看到不同的 Modifier 实例对象（整个 `then` 参数变化），无法按属性更新复用，触发完整的布局重建而非仅更新 testTag 字符串。

**修复**：统一为属性条件写法，Modifier 类型不变，只改 String 值（同时配合 HIGH-01 修复后移至链首）：
```kotlin
.testTag(if (!isOccupied) "mic_slot_empty_${slot.index}" else "mic_slot_occupied_${slot.index}")
```

#### [HIGH-01-R2] `onClickLabel` ≠ `contentDescription`，无障碍描述仍未设置

**文件**：`MicSlotCard.kt:66`

**问题根因**：

R1 修复建议使用 `clickable(onClickLabel = contentDesc)`，TDD 照此实现，但两个 API 操作的是**完全不同的 Compose 语义属性**：

| API | 设置的语义属性（Compose internal key） | TalkBack 朗读时机 |
|-----|--------------------------------------|-----------------|
| `clickable(onClickLabel = "foo")` | `SemanticsActions.OnClick.label` | "双击以 foo"（动作标签） |
| `semantics { contentDescription = "foo" }` | `SemanticsProperties.ContentDescription` | 节点获得焦点时的**元素描述** |
| `onNodeWithContentDescription("foo")` | **仅**匹配 `SemanticsProperties.ContentDescription` | — |

当前代码中 `contentDesc` 只传给了 `onClickLabel`，全文件**从未设置任何节点的 `SemanticsProperties.ContentDescription`**。

**直接后果**：

1. **MC-08/09/10 运行时失败**：  
   `onNodeWithContentDescription("麦位 5，空位，点击上麦").assertExists()` 找不到任何节点，  
   因为没有节点的 `ContentDescription` 等于该字符串 → `assertExists()` 抛 `AssertionError`。  
   CI 仅编译不跑设备，故目前未暴露。

2. **TalkBack 仍然无效**：  
   用户聚焦麦位时 TalkBack 不播报元素名称；只有双击时播报动作标签，体验严重受损。

**正确修复方案**：  
将 `semantics { contentDescription = contentDesc }` 放在 `clickable` **之后**（即链条更靠内侧），使其成为 clickable 语义节点的 descendant，从而被 `mergeDescendants = true` 合并进可点击节点：

```kotlin
Box(
    modifier = modifier
        .size(80.dp)
        .testTag(if (!isOccupied) "mic_slot_empty_${slot.index}" else "mic_slot_occupied_${slot.index}")
        .clickable(onClickLabel = contentDesc) { onClick(slot.index) }
        .semantics { contentDescription = contentDesc },  // AFTER clickable → descendant → 被合并
    contentAlignment = Alignment.Center,
)
```

> **Modifier 链顺序原则**：`M1.M2.M3` 中 M1 是最外层（ancestor），M3 是最内层（descendant）。  
> `clickable`（mergeDescendants=true）只向内合并 descendant；  
> `semantics` 放在 `clickable` **后面** = 内层 = descendant = 被合并 ✅  
> `semantics` 放在 `clickable` **前面** = 外层 = ancestor = 不被合并 ❌（R1 原始 bug）
